# 보상 모델링: 사람의 선호도를 컴퓨터에게 가르치기

## 핵심 아이디어

보상 모델은 일종의 '작은 판사'입니다. 동일한 질문에 대한 두 가지 답변을 보여주고 사람이 어떤 것을 더 좋아했는지 알려주면, 모델은 시간이 지남에 따라 사람이 선호할 만한 답변에 더 높은 점수를 주는 법을 배웁니다.

왜 이런 판사가 필요할까요? 우리가 언어 모델에게 원하는 것 중 상당수는 수학 공식으로 쓰기 어렵기 때문입니다. "도움이 된다", "정중하다", "글을 잘 쓴다"는 것을 정의하는 단 하나의 방정식은 존재하지 않습니다. 하지만 사람은 두 가지 옵션 중 어느 것이 더 나은지 거의 항상 짚어낼 수 있습니다. 보상 모델은 이러한 단순한 "이게 더 낫다"는 투표 결과를 학습 알고리즘이 사용할 수 있는 숫자로 바꿔줍니다.

## 실생활 비유

친구에게 브라우니 굽는 법을 가르친다고 상상해 보세요.

여러분은 친구에게 "무엇이 좋은 브라우니인가"에 대한 50페이지짜리 규칙 책을 건네지 않습니다. 대신, 두 판의 브라우니를 맛보고 이렇게 말하죠.

"이게 더 맛있어."

이 과정을 몇 번 반복하면 친구는 패턴을 알아차리기 시작합니다. 아마도 더 쫀득한 것이 항상 이길 수도 있고, 너무 구워진 것은 항상 탈락할 수도 있습니다. 친구는 여러분의 비교를 통해 마음속에 자신만의 점수 체계를 구축하게 됩니다.

보상 모델은 정확히 이 일을 숫자로 수행합니다. 모델은 *왜* 선택된 답변이 더 나은지 알 필요가 없습니다. 그저 수많은 "A가 B를 이김" 사례를 접하다 보면, 선호도와 일치하는 점수를 점차 학습하게 됩니다.

## 학습 원리 (직관적 이해)

각 예시는 프롬프트, **선택된(chosen)** 응답, 그리고 **거부된(rejected)** 응답이라는 세 가지 요소로 구성됩니다. 우리는 모델이 거부된 응답보다 선택된 응답에 더 높은 점수를 주기를 원합니다. 그 점수 차이가 얼마이든 상관없이요.

훈련 과정에서의 유도 방식은 본질적으로 단순합니다.

- 선택된 응답의 점수가 너무 낮다? 점수를 올립니다.
- 거부된 응답의 점수가 너무 높다? 점수를 내립니다.
- 이미 명확한 차이로 올바른 순서가 매겨져 있다? 그대로 둡니다.

이러한 유도 방식을 브래들리-테리(Bradley-Terry) 손실이라고 하며, 현대 RLHF 시스템의 표준 레시피입니다.

## 실험 결과

우리는 2,000개의 가상 선호도 쌍을 사용하여 보상 모델을 훈련시켰습니다. 아래 그래프는 동일한 훈련 과정에 대한 세 가지 관점을 보여줍니다.

![Reward model training](outputs/reward_modeling.png)

- **왼쪽** - 손실(loss)이 빠르게 감소합니다. 모델이 자신의 순위 매기기에 대해 더 확신을 갖게 되었음을 의미합니다.
- **가운데** - 선호도 정확도가 거의 100%까지 올라갑니다. 거의 모든 쌍에 대해 선택된 응답이 거부된 응답보다 높은 점수를 받게 됩니다.
- **오른쪽** - 선택된 응답과 거부된 응답의 점수 분포가 서로 멀어집니다. 처음에는 겹쳐 있었지만, 훈련 후에는 선택된 응답들이 확연히 오른쪽(높은 점수 쪽)에 위치합니다.

이러한 분리가 핵심입니다. 이제 이후 단계(PPO나 DPO)에서 이 점수를 최적화 목표로 사용할 수 있게 되었습니다.

## RLHF 파이프라인에서의 위치

로드맵에서는 RLHF를 "인간의 선호도에 맞춰 모델을 정렬하는 것"으로 설명합니다. 보상 모델은 다음 세 단계 중 첫 번째 단계입니다.

1. **보상 모델 (본 파일)** - 선호도 투표를 수치 점수로 변환합니다.
2. **PPO 미세 조정** - 언어 모델이 원래의 특성을 유지하면서 더 높은 점수를 받도록 밀어붙입니다.
3. **DPO** - 보상 모델 단계를 완전히 건너뛰는 최신 방식의 지름길입니다.

따라서 보상 모델링은 *인간의 판단*과 *기계의 최적화*를 잇는 다리입니다. 이 다리를 잘못 만들면 이후의 모든 단계가 엉뚱한 방향으로 흘러가게 됩니다.

## 실험실 밖에서의 중요성

동일한 아이디어가 많은 곳에서 쓰이고 있습니다.

- **추천 시스템**은 여러분의 클릭, 건너뛰기, 시청 시간을 통해 여러분이 무엇을 좋아하는지 배웁니다.
- **검색 엔진**은 "어떤 결과를 클릭했는가"를 통해 순위를 학습합니다.
- **음식점**은 손님이 무엇을 좋아했는지에 대한 에세이를 써주길 기다리는 대신, 재주문 횟수를 통해 인기 메뉴를 파악합니다.

**평가(rating)**하는 것보다 **비교(comparing)**하는 것이 더 쉬울 때마다 보상 모델은 가장 적합한 도구가 됩니다.
