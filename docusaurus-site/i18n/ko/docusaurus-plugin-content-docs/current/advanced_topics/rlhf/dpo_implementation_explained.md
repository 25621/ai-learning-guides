# DPO: 판사를 건너뛰고 바로 소스로 가기

## 핵심 아이디어

기존의 RLHF(인간 피드백을 통한 강화학습)는 두 단계로 이루어집니다. 먼저 보상 모델(reward model)을 훈련시킨 다음, PPO를 사용하여 그 점수를 높이도록 학습합니다. DPO(Direct Preference Optimization)는 여기서 영리한 질문을 던집니다.

*만약 보상 모델이 단지 중간 단계일 뿐이라면, 그걸 생략할 수는 없을까?*

결론부터 말하자면, 가능합니다. DPO는 별도의 판사(보상 모델)나 PPO 샘플링 루프, 튜닝할 KL 계수 없이 선호도 쌍(preference pairs) 데이터로부터 직접 언어 모델을 훈련시킵니다. 하나의 우아한 공식을 사용하며, 마치 지도 학습(supervised learning)처럼 작동합니다.

이로 인해 DPO는 실행이 더 간단하고, 안정적이며, 빠릅니다. 이것이 바로 많은 오픈 소스 정렬 모델들이 DPO를 기본 선택지로 채택하게 된 이유입니다.

## 실생활 비유

학생에게 에세이 쓰는 법을 가르친다고 가정해 봅시다.

**PPO 방식**은 이렇습니다: 에세이를 채점할 선생님을 고용합니다. 그런 다음 학생에게 에세이를 계속 쓰게 하고 선생님의 점수에 따라 내용을 수정하게 합니다.

**DPO 방식**은 이렇습니다: 학생에게 한 번에 두 개의 에세이를 보여주며 말합니다. "이게 더 나은 거야. 이것처럼 쓰려고 노력하고, 저런 방식은 피해." 중간에 선생님이 없습니다. 학생은 비교를 통해 직접 내용을 수정합니다.

둘 다 효과가 있을 수 있습니다. 하지만 DPO는 별도의 선생님을 훈련시키고 유지할 필요가 없기 때문에 대개 더 빨리 끝납니다.

## 학습 원리 (직관적 이해)

DPO는 보상 모델링과 동일한 선호도 쌍(프롬프트, 선택된 응답(chosen), 거부된 응답(rejected))을 사용합니다. 각 쌍에 대해 두 가지 질문을 던집니다.

1. 모델이 기준 모델(reference model)보다 선택된 응답을 생성할 **확률이 높아졌는가?**
2. 모델이 기준 모델보다 거부된 응답을 생성할 **확률이 낮아졌는가?**

훈련은 이 두 숫자를 동시에 올바른 방향으로 밀어냅니다. 중요한 점은 비교 과정에서 기준 모델이 항상 존재한다는 것입니다. 이는 PPO에서의 KL 페널티와 같은 역할을 합니다. 모델의 변화를 허용하되, 그 변화는 항상 시작점을 기준으로 상대적이어야 합니다.

DPO 논문의 미묘하고도 아름다운 결과 중 하나는, 이 단일 손실 함수(loss function)가 "보상 모델을 훈련시킨 후 KL 페널티와 함께 PPO를 실행하는 것"과 수학적으로 동일하다는 것을 증명한 점입니다. 목적지는 같지만, 과정은 더 단순해졌습니다.

## 실험 결과

우리는 2,000개의 선호도 쌍 데이터셋을 사용하여 300 에포크(epochs) 동안 정책을 직접 훈련시켰습니다.

![DPO training](outputs/dpo_implementation.png)

- **왼쪽** - 모델이 거부된 응답보다 선택된 응답을 선호하도록 학습함에 따라 DPO 손실이 감소합니다.
- **가운데** - 선호도 정확도(정책이 선택된 응답에 더 높은 암묵적 보상을 부여하는 빈도)가 약 99%까지 상승합니다.
- **오른쪽** - 암묵적 보상의 차이(margin)가 커집니다. DPO는 명시적인 "보상"을 명명하지 않지만, 베타(beta)로 스케일링 된 선택된 응답과 거부된 응답의 로그 확률 차이를 보상으로 해석할 수 있습니다. 이 차이가 꾸준히 벌어지는 것은 모델이 자신의 선호에 대해 더 확신을 갖게 됨을 의미합니다.

PPO와 비교했을 때 이 과정이 얼마나 깔끔한지 주목해 보세요. 샘플링 루프도 없고, 탐험 노이즈도 없으며, 별도의 보상 모델이 실행되지도 않습니다. 각 에포크는 선호도 데이터셋에 대한 순수한 지도 학습 방식의 업데이트입니다.

## RLHF 파이프라인에서의 위치

DPO는 기존 파이프라인의 두 번째 단계들에 대한 **대안**입니다.

- **기존 방식:** 선호도 데이터 → 보상 모델 → PPO → 정렬된 모델.
- **DPO 방식:** 선호도 데이터 → 정렬된 모델. (끝.)

단점은 DPO가 고정된 선호도 데이터셋에서 훈련된다는 점입니다. PPO는 매 라운드 새로운 응답을 샘플링하기 때문에 원칙적으로는 더 넓게 탐험할 수 있습니다. 하지만 실제로는 "정제된 선호도 데이터셋에 맞게 정렬하는" 대부분의 사용 사례에서 DPO가 승리합니다.

## 실험실 밖에서의 중요성

"중간 측정 단계를 건너뛰는" 패턴은 도처에 있습니다.

- 코치가 수영 선수의 자세를 교정할 때, 매 랩마다 스톱워치로 점수를 매기는 대신 옆에서 직접 시범을 보여주며 비교하는 것.
- 사진작가가 "좋은 사진" 채점 기준표를 만드는 대신, 두 장의 사진을 나란히 놓고 더 나은 것을 고르며 편집하는 것.
- 채용 담당자가 30가지 항목의 체크리스트로 점수를 매기는 대신, 두 개의 이력서를 직접 비교하는 것.

**순위만 매기면 될 때는 절대적인 척도가 필요하지 않습니다.** DPO는 바로 그 통찰력을 언어 모델에 적용한 것입니다.

## 한 줄 요약

**DPO는 중간에 보상 모델을 두지 않고 선호도 쌍을 직접 더 나은 모델로 바꿉니다. PPO보다 단순하면서도 종종 그만큼 훌륭한 성능을 냅니다.**
