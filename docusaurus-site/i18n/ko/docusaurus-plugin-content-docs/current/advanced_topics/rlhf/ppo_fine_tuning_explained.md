# PPO 미세 조정: 망가뜨리지 않고 모델을 다듬기

## 핵심 아이디어

응답에 점수를 매기는 보상 모델을 만들었다면, 이제 언어 모델이 더 높은 점수를 받는 응답을 생성하도록 만들고 싶을 것입니다. PPO(Proximal Policy Optimization)는 바로 이 일을 수행합니다. 다만, 모델이 점수만 쫓다가 일반적인 텍스트 작성 능력을 잃어버리지 않도록 '안전벨트'를 채워줍니다.

이를 '다듬기' 단계라고 생각하세요. 모델은 이미 유창하게 말할 줄 압니다. 우리는 그저 모델의 고유한 목소리를 유지하면서, 보상 모델이 좋아하는 방식으로 말하도록 살짝 유도하는 것입니다.

## 실생활 비유

이미 요리를 잘하지만, 특정 음식 평론가의 입맛을 맞추는 법을 배우고 있는 셰프를 상상해 보세요.

평론가가 요리를 먹을 때마다 점수를 줍니다. 셰프는 두 가지 압박을 받습니다.

1. **더 높은 점수 받기.** 평론가가 좋아하는 방식으로 요리합니다.
2. **정체성을 잃지 않기.** 평론가 점수만 따려고 평소 스타일을 완전히 버리고 소금을 컵째로 들이붓는다면, 요리는 이상해질 것입니다. 결국 손님들도 끊기겠죠.

PPO는 이 두 가지 압박을 모두 반영합니다.

- **보상 (reward)** 부분은 모델을 판사가 좋아하는 응답 쪽으로 밀어줍니다.
- **KL 페널티 (KL penalty)** 부분은 모델이 훈련 시작 전의 말투에서 너무 멀어지지 않도록 끌어당깁니다. KL은 "새로운 행동이 이전 행동과 얼마나 다른가"를 측정하는 방법일 뿐입니다.

종합하면 이렇습니다: *더 나아지되, 너 자신을 잃지 마라.*

## 학습 원리 (직관적 이해)

각 훈련 라운드는 다음과 같이 진행됩니다.

1. 몇 가지 프롬프트를 가져와서 현재 모델이 응답을 생성하게 합니다.
2. 보상 모델로 응답의 점수를 매깁니다.
3. 훈련 전의 모델 복사본인 **기준 모델(reference model)**과 비교합니다. 만약 새로운 응답이 너무 많이 달라졌다면, 보상에서 KL 페널티를 뺍니다.
4. 좋은 점수를 받은 응답 쪽으로 모델을 아주 조금씩 이동시킵니다.

PPO의 "Proximal(근접한)"이라는 단어는 *너무 큰 도약을 하지 마라*는 뜻입니다. 각 업데이트는 작고 신중한 단계입니다. 정책 훈련에서 급격한 변화는 모델의 붕괴를 초래하기 때문에, 기존의 일반적인 정책 그래디언트 방식이 매우 불안정했던 것입니다.

## 실험 결과

우리는 새로운 정책과 훈련된 보상 모델로 시작합니다. PPO를 150회 반복하며 응답 배치를 샘플링하고 정책을 업데이트합니다.

![PPO training](outputs/ppo_fine_tuning.png)

- **왼쪽** - 모델이 판사가 좋아하는 응답을 생성하는 법을 배우면서 평균 보상 모델 점수가 꾸준히 상승합니다.
- **가운데** - 기준 모델과의 KL 발산(divergence)이 커집니다. 정책이 시작점으로부터 멀어지고 있음을 의미합니다. 이는 예상된 결과이지만, 제어 없이 커지기만 한다면 모델은 횡설수설하게 될 것입니다.
- **오른쪽** - 보정된 보상(원래 보상에서 KL 페널티를 뺀 값)은 처음에는 원래 보상을 잘 따라가다가, KL이 높아짐에 따라 뒤처지기 시작합니다. 페널티가 제 역할을 하고 있는 것이죠: 모델이 너무 멀리 벗어나는 것에 대해 "비용"을 치르게 하는 것입니다.

실제 RLHF 시스템에서는 점수는 올라가되 모델이 일관성을 유지할 수 있도록 KL 계수를 튜닝합니다. 페널티가 너무 작으면 모델은 이상한 문구를 반복하며 보상을 해킹하려 들고, 너무 크면 모델은 전혀 발전하지 못합니다.

## RLHF 파이프라인에서의 위치

이것은 클래식 RLHF 레시피의 두 번째 단계입니다.

1. 선호도 데이터를 통해 보상 모델을 훈련합니다.
2. **그 보상 모델을 사용하여 PPO로 언어 모델을 미세 조정합니다.**
3. (선택 사항) 더 간단한 경로를 원한다면 2단계를 DPO로 대체할 수 있습니다.

PPO는 OpenAI나 Anthropic과 같은 회사들이 InstructGPT나 초기 ChatGPT를 포함한 첫 번째 정렬 모델들을 만들 때 사용했던 핵심 기술입니다.

## 실험실 밖에서의 중요성

"개선하되, 정체성을 잃지 말 것"이라는 패턴은 어디에나 있습니다.

- 피아니스트가 어려운 구절을 연습할 때, 그 한 부분을 잘 치려고 전체 연주 기법을 바꾸지는 않습니다. 그러면 독주회 전체를 망칠 테니까요.
- 회사가 가입률을 높이려고 웹사이트를 수정할 때도 기존 사용자들이 브랜드를 알아볼 수 있게 유지해야 합니다.
- 공장에서 공정 노브 하나를 조정할 때도 다른 설정값들은 검증된 범위 내에서 유지합니다.

PPO는 이러한 보편적인 아이디어를 수학으로 표현한 정교한 버전입니다.

## 한 줄 요약

**PPO 미세 조정은 모델을 더 높은 보상 쪽으로 밀어붙이는 동시에, KL 페널티를 통해 원래의 행동을 유지하게 합니다. 즉, 자신을 유지하면서 더 나아지는 과정입니다.**
