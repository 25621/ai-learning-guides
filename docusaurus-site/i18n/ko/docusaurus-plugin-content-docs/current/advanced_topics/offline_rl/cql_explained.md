# Conservative Q-Learning (보수적 Q-러닝, CQL) 🛡️

## 무엇인가요?

여러분이 다른 사람들이 과거에 했던 수많은 주식 거래 기록이 담긴 거대한 장부를 보며 투자하는 법을 배우고 있다고 상상해 보세요. 장부에는 매수, 매도, 보유 기록이 있지만, **아무도 실제로 하지 않은 거래에 대한 기록은 없습니다**.

그런데 한 과신하는 학생이 장부를 보더니 이렇게 말합니다.
*"누군가가 매주 월요일마다 복권을 샀다면 어땠을까요? 그건 정말 대단한 거래였을 거예요!"*

문제는 이렇습니다. **장부에는 '월요일에 복권 사기'에 대한 데이터가 전혀 없으므로**, 이 학생은 그저 환상을 보고 있는 것입니다. 하지만 그 환상 속의 거래는 서류상으로는 아주 좋아 보이기 때문에, 학생의 "투자 정책"은 계속해서 그 일을 하고 싶어 합니다.

이 환상 문제는 **분포 변화(distribution shift)**라고 불립니다. 오프라인 학습자는 데이터셋이 테스트해보지 않은 행동을 매우 좋아하게 되는데, 이는 그 낙관주의를 반박할 데이터가 없기 때문입니다. CQL은 이에 대한 해결책입니다.

---

## 오프라인에서 Q-러닝이 잘못되는 방식

일반적인 Q-러닝의 목표값(target)은 다음과 같습니다.

```
target(s, a) = r + γ · max_{a'} Q(s', a')
```

저 `max_{a'}` 부분이 위험합니다. 데이터셋에 상태 `s'`에서의 행동 `a'`에 대한 기록이 전혀 없을 때, 네트워크는 그저 Q-값을 *추측*합니다. 그리고 신경망은 **보지 못한 입력에 대해 Q-값을 과대평가(over-estimate)**하는 경향이 있습니다. 목표값은 이 과대평가된 값을 물려받고, 네트워크는 더 큰 값을 예측하도록 학습하며, 다음 단계에서 우리는 데이터가 뒷받침하는 수준을 훨씬 넘어서는 값을 **외삽(extrapolate)**하여 더 높게 잡습니다. 결국 정책은 환상을 쫓게 됩니다.

데이터를 계속 수집할 수 있다면 이 문제는 스스로 해결될 것입니다 (환상 속의 행동이 실제로 나쁘다는 것을 알게 되니까요). 하지만 **오프라인 RL에서는 데이터를 더 수집할 수 없습니다.** 환상은 영원히 남습니다.

---

## CQL의 비결

CQL (Kumar et al., 2020)은 손실 함수(loss)에 페널티 항을 추가합니다.

```
cql_loss(s)  =  log Σ_a exp Q(s, a)   -   Q(s, a_dataset)
```

두 부분으로 나뉩니다.

1. **`log Σ_a exp Q(s, a)`** (읽기: *"모든 행동에 대한 log-sum-exp"*): 이는 모든 행동에 대한 **소프트 맥심(soft maximum)**입니다. 하나의 승자만 선택하는 `max`와 달리, 모든 행동을 동시에 고려하는 매끄럽고 미분 가능한 근사치입니다. 여기에 페널티를 주면 **모든 행동에 대한 Q-값을 전반적으로** 낮춥니다 (모든 예측값을 균등하게 아래로 누름). 특히 Q-값이 가장 높은 행동들을 강하게 억제하는데, 이곳이 바로 환상이 존재하는 지점입니다.
2. **`- Q(s, a_dataset)`**: 데이터셋에 실제로 기록된 행동에 대해서는 높은 Q-값을 갖도록 보상합니다. 이는 위에서 언급한 전체적인 억제로부터 실제 데이터 분포 내의 Q-값들을 보호합니다.

최종 효과: **보지 못한 행동에 대한 Q-값은 내려가고, 본 행동에 대한 Q-값은 올라갑니다.** 학습된 Q-값은 실제 Q-값의 *하한선(lower bound)*이 됩니다. **`argmax`** 정책(단순히 Q-값이 가장 높은 행동을 선택하는 규칙)은 환상을 쫓는 것을 멈춥니다.

전체 손실 함수:

```
L  =  Bellman_MSE   +   α · cql_loss
```

(**`Bellman_MSE`**는 일반적인 Q-러닝의 표준 오차로, 네트워크의 현재 추측과 미래 추측이 얼마나 일치하지 않는지를 측정합니다).

`α`는 보수성(conservatism)을 조절하는 노브입니다. 너무 작으면 분포 변화 문제가 다시 나타납니다. 너무 크면 에이전트가 너무 보수적이어서 데이터보다 더 나은 성능을 내지 못하게 됩니다.

## 실생활 예시

- **보수적인 체스 코치.** 이미 플레이된 게임들로부터만 배울 수 있습니다. 무모한 코치는 "전례 없는 이 가상의 수야말로 천재적일 수 있어!"라고 말합니다. CQL은 "그에 대한 데이터가 전혀 없으니, 실제 플레이어들이 시도했던 수들을 따르자"고 말하는 코치와 같습니다.
- **식당 메뉴 선택.** Yelp 리뷰는 숨겨진 메뉴(off-menu)를 다루지 않는 경우가 많습니다. 나이브한 정책은 환상적인 별 5개 평점을 바탕으로 숨겨진 메뉴를 추천할 것입니다. CQL은 신뢰할 수 있을 만큼 충분히 주문된 메뉴만 추천합니다.
- **로그 데이터를 통한 로봇의 물체 쥐기.** 로봇에게 컵, 병, 책을 쥐는 영상은 있지만 칼을 쥐는 영상은 없습니다. CQL은 "칼날 부분을 잡으라"고 자신 있게 추천하는 것을 거부합니다.

## 우리 코드의 작동 방식

`cql.py` 스크립트는 다음과 같이 작동합니다.

1. `d4rl_dataset.py`에서 생성된 **네 가지 데이터셋을 로드**합니다.
2. **`medium-replay`**를 훈련 세트로 선택합니다. 이는 가장 현실적(혼합된 품질)이면서 나이브한 방식에 가장 큰 타격을 주는 데이터셋입니다.
3. 동일한 조건에서 `α`값만 다르게 하여 **세 명의 에이전트를 순수하게 오프라인으로 훈련**합니다.
   - `α = 0`   →  나이브 오프라인 DQN (페널티 없음 — 실패하는 베이스라인)
   - `α = 1.0` →  약한 CQL
   - `α = 5.0` →  강한 CQL
4. **2,500번의 그래디언트 단계마다 평가**를 진행합니다. 실제 환경에서 탐욕적으로 10번의 에피소드를 실행합니다. 이것이 환경과의 유일한 접점이며, 훈련 자체는 환경을 전혀 보지 못합니다.
5. 학습 곡선을 `outputs/cql.png`에 **그래프로 그립니다**.

## 예상 결과

일반적인 실행 결과는 다음과 같습니다.

```
최종 평가 리턴 (10회 에피소드 평균, greedy):
  naive offline DQN (alpha=0)         ->  ~30-150  (불안정함, 종종 무너짐)
  CQL (alpha=1.0)                     ->  ~300-450
  CQL (alpha=5.0)                     ->  ~450-500
```

학습 곡선 그래프에서 볼 수 있는 특징:

- **빨간색 곡선** (`α = 0`)은 초기에 상승하다가 분포 변화 환상이 **Bellman target**을 감염시키면서 **낭떠러지로 떨어지는** 경우가 많습니다. (Bellman target은 Q-네트워크를 훈련할 때 "정답"으로 사용하는 숫자입니다: `r + γ · max Q(s', ·)`). 환상 속의 Q-값이 타겟을 오염시키면, 매 단계의 학습이 상황을 악화시킵니다. **Bellman loss** (Q-네트워크의 예측값과 Bellman target 사이의 오차)는 괜찮아 보일 수 있는데, 이것이 이 문제의 **배신감** 느껴지는 부분입니다. 네트워크가 스스로의 잘못된 믿음과 완벽하게 일치하게 학습되므로 손실 함수는 아무런 경고를 주지 않습니다.
- **주황색 곡선** (`α = 1.0`)은 더 천천히 상승하지만 **성능을 유지**합니다.
- **초록색 곡선** (`α = 5.0`)은 가장 안정적이며 대개 가장 좋은 성능을 보입니다.

Bellman-loss 패널은 또 다른 징후를 보여줍니다. 나이브 DQN의 손실은 작게 유지될 수 있지만 정책은 끔찍할 수 있는데, 이는 네트워크가 내부적으로 자신의 환상과 일치하기 때문입니다.

## 이 분야에서의 CQL 위치

CQL은 분포 변화 문제에 대해 원칙적이고 간단한 해결책을 제시했기 때문에 매우 중요한 연구였습니다. 그 계보는 다음과 같습니다.

```
DQN (온라인)
   │
   ▼
나이브 오프라인 DQN  ── 분포 변화 문제로 인해 실패
   │
   ▼
CQL (Kumar 2020)    ── 보수적 페널티 추가: Q는 하한선이 됨
   │
   ▼
IQL (Kostrikov 2021)  ── 애초에 보지 못한 행동에 대해 Q값을 묻지 않음
   │
   ▼
Decision Transformer (Chen 2021)  ── Q값을 완전히 생략하고 RL을 시퀀스 모델링으로 취급함
                                      (LLM이 다음 단어를 예측하듯이, 과거 상태와
                                       원하는 총 리턴이 주어졌을 때 *다음 행동*을 예측)
```

이 계보의 각 단계는 동일한 질문에 대한 서로 다른 대답입니다.
**"어떻게 하면 내 Q-네트워크가 한 번도 본 적 없는 것에 대해 대답하는 상황을 피할 수 있을까?"**

---

## 기억해야 할 주요 용어

| 용어 | 의미 |
|------|---------|
| **Distribution shift (분포 변화)** | 훈련된 정책이 데이터 범위를 벗어난 행동을 원하는 현상 |
| **Out-of-distribution (OOD, 분포 외)** | 데이터셋에 기록되지 않은 (s, a) 쌍 |
| **True Q (실제 Q)** | 상태 s에서 행동 a를 취했을 때 기대되는 실제 미래 리턴 |
| **Conservative Q (보수적 Q)** | 실제 Q값을 과대평가하지 않고 그보다 낮게 유지하려는 학습된 Q-함수 |
| **Logsumexp** | `max`를 매끄럽고 미분 가능하게 근사한 것 |
| **Alpha (α)** | CQL의 보수성 조절 노브 — OOD 행동의 Q값을 얼마나 강하게 누를지 결정 |

---

## 한 줄 요약

> **CQL은 데이터셋이 시도하지 않은 행동에 대해 높은 Q-값을 부여하지 못하도록 하는 "비관주의 페널티"를 추가하여, 정책이 환상에 빠지지 않도록 합니다.**
