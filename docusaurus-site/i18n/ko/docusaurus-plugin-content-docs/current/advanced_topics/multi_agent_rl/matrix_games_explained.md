# 행렬 게임 (Matrix Games): 가장 단순한 다중 에이전트 세상 🎲

## 행렬 게임이란 무엇인가요?

여러분과 친구가 동시에 손동작 하나를 고른다고 상상해 보세요. **가위, 바위, 보**처럼요. 서로 무엇을 낼지 보지 못한 상태에서 결정합니다. 승자는 다음과 같은 작은 표에 의해 결정됩니다.

|        | 바위 | 보 | 가위 |
|--------|:----:|:-----:|:--------:|
| 바위     |  0,0  | -1,+1 | +1,-1 |
| 보      | +1,-1 |  0,0  | -1,+1 |
| 가위     | -1,+1 | +1,-1 |  0,0  |

이 표가 게임의 *세상 전부*입니다. 움직임도, 시간도, 지도도 없습니다. 단 한 번의 결정만 있을 뿐입니다. 보상 행렬(payoff matrix)이 환경의 전부이기 때문에 우리는 이를 **행렬 게임(matrix game)**이라고 부릅니다.

행렬 게임은 **다중 에이전트 강화학습 (Multi-agent RL)**을 연구하기에 가장 깔끔한 장소입니다. 훈련 중에 변할 수 있는 유일한 것은 각 플레이어의 *정책(policy)*, 즉 각 행동을 선택할 확률뿐이기 때문입니다.

---

## 왜 "다중 에이전트"인가요?

단일 에이전트 강화학습에서 환경은 고정되어 있습니다. 바람은 항상 같은 방식으로 불고, 바닥은 결코 움직이지 않습니다. 에이전트는 실력을 쌓고 결국 승리합니다.

행렬 게임에서 여러분의 "환경"은 *학습하는 또 다른 에이전트*입니다. 상대방이 똑똑해짐에 따라 여러분에게 좋은 수가 무엇인지도 *변합니다*. 이를 **비정상성(non-stationarity)**이라고 하며, 다중 에이전트 강화학습의 핵심 문제입니다.

> 여러분이 계속 바위만 낸다면, 상대방은 결국 항상 보를 내기 시작할 것입니다. 그러면 여러분은 가위로 바꿉니다. 그러면 상대방은 바위로 바꿉니다. 그러면 여러분은 보로 바꾸고... 이런 식이 됩니다. "최선의 수"는 결코 한곳에 머물지 않습니다.

고전적인 해결책은 **혼합 전략(mixed strategies)**입니다. 어느 하나의 행동만 고집하지 않고, 상대방이 이용할 수 없도록 무작위로 섞어서 내는 것입니다.

---

## 우리가 플레이하는 세 가지 게임

### 1) 가위바위보 (Zero-sum, 제로섬)
- 한 플레이어의 이득은 곧 다른 플레이어의 손실입니다.
- **내쉬 균형(Nash equilibrium)**은 각 플레이어가 각 행동을 ⅓의 확률로 선택하는 것입니다. 여기서 벗어나면 상대방에게 이용당할 수 있습니다.
- 두 Q-러너 에이전트는 ⅓-⅓-⅓ 근처에서 요동칠 것입니다. 한쪽이 치우칠 때마다 다른 쪽이 반응하기 때문에 결코 완벽하게 멈춰 있지 않습니다.

### 2) 죄수의 딜레마 (General-sum, 일반 게임)
두 용의자가 분리되어 심문을 받습니다.

|           | 협력 | 배반 |
|-----------|:---------:|:------:|
| 협력       |   3, 3    |  0, 5  |
| 배반       |   5, 0    |  1, 1  |

- 상대방이 무엇을 하든 "배반"이 "협력"보다 유리합니다. 이를 **우월 전략(dominant strategy)**이라고 합니다.
- 두 플레이어 모두 합리적이라면 둘 다 배반하게 되고, 둘 다 1점을 얻습니다. (협력, 협력)을 했다면 각각 3점을 얻었을 텐데 말이죠. 이기적인 최적 반응이 집단의 복지를 파괴합니다.
- Q-러닝은 (배반, 배반)으로 깔끔하게 수렴할 것입니다.

### 3) 사슴 사냥 (Coordination, 협력 게임)
두 사냥꾼은 힘을 합쳐 사슴(큰 보상)을 잡거나, 각자 토끼(작지만 안전한 보상)를 잡을 수 있습니다.

|       | 사슴 | 토끼 |
|-------|:----:|:----:|
| 사슴  | 4, 4 | 0, 3 |
| 토끼  | 3, 0 | 2, 2 |

- (사슴, 사슴)은 **보상 우위(payoff-dominant)**입니다 — 둘 모두에게 최선입니다.
- (토끼, 토끼)는 **위험 우위(risk-dominant)**입니다 — 파트너를 믿지 못할 때 안전한 선택입니다.
- 결과는 초기 조건에 따라 달라집니다. 독립적인 Q-러너들은 토끼를 잡는 것이 배우기에 더 안전하기 때문에 종종 더 나쁜 균형점인 (토끼, 토끼)에 머물게 됩니다.

## 실생활 예시

- **과점 시장에서의 가격 결정.** 같은 거리에 있는 두 커피숍이 매일 아침 가격을 결정합니다. 보상 행렬의 모양에 따라 그들은 높은 "협력적" 가격(그들에게는 좋고 고객에게는 나쁨)에 머물거나, 서로 가격을 깎는 치열한 경쟁 가격에 머물게 됩니다.
- **네트워크 프로토콜.** 라우터와 송신자들은 타이밍 전략을 선택합니다. 네트워크의 혼잡 결과는 전송 성공 대 대기라는 행렬 게임 형태의 보상에 의해 결정됩니다.
- **경매 입찰.** 각 입찰자는 다른 사람의 입찰가를 모르는 상태에서 입찰가를 선택합니다. 보상은 전체 입찰 결과에 따라 달라집니다. 내쉬 균형은 하나의 숫자가 아니라 하나의 *입찰 전략*이 됩니다.

## 우리 코드의 작동 방식

각 게임에 대해 우리는 다음과 같이 진행합니다.
1. 두 개의 상태 없는(stateless) Q-러너를 만듭니다 (단판 게임이므로 Q는 각 행동에 대한 하나의 숫자일 뿐입니다).
2. 20,000번의 단계를 반복합니다. 매 단계마다 두 에이전트는 동시에 ε-greedy 행동을 선택하고, 보상을 받고, Q-값을 업데이트합니다.
3. 500단계의 슬라이딩 윈도우를 사용하여 각 에이전트의 **경험적 행동 빈도**를 추적합니다. 추상적인 확률만 보는 대신, 최근에 실제로 어떤 행동을 선택했는지(예: "최근 500회 동안 바위를 40% 냈다")를 집계합니다. 이를 통해 변화하는 전략을 실시간으로 파악할 수 있습니다.
4. 빈도의 변화를 그래프로 그려 `outputs/<game>.png`에 저장하고 최종 Q-값을 출력합니다.

### 예상 결과

| 게임 | 그래프의 예상 결과 |
|------|------------------------------|
| **가위바위보** | 두 플레이어 모두 ⅓-⅓-⅓ 근처에 머물지만 눈에 띄게 흔들립니다. 곡선들이 서로를 쫓아가는 전형적인 순환 행동을 보여줍니다. |
| **죄수의 딜레마** | 두 플레이어의 "배반" 빈도가 빠르게 1.0으로 상승합니다. "협력"은 완전히 사라집니다. |
| **사슴 사냥** | 대부분의 경우 (토끼, 토끼)에 머뭅니다. 운이 좋은 경우 (사슴, 사슴)에 도달하기도 합니다. 스크립트에서 시드(seed)를 바꿔가며 결과가 바뀌는 것을 관찰해 보세요. |

## 독립적 학습의 한계

우리의 에이전트들은 *독립적*입니다. 그들은 자신의 보상만 볼 수 있을 뿐, 상대방의 행동이나 Q-값은 보지 못합니다. 이것은 가장 단순한 베이스라인이지만 다음과 같은 한계가 있습니다.

- 일반 게임(general-sum games)에서 수렴을 보장할 수 없습니다.
- 나쁜 균형점(사슴 사냥의 토끼)에 갇힐 수 있습니다.
- 상대방을 모델링할 수 없습니다.

실제 다중 에이전트 알고리즘들은 상대방에 대해 명시적으로 추론함으로써 이를 해결합니다.

| 알고리즘 | 핵심 아이디어 | 실생활 비유 |
|-----------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------|
| **Fictitious play** | 상대방이 각 행동을 얼마나 자주 선택했는지 기록합니다. 내일도 상대가 평소처럼 행동할 것이라고 가정하고, 그 믿음에 대한 최적의 대응을 선택합니다. | 많은 체스 경기를 통해 상대방의 습관을 파악하고 그에 맞춰 오프닝을 조정하는 것. |
| **CFR (Counterfactual Regret Minimisation)** | 매 라운드 후 "다른 행동을 선택하지 않은 것을 얼마나 후회하는가?"를 자문합니다. 후회되는 행동의 확률을 점차 높입니다. 포커와 같은 **불완전 정보** 게임에서 사용됩니다. | 포커 판이 끝난 후 복기하며 "베팅을 더 했어야 했는데 — 다음엔 그렇게 해야지"라고 생각하는 것. |
| **LOLA (Learning with Opponent-Learning Awareness)** | 상대방도 학습하고 있다는 사실을 고려합니다. 상대방의 다음 업데이트를 예측하면서 자신의 업데이트를 최적화합니다. 상대보다 두 수 앞을 내다보는 방식입니다. | 협상을 할 때 "내가 X를 제안하면 상대는 Y로 대응할 테니, 나는 Z부터 시작해야겠다"라고 생각하는 것. |
| **MADDPG** | 각 에이전트의 *비평가(critic)*는 **전체 상황**을 보며 훈련됩니다. 하지만 실제 행동을 하는 *배우(actor)*는 여전히 자신의 정보만 사용합니다 (중앙 집중식 훈련, 분산식 실행 패턴). | 농구 코치는 코트 전체를 보지만(중앙 비평가), 각 선수에게는 자신이 볼 수 있는 상황에만 반응하도록 가르치는 것(분산 배우). |

하지만 독립적 Q-러닝은 가장 좋은 첫걸음입니다. 비정상성 문제를 직접 마주하게 해주며, 이를 통해 위의 해결책들이 왜 필요한지 이해할 수 있게 해줍니다.

## 기억해야 할 주요 용어

| 용어 | 의미 |
|------|---------|
| **Payoff matrix (보상 행렬)** | 다중 에이전트 게임의 보상을 정의한 표 |
| **Nash equilibrium (내쉬 균형)** | 어떤 에이전트도 혼자서 전략을 바꿔서는 이득을 볼 수 없는 상태 |
| **Mixed strategy (혼합 전략)** | 여러 행동을 확률적으로 섞어서 사용하는 정책 |
| **Non-stationarity (비정상성)** | 학습에 따라 환경(=다른 에이전트)이 계속해서 변하는 성질 |
| **Independent learner (독립적 학습자)** | 다른 학습자의 존재를 무시하고 학습하는 에이전트 |
| **Zero-sum (제로섬)** | 한쪽의 이득이 정확히 다른 쪽의 손실이 되는 게임 |
| **General-sum (일반 게임)** | 둘 다 이기거나 둘 다 지는 등 다양한 결과가 가능한 게임 |
