# Montezuma's Revenge 훈련하기 🏛️🔑

## 왜 이 게임이 유명한가요? (RL 계에서)

2015년, DeepMind의 DQN은 가공되지 않은 픽셀로부터 수십 개의 아타리(Atari) 게임을 인간 이상의 실력으로 플레이하는 법을 배웠습니다. 이는 대대적인 뉴스가 되었죠. 하지만 결과표 구석에는 DQN이 **0점**을 기록한 게임이 하나 있었습니다. 아무것도 하지 않은 것과 다를 바 없는 점수였죠. 바로 **Montezuma's Revenge (몬테주마의 복수)**였습니다.

왜 그랬을까요? 게임의 첫 번째 방에서 요구하는 행동들을 살펴봅시다.

1. 사다리를 타고 *내려갑니다*.
2. 난간을 따라 걷습니다.
3. 굴러오는 해골을 점프해서 피합니다 (타이밍을 놓치면 죽습니다).
4. 다른 사다리를 타고 *올라갑니다*.
5. 열쇠를 줍습니다.

이 과정은 대략 **100번의 정확한 버튼 조작**을 필요로 합니다. 하지만 게임은 열쇠를 손에 쥐기 전까지 **단 1점도 주지 않습니다**. 오프닝 시퀀스 내내 보상 신호는 아무런 특징 없는 **0**일 뿐입니다.

일반적인 RL 에이전트는 실제로 받은 보상에 맞춰 자신을 수정하며 배웁니다. 도달할 수 있는 모든 곳에서 보상이 0이라면 *배울 것이 아무것도 없습니다*. 마치 완전히 평평한 골짜기에서 내리막길 방향을 손으로 더듬어 바닥을 찾으려는 것과 같습니다. 그래서 DQN은 시작 플랫폼 주변에서 영원히 꼼지락거리기만 했습니다. Montezuma는 **어려운 탐험 (hard exploration)**의 상징적인 벤치마크가 되었습니다. 무작위가 아닌 *영리하게* 탐험해야만 깰 수 있는 게임이기 때문입니다.

돌파구는 2018년 **Random Network Distillation (RND)**과 함께 찾아왔습니다. 그 비결은 바로 우리가 다루었던 내용입니다: **내재적 호기심 보너스**를 추가하여 에이전트가 새로운 화면에 도달했을 때 *스스로에게* 보상을 주게 하는 것입니다. 그러면 갑자기 에이전트를 레벨 깊숙이 끌어들이는 조밀한 신호가 생깁니다. RND는 Montezuma에서 인간 이상의 점수를 기록했습니다. (이후 Go-Explore, Agent57 등이 등장했습니다.)

## "Montezuma 스타일"의 희소 보상(Sparse Reward) 실생활 예시

- **비밀번호 자물쇠 / 암호로 된 단서가 있는 보물 찾기.** 부분 점수가 없습니다. 보물을 찾기 전까지는 계속 0점입니다.
- **논문 게재 승인을 받거나, 스타트업이 흑자를 내는 것.** 수개월 동안 외부 보상이 전혀 없다가 (운이 좋으면) 한 번에 큰 보상이 찾아옵니다.
- **비디오 게임 스피드런 경로.** 기술이 성공하거나 실패하기 전까지 피드백 없이 수십 번의 프레임 단위 입력을 완벽하게 수행해야 합니다.
- **방 탈출 카페.** 여러 발견을 하나로 엮기 전까지 방은 여러분에게 거의 아무것도 알려주지 않습니다.

이 모든 상황에서 "그냥 이것저것 해보기"는 희망이 없습니다. **체계적으로** 탐험해야 하며, "오, 이거 새로운데? 계속해 보자"라는 내부 신호가 여러분을 체계적으로 움직이게 만듭니다.

## 왜 여기서 직접 픽셀 단위 Montezuma를 훈련하지 않나요?

*진짜* 게임을 제대로 훈련하려면 다음과 같은 것들이 필요합니다.

- 210×160 RGB 화면을 보는 컨볼루션 네트워크
- 프레임 스태킹 (해골이 어느 방향으로 움직이는지 알기 위해)
- RND 모듈 (두 개의 네트워크 추가: 고정된 무작위 "타겟"과 훈련된 "예측기")
- 그리고 **수천만 번의 환경 프레임** — 많은 GPU 시간

이것은 학습용 스크립트라기보다 연구 프로젝트에 가깝습니다. 그래서 `montezuma_revenge.py`는 대신 두 가지 솔직한 작업을 수행합니다.

### 1. 실제 게임을 *체험*해 봅니다 (`ale-py`가 설치된 경우)

Gymnasium을 통해 `ALE/MontezumaRevenge-v5`를 로드하고, **무작위 에이전트를 2,000단계 동안 실행**한 뒤 총 보상을 보고합니다. 출력되는 숫자는 거의 항상 **0.0**입니다. "희소 보상"이라는 추상적인 문구가 여러분이 직접 실행해 볼 수 있는 구체적인 사실로 변하는 순간입니다. 만약 Atari 패키지가 설치되어 있지 않다면, 설치를 위한 `pip install` 명령어를 출력하고 넘어갑니다.

### 2. 축소 모델인 `MiniMontezumaEnv`에서 테이블 기반 에이전트를 훈련합니다.

이는 Montezuma의 첫 번째 방과 동일한 *뼈대*를 가진 아주 작은 그리드 월드입니다.

```
###############
#S....#.......#
#.....#.......#
#.....#...T...#     S = 시작점
#.....D.......#     K = 열쇠      D = 문 (열쇠가 있어야만 통과 가능)
#..K..#.......#     T = 보물 (보상을 주는 유일한 타일)
###############
```

이기려면 다음 과정을 거쳐야 합니다: **열쇠**까지 걷기 (~6수), 줍기; **문**까지 걷기 (~4수) — 이제 문이 열립니다; 문을 통과해 **보물**에 도달하기 (~5수). 약 **15번의 완벽한 움직임**이 필요하며, **보물에 닿기 전까지 피드백은 0**입니다. `has_key` 플래그는 에이전트 상태의 일부이므로, 열쇠를 잡는 순간 발견해야 할 *새로운* 상태들로 가득 찬 두 번째 방이 열립니다. 이는 실제 게임에서 새로운 화면이 열리는 것과 같습니다.

우리는 기본적인 **테이블 기반 Q-러너**를 두 번 훈련시킵니다.

| 에이전트 | MiniMontezuma 결과 |
|-------|--------------------------|
| **호기심 없음 (epsilon-greedy)** | 1,500회 에피소드 내내 리턴이 **0**에 머뭅니다. 열쇠 근처에도 가지 못합니다. (익숙하죠? 실제 게임에서의 DQN과 같습니다.) |
| **예측 오차 호기심 보너스 포함** | 약 20~25회 에피소드 이내에 보물에 도달하고, **최적의 15수 경로**를 학습합니다. (이것이 Q-테이블에 맞게 축소된 RND의 핵심 아이디어입니다.) |

그래프는 두 학습 곡선을 나란히 보여주며, 호기심 많은 에이전트가 학습한 실제 경로(시작 → 열쇠 → 문 → 보물)를 그리드 위에 그려줍니다. 스크립트는 이 경로를 ASCII 프레임으로도 출력합니다.

## 교훈

> **"희소 보상"은 어떤 이상한 아타리 게임만의 특징이 아닙니다. 성공을 위해 길고 구체적인 행동 시퀀스가 필요한 모든 세상에서의 기본값입니다.** 보상에만 의존하는 에이전트(vanilla DQN)는 말 그대로 시작조차 할 수 없습니다. 따라갈 경사(gradient)가 없기 때문입니다. 호기심 보너스는 스스로 생성하는 "이건 새롭다, 계속 가라"는 조밀한 신호를 만들어내며, 이 신호가 에이전트를 0점의 사막을 건너 첫 번째 실제 보상까지 인도합니다. 그 이후의 모든 것(RND, Go-Explore, Agent57)은 이와 동일한 원리를 신경망 버전으로 확장한 것일 뿐입니다.

## 기억해야 할 주요 용어

| 용어 | 의미 |
|------|---------|
| **Hard exploration (어려운 탐험)** | 영리하게 탐험해야만 성공할 수 있는 문제. 무작위 탐험은 실패함 |
| **Sparse reward (희소 보상)** | 거의 모든 곳에서 보상이 0이며, 긴 올바른 시퀀스 끝에만 보상이 주어짐 |
| **Montezuma's Revenge** | 클래식 Deep-RL 에이전트들이 0점을 기록했던 아타리 게임 — 어려운 탐험의 전형적인 벤치마크 |
| **RND (Random Network Distillation)** | 예측 오차 호기심 보너스를 사용하여 Montezuma를 정복한 2018년의 방법 |
| **Go-Explore** | "유망한 상태를 기억하고, 그곳으로 돌아가서 탐험하라"는 방식의 Montezuma 해결 알고리즘 |
| **Scale model (축소 모델)** | 어려운 문제의 *구조*는 유지하되 빠르게 연구할 수 있도록 만든 작고 저렴한 환경 |

## 한 줄 요약

> **Montezuma's Revenge는 강화학습계에 "받아보지 못한 보상은 아무것도 가르쳐줄 수 없다"는 교훈을 남긴 게임이며, 그 해결책은 에이전트가 실제 보상을 찾을 때까지 탐험에 대해 스스로 보상하게 하는 호기심 보너스입니다.**
