# Dyna-Q: 상상을 통해 더 빨리 배우기 🧠

## 무엇인가요?

새로운 학교에서 길을 찾는 법을 배우는 미아라는 아이를 상상해 보세요. 매일 미아는 복도를 걸어 다니며 새로운 사실을 발견합니다: "도서관은 매점 다음에 있네", "스미스 선생님 교실은 계단 근처 2층에 있어."

**일반적인 Q-러닝** 학생은 *오늘* 실제로 한 일로부터만 배웁니다. 오늘 교실에서 매점까지만 걸어갔다면, 그 한 가지 경로에 대한 기억만 업데이트합니다.

**Dyna-Q** 학생은 다릅니다. 실제로 한 번 걷고 난 후, 잠시 앉아서 자기가 기억하는 과거의 경험들을 **머릿속으로 재현(replay)**해 봅니다. 재현을 할 때마다 마음속의 지도는 더 견고해집니다. 몇 주가 지나면 미아는 학교를 속속들이 알게 됩니다. 더 많이 걸어 다녀서가 아니라, **이미 본 것에 대해 더 많이 생각했기 때문**입니다.

이것이 바로 Dyna-Q가 에이전트를 위해 하는 일입니다. 에이전트는 실제 경험 **그리고** 그동안 구축한 모델로부터 끌어온 상상 속의 경험을 통해 학습합니다.

---

## 세 가지 구성 요소

Dyna-Q는 "Q-러닝 + 모델 + 계획(planning)"의 조합입니다. 한 번의 실제 단계(step)가 **세 가지** 일을 수행합니다.

1. **직접 RL (Direct RL)** — `(s, a, r, s')`로부터 평범한 Q-러닝 업데이트를 수행합니다.
2. **모델 학습 (Model learning)** — 다음과 같이 기록합니다: "상태 *s*에서 행동 *a*를 했을 때 보상 *r*을 받았고 상태 *s'*에 도착했다."
3. **계획 (Planning)** — 모델의 기억에서 *n*개의 무작위 `(s, a)` 쌍을 골라 *n*번의 Q-러닝 업데이트를 추가로 수행합니다. 마치 그 일들이 방금 일어난 것처럼 **가정**하는 것입니다.

이 세 번째 단계가 마법과도 같습니다. `n = 50`이라면, 세상에서의 한 번의 실제 움직임이 Q-테이블에 대한 **51번의 업데이트**를 일으킵니다. 에이전트는 순수 Q-러닝 에이전트보다 실제 움직임 횟수 대비 약 50배 더 빨리 배웁니다.

## 루프의 도식화

```
                   ┌────────────────────────────────────┐
                   │                                    │
   실제 세상   ──► 행동 a 취함 ──► 결과 관찰 (r, s')     │
                            │                           │
              ┌─────────────┼──────────────┐            │
              ▼             ▼              ▼            │
        Q-러닝 업데이트    Model[s,a] ← (r,s')   계획 (Planning) ─┘
                                           (n번의 상상 업데이트)
```

모델은 단순히 `(상태, 행동) → (보상, 다음 상태)`를 저장하는 룩업 테이블(lookup table)입니다. 구축하기 쉽고 조회하기도 쉽습니다.

---

## 실생활 예시

- **체스 공부.** 그랜드마스터들은 자신의 경기나 거장들의 경기를 머릿속으로 재현하는 데 수많은 시간을 보냅니다. 각 재현은 이미 일어난 경험으로부터 더 배우는 "계획(planning)" 과정입니다.
- **악기 스케일 연습.** 까다로운 마디를 한 번 연주한 후, 다음으로 넘어가기 전 머릿속으로 열 번 더 리허설을 합니다. 손가락은 움직이지 않지만 뇌는 업데이트되고 있습니다.
- **자율 주행 자동차.** 빨간불에 멈춰 있는 동안 지난 수백 번의 차선 변경 상황을 시뮬레이션으로 다시 돌려보며, 타이어를 태우지 않고도 정책을 미세 조정합니다.

## 우리 코드의 작동 방식

우리는 클래식한 **Dyna Maze** ([Sutton & Barto, Figure 8.2](http://incompleteideas.net/book/the-book.html)) 환경을 사용합니다. 이는 몇 개의 벽이 있는 6×9 그리드로, 중앙 왼쪽에 시작점 `S`, 오른쪽 상단에 목표점 `G`가 있습니다.

30개의 무작위 시드(seed)에 대해 평균을 낸 세 가지 변형을 실행합니다.

| 설정 | 실제 단계당 계획 단계 수 | 의미 |
|---------|------------------------------|---------|
| `n = 0` | 0 | 일반 Q-러닝 |
| `n = 5` | 5 | 약간의 상상 연습 |
| `n = 50` | 50 | 많은 상상 연습 |

스크립트는 훈련이 진행됨에 따라 **에피소드당 실제 단계 수의 평균**을 기록합니다. 단계 수가 적을수록 에이전트가 목표까지 더 직접적인 경로를 찾아냈다는 뜻입니다.

### 실행 시 예상 결과

이 미로의 최단 경로는 약 9단계입니다. ε-greedy 탐험을 고려하면 잘 훈련된 에이전트는 에피소드당 평균 약 10단계를 기록합니다. 50번의 에피소드를 실행하면 세 설정 모두 이 수치에 도달하지만, 차이점은 **얼마나 빨리** 도달하느냐에 있습니다.

| 설정 | 에피소드당 단계 수 (마지막 10회 평균) | 의미 |
|---------|--------------------------------:|---------------|
| `n = 0`   | ~10 | 수렴함 — 하지만 여기까지 오는데 약 30~50 에피소드의 배회가 필요했음 |
| `n = 5`   | ~10 | 약 10 에피소드 이내에 수렴함 |
| `n = 50`  | ~10 | 약 3~5 에피소드 이내에 수렴함 |

중요한 신호는 최종 숫자가 아니라 *학습 곡선*입니다. `outputs/dyna_q.png`에 저장된 그래프를 보면 세 곡선이 서로 다른 속도로 바닥을 향해 떨어지는 것을 볼 수 있습니다. `n = 50`은 단 몇 번의 에피소드 만에 바닥에 도달하는 반면, `n = 0`은 한참 동안 완만하게 내려갑니다. (이런 작고 결정론적인 미로에서는 일반 Q-러닝도 결국 도달하긴 합니다. 하지만 Dyna-Q는 훨씬 적은 실제 에피소드만으로 도달하며, 이것이 실제 비용이 많이 드는 환경에서 Dyna-Q가 필요한 이유입니다.)

---

## 왜 이 미로에서 효과가 그렇게 좋을까요?

두 가지 이유가 있습니다.

1. **환경이 결정론적(deterministic)입니다.** 각 `(s, a)`는 항상 똑같은 `(r, s')`를 주므로, 한 번만 방문해도 모델이 정확해집니다. 상상 속의 경험이 실제 경험만큼이나 가치가 있습니다.
2. **실제 움직임은 비싸지만, 상상은 공짜입니다.** 상상 업데이트는 그저 테이블을 몇 번 조회하는 것뿐이지만, 실제 단계는 에이전트가 직접 움직여야 합니다. 실제 상호작용에 비용이 많이 드는 경우(실제 로봇, 실제 게임 등), Dyna-Q는 엄청난 샘플 효율성을 보여줍니다.

---

## Dyna-Q가 어려움을 겪는 경우

- **확률적(stochastic) 환경.** 만약 `(s, a)`가 여러 다른 `s'` 상태로 이어질 수 있다면, "마지막 결과만 기억하는" 모델은 거짓말을 하게 됩니다. 해결책: 방문 횟수를 저장하거나 확률 모델을 훈련시킵니다.
- **비정상(non-stationary) 환경.** 세상이 변하는 경우(예: 열려 있던 문이 갑자기 닫히거나 지름길이 나타남), 모델은 낡은 것이 되어 틀린 예측을 하게 됩니다. **Dyna-Q+**는 여기에 *탐험 보너스*를 추가하여 이를 해결합니다. 오랫동안 다시 방문하지 않은 상태에 작은 추가 보상을 주어, 세상이 변했는지 에이전트가 다시 확인하러 가도록 유도합니다.
- **거대한 상태 공간.** `(s, a)`를 키로 하는 딕셔너리는 수백만 개의 상태나 연속적인 상태 공간으로 확장할 수 없습니다. 이것이 바로 **학습된 (신경망 기반) 세계 모델(world models)**이 채워주는 간극입니다. 다음의 `world_model.py`를 참조하세요.

## 기억해야 할 주요 용어

| 용어 | 의미 |
|------|---------|
| **Model**       | `(상태, 행동) → (보상, 다음 상태)`에 대한 기억 |
| **Planning step** | 모델에서 가져온 가상의 데이터로 Q-업데이트를 수행하는 것 |
| **Direct RL**   | 실제 데이터로 Q-업데이트를 수행하는 것 |
| **Sample efficiency** | AI 모델이나 알고리즘이 특정 수준의 성능에 도달하기 위해 훈련 데이터를 얼마나 효과적으로 사용하는지 나타내는 척도 |
| **Dyna**        | 학습과 계획을 교차로 수행하는 Sutton의 아키텍처 |
