# 학습된 모델을 계획에 활용하기 (MPC) 🔮

## 핵심 아이디어

여러분은 미래를 예측하는 **세계 모델(world model)**인 신경망을 가지고 있습니다. 이제 이것을 어떻게 활용할까요?

가장 직접적인 용도는 **계획(planning)**입니다. 매 순간 모델에게 "내가 *이* 계획을 시도하면 어떻게 될까? *저* 계획은? 또 *다른* 계획은?"이라고 묻는 것입니다. 그런 다음 가장 좋아 보이는 계획을 선택하되, **그 계획의 아주 첫 번째 단계만 수행**합니다. 모델이 완벽하지 않기 때문에, 행동 하나만 실행하고 실제 환경에서 새로운 상태를 관찰한 뒤, 다시 처음부터 계획을 세웁니다.

이러한 기법을 **모델 예측 제어 (Model Predictive Control, MPC)**라고 부릅니다.

---

## 실생활 비유

식당에서 메뉴판을 보고 있습니다. 여러분은 그 자리에서 코스 요리 5개를 한꺼번에 주문하지 않습니다. 먼저 첫 번째 요리를 주문해서 먹어보고, 배가 얼마나 부른지 확인한 다음 디저트를 먹을지 다시 결정합니다.

또는, 구불구불한 길에서 운전하고 있다고 해봅시다. 다음 30초 동안의 핸들 조작을 미리 확정해두지 않습니다. 계속해서 앞을 내다보고, 몇 초 뒤까지의 계획을 세우고, 다음 핸들 조작을 수행한 뒤, 다시 계획을 세웁니다.

이렇게 **멀리 계획하고 / 가까운 것만 실행하고 / 다시 계획하는** 루프가 바로 MPC입니다.

---

## "무작위 슈팅 (Random Shooting)"의 작동 원리

다음과 같은 더 정교한 계획 알고리즘들도 있습니다.
- **CEM** (Cross-Entropy Method): 매 라운드 상위 k개의 계획만 남기며 계획 분포를 반복적으로 개선하는 방식.
- **MCTS** (Monte Carlo Tree Search): 시뮬레이션 통계에 따라 탐색 트리를 구축하는 방식으로, AlphaGo와 MuZero에서 사용됨.
- **경사 기반 계획기 (Gradient-based planners)**: 모델의 예측값을 행동에 대해 미분하여 경사를 직접 따라가는 방식.

우리는 가장 단순하면서도 효과적인 **무작위 슈팅(random shooting)** 방식을 사용합니다.

```
현재 상태 s가 주어지면:
    1. 길이가 H=15인 무작위 행동 시퀀스를 N=200개 생성합니다.
    2. 각 시퀀스에 대해 세계 모델을 통해 s에서부터 시뮬레이션하며, 매 단계의 보상을 합산합니다.
       (200개의 상상을 병렬로 수행 — 매우 빠릅니다!)
    3. 예측된 총 보상이 가장 높은 시퀀스를 찾습니다.
    4. 그 시퀀스의 첫 번째 행동만 실제 환경에서 실행합니다.
    5. 실제 다음 상태를 관찰합니다. 나머지 계획은 버립니다.
    6. 1단계로 돌아가서 처음부터 다시 계획을 세웁니다.
```

200개의 계획 × 15단계 = 실제 한 단계마다 3,000번의 상상 속 전환이 일어납니다. 세계 모델은 신경망의 한 번의 배치(batch) 연산으로 이 모든 것을 처리하며, 대개 몇 밀리초밖에 걸리지 않습니다.

---

## 왜 매 단계마다 다시 계획을 세우나요?

모델이 불완전하기 때문입니다. (`world_model.py`의 결과인 `outputs/world_model.png`에서 보았듯이) 오차는 시간이 갈수록 누적됩니다. 0단계에서의 계획은 처음 몇 수 동안만 신뢰할 수 있으며, 15단계쯤 가면 모델은 환상을 보고 있을지도 모릅니다. 그래서 우리는 오직 **첫 번째 움직임**만 신뢰하고, 실제 상태를 확인한 뒤 계획을 갱신하는 것입니다.

이것이 인간이 체스에서 100수 앞을 내다보는 계획을 짜고 그대로 고수하지 않는 이유와 같습니다. 상황은 변하고, 멀리 예측할수록 현실과 맞지 않게 되기 때문입니다.

---

## 한 가지 문제: 보상이 계획기(planner)에게 정보를 주어야 함

CartPole에서 실제 보상은 폴이 쓰러지기 전까지 매 단계 `+1`입니다. 모델은 거의 모든 계획에 대해 충실하게 `+1, +1, +1, ...`을 예측할 것입니다. 무작위 계획이 모델 내부에서 그렇게 빨리 끝나지 않기 때문입니다. 그러면 계획기는 모든 계획의 점수가 같아서 무엇을 선택해야 할지 모르게 됩니다.

해결책: 계획을 세우는 동안 실제 보상을 **매끄러운 대리 보상(proxy reward)**으로 대체합니다.

```python
reward_proxy(state) = 1
                    - |폴_각도| / 0.21          # 폴이 수직인가? (1=예)
                    - 0.1 * |카트_위치| / 2.4  # 카트가 중앙인가? (1=예)
```

이제 폴이 쓰러질 것 같은 계획은 서 있는 계획보다 눈에 띄게 낮은 점수를 받게 됩니다. 계획기가 순위를 매길 수 있게 된 것이죠.

> **실생활 교훈.** "1초 더 버텼다"와 같은 단순한 보상 신호는 단기 계획에 무용지물입니다. 조밀하고 구체적인 신호가 도움이 됩니다.

## 우리 코드의 작동 방식

`model_based_planning.py`:

1. `world_model.py`에서 저장한 세계 모델 가중치를 **로드**합니다. (없다면 즉석에서 훈련합니다.)
2. 실제 CartPole-v1 환경에서 MPC를 **20회 에피소드** 동안 실행합니다.
3. 비교를 위해 완전히 무작위인 정책으로도 **20회 에피소드**를 실행합니다.
4. 두 결과를 나란히 **그래프로 그리고** 평균치를 출력합니다.

### 실행 시 예상 결과

| 정책 | 평균 보상 (생존 단계) |
|--------|-------------------------------:|
| 무작위 (Random) | ~22 (CartPole에서 일반적인 수치 — 폴이 금방 쓰러짐) |
| MPC (우리 방식) | ~150–500 (시드에 따라 다르며, 많은 에피소드가 500에 도달함) |
| 최대 가능 점수 | 500 |

이 **5~25배의 향상**은 정책 네트워크나 가치 함수 없이, 추가적인 훈련도 없이 달성된 것입니다. 오직 세계 모델과 단계당 200번의 상상만으로요.

`outputs/model_based_planning.png` 그래프를 보면 에피소드마다 MPC가 무작위 방식보다 거의 항상 높으며, 많은 에피소드에서 500단계 천장을 치는 것을 볼 수 있습니다.

## 모델 기반 계획(Model-Based Planning)의 장점

- **샘플 효율성.** 모든 학습은 무작위 전환 데이터 한 배치에서 이루어졌습니다. 유용한 정책을 얻기 위해 환경과의 추가적인 상호작용이 필요하지 않았습니다.
- **목표 변경이 쉬움.** 에이전트를 다르게 제어하고 싶나요? 재훈련 없이 대리 보상만 바꾸면 됩니다. (재미 삼아 카트 속도를 최대화하도록 바꿔보세요.)
- **해석 가능성.** 에이전트가 고려한 계획, 예측된 궤적, 그리고 점수들을 직접 확인할 수 있습니다.

## 단점 (그리고 사람들이 하는 해결책)

- **무작위 슈팅은 단순합니다.** 계획을 맹목적으로 샘플링하기 때문입니다. 더 높은 차원에서는 **CEM**이나 **iLQR**(모델을 국지적으로 선형으로 근사하여 해석적으로 푸는 고전적인 최적 제어 방법), 또는 모델을 미분하여 행동을 개선하는 **경사 기반** 계획기를 사용합니다.
- **누적되는 모델 오차.** 장기 예측은 빗나가기 마련입니다. 사람들은 **확률적 앙상블** (동일한 데이터로 여러 모델을 훈련함, PETS 논문 등)을 사용하여 모델들 사이의 의견 불일치를 감지하고, 모델이 불확실해하는 계획에 페널티를 줍니다.
- **결국 우리가 원하는 것은 실제 보상입니다.** 보상 쉐이핑(shaping)이 도움이 되지만, 더 복잡한 과제에서는 모델 *내부에서* 훈련된 **가치 함수 (value function)**를 사용합니다. 즉, 수작업 대리 보상 없이도 어떤 상태에서 기대되는 장기 리턴을 추정하는 학습된 비평가(critic)를 사용하는 것입니다. **Dreamer** (잠재 상상 속에서만 배우-비평가를 훈련함)와 **MuZero** (MCTS를 학습된 가치 네트워크와 결합함) 모두 이 아이디어를 사용합니다.

## 현대 시스템과의 연결

여러분이 방금 실행한 **학습된 역학 + 계획** 레시피는 현대 AI 연구에서 가장 강력한 강화학습 시스템들의 근간입니다.

- **MuZero** (DeepMind): 학습된 세계 모델과 몬테카를로 트리 탐색을 결합합니다. 게임 규칙을 미리 알지 못해도 바둑, 체스, 쇼기, 아타리를 마스터했습니다.
- **Dreamer / DreamerV3** (Hafner et al.): 가공되지 않은 이미지나 상태를 추상적인 표현으로 압축하는 **잠재 공간(latent-space)** 세계 모델 내부에서 정책을 훈련합니다. 100개 이상의 벤치마크에서 최고 수준의 성능을 보여줍니다.
- **PETS / PlaNet / TD-MPC**: 이 아이디어를 로봇 공학 같은 복잡한 연속 제어 작업으로 확장한 알고리즘들입니다.

여러분은 불과 몇 백 줄의 코드로 이 거대한 가족의 막내 모델을 직접 구축한 것입니다.

## 주요 용어

| 용어 | 의미 |
|------|---------------|
| **MPC**           | 모델 예측 제어 — 미리 계획하고, 하나만 실행하고, 다시 계획함 |
| **Random shooting** | 수많은 무작위 계획의 점수를 매겨 최고를 선택함 |
| **Horizon (H)**   | 계획이 내다보는 단계 수 |
| **N samples**     | 단계당 고려하는 후보 계획의 수 |
| **Receding horizon** | 하나의 계획에 집착하지 않고 매 단계 다시 계획을 세우는 방식 |
| **Reward proxy / shaping** | 계획기가 최적화할 수 있는 유용한 신호를 주는 매끄러운 대리 보상 |

## 한 줄 요약

> **세계 모델이 있다면, 계획이란 단지 "백 가지 미래를 상상하고, 가장 좋은 첫 단계를 선택하고, 이를 반복하는 것"입니다.**

이것이 모델 기반 강화학습의 전부라고 해도 과언이 아닙니다.
