# 월드 모델(World Model) 학습: 에이전트에게 꿈꾸는 법 가르치기 🌍

## "월드 모델"이란 무엇인가요?

**월드 모델(world model)**은 에이전트가 가진 *우주의 내부 복사본*입니다. 에이전트에게 상태와 행동을 주면, 다음에 어떤 일이 일어날지 예측합니다:

```
(상태, 행동)  ──►  신경망  ──►  (다음 상태, 보상)
```

이것은 실제 세상이 아닙니다 — 에이전트가 현실을 관찰하고 그것을 흉내 내는 법을 배워서 **스스로 구축한 시뮬레이터**입니다.

일단 학습이 되면, 모델은 에이전트가 실제 행동을 취하지 않고도 "만약에(what-if)"라는 질문을 던질 수 있게 해줍니다:

> *"만약 지금 왼쪽으로 밀고 그다음에 오른쪽으로 두 번 밀면, 나는 어디에 있게 될까? 막대기가 넘어질까?"*

에이전트는 실제 움직임 한 번을 취할 시간에 모델 안에서 수백 개의 계획을 숙고해볼 수 있습니다. 이것이 월드 모델의 핵심입니다.

---

## 실생활 비유

당신이 퍼즐을 맞추는 방식을 생각해 보세요. 모든 조각을 모든 구멍에 일일이 끼워보지는 않을 것입니다. 당신은 조각 A가 여기에 들어갔을 때 어떤 일이 일어날지 **상상**해 봅니다. 그 정신적 시뮬레이션이 틀린 것 같으면, 손가락 하나 까딱하기 전에 그 계획을 거부합니다.

당신의 뇌는 물체가 어떻게 행동하는지 수년 동안 보아오면서 구축된 — 학습된 월드 모델을 가지고 있으며, 이를 통해 실제 행동에 옮기기 전에 결과를 시뮬레이션할 수 있습니다.

다른 예시들:

- **체스 선수**는 몇 수 앞을 내다보는 움직임을 상상합니다.
- **운전자**는 "지금 브레이크를 밟으면 뒷차에 충분한 공간이 생기겠지"라고 생각합니다.
- **블록을 쌓는 아이**: "큰 걸 위에 올리면 탑이 흔들릴 거야." (아이는 이전에 탑을 무너뜨려 보면서 이 모델을 배웠습니다.)

모든 경우에, **정신적 모델 + 상상력 = 더 적은 위험으로 더 나은 결정**을 내리게 해줍니다.

---

## 에이전트는 어떻게 모델을 구축하나요?

에이전트는 그냥 **지켜봅니다**. 구체적으로는:

1. **데이터 수집.** 어떤 정책(무작위라도 상관없음)이든 실제 환경과 일정 기간 상호작용하게 합니다. 모든 전이(transition) 과정을 저장합니다:
   ```
   (상태, 행동, 보상, 다음 상태)
   ```
2. **신경망 학습.** `(상태, 행동)`으로부터 `다음 상태`와 `보상`을 예측하도록 신경망을 학습시킵니다. 이것은 지도 학습(supervised learning)입니다: 저장된 각 전이는 입력이 "에이전트가 보고 한 일"이고 레이블이 "실제로 다음에 일어난 일"인 정답이 있는 예시가 됩니다.
3. **검증.** 데이터의 10%를 떼어두고 모델의 예측이 실제와 얼마나 잘 맞는지 확인합니다. 오차가 적다는 것은 모델이 환경의 **역학(dynamics)**, 즉 행동에 따라 상태가 어떻게 변하는지를 잘 파악했다는 뜻입니다.

우리가 사용하는 트릭: `다음 상태`를 직접 예측하는 대신, **변화량(delta)**인 `다음 상태 − 상태`를 예측합니다. 대부분의 물리 현상은 점진적이며("카트가 아주 조금 움직임"), 작은 타겟값이 신경망이 학습하기에 더 수월하기 때문입니다.

## 실험 설정

| 항목 | 값 | 이유 |
|--------|-------|-----|
| 환경 | `CartPole-v1` | 4차원 상태, 2개의 행동 — 모델링하기 쉬움 |
| 데이터 | 무작위 정책으로부터 얻은 20,000개의 전이 | 상태 공간을 폭넓게 커버함 |
| 네트워크 | MLP, 2 × 128 ReLU 은닉층 | MLP = 다층 퍼셉트론 (표준적인 "바닐라" 신경망). ReLU 활성화 함수를 사용하는 128개 뉴런의 은닉층 2개. 충분한 용량과 빠른 학습 속도. |
| 손실 함수 | `(변화량 상태, 보상)`에 대한 MSE | MSE = 평균 제곱 오차 (예측 오차의 제곱 평균). 표준적인 회귀 손실 함수. |
| 옵티마이저 | Adam, lr = 1e-3, 30 에포크 | Adam = 적응형 옵티마이저 (매개변수별로 학습률을 자동으로 조정). 별도의 튜닝 없이도 잘 작동함. |

전체 학습은 CPU에서 몇 초 안에 끝납니다.

## "좋은 모델"이란 어떤 모습인가요?

두 가지 진단 지표가 중요합니다:

### 1. 단일 단계 정확도 (검증 MSE)

이것은 "모델이 미래의 한 단계를 얼마나 잘 예측하는가?"를 나타냅니다. 30 에포크 후에 검증 MSE가 **1e-4에서 1e-3** 범위에 있다면 아주 훌륭한 것입니다 — 막대기의 각도와 카트의 위치가 소수점 몇 자리까지 정확하다는 뜻입니다.

### 2. k-단계 롤아웃(rollout)에서의 누적 오차

이것이 *진짜* 테스트입니다. 하나의 상태를 취해 모델에 넣고, 그 예측값을 다시 모델에 넣는 과정을 `k`번 반복합니다. 매 단계마다 이전 예측 위에 약간의 노이즈가 더해지기 때문에 오차는 점점 커집니다.

```
1단계:  L2 오차 ≈ 0.01   (거의 완벽함)
5단계:  L2 오차 ≈ 0.05
10단계: L2 오차 ≈ 0.15
20단계: L2 오차 ≈ 0.40   (눈에 띄게 어긋남)
```

*(L2 오차 = 예측된 다음 상태와 실제 상태 사이의 유클리드 거리 — 4차원 상태 공간에서 모델의 추측이 얼마나 멀리 떨어져 있는지를 나타낸다고 생각하면 됩니다.)*

**이것이 중요한 이유.** 모델을 가지고 15단계 앞을 내다보는 계획을 세울 때, 15단계째의 *정확한* 상태는 틀릴 수 있습니다 — 하지만 "좋은 계획과 나쁜 계획"의 상대적인 순위만 유지된다면 계획은 여전히 유효합니다. (이것이 `model_based_planning.py`가 활용하는 점입니다.)

`outputs/world_model.png`의 그래프는 두 가지 진단 결과를 나란히 보여줍니다: 학습 손실 곡선은 로그 스케일에서 예쁘게 내려가고, 롤아웃 오차 곡선은 예쁘게 올라갑니다.

## 왜 *변화량(Delta)*을 예측하나요?

신경망에게 동일한 문제를 제시하는 두 가지 방식을 비교해 보세요:

| 예측 대상 | 전형적인 크기 | 쉬움 혹은 어려움? |
|--------|------------------:|--------------|
| `다음 상태`        | 0–2.4 (카트 위치) | 네트워크가 위치 정보 **와** 미세한 변화를 모두 재현해야 함 |
| `다음 상태 - 상태`| ~0.02            | 네트워크는 오직 미세한 변화만 배우면 됨 |

변화량을 예측한다는 것은 또한 다음과 같은 의미가 있습니다: 만약 네트워크가 (초보 네트워크가 흔히 그렇듯) 0을 출력한다면, 예측값은 단순히 "아무것도 움직이지 않음"이 됩니다 — 이는 단일 타임스텝에 대해 합리적이고 안전한 기본값이 됩니다. 반대로 절대적인 `다음 상태`를 직접 예측하게 되면 초기에는 완전히 무작위인 쓰레기 값을 출력하여 학습 초기 단계가 매우 불안정해질 것입니다.

## 월드 모델이 주는 이점

학습된 월드 모델은 다음과 같은 것들의 기초가 됩니다:

- **계획(Planning)** — 상상된 행동 시퀀스에 대해 탐색함 (`model_based_planning.py` 참조).
- **Dyna 스타일 증강(Augmentation)** — 상상된 데이터로 Q-네트워크를 학습시켜 샘플 효율성을 수배로 높임.
- **호기심 / 탐험** — 모델이 잘 예측하지 못하는 상태를 방문함.
- **Dreamer / World-Models 논문들** — 초기 데이터 수집 외에는 실제 세계와의 상호작용 없이 오직 모델 내부에서만 *정책*을 학습시킴.

## 한계 및 주의사항

- **분포 외 드리프트(Out-of-distribution drift).** 모델은 자신이 본 세상의 일부분만 알고 있습니다. 너무 공격적으로 계획을 세우다 보면 모델이 한 번도 가보지 않은 영역에 도달하게 되고, 그곳의 예측은 순전히 환상에 불과할 수 있습니다.
- **누적 오차.** 차트에서 보듯, 누적되는 오차 때문에 먼 **지평선(horizon)**(미래의 많은 단계)에 대해 계획을 세우는 것은 신뢰하기 어렵습니다. 현대적인 시스템은 **확률적 앙상블**(여러 모델을 학습시키고 서로 동의하는지 확인하는 방식, PETS나 Dreamer 등)을 사용하여 이를 해결합니다. 이를 통해 플래너는 매 단계에서 모델이 얼마나 *불확실한지* 정확히 알고 위험한 미지의 경로를 피할 수 있습니다.
- **Stochastic environments.** A standard deterministic regressor predicts only the *mean* average
  outcome and completely misses the spread of possible outcomes. Complex, real-world environments require probabilistic
  models (like those with Gaussian outputs, or **latent stochastic models** — networks that
  encode the world state as a probability distribution in a compressed space,
  letting them capture genuine randomness rather than averaging it away) to accurately represent uncertainty and randomness.

## 주요 단어

| 용어 | 쉬운 설명 |
|------|---------------|
| **월드 모델** | 환경을 흉내 내는 신경망 |
| **역학(Dynamics)** | 함수 `(s, a) → s'` |
| **보상 모델** | 함수 `(s, a) → r` (보통 같이 포함됨) |
| **단일 단계 예측** | 실제 상태에서 모델이 출력하는 값 |
| **롤아웃(Rollout)** | 출력을 다시 입력으로 넣으며 반복하는 단일 단계 예측 |
| **누적 오차** | 롤아웃이 진행됨에 따라 커지는 작은 오차들 |
