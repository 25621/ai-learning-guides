# 처음부터 배우는 Deep Q-Network (DQN) 🧠

## 선형 방식(Linear)의 한계

이전에 사용했던 선형 공식을 기억하시나요?

> 점수 = w₁ × 카트_위치 + w₂ × 카트_속도 + w₃ × 폴_각도 + w₄ × 폴_속도

이 공식은 CartPole에서는 어느 정도 작동하지만, 수천 개의 픽셀이 보이는 비디오 게임에서는 어떨까요? 그런 복잡한 상황에 대해 이런 단순한 공식을 쓸 수는 없습니다!

우리는 복잡한 상황을 보고 최선의 행동을 찾아낼 수 있는 무언가가 필요합니다. 그것이 바로 **신경망 (Neural Network)**입니다.

---

## 신경망이란 무엇인가요?

여러분의 뇌를 생각해 보세요. 뉴런이라고 불리는 수백만 개의 작은 세포들이 서로 대화합니다. 뜨거운 것을 만지면 뉴런들이 신호를 보냅니다. "뜨거워! → 당장 손을 떼!" 각 뉴런은 정보를 전달하고, 이들이 모여 스마트한 결정을 내립니다.

**컴퓨터의 신경망**도 같은 방식으로 작동합니다.

```
입력 레이어        은닉 레이어 1      은닉 레이어 2      출력 레이어
[카트 위치]  →    [128개 뉴런]  →  [128개 뉴런]  →  [왼쪽 밀기 점수]
[카트 속도]  →    [  ...       ]    [  ...       ]    [오른쪽 밀기 점수]
[폴 각도]    →
[폴 속도]    →
```

각 화살표에는 **가중치 (weight)**가 있습니다 (연결이 얼마나 강한지를 나타냅니다). 수천 개의 가중치가 존재하며, 네트워크는 이 모든 것을 스스로 학습합니다!

**실생활 예시:** 식당의 셰프가 음식 맛을 보고 수백 가지 재료를 한꺼번에 조절하는 것과 같습니다. 각 미뢰(taste bud)는 뉴런과 같아서, 이들이 모여 셰프에게 "소금을 더 넣어"라거나 "후추를 줄여"라고 알려줍니다. 네트워크를 훈련하는 것은 셰프가 수천 번의 요리를 통해 배우는 과정과 비슷합니다.

---

## DQN = Deep Q-Network

**DQN** (Deep Q-Network)은 2013년 DeepMind에 의해 발명되었습니다. 그들은 기존의 Q-러닝 공식에서 Q-테이블을 신경망으로 교체했습니다!

기존 방식:
> Q-테이블[상태][행동] = 점수

DQN 방식:
> Q-네트워크(상태) → [왼쪽_점수, 오른쪽_점수]

네트워크는 상태를 입력으로 받아 모든 행동에 대한 Q-값을 한꺼번에 출력합니다. 이는 각 행동을 따로 계산하는 것보다 훨씬 효율적입니다!

---

## 이 스크립트: "나이브(Naive)" 버전

이 스크립트는 특별한 기교가 없는 **가장 기본적인** DQN을 보여줍니다. 과정은 단순합니다.
1. 상태를 확인합니다.
2. 네트워크에 "왼쪽이 좋아, 오른쪽이 좋아?"라고 묻습니다.
3. 더 높은 점수의 행동을 수행합니다.
4. 보상을 받고 네트워크를 업데이트합니다.

**이 방식은 의도적으로 불안정하게 설계되었습니다!** 마치 무언가 새로운 것을 배울 때마다 이전 교훈을 즉시 잊어버리는 학생과 같습니다. 매 단계마다 네트워크를 업데이트하기 때문에 혼란이 발생합니다.

**실생활 예시:** 한 입 먹을 때마다 레시피 전체를 바꾸면서 요리를 배운다고 상상해 보세요. "너무 짜네"라고 해서 소금을 다 뺐다가, 다시 "싱겁네"라며 소금을 왕창 넣는 과정을 반복하게 되어, 적절한 양을 절대 찾지 못할 수도 있습니다. 이것이 바로 여기서 일어나는 현상입니다!

## ε-Greedy (입실론 그리디) 트릭 🎲

로봇은 항상 최선의 행동만을 고르지 않습니다. 가끔은 무작위로 선택합니다!

왜 그럴까요? 항상 최선이라고 생각하는 것만 고수하면, 더 나은 옵션을 절대 발견할 수 없을지도 모르기 때문입니다.

> ε (입실론)의 확률로: 무작위 행동을 선택 (탐험, Explore!)
> 1-ε의 확률로: 알려진 최선의 행동을 선택 (활용, Exploit!)

우리는 ε = 1.0 (100% 무작위)에서 시작하여 서서히 ε = 0.01 (1% 무작위)까지 줄여나갑니다. 이렇게 하면 로봇은 처음에는 많이 탐험하고, 나중에는 배운 것에 집중하게 됩니다.

**실생활 예시:** 새로운 도시에 갔을 때, 처음에는 무작위로 식당에 가봅니다 (탐험). 시간이 지나면 가장 좋았던 곳들 위주로 갑니다 (활용). 하지만 여전히 가끔은 숨겨진 맛집이 있을지 모르니 새로운 곳을 시도해 봅니다!

## 무엇이 빠졌나요? (그리고 왜 중요한가요?)

이 나이브 DQN에는 두 가지 큰 문제가 있습니다.

1. **상관관계가 있는 업데이트**: 모든 경험이 순서대로 들어옵니다 (1단계, 2단계, 3단계...). 만약 5단계가 나빴다면, 그 주변의 모든 업데이트가 함께 혼란에 빠집니다.
   
2. **움직이는 타겟**: 매 업데이트 후 네트워크가 변합니다. 그런데 다음 업데이트는 타겟(정답)을 계산하기 위해 바로 그 변해버린 네트워크를 사용합니다. 마치 움직이는 과녁을 맞히려는 것과 같습니다!

이 문제들은 다음 스크립트들에서 다룰 **경험 재현 (Experience Replay)**과 **타겟 네트워크 (Target Network)**를 통해 해결됩니다. 이 기능들이 합쳐지면 비로소 DQN은 비틀거리는 초보자에서 챔피언 게임 플레이어로 거듭나게 됩니다!
