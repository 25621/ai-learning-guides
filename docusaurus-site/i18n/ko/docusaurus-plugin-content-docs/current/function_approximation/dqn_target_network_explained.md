# 타겟 네트워크 (Target Network): 과녁 안정시키기 🎯

## 움직이는 골대 문제

활과 화살을 가지고 과녁을 맞히려고 노력한다고 상상해 보세요. 화살을 쏘고, 화살이 떨어진 곳을 확인하고, 다음 번에는 조준을 수정합니다. 간단하죠?

그런데 여러분이 화살을 쏠 때마다 과녁이 움직인다고 상상해 보세요! 여러분이 쏜 화살 하나하나가 다음 샷을 위한 과녁의 위치를 조금씩 바꿔버립니다. 그렇다면 여러분은 결코 실력이 늘지 않을 것입니다. 항상 도망가는 타겟을 쫓아다니기만 할 테니까요.

이것이 바로 타겟 네트워크가 없는 DQN의 문제입니다!

---

## 왜 Q-타겟이 계속 움직일까요?

DQN에서 각 업데이트의 타겟(정답)은 다음과 같습니다.
> target = reward + γ × max(Q(next_state))

여기서 **γ (감마)**는 **할인 인자(discount factor)**입니다. 0과 1 사이의 숫자(대개 0.99)로, 에이전트가 *미래*의 보상을 *즉각적인* 보상에 비해 얼마나 중요하게 여기는지를 조절합니다.

**실생활 예시:** 누군가 여러분에게 지금 쿠키 하나를 줄지, 아니면 내일 쿠키 두 개를 줄지 제안한다고 해봅시다. 당장 쿠키를 먹고 싶다면 여러분의 γ는 낮습니다 (미래를 크게 할인함). 만약 인내심이 있고 기다리는 것이 즐겁다면 여러분의 γ는 높습니다 (미래의 보상이 지금만큼 중요함). 강화학습에서 γ = 0.99는 "다음 단계의 보상은 지금 받는 보상의 99% 가치가 있다"는 뜻입니다.

그런데 위 식의 오른쪽에 있는 Q-값은... 바로 우리가 훈련시키고 있는 그 네트워크에서 나옵니다!

따라서 Q-값을 더 좋게 만들기 위해 네트워크를 업데이트할 때마다 타겟도 함께 변해버립니다. 다음과 같은 피드백 루프가 형성됩니다.

1. 네트워크 업데이트 → Q-값 변화
2. Q-값 변화 → 타겟 변화
3. 타겟 변화 → 네트워크를 이전과 다르게 업데이트
4. 영원히 반복 — 불안정함!

**실생활 예시:** 몸무게를 재려고 체중계에 올라갈 때마다 측정 기준이 바뀌는 체중계와 같습니다. 여러분의 진짜 몸무게를 절대 알 수 없겠죠!

## 해결책: 과녁을 얼려버리세요! ❄️

**타겟 네트워크 (Target Network)**는 메인 Q-네트워크를 그대로 복사해서 제자리에 고정해 둔 것입니다.

- **Online 네트워크** (`qnet`): 매 훈련 단계마다 업데이트됨 — 빠르게 학습함
- **Target 네트워크** (`target_net`): 고정된 복사본 — 100단계마다 한 번씩만 업데이트됨

우리는 타겟(정답)을 계산할 때 이 고정된 타겟 네트워크를 사용합니다.
> target = reward + γ × max(Q_TARGET(next_state))

이제 타겟은 100단계 동안 제자리에 멈춰 있습니다! 덕분에 Online 네트워크는 조준할 수 있는 안정적인 목표를 가질 수 있습니다. 100단계가 지나면 Online 네트워크의 가중치를 Target 네트워크로 복사하고, 다시 고정하는 과정을 반복합니다.

**실생활 예시:** 학생과 선생님을 생각해 보세요. 선생님은 숙제(타겟)를 내줍니다. 학생은 그것을 배우고 실력을 쌓습니다. 100번의 수업 후에 선생님은 숙제를 더 어렵게 업데이트합니다. 선생님이 매 분마다 기준을 바꾸지는 않습니다. 그렇게 하면 너무 혼란스러울 테니까요!

## 완전한 DQN 레시피 🍕

경험 재현(Experience Replay)과 타겟 네트워크가 모두 포함된 완전한 DQN 알고리즘은 다음과 같습니다.

```
1. Online 네트워크 Q와 Target 네트워크 Q_target을 초기화함 (가중치 동일)
2. 리플레이 버퍼(기억 상자)를 생성함

매 단계마다:
  a. Q 네트워크를 사용해 ε-greedy 방식으로 행동을 선택함
  b. (상태, 행동, 보상, 다음 상태)를 버퍼에 저장함

매 4단계마다:
  c. 버퍼에서 무작위 미니 배치를 샘플링함
  d. 고정된 Q_TARGET을 사용하여 타겟(정답)을 계산함
  e. 손실(loss)을 최소화하도록 Q를 업데이트함

매 100단계마다:
  f. Q의 가중치를 Q_TARGET으로 복사함 (타겟 동기화)
```

이것이 바로 2015년 DeepMind의 DQN 논문에 실린 정확한 알고리즘입니다!

## 결과 비교

`dqn_target_network.py`를 실행하면 다음과 같은 결과를 볼 수 있습니다.

**타겟 네트워크가 없는 경우 (DQN + 리플레이만 사용):**
- 훈련이 어느 정도 되는 것 같다가도 주기적으로 성능이 무너집니다.
- Q-값이 발산(폭발하거나 요동침)할 수 있습니다.
- 학습 결과를 예측하기 어렵습니다.

**완전한 DQN (리플레이 + 타겟 네트워크):**
- 훨씬 더 일관되게 성능이 향상됩니다.
- Q-값이 합리적인 범위 내에서 유지됩니다.
- 목표치(CartPole의 경우 195점 이상)에 더 빠르게 도달합니다.

## "치명적인 삼중주 (Deadly Triad)" ☠️

강화학습에서 다음 세 가지 요소가 결합하면 불안정성이 발생합니다.

1. **함수 근사 (Function approximation)**: 테이블 대신 신경망을 사용함 ← 우리는 이것을 사용함
2. **부트스트래핑 (Bootstrapping)**: Q-값을 추정하기 위해 다시 Q-값을 사용함 ← 우리는 이것을 사용함
3. **오프-폴리시 학습 (Off-policy learning)**: 실제 취한 행동이 아닌 최대값을 사용함 ← 우리는 이것을 사용함

이 세 가지가 모이면 "치명적인 삼중주"가 됩니다. DQN은 이를 다음과 같이 다스립니다.
- 경험 재현 → 상관관계 제거
- 타겟 네트워크 → 피드백 루프 차단

문제를 완전히 해결하는 것은 아니지만, 다룰 수 있는 수준으로 만들어 줍니다!

## 주요 용어

| 용어 | 의미 |
|------|---------|
| **Target Network** | 타겟(정답) 계산에만 사용되는 고정된 Q-네트워크 복사본 |
| **Online Network** | 현재 활발하게 훈련되고 있는 Q-네트워크 |
| **Sync (동기화)** | Online 네트워크의 가중치를 Target 네트워크로 복사하는 것 |
| **Feedback Loop** | 시스템의 출력이 다시 입력에 영향을 주어 변화를 일으키는 과정 (불안정의 원인이 됨) |
| **Deadly Triad** | 함수 근사 + 부트스트래핑 + 오프-폴리시 학습이 결합되어 불안정을 초래하는 현상 |
