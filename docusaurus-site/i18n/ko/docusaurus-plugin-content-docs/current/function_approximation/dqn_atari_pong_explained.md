# Atari Pong에서의 DQN 🏓

## Atari Pong이란 무엇인가요?

퐁(Pong)은 역사상 가장 오래된 비디오 게임 중 하나로, 디지털 탁구와 같습니다! 두 개의 패들이 공을 주고받으며 튕깁니다. 상대방이 공을 놓치면 점수를 얻습니다. 누군가 21점에 먼저 도달하면 게임이 끝납니다.

우리의 버전에서 AI는 한쪽 패들을 제어하고, 상대방(컴퓨터)은 반대쪽을 제어합니다. 게임은 항상 최악의 점수인 -21점에서 시작합니다. 훌륭한 에이전트는 0점이나 +21점에 도달합니다.

---

## 왜 에이전트에게 Pong이 어려울까요?

CartPole에서 로봇은 숫자들을 직접 볼 수 있었습니다 (폴의 각도, 카트의 속도 등). 하지만 Pong에서 에이전트가 보는 것은 오직 **가공되지 않은 픽셀(raw pixels)**뿐입니다. 화면 위의 수천 개의 작은 색 점들이죠!

```
CartPole 입력: [0.02, −0.14, 0.01, −0.23]   ← 4개의 숫자, 쉬움!
Pong 입력:     [픽셀 그리드: 210×160×3]        ← 100,800개의 숫자, 훨씬 어려움!
```

로봇은 픽셀로부터 다음을 파악해야 합니다.
- 내 패들은 어디에 있는가?
- 공은 어디에 있는가?
- 공이 왼쪽으로 움직이는가, 오른쪽으로 움직이는가?
- 얼마나 빠르게 움직이는가?

인간은 이를 자동으로 해냅니다 (우리는 놀라운 시각 능력을 갖추고 있으니까요!). 하지만 AI에게 이것은 엄청난 도전입니다.

---

## 움직임 포착하기: 프레임 스태킹 (Frame Stacking) 🎬

단 한 장의 프레임(스크린샷)으로는 공이 어느 방향으로 움직이는지 알 수 없습니다. 움직임을 이해하려면 여러 장의 프레임을 동시에 봐야 합니다. 마치 플립북(flip book)이 여러 페이지를 빠르게 넘겨야 작동하는 것과 같습니다.

**프레임 스태킹 (Frame Stacking):** 마지막 4개의 프레임을 네트워크에 동시에 입력합니다.

```
프레임 1: 공의 위치 40
프레임 2: 공의 위치 43    → 이 4개의 프레임을 쌓음(stack) → 네트워크가 움직임을 감지!
프레임 3: 공의 위치 46
프레임 4: 공의 위치 49
```

이제 네트워크는 "공이 오른쪽으로 3의 속도로 움직이고 있군"이라고 추론할 수 있습니다.

**실생활 예시:** 영화를 보는 것과 한 장면의 사진을 보는 것의 차이입니다. 카 레이싱의 사진 한 장은 그저 흐릿한 이미지일 뿐이지만, 4개의 프레임을 이어서 보면 어떤 차가 더 빠른지 알 수 있습니다!

---

## CNN으로 세상 보기 🔍

픽셀 입력을 처리하기 위해 우리는 **컨볼루션 신경망 (Convolutional Neural Network, CNN)**이라는 특수한 신경망을 사용합니다. CNN은 모든 픽셀을 한 번에 보는 대신, 이미지를 훑는 슬라이딩 윈도우(sliding windows)를 사용하여 패턴을 찾아냅니다. 마치 눈이 이미지를 스캔하는 것과 비슷합니다.

```
가공되지 않은 픽셀 (84×84×4 프레임)
       ↓
Conv Layer 1 (8×8 필터, 스트라이드 4) → 에지(edges)와 모양 탐색
       ↓
Conv Layer 2 (4×4 필터, 스트라이드 2) → 객체(패들, 공) 탐색
       ↓
Conv Layer 3 (3×3 필터, 스트라이드 1) → 객체 간의 관계 탐색
       ↓
Flatten → 512개의 뉴런 → Q-값 (행동당 하나씩)
```

**실생활 예시:** 군중 속에서 친구를 찾을 때, 뇌는 먼저 사람의 형체를 인식하고, 그다음 머리색 같은 특징을, 마지막으로 얼굴 같은 세부 정보를 인식합니다. CNN도 같은 방식으로 작동합니다 — 단순한 패턴에서 복잡한 패턴 순으로요!

---

## 전처리: 세상 줄이기

Pong 프레임은 210×160 픽셀의 컬러 이미지입니다. 이는 너무 큽니다! 우리는 각 프레임을 전처리합니다.

1. **흑백 전환 (Grayscale)** — Pong에서 색상은 중요하지 않습니다 (어차피 공은 항상 흰색입니다).
2. **84×84로 크기 조절** — 크기가 작을수록 훈련이 빨라지지만, 여전히 사물을 식별하기엔 충분합니다.
3. **[0,1]로 정규화** — 픽셀 값을 255로 나누어 작은 숫자로 만듭니다.

**실생활 예시:** 복사기를 사용할 때 50% 축소 복사를 하는 것과 같습니다. 중요한 세부 사항(공, 패들)은 여전히 보이지만 크기만 작아집니다. 복사기는 색상 정보도 신경 쓰지 않죠!

---

## 보상 클리핑 (Reward Clipping): 모든 게임을 공평하게 대하기 ✂️

Pong에서는 득점 시 +1, 실점 시 -1을 얻습니다. 하지만 다른 아타리 게임에서는 점수가 수천 점에 달하기도 합니다!

우리는 **보상을 [-1, +1] 범위로 클리핑(clipping)**하여 네트워크가 보상의 규모에 신경 쓰지 않도록 만듭니다. 이렇게 하면 동일한 코드로 보상 규모를 튜닝할 필요 없이 어떤 아타리 게임이든 훈련할 수 있습니다.

---

## 훈련은 얼마나 걸리나요?

| 훈련 기간 | 에이전트가 배우는 것 |
|---|---|
| 10만 단계 | 대부분 무작위 행동, 거의 반응하지 않음 |
| 100만 단계 | 가끔 공을 향해 움직이기 시작함 |
| 500만 단계 | 공을 몇 번 받아침 |
| 1,000만 단계 | 경쟁력 있는 플레이를 보여주며, 가끔 승리함 |
| 2,000만 단계 이상 | 종종 컴퓨터 상대에게 승리함 |

우리의 데모는 **30만 단계** 동안 실행됩니다. 이는 훈련 아키텍처가 작동하는 것을 확인하고 초기 학습 과정을 관찰하기엔 충분하지만, 게임을 완벽히 마스터하기에는 부족한 시간입니다.

**실생활 예시:** 피아노를 배우는 데는 수개월이 걸립니다. 10분 동안의 연습은 여러분이 올바르게 연습하고 있다는 것을 보여주지만, 당장 콘서트를 열 수는 없는 것과 같습니다!

---

## 우리 코드의 결과

Pong에서 30만 단계 훈련 후:
- 에이전트는 약 -20점 근처에서 시작합니다 (공을 거의 받아치지 못함).
- 끝날 때쯤에는 대개 -15점에서 -10점 정도로 향상됩니다.
- 학습 곡선은 무작위 플레이에서 점진적으로 개선되는 모습을 보여줍니다.

진짜 경쟁력 있는 성능을 보려면 GPU와 함께 약 1,000만 단계 이상의 실행이 필요합니다. 구현 자체는 완벽하고 정확하며, 단지 시간이 더 필요할 뿐입니다!

## 주요 용어

| 용어 | 의미 |
|------|---------|
| **CNN** | 컨볼루션 신경망 — 이미지 입력을 처리하는 데 최적화된 신경망 |
| **Frame Stacking** | 움직임을 포착하기 위해 연속된 여러 프레임을 입력으로 사용하는 것 |
| **Preprocessing** | 신경망에 넣기 전 프레임을 변형(흑백화, 크기 조절, 정규화)하는 과정 |
| **Reward Clipping** | 서로 다른 게임들에 범용적으로 적용되도록 보상을 [-1, +1]로 제한하는 것 |
| **ALE** | Arcade Learning Environment — 아타리 게임을 실행하는 라이브러리 |

## 역사적인 성과

2015년 DeepMind가 DQN을 발표했을 때 전 세계는 경악했습니다. **단 하나의 알고리즘**과 **동일한 아키텍처**를 사용하여, 가공되지 않은 픽셀과 점수 데이터만으로 49개의 서로 다른 아타리 게임을 학습했기 때문입니다. 그중 많은 게임에서 인간 수준을 뛰어넘는 실력을 보여주었습니다.

DQN 이전에는 각 게임에 맞는 전략을 사람이 직접 코딩해야 한다고 생각했습니다. 하지만 DQN은 범용 학습 에이전트가 스스로 모든 것을 알아낼 수 있음을 증명했습니다. AI 역사에 남을 기념비적인 순간이었습니다!
