# 경험 재현 (Experience Replay): 로봇의 기억력 기르기 🎒

## 문제점: 망각 (그리고 혼란)

나이브(Naive) DQN이 왜 그렇게 불안정했는지 기억하시나요? 가장 큰 이유는 **상관관계가 있는 학습 (correlated learning)** 때문입니다.

로봇이 게임을 할 때, 로봇은 일어나는 일을 순서대로 경험합니다.
> 1단계 → 2단계 → 3단계 → 4단계 → ...

이 단계들은 서로 연결되어 있습니다! 만약 로봇이 10단계에서 왼쪽으로 기울어졌다면, 11단계에서도 여전히 왼쪽으로 기울어져 있을 것입니다. 이들은 독립적이지 않고 서로 의존적입니다.

이렇게 서로 관련 있는 단계들을 사용하여 네트워크를 업데이트하는 것은, 역사 공부를 할 때 똑같은 페이지만 반복해서 읽는 것과 같습니다. 한 페이지만 아주 잘 알게 되고 나머지 모든 내용은 잊어버리게 될 것입니다!

**실생활 예시:** 어제 숙제만 계속 연습하면서 시험 공부를 한다고 상상해 보세요. 그 숙제 문제들은 기가 막히게 잘 풀겠지만, 시험에는 다른 문제들이 나옵니다! 다양한 문제들을 **섞어서** 연습해야 합니다.

---

## 해결책: 기억 상자 📦

**경험 재현 (Experience Replay)**은 로봇에게 커다란 기억 상자(**리플레이 버퍼, replay buffer**)를 만들어 줍니다.

가장 최근의 경험에서 바로 배우는 대신, 로봇은 다음과 같이 행동합니다.
1. 모든 경험을 기억 상자에 **저장**합니다: (상태, 행동, 보상, 다음 상태)
2. 상자에서 한 움큼의 기억을 **무작위로 꺼냅니다**.
3. 단지 마지막 단계가 아니라, 그 **무작위로 섞인 기억들로부터 학습**합니다.

```
게임 1단계 → [상자에 저장]
게임 2단계 → [상자에 저장]
게임 3단계 → [상자에 저장]
...
게임 50단계 → [상자에 저장] → 무작위로 64개의 기억을 꺼냄 → 네트워크 업데이트
게임 51단계 → [상자에 저장] → 무작위로 64개의 기억을 꺼냄 → 네트워크 업데이트
```

**실생활 예시:** 사진 앨범을 생각해 보세요. 오늘 찍은 사진만 보면서 인생을 배우지는 않습니다. 예전 사진들도 훑어보면서 좋은 기억과 힘들었던 순간들을 섞어서 되새깁니다. 이를 통해 오늘 하루뿐만 아니라 인생 전체의 패턴을 이해할 수 있게 됩니다.

---

## 무작위 샘플링이 도움이 되는 이유

기억을 무작위로 꺼내면 상관관계를 깨뜨릴 수 있습니다. 로봇은 다음과 같은 다양한 기억들로부터 학습하게 됩니다.
- 폴(pole)이 아주 똑바르게 서 있던 기억 (500단계 전)
- 폴이 막 쓰러지려던 기억 (20단계 전)
- 운 좋게 잘 버텼던 기억 (3단계 전)

이렇게 무작위로 섞으면 다음과 같은 장점이 있습니다.
✅ 로봇이 다양한 상황으로부터 배울 수 있습니다.
✅ 각 기억을 여러 번 "재현(replay)"할 수 있습니다 (경험을 효율적으로 사용).
✅ 네트워크가 최근 사건에만 과하게 집중(overfit)하지 않습니다.

---

## 미니 배치 (Mini-Batch) 학습

한 번에 하나의 경험으로 업데이트하는 대신, **한 번에 64개의 경험**을 묶어서 업데이트합니다 (이를 "미니 배치"라고 부릅니다). 이는 마치 다음과 같습니다.
- 옛날 방식: 플래시 카드 한 장을 읽고 스스로 퀴즈를 냄
- 새로운 방식: 64장의 서로 다른 플래시 카드를 읽고, 그 섞인 내용으로 퀴즈를 냄

미니 배치를 사용하면 학습 신호가 훨씬 더 안정적이고 노이즈가 줄어듭니다.

---

## 워밍업 기간 (Warmup Period)

게임을 시작하자마자 바로 학습을 시작하지는 않습니다! 먼저 리플레이 버퍼에 기억이 어느 정도 쌓여야 합니다. 상자에 최소 **500개의 경험**이 쌓일 때까지 기다린 후에 비로소 훈련을 시작합니다.

**실생활 예시:** 요리 재료를 다 모으기도 전에 요리를 시작하지는 않을 것입니다. 워밍업 기간은 요리를 하기 전 장을 보는 시간과 같습니다!

## 결과 비교

`dqn_experience_replay.py`를 실행하면 두 가지 학습 곡선을 볼 수 있습니다.

| 나이브 DQN | DQN + 리플레이 |
|-----------|-------------|
| 매우 울퉁불퉁함 | 더 매끄러움 |
| 자주 성능이 무너짐 (모든 것을 잊어버림) | 더 일관되게 성능이 향상됨 |
| 편차가 큼 | 편차가 작음 |

리플레이 버전은 대개 다음과 같은 특징을 보입니다.
- 좋은 점수에 더 안정적으로 도달합니다.
- 점수가 500점에서 갑자기 30점으로 뚝 떨어지는 일이 적습니다.
- 더 안정적인 학습 과정을 보여줍니다.

## 코드로 보는 리플레이 버퍼

```
ReplayBuffer:
  - 용량(capacity): 10,000개의 기억 (가득 차면 가장 오래된 것부터 잊어버림)
  - push(상태, 행동, 보상, 다음 상태, 종료 여부)
  - sample(배치 크기=64) → 무작위 배치 추출
```

이는 10,000줄짜리 공책과 같다고 생각하면 됩니다. 공책이 가득 차면 가장 오래된 줄을 지우고 최신 내용을 적습니다. 그리고 공부할 때는 항상 무작위로 페이지를 펼쳐서 공부하는 것이죠!

---

## 주요 용어

| 용어 | 의미 |
|------|---------|
| **Experience Replay** | 과거의 경험을 저장해두고 훈련을 위해 무작위로 다시 사용하는 기법 |
| **Replay Buffer** | 과거의 (상태, 행동, 보상, 다음 상태) 튜플들을 저장하는 기억 상자 |
| **Correlated updates** | 훈련 데이터가 서로 의존적일 때 (학습에 좋지 않음!) |
| **Mini-batch** | 한 번의 업데이트 단계에 사용되는 무작위로 추출된 작은 기억 꾸러미 |
| **Decorrelation** | 연속된 경험들 사이의 연결 고리를 끊는 것 |

## 아직 무엇이 부족한가요?

리플레이 버퍼가 있더라도 여전히 문제가 하나 남아 있습니다. 바로 **움직이는 타겟 (moving target)** 문제입니다.

네트워크를 업데이트할 때마다 Q-값이 변합니다. 그런데 업데이트된 그 Q-값들이 다음 업데이트의 타겟(정답)을 계산하는 데에도 사용됩니다. 서로가 서로를 바꾸는 혼란의 소용돌이가 발생하는 것이죠!

이 문제는 **타겟 네트워크 (Target Network)**를 통해 해결됩니다. 이는 100단계마다 한 번씩만 업데이트되는 고정된 네트워크 복사본입니다. 이를 통해 "과녁(bullseye)"을 잠시 멈춰두어 로봇이 안정적으로 조준할 수 있게 해줍니다!
