# Double DQN: 과신 문제 해결하기 🤔

## 문제: DQN은 자신이 실제보다 더 훌륭하다고 생각합니다

여러분이 이런 질문을 받았다고 상상해 보세요. "이 동네에서 가장 맛있는 식당이 어디인가요?"

여러분은 "피자 팰리스가 최고예요. 분명 10점 만점에 10점일 거예요!"라고 대답할지도 모릅니다. 하지만 여러분은 그곳에 딱 두 번 가봤을 뿐입니다. 그곳이 *정말로* 10점 만점인지는 사실 모릅니다. 그저 운 좋게 방문했을 때 피자가 맛있어서 실제보다 과대평가했을 수도 있습니다.

이와 똑같은 문제가 DQN에서도 발생합니다. 에이전트가 **Q-값을 과대평가(overestimate)**하는 것입니다.

---

## DQN은 왜 과대평가를 하나요?

DQN이 타겟(target)을 계산할 때 다음과 같은 식을 사용합니다.
> target = reward + γ × **max** Q(next_state)

여기서 `max`가 문제입니다! 노이즈가 섞인 여러 추정값 중에서 최대값을 선택할 때, 우리는 거의 항상 무작위 오차가 가장 큰(위쪽으로 치우친) 값을 선택하게 됩니다.

**실생활 예시:** 친구 5명에게 건물의 높이를 맞춰보라고 합니다. 친구들의 추측값은 40m, 38m, 45m (운 좋게 높게 부름!), 39m, 41m였습니다. 실제 높이는 40m입니다. 만약 여러분이 `max(추측값)`인 45m를 선택한다면, 실제보다 훨씬 높게 측정하는 셈입니다. 노이즈가 섞인 추측값들 중 최대값은 거의 항상 과대평가된 값입니다.

수천 번의 업데이트를 거치면서 DQN은 이렇게 부풀려진 타겟을 향해 계속 학습하게 되고, 상황이 실제보다 더 좋다고 믿게 됩니다. 이는 학습 속도를 늦추거나 에이전트가 근거 없는 자신감으로 나쁜 결정을 내리게 만듭니다.

---

## Double DQN의 해결책

**Double DQN** (Hasselt et al., 2016)은 `max` 계산을 두 단계로 나눕니다.

**1단계 — 어떤 행동을 할까?** **Online 네트워크**를 사용하여 가장 좋은 행동을 선택합니다.
> best_action = argmax Q_online(next_state)

**2단계 — 그 가치는 얼마일까?** **Target 네트워크**를 사용하여 그 행동의 가치를 평가합니다.
> target = reward + γ × Q_target(next_state, best_action)

```
일반 DQN (Vanilla DQN):   target = r + γ × max_a Q_target(s', a)
                                 ↑ 동일한 네트워크가 선택하고 평가함 → 편향 발생

Double DQN:    best_a = argmax_a Q_online(s', a)   ← Online이 선택
               target = r + γ × Q_target(s', best_a) ← Target이 평가
                                 ↑ 서로 다른 네트워크를 사용 → 편향 감소
```

**실생활 예시:** 취업 면접에서 구직자가 자신의 수행 능력을 직접 채점하게 하지 않습니다 (이는 일반 DQN의 문제입니다!). 대신 지원자는 자신의 가장 뛰어난 결과물을 *제출*하고, *별도의* 면접관이 이를 평가합니다. 두 명의 서로 다른 사람이 관여하므로 더 공정한 평가가 가능해집니다!

---

## 왜 분리하는 것이 도움이 되나요?

두 네트워크(Online과 Target)는 Target이 덜 자주 업데이트되기 때문에 서로 다른 가중치를 가지고 있습니다. 즉, 어떤 행동이 최선인지에 대해 서로 다른 "의견"을 가질 수 있습니다.

두 네트워크의 의견이 일치하지 않을 때:
- Online: "행동 A가 정말 좋아 보여요!"
- Target: "사실 행동 A는 괜찮은 정도예요. 10점이 아니라 7점 정도의 가치죠."

Online 네트워크가 선택한 행동에 대해 Target 네트워크의 가치 추정치를 사용함으로써, 우리는 더 정직하고 덜 부풀려진 수치를 얻을 수 있습니다.

---

## 코드 차이점: 단 한 줄!

일반 DQN에서 Double DQN으로의 유일한 코드 변경은 타겟 계산 부분에 있습니다.

```python
# 일반 DQN (Vanilla DQN):
q_next = target_net(s_next).max(dim=1).values

# Double DQN:
best_actions = online_net(s_next).argmax(dim=1, keepdim=True)   # Online으로 선택
q_next = target_net(s_next).gather(1, best_actions)              # Target으로 평가
```

단 두 줄만 바뀌었을 뿐이지만, 안정성과 정확성에 미치는 영향은 상당합니다!

## 결과 비교

`double_dqn_cartpole.py`를 실행하면 두 개의 그래프를 볼 수 있습니다.

**그래프 1: 학습 곡선 (Learning Curves)**
- 일반 DQN과 Double DQN 모두 CartPole 문제를 해결할 것입니다.
- Double DQN이 종종 더 빠르고 안정적으로 수렴합니다.
- CartPole은 단순한 환경이라 차이가 미미할 수 있지만, 아타리(Atari) 게임에서는 그 차이가 매우 극적으로 나타납니다.

**그래프 2: Q-값 추정치 (Q-value Estimates)**
- 일반 DQN: Q-값이 시간이 지남에 따라 위로 치솟는 경향이 있습니다 (과대평가).
- Double DQN: Q-값이 더 적절하고 정확한 수준을 유지합니다.

Q-값 과대평가 그래프가 핵심 통찰력을 제공합니다. 일반 DQN이 부풀려진 가치를 학습하고 이것이 결국 성능 저하로 이어지는 과정을 보여줍니다.

## Double DQN은 얼마나 더 좋은가요?

| 지표 | 일반 DQN (Vanilla) | Double DQN |
|--------|------------|------------|
| Q-값 정확도 | 과대평가함 | 더 정확함 |
| 학습 안정성 | 변동이 큼 | 변동이 작음 |
| CartPole 성능 | 좋음 | 약간 더 좋음 |
| Atari 성능 (50개 게임) | 기준점 | 인간 수준에 도달한 게임 수 2.6배 증가 |

복잡한 아타리 게임에서 Double DQN은 CartPole에서보다 훨씬 더 큰 차이를 만들어냈습니다 (아타리 게임의 Q-값 추정치에 노이즈가 훨씬 더 많기 때문입니다).

## DQN 개선의 계보

Double DQN은 일반 DQN을 개선한 여러 방법 중 하나일 뿐입니다. "Rainbow" 논문 (2017)은 6가지 개선 사항을 하나로 합쳤습니다.

1. **Double DQN** (과대평가 해결) ← 이 스크립트의 내용입니다!
2. **Prioritized Replay** (놀라운 경험으로부터 더 많이 배우기)
3. **Dueling Networks** ("이 상태가 얼마나 좋은가?"와 "어떤 행동이 최선인가?"를 분리)
4. **Multi-step returns** (더 먼 미래를 내다보기)
5. **Distributional RL** (평균 리턴뿐만 아니라 전체 리턴 분포를 학습)
6. **NoisyNets** ([ε-greedy](../foundations/multi_armed_bandit_explained.md#the-epsilon-greedy-strategy) 대신 학습된 탐험 방식 사용)

Rainbow는 이 모든 것을 결합하여 당시 아타리 게임에서 최고의 성능을 기록했습니다!

## 주요 용어

| 용어 | 의미 |
|------|---------|
| **Overestimation (과대평가)** | Q-값이 실제 가치보다 높게 측정되는 현상 (지나치게 낙관적임) |
| **Double DQN** | 행동 선택에는 Online 네트워크를, 가치 평가에는 Target 네트워크를 사용하는 방식 |
| **Decoupling (분리)** | 동일한 네트워크가 수행하던 두 가지 작업을 나누는 것 |
| **Bias (편향)** | 한 방향으로 치우친 체계적인 오류 (항상 너무 높거나 낮은 경우) |
| **Rainbow** | 최대 성능을 위해 6가지 DQN 개선 사항을 결합한 변형 알고리즘 |
