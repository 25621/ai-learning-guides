# A2C: Actor와 Critic이 함께 일하는 방식

## 핵심 아이디어

REINFORCE 알고리즘은 게임이 완전히 끝날 때까지 기다린 후에 업데이트를 진행합니다. 이는 마치 축구 경기 내내 침묵하며 지켜보다가 경기가 다 끝난 후에야 피드백을 주는 코치와 같습니다.

**A2C (Advantage Actor-Critic)**는 게임 진행 중에 피드백을 줍니다. 몇 단계마다 코치가 멈춰서 "그 패스는 아주 좋았어!", "그 태클은 별로였어!"라고 말해주는 것과 같습니다.

이 방식은 훨씬 빠르고 효율적입니다.

---

## 두 명의 등장인물

> **LunarLander란 무엇인가요?** 이 문서 전체에서 우리는 **LunarLander** 환경을 사용합니다. 이는 물리 시뮬레이션으로, 여러분은 작은 우주선을 제어하여 3개의 엔진(왼쪽, 메인, 오른쪽)을 사용해 달의 목표 지점에 부드럽게 착륙시켜야 합니다. 이는 Gymnasium 라이브러리에서 제공되는 강화학습의 표준 벤치마크입니다.

### Actor (배우) 🎭
**Actor**는 정책(Policy)입니다. 어떤 행동을 취할지 결정합니다.

> "지금 이런 상태인데, 왼쪽 엔진을 켤까 아니면 오른쪽 엔진을 켤까?"

**실생활 예시:** 핸들을 돌리고 페달을 밟는 자동차의 *운전자*.

### Critic (비평가) 🎬
**Critic**은 현재 상황이 얼마나 좋은지 추정합니다. 즉, 가치 V(s)를 평가합니다.

> "이 상태에 있는 것은 앞으로 얻을 총 보상 관점에서 약 +150점 정도의 가치가 있어."

**실생활 예시:** 운전자 옆에 앉아 "좋은 길로 가고 있네요. 30분 뒤면 도착하겠어요." 또는 "교통 체증이 시작되네요. 좀 늦겠어요."라고 말하는 *내비게이터*.

### 두 캐릭터는 뇌를 공유합니다
우리의 구현에서는 두 캐릭터 모두 **동일한 신경망 백본(backbone)**을 사용합니다.

```
          상태 (LunarLander의 경우 8개의 숫자)
                       ↓
          ┌─────────────────────────┐
          │  공유 레이어 (Shared Layers) │
          │  [256 neurons] → ReLU   │
          │  [256 neurons] → ReLU   │
          └────────┬────────┬───────┘
                   ↓        ↓
          Actor Head    Critic Head
          [4 outputs]   [1 output]
          (행동 확률)    (V(s))
```

- **ReLU** (Rectified Linear Unit): 각 레이어 뒤에 적용되는 활성화 함수입니다. `max(0, x)`를 출력하여 양수 값은 유지하고 음수는 0으로 만듭니다. 이를 통해 네트워크가 비선형 패턴을 학습할 수 있게 합니다.
- **행동 확률 (action probs)**: 4가지 행동 각각을 취할 확률입니다. Actor는 매 단계 이 분포에서 샘플링하여 행동을 선택합니다.

**실생활 예시:** 운전(actor)도 하고 경로가 좋은지도 아는(critic) 택시 기사처럼, 하나의 뇌로 두 가지 일을 처리합니다. 뇌를 공유하면 더 빨리 배울 수 있습니다!

## Advantage: 예상보다 좋았는가?

Baseline을 사용하는 REINFORCE와 마찬가지로, A2C는 **Advantage**를 계산합니다.

> A(s, a) = "실제 결과" − "우리가 예상한 결과"

하지만 여기서 "실제 결과"는 Critic의 **n-step bootstrap**에서 옵니다. (**bootstrapping**이란 에피소드가 끝날 때까지 기다리는 대신, Critic 자신의 예측값 V(s)를 사용하여 미래 단계의 가치를 근사하는 것을 의미합니다. 마치 기말고사 점수를 현재 성적을 바탕으로 학기 중간에 추정하는 것과 같습니다.)

```
실제 TD return: r_t + γ · r_{t+1} + γ² · r_{t+2} + ... + γⁿ · V(s_{t+n})
Advantage A_t = TD return - V(s_t)
```

**실생활 예시:** 이번 경기에서 3골을 넣을 것으로 예상했는데(V(s)), 실제로 5골을 넣었다면 advantage는 +2입니다. 만약 1골만 넣었다면 advantage는 -2입니다.

Positive advantage → "그 행동이 예상보다 더 도움이 되었어 → 더 자주 해!"
Negative advantage → "그 행동이 예상보다 덜 도움이 되었어 → 덜 해!"

## 왜 여러 개의 병렬 환경을 사용하나요?

우리의 A2C 구현은 8개의 LunarLander 복사본을 동시에 실행합니다!

**왜 그럴까요?** 단일 환경에서의 경험은 서로 **상관관계(correlated)**가 높기 때문입니다. 즉, 한 단계는 이전 단계와 매우 밀접하게 연결되어 있습니다. 이러한 상관관계는 신경망이 특정 패턴이 실제보다 더 신뢰할만하다고 오해하게 만들 수 있습니다.

8개의 환경을 사용하면, 각 단계마다 매우 다른 상황에서 얻은 8개의 독립적인 경험을 얻을 수 있습니다. 이는 상관관계를 깨뜨리고 훈련을 획기적으로 안정화합니다.

**실생활 예시:** 날씨에 대해 배우려면 다음 중 무엇이 더 좋을까요?
- 한 도시를 8시간 동안 연속으로 관찰하기 (상관관계 높음 — 오후 2시에 맑았다면 3시에도 맑을 확률이 높음)
- 8개의 도시를 동시에 관찰하기 (상관관계 낮음 — 서로 다른 날씨 패턴, 더 많은 정보!)

```
환경 1: [달에 착륙, 왼쪽 엔진, 충돌, 리셋...]
환경 2: [너무 빨리 낙하, 양쪽 엔진, 호버링, 착륙...]
환경 3: [오른쪽으로 기울어짐, 오른쪽 엔진, 안정화, 착륙...]
...
환경 8: [왼쪽으로 표류, 왼쪽 엔진, 안정 유지, ...]
```

8개 환경 모두 동시에 네트워크를 업데이트합니다. 업데이트당 8배 더 다양한 경험을 얻는 셈입니다!

## N-Step 업데이트: 게임이 끝날 때까지 기다리지 마세요

REINFORCE는 전체 에피소드가 끝날 때까지 기다립니다 (최대 1000단계가 될 수 있음).

A2C는 매 **n_steps = 128 단계**마다 업데이트를 진행합니다.

```
8개 환경에서 128단계를 플레이합니다
    → 128 × 8 = 1024개의 경험 튜플(tuple)을 얻습니다
    → Advantage와 return을 계산합니다
    → Actor와 Critic을 업데이트합니다
    → 다시 128단계를 플레이합니다...
```

**실생활 예시:** 시험 공부를 하는 학생
- REINFORCE 스타일: 교과서 전체를 다 읽은 후에 연습 문제를 풉니다.
- A2C 스타일: 10페이지를 읽고 퀴즈를 풀고, 다시 10페이지를 읽고 퀴즈를 풉니다.

더 잦은 피드백은 더 빠른 학습으로 이어집니다!

## 결합된 세 가지 손실 (Losses)

A2C는 세 가지 손실 항을 동시에 사용하여 훈련합니다.

**손실(Loss)**은 옵티마이저가 최소화하려고 노력하는 숫자입니다. 손실이 작을수록 네트워크의 현재 행동이 훈련 목표에 더 가깝다는 것을 의미합니다.

### 1. Actor Loss (Policy Gradient)
이득이 되는(advantageous) 행동의 확률을 높입니다.
```
L_actor = -E[log π(a|s) · A(s,a)]
```
만약 A > 0이면, 해당 행동의 확률을 높입니다.
만약 A < 0이면, 해당 행동의 확률을 낮춥니다.

### 2. Critic Loss (Value Function MSE)
가치 예측을 더 정확하게 만듭니다. (**MSE** = Mean Squared Error: 예측 오차를 제곱하여 평균을 낸 것입니다. 제곱을 하면 작은 실수보다 큰 실수에 더 큰 페널티를 부여하게 됩니다.)
```
L_critic = E[(V(s) - return)²]
```
어떤 **회귀(regression)** 모델을 훈련하는 것과 같습니다. (회귀 = 연속적인 숫자를 예측하는 것, 여기서는 예상되는 리턴인 V(s)를 의미함) — 예측 오차를 최소화합니다.

### 3. Entropy Bonus (Exploration)
정책이 너무 빨리 확신에 빠지는 것을 방지합니다.
```
L_entropy = -H[π(·|s)] = E[log π(a|s)]
```
높은 엔트로피 (High entropy) = 다양한 행동 선택 = 탐험(Exploration)
낮은 엔트로피 (Low entropy) = 확신에 찬 좁은 선택 = 활용(Exploitation)

**실생활 예시:** 엔트로피 보너스는 선생님이 "객관식 문제에서 그냥 A만 찍지 마! 무엇이 정답인지 알기 위해 다양한 답을 시도해 봐."라고 말하는 것과 같습니다.

```
Total loss = L_actor + 0.5 × L_critic - 0.01 × entropy
```

## LunarLander: 더 어려운 도전

**LunarLander-v3**는 Gymnasium (구 OpenAI Gym) 환경입니다. "v3"는 이 환경의 세 번째 개정판임을 나타내는 버전 번호입니다. 에이전트는 달의 지정된 패드에 안전하게 착륙해야 하는 작은 우주선을 제어합니다. 이는 CartPole보다 훨씬 어렵습니다.
- 8차원 상태 공간 (위치, 속도, 각도, 다리 접촉 여부, 연료)
- 4가지 이산적 행동 (아무것도 안 함, 왼쪽 엔진, 메인 엔진, 오른쪽 엔진)
- 보상: 착륙 시 +100, 충돌 시 -100, 소량의 연료 페널티

훈련 곡선은 매우 낮은 음수의 보상에서 양수의 보상으로 점진적으로 개선되는 모습을 보여줍니다. LunarLander에서 A2C가 기본적인 안정성을 배우기 위해서는 상당한 양의 경험이 필요합니다.

## 주요 방정식

```
n-step return:  G_t = r_t + γ·r_{t+1} + ... + γⁿ·V(s_{t+n})
Advantage:      A_t = G_t - V(s_t)
Actor 업데이트:   θ_π ← θ_π - α ∇ L_actor
Critic 업데이트:  θ_V ← θ_V - α ∇ L_critic
```

## 핵심 요약

| 개념 | 설명 |
|---------|---------------|
| **Actor** | 정책(Policy) — 무엇을 할지 결정 |
| **Critic** | 가치 함수 — 상황이 얼마나 좋은지 판단 |
| **Advantage** | "예상보다 좋았는가?" (실제 - 예상) |
| **N-step return** | V(s)로 부트스트래핑하기 전 n단계 미래를 내다봄 |
| **병렬 환경** | 상관관계가 제거된 다양한 경험을 위한 여러 환경들 |
| **Entropy bonus** | Actor가 새로운 것을 계속 시도하도록 장려 |

## 다음 단계는?

A2C는 훌륭하지만 한 가지 큰 약점이 있습니다. 때때로 정책을 너무 공격적으로 업데이트한다는 점입니다. 단 한 번의 잘못된 업데이트가 이전의 모든 좋은 학습 내용을 망가뜨릴 수 있습니다.

**PPO (Proximal Policy Optimization)**는 단일 업데이트가 정책을 너무 많이 변경하는 것을 방지하는 영리한 "안전 클립(safety clip)" 기능을 통해 이 문제를 해결합니다.

PPO 구현에 대해서는 `ppo_scratch.py`를 참조하세요!
