# PPO 하이퍼파라미터 민감도: 무엇이 가장 중요할까?

## 하이퍼파라미터가 중요한 이유

초콜릿 케이크를 굽는다고 상상해 보세요. 레시피는 다음과 같습니다.
- 계란 2개
- 밀가루 200g
- 베이킹파우더 1 티스푼
- 180°C에서 35분간 굽기

만약 계란을 10개 넣으면 케이크는 터져버릴 것입니다. 베이킹파우더를 0.1 티스푼만 넣으면 부풀어 오르지 않겠죠. 300°C에서 10분만 구우면 겉은 타고 속은 익지 않을 것입니다.

**PPO의 하이퍼파라미터는 요리 재료 및 오븐 설정과 같습니다.** 올바른 조합은 훌륭한 결과를 만들어내지만, 잘못된 설정은 학습 자체를 불가능하게 만들 수 있습니다.

이 스크립트는 한 번에 **단 하나**의 하이퍼파라미터만 바꾸고, 각 설정을 3개의 서로 다른 무작위 시드(seed)로 실행하여 결과를 체계적으로 비교합니다.

## 세 가지 실험

### 실험 1: Clip Epsilon (ε)

```
ε = 0.05   (매우 보수적 — 아주 작은 정책 변화만 허용)
ε = 0.2    (표준 — 안전성과 속도의 균형)
ε = 0.4    (공격적 — 큰 정책 변화를 허용)
```

**ε는 무엇을 조절하나요?**

ε는 이전 정책 주변의 "안전 창(safety window)"의 크기입니다.
```
비율(ratio)은 반드시 [1 - ε, 1 + ε] 범위 내에 있어야 함
ε=0.05: 비율이 [0.95, 1.05] 사이  ← 미세한 변화
ε=0.2:  비율이 [0.80, 1.20] 사이  ← 표준
ε=0.4:  비율이 [0.60, 1.40] 사이  ← 큰 변화
```

**실생활 예시:** ε를 "자동차 핸들을 한 번에 얼마나 꺾을 수 있는지"라고 생각해보세요.
- ε=0.05: 빙판길 운전 — 아주 조금씩만 조작함
- ε=0.2:  일반적인 운전 — 적당한 회전
- ε=0.4:  레이싱 드라이버 — 공격적인 핸들링, 차가 **스핀**할 위험이 있음 (변화가 너무 급격해서 통제력을 잃음)

**예상 결과:**
- ε=0.05: 느리지만 안정적인 학습 (지나치게 신중함)
- ε=0.2:  좋은 균형 (**"골디락스(Goldilocks)" 값** — 너무 작지도 크지도 않은 딱 적당한 상태. 너무 뜨겁지도 차갑지도 않은 죽을 선택한 동화 주인공의 이름에서 유래함)
- ε=0.4:   빠르게 배울 수도 있지만 **오버슈트(overshoot)와 진동(oscillation)**이 발생할 수 있음 (오버슈트 = 최적 지점을 지나쳐버림, 진동 = 정착하지 못하고 주변을 왔다 갔다 함)

### 실험 2: 학습률 (Learning Rate)

```
lr = 1e-4  (느리지만 안정적)
lr = 3e-4  (표준)
lr = 1e-3  (빠르지만 위험함)
```

**학습률은 무엇을 조절하나요?**

학습률은 언덕을 오를 때의 "보폭"과 같습니다 (한 걸음 = 신경망 가중치의 한 번의 업데이트).
- 너무 작으면: 정상에 도달하는 데 영원히 걸립니다 (수렴이 느림).
- 너무 크면: 정상을 지나쳐 반대편으로 떨어집니다 (**발산, diverge** — 학습 보상이 꾸준히 오르지 않고 무너지거나 격렬하게 요동침).
- 적당하면: 정상을 향해 꾸준히 나아갑니다.

**실생활 예시:** 기타 줄 튜닝하기.
- lr=1e-4: 튜닝 **페그**(줄을 조이는 손잡이)를 아주 미세하게 돌림 — 정확하지만 시간이 오래 걸림
- lr=3e-4: 일반적인 튜닝 — 몇 번 돌려서 맞는 음을 찾음
- lr=1e-3: 페그를 확 잡아당김 — 줄이 **끊어질** 수도 있음 (업데이트가 너무 커서 훈련이 회복 불가능하게 망가짐)!

**예상 결과:**
- lr=1e-4: 결국 좋아지긴 하지만 매우 느림
- lr=3e-4: 전반적으로 가장 좋은 성능
- lr=1e-3: 초기에는 빠르게 진행되다가 불안정해짐

---

### 실험 3: 업데이트 에포크 (K)

```
K = 3   (보수적 — 각 배치당 적은 횟수의 학습)
K = 10  (표준)
K = 20  (공격적 — 각 배치당 많은 횟수의 학습)
```

**업데이트 에포크는 무엇을 조절하나요?**

**롤아웃**(rollout, 일정 시간 동안 게임을 플레이하여 새로운 경험을 수집하는 것)을 마친 후, PPO는 그 경험들을 **배치**(batch)로 묶습니다. 그런 다음 동일한 데이터에 대해 K번의 **패스**(pass, 전체 데이터를 한 번 훑으며 네트워크를 업데이트하는 것)를 수행합니다.
에포크가 많을수록 각 배치에서 더 많은 학습을 짜낼 수 있지만, **낡은 데이터에 과적합(overfitting)**될 위험이 있습니다. (낡은 데이터에 과적합 = 이전 정책하에서는 맞았지만 정책이 바뀐 지금은 더 이상 유효하지 않은 패턴을 암기해버리는 것. 마치 작년 시험 문제를 달달 외웠다가 새로운 유형의 시험에서 낙제하는 학생과 같음).

**실생활 예시:** 20개의 수학 문제를 푸는 학생.
- K=3:  각 문제를 3번씩 풀어봄 → 여전히 배우는 중, 문제 자체를 외우지는 않음
- K=10: 각 문제를 10번씩 풀어봄 → 이 특정 문제들에 대해 완벽히 마스터함
- K=20: 각 문제를 20번씩 풀어봄 → **수학 원리를 이해하는 게 아니라 답을 외워버림** (현재 배치에는 완벽히 들어맞지만 새로운 상황에 대처하는 능력을 잃음)!

> ⚠️ **"K=20 결과도 괜찮아 보이는데 왜 신경 써야 하나요?"**
> PPO의 클리핑 트릭이 한 번의 패스당 정책 변화를 제한하므로 K=20이 당장 큰 붕괴를 일으키지는 않습니다.
> 하지만 에이전트는 현재 정책이 실제로 겪게 될 상황과 동떨어진 데이터에 조용히 과도하게 적응하고 있는 것입니다.
> 이는 **장기적인 학습 속도를 늦춥니다**. 뒤로 갈수록 점점 더 낡은 정보를 재활용하게 되어 각 롤아웃이 에이전트에게 가르쳐줄 수 있는 양이 줄어들기 때문입니다.
> 피해가 서서히 누적되기 때문에 짧은 실험에서는 놓치기 쉽습니다.

클리핑이 치명적인 과적합은 막아주지만, 너무 많은 에포크는 여전히 전체적인 학습 효율을 떨어뜨릴 수 있습니다.

**예상 결과:**
- K=3:  덜 효율적임 (배치당 뽑아낼 수 있는 학습 잠재력을 낭비함)
- K=10: 좋은 균형
- K=20: 정책이 **낡은 데이터에 대해 너무 강한 확신을 가질** 위험 (네트워크 업데이트가 현재 정책이 마주할 실제 상황과 일치하지 않는 경험들에 의해 주도되어, 샘플 효율성을 서서히 갉아먹음)

## 결과 해석 방법

그래프는 한 가지 하이퍼파라미터를 변화시킨 세 가지 차트를 보여줍니다.

```
왼쪽 그래프:    Clip Epsilon — 어떤 ε이 가장 빨리 배우는가?
가운데 그래프:  Learning Rate — 어떤 lr이 가장 안정적인가?
오른쪽 그래프:  Update Epochs — 어떤 K가 최고의 정책을 찾는가?
```

각 선은 무작위성을 줄이기 위해 **3개 시드의 평균 보상**을 나타냅니다.

**확인해야 할 점:**
1. **학습 속도:** 어떤 선이 가장 빨리 높은 보상에 도달하는가?
2. **최종 성능:** 어떤 선이 가장 높은 최종 보상을 달성하는가?
3. **안정성:** 어떤 선의 요동(진동)이 가장 적은가?

좋은 하이퍼파라미터는 이 세 가지의 균형을 잘 맞춘 것입니다!

## 방법론: 과학적 실험 설계

이 실험은 **어블레이션 연구 (ablation study)** 설계를 사용합니다 (기능의 개별 영향을 측정하기 위해 한 번에 하나의 구성 요소만 제거하거나 변경하는 방법):
1. 기본값 설정: ε=0.2, lr=3e-4, K=10
2. 한 번에 **하나**의 파라미터만 변경
3. 나머지는 모두 고정
4. 결과 비교

이를 통해 각 파라미터가 독립적으로 미치는 영향을 파악할 수 있습니다.

**실생활 예시:** 새로운 비료가 식물에 도움이 되는지 테스트하기:
- 비료만 바꾸고 나머지(흙, 물, 햇빛)는 똑같이 유지함
- 식물이 더 잘 자란다면 → 비료 덕분임!

## 실무에서의 일반적인 발견

| 하이퍼파라미터 | 너무 작을 때 | 적정 지점 (Sweet Spot) | 너무 클 때 |
|----------------|-----------|------------|-----------|
| **ε (clip)** | 수렴이 느림 | ε ≈ 0.2 | 불안정함 |
| **lr** | 너무 느림 | 2.5e-4 ~ 3e-4 | 발산함 |
| **K (epochs)** | **데이터 낭비** (충분히 배우기 전에 데이터 버림) | K = 4-10 | 낡은 데이터에 과적합 |
| **n_steps** | 노이즈가 심함 | 128-2048 | **메모리 부족 (OOM)** 오류 발생 |
| **batch_size** | 노이즈가 심함 | 32-256 | **메모리 부족 (OOM)** 오류 발생 |

이러한 "적정 지점"은 환경에 따라 달라질 수 있습니다!

## 핵심 통찰: PPO는 비교적 견고합니다

(타겟 네트워크가 없는 DQN과 같은) 이전 알고리즘들에 비해, PPO는 하이퍼파라미터 선택에 대해 비교적 견고(robust)합니다. 클리핑 메커니즘이 자연스러운 안전망 역할을 해주기 때문입니다.

**실생활 예시:** **ABS** (브레이크 잠김 방지 장치 — 급제동 시 바퀴가 잠기는 것을 막아 운전자가 계속 조종할 수 있게 돕는 안전 장치)가 있는 차와 없는 차의 제동:
- ABS가 없을 때 (DQN): 한 번만 핸들을 잘못 꺾어도(나쁜 하이퍼파라미터) 차가 스핀함
- ABS가 있을 때 (PPO): 차가 스스로 보정함 — 웬만한 하이퍼파라미터 설정에서도 어느 정도 잘 작동함

이러한 견고함이 바로 실무에서 PPO가 가장 인기 있는 강화학습 알고리즘인 주요 이유입니다!

## 핵심 요약

| 개념 | 설명 |
|---------|---------------|
| **어블레이션 연구** | 효과를 확인하기 위해 한 번에 하나씩만 바꾸는 실험 방법 |
| **Clip epsilon ε** | 안전 경계선 — 보통 0.2가 가장 좋음 |
| **학습률 (lr)** | **보폭** — 매 배치 후 네트워크 가중치를 얼마나 조정할지 결정 (목표를 향해 걸어갈 때의 발걸음 크기라고 생각하세요). **2.5e-4 ~ 3e-4**는 0.00025 ~ 0.0003을 나타내는 과학적 표기법이며, 시간 단위가 아닌 비율을 나타내는 숫자입니다. |
| **업데이트 에포크 K** | 각 배치를 몇 번이나 재사용할지 — 보통 4-10회가 표준임 |
| **무작위 시드 (Random Seeds)** | 각 실험은 서로 다른 **무작위 시드**로 반복됩니다. (시드는 무작위 수 생성기의 시작 번호로, 훈련 중의 모든 무작위 선택을 결정합니다). 여러 시드를 사용하면 결과가 일시적인 행운인지 아니면 일관된 것인지 확인할 수 있습니다. |
