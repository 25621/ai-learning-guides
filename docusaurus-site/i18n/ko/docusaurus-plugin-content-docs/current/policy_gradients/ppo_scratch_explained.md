# PPO: 안전하고 꾸준한 정책 업데이트

## A2C의 문제점

손가락 위에 빗자루를 세우고 균형을 잡는 법을 배우고 있다고 상상해 보세요. 수주간의 연습 끝에 이제 30초 동안은 버틸 수 있게 되었습니다!

이때 코치가 조언을 해줍니다: "손목을 왼쪽으로 아주 살짝만 더 기울여봐."

**좋은 조언 → 신중한 변화 → 여전히 30초 동안 균형 유지 ✓**

그런데 만약 코치가 흥분해서 이렇게 소리친다면 어떨까요? "당장 왼쪽으로 확 꺾어버려!"
여러분은 너무 과하게 반응하게 되고 → 빗자루는 쓰러집니다 → 결국 수주간의 노력이 물거품이 됩니다.

이것이 바로 A2C의 문제입니다: **급격한 그래디언트 업데이트가 공들여 만든 좋은 정책을 망가뜨릴 수 있다는 것이죠.**

**PPO (Proximal Policy Optimization)**는 이를 방지하는 안전장치와 같습니다.

---

## 핵심 아이디어: 잘 작동하던 것과 비슷하게 유지하기

PPO의 핵심 제약 조건은 다음과 같습니다.

> **"한 번의 업데이트에서 정책을 너무 많이 바꾸지 마라."**

업데이트 전의 정책을 "이전 정책(π_old)"이라고 하고,
업데이트 후의 정책을 "새로운 정책(π_new)"이라고 합시다.

PPO는 **확률 비율(probability ratio)**을 통해 정책이 얼마나 변했는지 측정합니다.

```
r(θ) = π_new(a|s) / π_old(a|s)
```

- r = 1.0: 정책이 변하지 않음
- r = 1.5: 새로운 정책이 해당 행동을 취할 확률이 50% 높아짐
- r = 0.5: 새로운 정책이 해당 행동을 취할 확률이 50% 낮아짐

**실생활 예시:** 요리사가 레시피를 수정하는 상황.
- r = 1.0: 이전과 같은 양의 소금을 넣음
- r = 2.0: 소금을 두 배로 넣음 — 너무 급격함!
- r = 0.9: 소금을 10% 덜 넣음 — 작고 안전한 변화

## 클리핑 (Clipping) 트릭 {#the-clipping-trick}

PPO는 비율(ratio)이 [1-ε, 1+ε] 범위(보통 ε = 0.2) 안에 머물도록 제한(clip)합니다.

```
L_CLIP = E[min(r(θ) · A,  clip(r(θ), 1-ε, 1+ε) · A)]
```

내용을 나누어 살펴봅시다.

**경우 1: 행동이 좋았을 때 (A > 0)**

우리는 이 행동을 더 많이 하고 싶어 합니다 (r > 1). 하지만 증가폭에 제한을 둡니다.
```
만약 r > 1.2이면: 1.2로 제한(clip), 더 이상 늘리도록 유도하지 않음
```
이는 한 방향으로 너무 과하게 쏠리는 것을 방지합니다.

**경우 2: 행동이 나빴을 때 (A < 0)**

우리는 이 행동을 덜 하고 싶어 합니다 (r < 1). 이번에도 하한선에 제한을 둡니다.
```
만약 r < 0.8이면: 0.8로 제한(clip), 더 이상 페널티를 주지 않음
```

**시각화:**
```
ε = 0.2인 경우, 안전한 비율 창은 0.8에서 1.2 사이입니다.

좋은 행동 (A > 0): 행동 확률을 높이되, 1.2가 넘어가면 보상을 멈춤
비율 r:       0.6      0.8      1.0      1.2      1.4
유도 방향:      ↑        ↑        ↑        ↑        -
의미:        너무 낮음   적당함     이전     최대    제한됨(Clipped)

나쁜 행동 (A < 0): 행동 확률을 낮추되, 0.8 아래로 내려가면 페널티를 멈춤
비율 r:       0.6      0.8      1.0      1.2      1.4
유도 방향:      -        ↓        ↓        ↓        ↓
의미:      제한됨(Clipped) 최대     이전    적당함   너무 높음
```

`-` 표시된 부분은 평평한 클리핑 영역입니다. 이 영역에서는 확률 비율을 더 극단적으로 만들어도 목표 함수가 개선되지 않으므로, PPO는 더 이상 그 방향으로 밀어붙이지 않습니다.

**실생활 예시:** 자동차의 속도 제한 장치. 가속은 할 수 있지만, 시속 120km에 도달하면 제한 장치가 작동하여 더 빨라지지 않게 막아줍니다. 여러분의 움직임을 멈추지는 않으면서도 안전하게 지켜주는 것이죠.

## 왜 이것이 파괴적인 업데이트를 막아주나요?

**파괴적인 업데이트(catastrophic update)**란, 단 한 번의 커다란 정책 변화로 에이전트가 그동안 배운 모든 것을 망쳐버리는 상황을 말합니다. 수시간의 훈련 결과가 단 한 번의 그래디언트 단계 만에 사라지는 것이죠.

클리핑이 없다면: 한 번의 큰 업데이트로 정책이 급격하게 변할 수 있습니다.
클리핑이 있다면: [1-ε, 1+ε] 범위를 벗어나면 그래디언트가 0이 되므로, 정책은 단계마다 조금씩만 움직일 수 있습니다.

**실생활 예시:** 훌륭한 외과 의사는 크고 휘두르는 절개가 아닌, 작고 정밀한 절개를 합니다. PPO는 강화학습 최적화 도구 중 "신중한 외과 의사"와 같습니다.

## GAE: 더 똑똑한 Advantage 추정 {#gae-smarter-advantage-estimates}

PPO는 **GAE (Generalized Advantage Estimation)**를 사용하여 advantage를 계산합니다.

```
δ_t = r_t + γ · V(s_{t+1}) - V(s_t)          (TD 오차)
A_t = δ_t + γλ · δ_{t+1} + (γλ)² · δ_{t+2} + ...
```

GAE에는 λ (람다)라는 파라미터가 있습니다.
- λ = 0: 한 단계 앞만 보는 TD 오차 사용 (낮은 분산, 높은 편향)
- λ = 1: 전체 에피소드 리턴 사용 (높은 분산, 낮은 편향)
- λ = 0.95: 두 방식 사이의 좋은 균형점!

**실생활 예시:** 자동차 여행 계획 짜기.
- λ=0: 바로 앞 5km만 봄 (안전하지만 나중에 나타날 지름길을 놓칠 수 있음)
- λ=1: 500km 전체 여정을 고려함 (정보는 많지만 변수가 너무 많아 불확실함)
- λ=0.95: 멀리 내다보되 가까운 길에 더 높은 가중치를 둠 ← 가장 좋은 균형!

## 여러 번의 에포크: 데이터를 효율적으로 재사용하기

REINFORCE는 경험 뭉치(롤아웃)를 수집한 뒤, 단 한 번의 업데이트 후에 바로 버립니다.

반면 PPO는 각 배치를 **K 에포크** (보통 같은 데이터에 대해 4~10번 반복) 동안 재사용합니다.

```
4개의 병렬 환경에서 512단계 수집 = 2048개의 전환 데이터(transitions)
에포크 1: 32개의 미니 배치로 나누어 각각 업데이트
에포크 2: 데이터를 섞은 뒤 다시 32개의 미니 배치로 업데이트
에포크 3: ...
에포크 4: ...
```

**"미니 배치(minibatch)"란 무엇인가요?** 2048개의 모든 데이터를 한꺼번에 업데이트하는 것은 느리고 메모리를 많이 차지합니다. 반면 데이터를 하나씩 업데이트하는 것은 노이즈가 너무 심하죠. **미니 배치**는 그 중간의 적당한 덩어리입니다. 여기서는 2048 ÷ 32 = **미니 배치당 64개의 데이터**가 되겠네요. 각 미니 배치마다 한 번의 그래디언트 단계를 계산하므로, 한 에포크마다 1번의 거대한 업데이트 대신 32번의 작고 안정적인 업데이트를 수행하게 됩니다. (이는 딥러닝 전반에서 사용되는 미니 배치 경사 하강법과 동일한 아이디어입니다.)

이때 클리핑 덕분에 여러 번 반복해서 학습해도 정책이 너무 멀리 가버리지 않습니다. 클리핑이 없다면 여러 번의 에포크는 정책을 과하게 밀어붙여 망가뜨렸을 것입니다!

**실생활 예시:** 학생에게 30개의 연습 문제가 있습니다.
- REINFORCE: 각 문제를 한 번씩만 풀고 조금 배운 뒤 버립니다.
- PPO: 각 문제를 4번씩 (매번 다른 관점에서) 풀어봅니다. 이때 정답만 달달 외우지 않도록 변화폭을 제한하여 원리를 익히게 합니다.

## 전체 PPO 손실 함수

```
L = L_CLIP - c₁ · L_entropy + c₂ · L_critic

L_CLIP    = 클리핑된 정책 그래디언트
L_entropy = 엔트로피 보너스 (탐험 장려)  
L_critic  = V(s)와 리턴 사이의 평균 제곱 오차 (MSE)
```

일반적인 계수 값: c₁ = 0.01 (엔트로피), c₂ = 0.5 (크리틱)

**두 가지 항에 대한 자세한 설명:**

- **정책 그래디언트 (Policy gradient)** — 손실 함수의 "액터(actor)" 부분입니다. 그래디언트 신호를 사용하여 정책이 advantage가 높은 행동의 확률은 높이고, 낮은 행동의 확률은 낮추도록 밀어줍니다. 이는 REINFORCE에서 소개된 것과 동일한 핵심 아이디어입니다. [REINFORCE 가이드](./reinforce_cartpole_explained.md#the-old-way-vs-the-new-way)에서 직관적인 설명을 확인해 보세요. PPO는 여기에 클리핑이라는 외투를 입힌 것입니다.
- **MSE (평균 제곱 오차)** — 손실 함수의 "크리틱(critic)" 부분입니다. 크리틱 V(s)는 어떤 상태에서의 기대 리턴을 예측합니다. 우리는 이 예측값과 실제 리턴을 비교하고 그 차이를 제곱합니다: `MSE = 평균((V(s) - 실제리턴)²)`. 차이를 제곱하면 작은 실수보다 큰 실수에 더 큰 페널티를 부여하게 되며, 훈련을 위한 매끄럽고 미분 가능한 신호를 제공합니다. (표준적인 회귀 손실 함수입니다.)

## 결과

```
업데이트  200 | 평균 보상: ~120
업데이트  400 | 평균 보상: ~280
업데이트  800 | 평균 보상: ~280-300
```

CartPole에서의 PPO는 꾸준한 향상을 보이지만 약 280-300점 근처에서 정체되는 경향이 있습니다. (**정체(plateau)**란 학습 곡선이 평평해지는 현상으로, 훈련을 계속해도 보상이 더 이상 오르지 않는 상태를 말합니다. 정책이 국지적인 최적 전략을 찾았지만 더 이상의 진전을 이루지 못하는 상태입니다.) 사실 이는 예상된 결과입니다. PPO는 더 어렵고 에피소드가 긴 환경을 위해 설계되었기 때문입니다.

한 가지 흥미로운 관찰: **REINFORCE가 CartPole을 더 빨리 해결했습니다!** (평균 500점 도달 vs PPO 300점)

왜 그럴까요? CartPole 에피소드는 짧기(500단계 이하) 때문에 REINFORCE의 정확한 리턴 계산이 매우 잘 맞습니다. 반면 PPO의 부트스트래핑 추정치는 불필요한 복잡함을 더하게 되죠. PPO는 에피소드가 완전히 끝나기를 기다리는 것이 비현실적인 환경(예: BipedalWalker)에서 진정한 진가를 발휘합니다.

**"BipedalWalker"란 무엇인가요?** BipedalWalker-v3는 강화학습의 고전적인 벤치마크 환경입니다. 두 다리가 달린 로봇이 울퉁불퉁한 지형을 넘어지지 않고 앞으로 걸어가는 법을 배워야 합니다. 왼쪽/오른쪽 두 가지 선택지만 있는 CartPole과 달리, BipedalWalker는 **연속적인(continuous)** 행동을 가집니다. 즉, 각 다리 관절에 대해 [-1, 1] 사이의 실수 값인 4개의 토크(회전력) 값을 결정해야 합니다. 에피소드가 수천 단계까지 이어질 수 있으므로, PPO의 데이터 효율성과 안정성이 빛을 발하는 전형적인 사례입니다.

---

## 주요 방정식

```
비율(Ratio): r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)
클립 손실:    L_CLIP = E[min(r_t A_t, clip(r_t, 1-ε, 1+ε) · A_t)]
GAE:         A_t = Σ_{l=0}^{∞} (γλ)^l · δ_{t+l}
```

## 핵심 요약

| 개념 | 설명 |
|---------|---------------|
| **비율 r(θ)** | 이번 행동에 대해 정책이 얼마나 변했는가 |
| **클립 ε** | 안전 경계선 — 정책을 이보다 더 많이 바꾸지 마라 |
| **GAE** | 여러 단계 앞을 내다보며 advantage를 똑똑하게 추정하는 방법 |
| **데이터 효율성** | 여러 병렬 환경에서 롤아웃을 수집하고(상관관계 제거), 이를 K번의 미니 배치 업데이트에 재사용함. 이때 클리핑이 반복 학습을 안전하게 지켜줌 |
