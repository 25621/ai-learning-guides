# 베이스라인을 포함한 REINFORCE: 노이즈 제거하기

## 일반 REINFORCE의 문제점

여러분이 시험을 치르고 난 뒤 자신의 답이 좋았는지 판단하려는 학생이라고 상상해 보세요.

**나쁜 피드백:** "7점을 받았습니다!"

7점은 좋은 점수일까요? 만점이 10점이라면 좋겠죠! 하지만 다른 사람들이 다 9점을 받았다면 나쁜 점수입니다. 맥락이 없다면 여러분은 자신의 답변 스타일을 바꿔야 할지 말지 알 수 없습니다.

이것이 바로 REINFORCE의 문제입니다: 행동을 평가할 때 **가공되지 않은 리턴(raw returns)**인 G_t를 그대로 사용한다는 점입니다. 200점이라는 점수는 상황에 따라 대단할 수도, 형편없을 수도 있습니다.

---

## 베이스라인(Baseline)의 도입

**베이스라인** b(s)는 하나의 기준점입니다: "이 상황에서 나는 **어느 정도의** 보상을 기대하는가?"

"이 행동이 좋았나?"라고 묻는 대신, 우리는 이렇게 묻습니다.

> **"이 행동이 평소에 기대하던 것보다 더 좋았나?"**

```
기존 신호: 업데이트 ∝ G_t
새로운 신호: 업데이트 ∝ (G_t - b(s_t))
```

**실생활 예시:** 수학 시험에서 85점을 받았습니다.
- 반 평균이 60점이라면 → 당신은 **평균보다 25점 높습니다** → 훌륭합니다!
- 반 평균이 90점이라면 → 당신은 **평균보다 5점 낮습니다** → 더 노력이 필요합니다!

**advantage** (G_t - b(s))는 예상보다 잘했을 때 양수(+), 못했을 때 음수(-)가 됩니다. 이는 훨씬 더 깔끔한 학습 신호가 됩니다!

---

## 베이스라인이란 무엇인가요?

가장 자연스러운 베이스라인은 **가치 함수 V(s)**입니다.

> V(s) = "내가 상태 s에 있고 현재 정책을 따를 때 기대되는 총 보상"

우리는 이를 별도의 **가치 네트워크 (Value Network)** (베이스라인 네트워크 또는 크리틱이라고도 함)를 통해 학습합니다.

```
상태  →  [128개 뉴런]  →  [128개 뉴런]  →  V(s)   (단일 숫자)
```

에이전트가 방문하는 각 상태에 대해, V(s)는 기대되는 리턴을 예측합니다. 만약 실제 리턴 G_t가 V(s)보다 높다면, 그 행동은 예상보다 좋았던 것입니다!

---

## 함께 배우는 두 네트워크

```
에피소드 발생
     ↓
실제 리턴 G_t 계산
     ↓
         ┌─────────────────────────────┐
         │ Advantage = G_t - V(s_t)    │
         │  +: 행동이 예상보다 좋았음   │
         │  -: 행동이 예상보다 나빴음   │
         └─────────────────────────────┘
              ↓                  ↓
    정책 네트워크 업데이트      가치 네트워크 업데이트
   (좋은 행동의 확률을      (다음번에 더 정확하게
    조절함)               예측하도록 학습함)
```

**실생활 예시:** 두 친구가 함께 식당에 갔습니다.

- 친구 1 (가치 네트워크): "이 요리는 10점 만점에 7점 정도일 것 같아."
- 친구 2 (정책 네트워크): 요리를 직접 먹어보고 9점을 줍니다.
- Advantage = 9 - 7 = +2 → "생각보다 맛있네! 다음에 또 시키자!"

다음 방문 때, 친구 1은 자신의 예측치를 9점에 가깝게 수정합니다.
친구 2는 다음에 그 요리를 더 자주 주문하게 될 것입니다.

---

## 왜 이것이 분산을 줄여주나요?

**수학적 증명 (직관적 이해):**

베이스라인이 없을 때: `그래디언트 ∝ ∇log π(a|s) × G_t`

G_t 값은 에피소드마다 차이가 매우 큽니다.
```
에피소드 1: G = [45, 44, 43, ..., 1]   (중간 정도의 게임)
에피소드 2: G = [500, 499, ..., 1]      (아주 잘한 게임!)
에피소드 3: G = [12, 11, ..., 1]        (형편없는 게임)
```

G_t가 크고 노이즈가 심하기 때문에 그래디언트 추정치가 이리저리 튑니다.

베이스라인이 있을 때: `그래디언트 ∝ ∇log π(a|s) × (G_t - V(s_t))`

Advantage (G_t - V(s_t))는 훨씬 작고 0 근처에 모여 있습니다.
```
에피소드 1: advantage ≈ [-2, +1, -3, ..., 0]   (작고 중심에 모임)
에피소드 2: advantage ≈ [+10, +8, ..., +3]      (이번 판은 잘했음)
에피소드 3: advantage ≈ [-5, -6, ..., -2]       (이번 판은 못했음)
```

**실생활 예시:** 달리기 속도 측정하기.
- 베이스라인 없음: "시속 8km로 달렸어" (맥락 없이는 무의미함)
- 베이스라인 있음: "내 평균보다 시속 2km 더 빠르게 달렸어" (확실히 잘함!)

Advantage는 항상 비교값입니다. 따라서 본질적으로 더 작고 안정적입니다.

---

## 중요한 사실: 편향(Bias)이 없습니다!

베이스라인은 알고리즘이 *무엇*을 배우는지는 바꾸지 않습니다. 단지 *얼마나 빠르고 안정적으로* 배우는지만을 바꿉니다.

**왜일까요?** 기대되는 advantage의 평균(기대값)은 항상 0이기 때문입니다.

> E[G_t - V(s_t)] = E[G_t] - V(s_t) = V(s_t) - V(s_t) = 0

행동(action)에 의존하지 않는 어떤 b(s)라도 유효한 베이스라인으로 사용할 수 있습니다!

**실생활 예시:** 상대평가(평균점수 조정)를 한다고 해서 누가 1등인지가 바뀌지는 않습니다. 단지 점수를 해석하기 쉽게 만들 뿐이죠. 순위는 그대로 유지되고 척도만 바뀝니다.

## 결과

```
베이스라인 없음 — 최종 100회 평균: 500.0, 그래디언트 분산: 599.3
베이스라인 있음 — 최종 100회 평균: 491.4, 그래디언트 분산: 578.8
```

두 방법 모두 CartPole에서 거의 완벽한 성능에 도달하지만, 다음을 주목하세요.
1. **그래디언트 분산(variance)**이 눈에 띄게 줄어듭니다 (그래프 오른쪽에서 훈련 과정 중의 분산을 확인할 수 있습니다).
2. 베이스라인을 사용하면 에이전트가 높은 성능에 **더 안정적으로** 도달합니다 — 훈련 중에 갑자기 보상이 뚝 떨어지는 현상이 줄어듭니다.

분산 감소 효과는 더 어려운 환경(LunarLander, MuJoCo)에서 훨씬 더 드라마틱하게 나타납니다.

---

## 주요 방정식

```
베이스라인 값:   V(s) ← V(s) + α(G_t - V(s))   [MSE 최소화]
정책 그래디언트:  θ ← θ + α ∇log π(a_t|s_t) · (G_t - V(s_t))
Advantage:      A_t = G_t - V(s_t)
```

## 핵심 요약

| 개념 | 설명 |
|---------|---------------|
| **베이스라인 b(s)** | 상태 s에서 기대되는 보상 — 우리의 기준점 |
| **Advantage A_t** | "이 행동이 예상보다 좋았는가?" |
| **가치 네트워크** | 기대 리턴을 예측하도록 학습하는 신경망 |
| **분산 감소** | 그래디언트 추정치의 노이즈 감소 → 더 안정적인 학습 |
| **편향 없음 (Unbiased)** | 베이스라인은 평균적으로 목표 정책을 바꾸지 않으며, 단지 학습 신호를 덜 시끄럽고 안정적으로 만듦 |

## 다음 단계는?

베이스라인은 사실 훨씬 더 강력한 방법인 **액터-크리틱 (Actor-Critic)** 방법의 시작입니다.

에피소드가 끝날 때만 V(s)를 계산하는 대신, 액터-크리틱은 **시간차(Temporal Difference)** 학습을 사용하여 매 단계마다 V(s)를 업데이트합니다. 이를 통해 업데이트 속도가 훨씬 빨라지고 에피소드가 끝나지 않아도 학습할 수 있게 됩니다!

전체 액터-크리틱 구현은 `a2c_lunarlander.py`를 참조하세요.
