# REINFORCE: 로봇에게 더 나은 선택을 하는 법 가르치기

## 우리가 하려는 것이 무엇인가요?

비디오 게임을 하는 로봇이 있다고 상상해 보세요. 매초 로봇은 선택해야 합니다:
**"버튼을 누를까, 말까?"**

Q-러닝처럼 모든 상황을 테이블(표)에 외우는 대신, 우리는 로봇이 **레시피**를 배우기를 원합니다. 즉, "이런 상황에서는 이 행동을 하라"고 직접 말해주는 규칙 세트 말이죠.

이 레시피를 **정책 (policy, π)**이라고 부릅니다. 강화학습에서 π는 "행동을 선택하는 규칙"을 의미합니다.

---

## 옛날 방식 vs. 새로운 방식 {#the-old-way-vs-the-new-way}

**옛날 방식 (Q-러닝 / DQN):** 각 행동이 얼마나 좋은지(Q-값)를 배우고, 그중 최고를 고릅니다.
> "왼쪽으로 미는 것은 7점, 오른쪽은 5점 → 왼쪽으로 밀자!"

**새로운 방식 (정책 그래디언트, Policy Gradient):** 어떤 행동을 선택할지 직접 배웁니다.
> "폴이 오른쪽으로 기울면, 80% 확률로 오른쪽으로 밀고, 20% 확률로 왼쪽으로 밀자."
*(**그래디언트(Gradient)**라는 단어는 이러한 확률을 올바른 방향으로 서서히 조정하기 위해 취하는 수학적인 "단계"를 의미합니다.)*

**실생활 예시:** 자전거 타기 배우기.
- 옛날 방식: "왼쪽으로 5도 기울이기" 대 "왼쪽으로 7도 기울이기"에 대한 *정확한 점수*를 계산함.
- 새로운 방식: 그냥 **몸**이 익힐 때까지 연습함 — 느낌이 오는 쪽으로 발을 구름!

---

## REINFORCE는 어떻게 작동하나요?

REINFORCE는 로봇이 게임 한 판을 처음부터 끝까지 수행하는 것(하나의 **에피소드**)을 지켜본 뒤 묻습니다: "어떤 행동들이 좋은 점수로 이어졌지? 그런 행동들을 더 많이 하자!"

### 단계별 과정

**1. 에피소드 플레이**

로봇은 선택을 내리고 경험을 수집합니다.
```
1단계: 상태 = [폴이 오른쪽으로 기움] → 행동 = 오른쪽으로 밀기 → 보상 = +1
2단계: 상태 = [폴이 거의 수직임] → 행동 = 오른쪽으로 밀기 → 보상 = +1
3단계: 상태 = [폴이 왼쪽으로 기움] → 행동 = 왼쪽으로 밀기  → 보상 = +1
...
47단계: 상태 = [폴이 쓰러짐!] → 에피소드 종료
```

**2. 리턴(Returns) 계산**

각 단계에 대해 G_t, 즉 **그 시점부터 앞으로 받을 총 보상**을 계산합니다.
```
47단계에서의 G = 1
46단계에서의 G = 1 + 0.99 × 1 = 1.99
45단계에서의 G = 1 + 0.99 × 1.99 = 2.97
...
1단계에서의 G  = 47 (대략 — 처음부터 시작했으므로 리턴값이 큼)
```

여기서 γ = 0.99인 **할인 인자(discount factor)**는 먼 미래의 보상을 현재보다 조금 덜 중요하게 취급한다는 뜻입니다.

**실생활 예시:** 학교 첫날 받는 참 잘했어요 스티커가 100일 뒤에 받을지도 모르는 스티커보다 더 신나게 느껴지는 것과 같습니다. 미래의 보상은 약간 "할인"됩니다.

**3. 정책 업데이트**

취해진 각 행동에 대해:
> 만약 G_t가 높았다면 (그 행동이 좋은 결과로 이어졌다면): **더 자주 하라!**
> 만약 G_t가 낮았다면 (그 행동이 나쁜 결과로 이어졌다면): **덜 하라!**

수식: `loss = -log_prob(action) × G_t`

그래디언트를 구하고 정책을 업데이트하는 것은 로봇에게 이렇게 말하는 것과 같습니다:
*"너 아까 20단계에서 했던 그 행동 말이야? 다음번에는 5% 정도 더 자주 하도록 해!"*

---

## 정책 네트워크(Policy Network)란 무엇인가요?

테이블 대신 우리는 **신경망**을 사용하여 정책을 표현합니다.

```
관측 정보         정책 네트워크          행동 확률
[카트 위치]  →   [128개 뉴런]  →  →  [왼쪽 밀기: 30%]
[카트 속도]  →   [128개 뉴런]        [오른쪽 밀기: 70%]
[폴 각도]    →
[폴 속도]    →
```

네트워크는 각 행동에 대한 **확률**을 출력합니다. 우리는 그 확률에 따라 샘플링합니다:
> 주사위를 굴림 → 1-30: 왼쪽으로 밀기, 31-100: 오른쪽으로 밀기

**실생활 예시:** 일기 예보 앱에서 "비 올 확률 70%"라고 합니다. 비가 올지 안 올지 확실히 아는 게 아니라, 확률에 따라 결정을 내리는 것이죠. 로봇도 똑같은 일을 합니다!

---

## 정규화(Normalization): 왜 평균을 뺄까요?

G_t를 사용하여 업데이트하기 전에 우리는 정규화를 거칩니다.
```
G_정규화 = (G - 평균(G)) / 표준편차(G)
```

**왜 그럴까요?** 만약 모든 보상이 양수라면 (CartPole처럼 항상 매 단계 +1인 경우), 모든 행동이 "좋아" 보이고 업데이트 신호가 혼란스러워집니다.

정규화를 거치면 어떤 리턴은 양수(평균 이상 → 더 많이 하기)가 되고, 어떤 리턴은 음수(평균 이하 → 덜 하기)가 됩니다. 신호가 훨씬 더 명확해지죠!

**실생활 예시:** 선생님이 시험 성적을 보정합니다. 평균이 70점인데 85점을 받았다면 훌륭한 거죠! 하지만 평균이 90점인데 85점을 받았다면 평균 이하입니다. 원점수만으로는 전체 이야기를 다 알 수 없습니다.

---

## 문제점: 높은 분산 (High Variance)

REINFORCE에는 커다란 약점이 하나 있는데, 바로 **분산(variance)**입니다. 리턴 G_t의 노이즈가 매우 심하다는 것이죠.

**실생활 예시:** 각 식당에서 딱 한 끼의 식사만 먹어보고 요리사를 평가한다고 상상해 보세요. 요리사가 컨디션이 안 좋았을 수도 있고, 재료가 신선하지 않았을 수도 있습니다. 단 한 번의 식사만으로는 그 식당이 정말 좋은지 신뢰성 있게 알 수 없습니다!

REINFORCE는 에피소드가 **완전히** 끝날 때까지 기다렸다가 업데이트를 합니다. 어떤 에피소드는 운이 아주 좋았을 수도 있고, 어떤 에피소드는 아주 운이 나빴을 수도 있습니다. 그래서 그래디언트가 이리저리 널뛰게 됩니다.

학습 곡선(그래프)이 뾰족뾰족하게 생긴 이유가 바로 이것입니다:
- 어떤 판은 500점 (대박!)
- 어떤 판은 50점 (쪽박!)

이러한 노이즈에도 불구하고 REINFORCE는 결국 배웁니다. 다만 인내심이 필요할 뿐이죠.

## 결과

```
에피소드  100 | 평균 보상 (최근 100회):  43.1
에피소드  200 | 평균 보상 (최근 100회): 193.9
에피소드  500 | 평균 보상 (최근 100회): 408.4
에피소드 1000 | 평균 보상 (최근 100회): 500.0  ← 해결!
```

로봇은 최대치인 500단계 동안 폴의 균형을 잡는 법을 성공적으로 배웠습니다!

분산 문제에도 불구하고 REINFORCE가 CartPole에서 효과적인 이유는 다음과 같습니다.
1. 에피소드가 짧습니다 (덕분에 한 번의 훈련에서 많은 에피소드를 겪을 수 있습니다).
2. 최적 정책이 단순합니다 (대체로 폴이 기우는 방향으로 밀면 됩니다).

## 핵심 요약

| 개념 | 설명 |
|---------|---------------|
| **정책 (Policy)** | 행동을 선택하기 위한 로봇의 레시피 |
| **에피소드 (Episode)** | 시작부터 끝까지의 한 판의 게임 |
| **리턴 (G_t)** | 현재 시점부터 앞으로 받을 총 보상 |
| **할인 인자 (γ)** | 미래의 보상을 현재보다 조금 낮게 가치 평가함 |
| **정규화 (Normalize)** | 평균을 빼서 학습 신호를 더 명확하게 만듦 |
| **분산 (Variance)** | 그래디언트 추정치가 얼마나 이리저리 튀는지 나타냄 |

## 다음 단계는?

REINFORCE의 큰 약점은 **분산**입니다. 다음 스크립트(`reinforce_baseline.py`)에서는 평균적으로 배우는 내용은 유지하면서도 이 노이즈를 획기적으로 줄여주는 **베이스라인(baseline)**을 추가할 것입니다.

핵심 아이디어는 이렇습니다: "이 행동이 좋았나?"라고 묻는 대신, "이 행동이 **예상보다** 좋았나?"라고 묻는 것입니다. 이 작은 변화가 학습을 훨씬 더 안정적으로 만들어 줍니다.
