# 연속 제어(Continuous Control)를 위한 PPO: BipedalWalker 걷게 만들기

## 이산적 행동 vs. 연속적 행동 (Discrete vs. Continuous Actions)

지금까지 우리가 해결한 모든 환경은 **이산적(discrete)** 행동을 가졌습니다.
- CartPole: 왼쪽으로 밀기 또는 오른쪽으로 밀기 (2가지 선택지)
- LunarLander: 아무것도 안 함 / 왼쪽 엔진 / 메인 엔진 / 오른쪽 엔진 (4가지 선택지)

하지만 현실 세계의 로봇은 **연속적(continuous)** 행동이 필요합니다.
- 휴머노이드 로봇: "각 관절을 얼마나 세게 밀 것인가" (-1에서 +1 사이의 모든 값)
- 자동차: "핸들을 정확히 몇 도나 꺾을 것인가" (-30°에서 +30° 사이의 모든 각도)
- 로봇 팔: "이 방향으로 정확히 2.3 뉴턴의 힘을 가하라"

**실생활 예시:** 키보드 타이핑 = 이산적 (A, B, C... 중 하나를 누름).
연필로 글씨 쓰기 = 연속적 (손을 오른쪽으로 2.3cm 이동, 40g의 힘으로 누름...).

## 연속적 행동을 위한 가우시안 정책 (Gaussian Policy)

연속적 행동의 경우, (N개의 카테고리 중 하나를 고르는) 범주형 분포(Categorical distribution) 대신 **정규 분포 (가우시안 분포, Normal/Gaussian distribution)**를 사용합니다.

```
행동 ~ 정규분포(μ, σ)
```

여기서 각 기호의 의미는 다음과 같습니다.
- **μ (뮤, 평균)**: 분포의 중심 — 네트워크가 "목표로 하는" 행동 값
- **σ (시그마, 표준편차)**: 퍼짐 정도 — 얼마나 많은 무작위성/탐험을 추가할 것인가

```
        확률 (Probability)
             │
        0.4 ─┤      ██████
             │    ████████████
        0.2 ─┤  ██████████████████
             │████████████████████████
             └──────────────────────── 행동 값 (Action value)
           -1  -0.5   0   0.5   1
                      ↑
                   평균 μ
```

**실생활 예시:** 숙련된 궁수는 과녁의 중심(μ)을 조준합니다. 하지만 화살이 모두 정확히 한 점에 박히지는 않습니다. 어느 정도의 퍼짐(σ)이 발생하죠. 연습을 거듭할수록 과녁 중심은 유지하면서 화살이 박히는 범위는 좁아집니다(σ 감소).

## 우리의 가우시안 액터-크리틱(Actor-Critic) 네트워크

```
상태 (24개의 숫자) → [256개 뉴런] → [256개 뉴런] →
    ├── Actor: 4개의 평균 값 (μ₁, μ₂, μ₃, μ₄)
    │          + 4개의 log_std 파라미터 (모든 상태에서 공유!)
    └── Critic: 1개의 가치 (V(s))
```

`log_std` (표준편차의 로그값 — 퍼짐 정도나 불확실성의 척도)는 상태에 의존하지 않는 **학습 가능한 파라미터**입니다. 이를 통해 모델을 단순하게 유지하면서도 훈련 과정에서 탐험의 정도를 변화시킬 수 있습니다.

**왜 std 대신 log_std를 쓰나요?** 표준편차는 항상 양수여야 합니다. `log_std`를 사용하면 네트워크가 양수나 음수 등 어떤 실수든 출력하게 할 수 있고, 여기에 `exp(log_std)` (로그의 역함수인 지수 함수)를 적용하여 항상 양수인 표준편차를 얻을 수 있습니다. 이는 표준편차가 음수나 0이 되는 것을 방지합니다.

## 연속적 행동의 로그 확률(Log Probability) 계산

이산적 행동의 경우: `log_prob = log(P(행동=왼쪽))`

연속적 행동의 경우, **정규 분포**는 평균을 중심으로 하는 매끄러운 종 모양 곡선을 나타냅니다. 연속 수학에서 정확히 한 점에 대한 확률은 0이므로, 대신 그 지점에서의 곡선 높이인 **pdf** (확률 밀도 함수, probability density function) 값을 사용합니다.
```
log_prob = Σᵢ log[정규분포(μᵢ, σᵢ).pdf(aᵢ)]
```

여기서 `log`는 자연로그를 의미합니다. 이는 매우 작은 밀도 값을 신경망이 최적화하기 쉬운 안정적인 숫자로 바꿔줍니다. 모든 행동 차원(BipedalWalker의 경우 4개)에 대해 합산하는데, 이는 전체 행동이 4개의 숫자로 된 하나의 벡터이기 때문입니다.

**실생활 예시:** 내일 기온이 정확히 5.732...°C일 확률은 얼마일까요? 연속적인 날씨 데이터에서는 정규 분포 곡선을 보고 해당 지점의 높이를 확인합니다. 평균에 가까운 기온일수록 더 높은 확률 밀도를 갖게 됩니다.

## BipedalWalker: 걷기 도전

BipedalWalker-v3는 넘어지지 않고 걷는 법을 배워야 하는 2D 로봇입니다.

```
          O (머리)
         /│\
        / │ \
       /  │  \
      L   │   R   ← 무릎 관절이 있는 두 다리
     / \  │  / \
    ●   ● │ ●   ●  ← 4개의 모터 (각 다리의 고관절/무릎)
```

**상태 공간 (24개의 숫자):**
- 선체(Hull): 각도, 각속도, 수평 속도, 수직 속도 (4개)
- 관절(Joints): 4개 모터의 각도 및 속도, 2개의 지면 접촉 센서 (10개)
- 10개의 LIDAR 거리 센서 (앞쪽 지면을 감지하는 거리 정보) (10개)

**행동 공간 ([-1, 1] 범위의 4개 연속 값):**
행동 값은 4개 관절에 가해지는 **토크(torque)**(모터에 의해 가해지는 회전력)를 제어합니다 (선체에는 직접적인 힘을 가하지 않습니다).
- 다리 1 고관절 토크, 다리 1 무릎 토크, 다리 2 고관절 토크, 다리 2 무릎 토크

**보상:**
- 목표 지점(오른쪽 끝) 도달 시 +300
- 넘어질 시 (몸체가 땅에 닿음) -100
- 앞으로 나아갈 때마다 소량의 보상
- 모터 사용 시 소량의 페널티 (효율적인 보상)

**해결 기준:** 100회 에피소드 평균 보상 > 300

## 이산적 PPO와의 주요 차이점

다음을 제외하고는 모두 동일합니다.

| | 이산적(Discrete) PPO | 연속적(Continuous) PPO |
|---|---|---|
| **정책 (Policy)** | Categorical(logits) | Normal(μ, σ) |
| **샘플링** | {0,1,...,N} 중 하나 선택 | 행동 = μ + σ × 노이즈 |
| **로그 확률** | log P(행동=k) | Σ log Normal(μᵢ, σᵢ).pdf(aᵢ) |
| **제한 (Clamp)** | 필요 없음 | 행동을 [-1, 1] 범위로 제한 |

**Logits**는 이산적 행동을 위한 가공되지 않은 점수입니다. 범주형 정책은 이를 **소프트맥스(softmax)** 함수를 통해 확률로 변환합니다. (소프트맥스는 어떤 숫자 세트든 합이 1인 양수 분포로 압축해 줍니다.) 예를 들어 로짓 [2.0, 1.0, 0.5]는 확률 [0.59, 0.24, 0.17]이 됩니다. 연속적 PPO는 행동 자체가 고정된 메뉴에서 선택되는 것이 아니므로 행동 자체에 대해서는 소프트맥스를 사용하지 **않습니다**. 대신 정책은 정규 분포의 평균과 표준편차를 출력하고, 거기서 실수 값인 토크를 샘플링합니다.

**Clamp**는 값을 유효한 범위 내로 강제하는 것을 의미합니다. 코드에서는 `action.clamp(-1, 1)`을 사용하여 환경이 허용된 범위를 벗어나는 모터 명령을 받지 않도록 합니다.

PPO에서 말하는 **Clip**은 다른 의미입니다. [PPO 클리핑 섹션](./ppo_scratch_explained.md#the-clipping-trick)에서 설명했듯이 손실 함수 내부에서 확률 비율을 제한하는 것을 뜻합니다. 행동 클램핑은 환경 인터페이스를 보호하고, PPO 클리핑은 정책 업데이트 과정을 보호합니다.

## 밑바닥부터 배우는 걷기: 에이전트의 학습 과정

**훈련 초기 (음수 보상):** 로봇이 무작위로 허둥대다 즉시 넘어집니다. 모든 에피소드가 몇 초 만에 충돌로 끝납니다.

**훈련 중기:** 로봇은 다리를 번갈아 움직이는 것이 앞으로 나아가는 원동력임을 발견합니다. 작고 서툰 걸음마를 시작하며 보상이 덜 부정적인 쪽으로 이동합니다.

**훈련 후기:** 부드럽고 효율적인 **보행(gait)**이 나타납니다. 보행이란 왼쪽과 오른쪽 발을 번갈아 내딛는 것과 같은 반복적인 움직임 패턴을 말합니다. 로봇은 LIDAR 센서를 활용하여 지형의 굴곡에 맞춰 실시간으로 발걸음을 조절합니다.

**실생활 예시:** 걷기를 배우는 아기:
1. 즉시 넘어짐 (음수 보상)
2. 한 걸음 떼고 넘어짐 (약간 나아짐)
3. 몇 걸음 걸음 (작은 양수 보상)
4. 방을 가로질러 걸어감 (큰 양수 보상!)

## 왜 BipedalWalker에 PPO가 필요한가요? (REINFORCE가 아닌 이유)

- **BipedalWalker 에피소드**는 최대 1600단계까지 이어질 수 있습니다 (CartPole보다 훨씬 깁니다!).
- **보상이 희소합니다** — 앞으로 나아갈 때 얻는 보상은 단계별로 매우 작습니다.
- **REINFORCE를 쓰려면** 유용한 신호를 얻기 위해 수천 개의 완벽한 에피소드가 필요할 것입니다.

PPO의 n-step 업데이트와 [GAE (Generalized Advantage Estimation)](./ppo_scratch_explained.md#gae-더-똑똑한-advantage-추정)를 사용하면 에피소드가 끝나지 않아도 학습할 수 있습니다.
> "50단계 만에 넘어졌더라도, 그 단계들 동안은 분명 앞으로 조금 나아갔어.
> 에피소드가 끝날 때까지 기다리지 말고 이 50단계의 리턴 추정치를 사용해서 배우자."

## 결과

500번의 업데이트(약 100만 번의 환경 단계) 후:
- 로봇은 무작위로 허둥대던 모습에서 벗어나 눈에 띄게 앞으로 나아갑니다.
- 학습 곡선이 꾸준히 상승합니다.
- 보상 300 이상의 완전한 수렴을 위해서는 더 많은 훈련(500만~1,000만 단계)이 필요합니다.

학습 곡선은 연속 제어의 특징적인 "S자 곡선"을 보여줍니다.
1. 느린 초기 진행 (학습 안정화 기간)
2. 급격한 향상 (보행 패턴 발견)
3. 점진적인 미세 조정 (보행 최적화)

## 핵심 요약

| 개념 | 설명 |
|---------|---------------|
| **가우시안 정책** | 메뉴에서 고르는 대신, 값의 범위 안에 다트를 던지는 방식 |
| **μ (평균)** | 정책이 "조준하는" 지점 |
| **σ (표준편차)** | 정책이 사용하는 무작위성/탐험의 정도 |
| **학습 가능한 파라미터로서의 log_std** | 경사 기반 최적화(보상에 대한 경사 상승, 또는 PPO 손실에 대한 경사 하강)에 의해 업데이트되는 전역 탐험률 — 다른 네트워크 가중치와 마찬가지로 학습됨 |
| **연속 제어 (Continuous control)** | 실수 값(토크, 힘, 각도 등)을 제어하는 것 |

## 다음 단계는?

PPO에는 수많은 **하이퍼파라미터(hyperparameters)**가 있습니다. 이는 훈련 시작 전에 직접 설정해야 하는 값들입니다 (네트워크 가중치처럼 자동으로 학습되는 *파라미터*와 대조됨). `clip_eps`, 학습률, 에포크 수, 배치 크기 등이 그 예입니다.

PPO가 이러한 설정값들에 얼마나 민감할까요? `ppo_hyperparams.py`는 각 하이퍼파라미터를 체계적으로 변경해가며 학습 속도와 안정성에 미치는 영향을 실험합니다.
