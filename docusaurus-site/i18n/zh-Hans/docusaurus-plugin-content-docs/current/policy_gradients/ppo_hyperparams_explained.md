# PPO 超参数敏感性：什么最重要？

## 为什么超参数很重要

想象一下烤巧克力蛋糕。食谱要求：
- 2 个鸡蛋
- 200 克面粉
- 1 茶匙发酵粉
- 180°C 烘烤 35 分钟

如果你用 10 个鸡蛋，蛋糕会爆炸。如果你只用 0.1 茶匙发酵粉，蛋糕就发不起来。如果你在 300°C 下烘烤 10 分钟，蛋糕外面焦了，里面还是生的。

**PPO 中的超参数就像食材和烤箱设置。** 正确的组合效果极佳；错误的设置可能会导致完全无法学习。

这个脚本通过一次只改变一个超参数，在 3 个不同的随机种子下运行每个设置，并比较结果，系统地测试了 3 个关键超参数。

## 三个实验

### 实验 1：裁剪 ε (Clip Epsilon)

```
ε = 0.05   (非常保守 —— 仅允许极小的策略变化)
ε = 0.2    (标准 —— 平衡了安全性和速度)
ε = 0.4    (激进 —— 允许较大的策略变化)
```

**ε 控制什么？**

ε 是旧策略周围“安全窗口”的大小：
```
概率比必须保持在 [1 - ε,  1 + ε] 之间
ε=0.05: 概率比在 [0.95, 1.05] 之间 ← 极小的变化
ε=0.2:  概率比在 [0.80, 1.20] 之间 ← 标准  
ε=0.4:  概率比在 [0.60, 1.40] 之间 ← 较大的变化
```

**现实生活中的例子**：把 ε 想象成“一次允许你转动方向盘多远”。
- ε=0.05：就像在冰上行驶 —— 只能做微小的调整。
- ε=0.2： 正常驾驶 —— 合理的转弯。
- ε=0.4： 赛车手 —— 激进的转向，有**甩尾/打滑 (spinning out)** 的风险（因为变化太剧烈而失去控制）。

**预期结果**：
- ε=0.05：学习缓慢但稳定（过于谨慎）。
- ε=0.2： 良好的平衡（**“金发姑娘”原则 (Goldilocks value)** —— 恰到好处，源自童话故事，指不冷不热，刚刚好）。
- ε=0.4： 学习可能很快，但可能会**超调和振荡 (overshoot and oscillate)**（超调 = 越过了最优策略；振荡 = 在最优策略周围来回摆动而无法稳定）。

---

### 实验 2：学习率 (Learning Rate)

```
lr = 1e-4  (慢但稳定)
lr = 3e-4  (标准)
lr = 1e-3  (快但冒险)
```

**学习率控制什么？**

学习率就像爬山时的“步长”（每一步 = 对神经网络权重的一次更新，向提高奖励的方向移动）：
- 太小：需要永远的时间才能到达山顶（收敛缓慢）。
- 太大：你会越过山峰掉到另一边（**发散 diverges** —— 训练奖励崩溃或剧烈波动，而不是稳定提高）。
- 恰到好处：稳步向顶峰迈进。

**现实生活中的例子**：给吉他调音。
- lr=1e-4：微调**弦钮 (peg)** —— 耗时很长但很精确。
- lr=3e-4：正常调音 —— 转几圈就能找到正确的音高。
- lr=1e-3：猛力**拽 (yanks)** 弦钮 —— 可能会**绷断 (snap)** 琴弦（就像过大的更新会不可逆地破坏训练一样）！

**预期结果**：
- lr=1e-4：最终效果不错，但非常缓慢。
- lr=3e-4：整体表现最佳。
- lr=1e-3：初期进展飞快，随后出现不稳定。

### 实验 3：更新轮数 (Update Epochs, K)

```
K = 3   (保守 —— 每个批次数据只跑几遍)
K = 10  (标准)
K = 20  (激进 —— 每个批次数据跑很多遍)
```

**更新轮数控制什么？**

在收集了一段**展开 (rollout)**（= 玩一段时间游戏以收集新经验）后，PPO 将这些经验打包成一个**批次 (batch)**。然后，它对同一组数据运行 K **轮 (passes)**（每轮遍历整个批次，每次更新一次网络）。
更多的轮数 = 从每个批次中压榨出更多的学习成果，但有**对陈旧数据过拟合 (overfitting to stale data)** 的风险（= 记住了在旧策略下成立但在策略更新后不再有效的模式）。

**现实生活中的例子**：一个学生练习 20 道数学题。
- K=3： 每道题做 3 遍 → 仍在学习，不会死记硬背练习集。
- K=10：每道题做 10 遍 → 熟练掌握了这些特定的题目。
- K=20：每道题做 20 遍 → **背下了答案，但并没有真正理解数学**（= 模型完美拟合了特定批次，但失去了泛化能力）！

> ⚠️ **“但是 K=20 的结果看起来还行 —— 为什么我要在意？”**
> PPO 的裁剪技巧限制了每轮策略的变化程度，因此 K=20 不会导致突然的崩溃。
> 然而，智能体仍然在悄悄地过度适应那些不再反映当前策略实际体验的数据。
> 这会**减慢长期的学习速度**：每次展开教给智能体的东西比它应得的要少，因为后面的轮次在回收利用越来越陈旧的信息。
> 这种损害是逐渐的，而不是戏剧性的 —— 这正是为什么在短期实验中很容易被忽视的原因。

裁剪防止了灾难性的过拟合，但过多的轮数仍然会减慢整体学习速度。

**预期结果**：
- K=3： 效率较低（每批次浪费了一些学习潜力）。
- K=10：良好的平衡。
- K=20：策略有对**陈旧数据过度自信**的风险（= 网络的更新是由不再符合当前策略遇到的体验驱动的，从而悄悄侵蚀了样本效率）。

## 如何阅读结果

图表显示了三个子图，每个子图改变一个超参数：

```
左图：   裁剪 ε —— 哪个 ε 学习最快？
中间图： 学习率 —— 哪个 lr 最稳定？
右图：   更新轮数 —— 哪个 K 找到了最好的策略？
```

每条线都是 **3 个随机种子的平均奖励**（以减少偶然性）。

**寻找什么**：
1. **学习速度**：哪条线最快达到高奖励？
2. **最终性能**：哪条线达到了最高的最终奖励？
3. **稳定性**：哪条线的振荡最少？

一个好的超参数可以平衡这三点！

## 方法论：科学实验

本实验采用**消融研究 (ablation study)** 设计（= 一次只改变或移除一个组件以衡量其单独影响的方法）：
1. 选择默认值：ε=0.2, lr=3e-4, K=10
2. 一次只改变一个参数。
3. 保持其他所有参数不变。
4. 比较结果。

这告诉了我们每个参数孤立存在时的影响。

**现实生活中的例子**：测试一种新肥料是否对植物有帮助。
- 改变肥料，保持其他一切相同（相同的土壤、水、阳光）。
- 如果植物长得更好 → 肥料起作用了！

## 实践中的常见发现

| 超参数 | 太小 | 黄金甜点位 | 太大 |
|----------------|-----------|------------|-----------|
| **ε (clip)** | 收敛缓慢 | ε ≈ 0.2 | 不稳定 |
| **lr (学习率)** | 太慢 | 2.5e-4 到 3e-4 | 发散 |
| **K (轮数)** | **浪费数据** (未提取完整信号就丢弃) | K = 4-10 | 对陈旧数据过拟合 |
| **n_steps** | 噪声太大 | 128-2048 | **显存溢出 (OOM)** |
| **batch_size** | 噪声太大 | 32-256 | **显存溢出 (OOM)** |

这些“甜点位”可能会根据环境的不同而发生变化！

## 核心洞察：PPO 相对稳健

与早期的算法（如没有目标网络的 DQN）相比，PPO 对超参数的选择相对稳健。裁剪机制提供了一个自然的“安全网”。

**现实生活中的例子**：带有 **ABS**（防抱死制动系统 —— 一种安全功能，防止车轮在猛烈刹车时锁死，保持驾驶员的控制）的汽车与没有 ABS 的汽车相比：
- 没有 ABS (DQN)：一个错误的动作（差的超参数）就会让你甩尾失控。
- 有 ABS (PPO)：汽车会自我纠正 —— 合理的超参数都能正常工作。

这种鲁棒性是 PPO 在实践中成为最受欢迎的强化学习算法的主要原因！

## 关键要点

| 概念 | 通俗解释 |
|---------|---------------|
| **消融研究** | 一次只改变一件事，看看它的效果 |
| **裁剪 ε (Clip epsilon)** | 安全边界 —— 0.2 通常是最好的 |
| **学习率 (Learning rate)** | **步长** —— 每次更新后网络权重调整的幅度。**2.5e-4 到 3e-4** 是 0.00025 到 0.0003 的科学计数法 —— 它们是无量纲的乘数，不是时间值 |
| **更新轮数 K** | 每个批次数据重复利用多少次 —— 标准是 4-10 次 |
| **随机种子 (Random Seeds)** | 每个实验都使用不同的随机种子重复运行。使用多个种子可以揭示结果是连贯一致的，还是仅仅靠运气 |

## 总结：策略梯度算法一览

```
REINFORCE              A2C                    PPO
     │                  │                      │
完整回合更新           N 步更新               N 步 + 裁剪
简单但噪声大           更快但不稳定           稳定且高效
适合简单环境           中等难度环境           困难环境
                      (工业标准)
```
