# REINFORCE：教会机器人做出更好的选择

## 我们想要做什么？

想象你有一个正在玩视频游戏的机器人。每一秒，机器人都必须做出选择：
**“我是应该按下按钮，还是不按？”**

我们不希望机器人像 Q-learning 那样在表中记住每种情况，而是希望它学习一套**策略 (Policy)**——一套直接说明“在这种情况下，执行这个动作”的规则。

这套规则被称为**策略** (π, pi)。在强化学习中，π 的意思是“选择动作的规则”。

---

## 旧方法 vs 新方法 {#the-old-way-vs-the-new-way}

**旧方法 (Q-learning / DQN)：** 学习每个动作有多“好”（Q 值），然后选择最好的。
> “向左推得 7 分，向右推得 5 分 → 向左推！”

**新方法 (策略梯度 - Policy Gradient)：** 直接学习应该“选择”哪个动作。
> “当杆子向右倾斜时，以 80% 的概率向右推，以 20% 的概率向左推。”
*（**梯度 Gradient** 这个词指的是我们为了让这些概率慢慢向正确方向调整而采取的数学“步骤”。）*

**现实生活中的例子：** 学习骑自行车。
- 旧方法：计算“向左倾斜 5 度”与“向左倾斜 7 度”的*精确分数*。
- 新方法：不断练习，直到你的**身体**学会——自然而然地做出感觉正确的动作！

---

## REINFORCE 是如何工作的？

REINFORCE 观察机器人从头到尾玩完一局游戏（一个**回合 episode**），然后问：“哪些动作导致了高分？让我们多做些那样的动作！”

### 分步解析

**1. 运行一个回合**

机器人做出选择并收集经验：
```
第 1 步：状态 = [杆子向右倾斜] → 动作 = 向右推 → 奖励 = +1
第 2 步：状态 = [杆子接近平衡] → 动作 = 向右推 → 奖励 = +1
第 3 步：状态 = [杆子向左倾斜] → 动作 = 向左推 → 奖励 = +1
...
第 47 步：状态 = [杆子倒了！] → 回合结束
```

**2. 计算回报**

对于每一步，计算 G_t —— **从该点开始的总奖励**：
```
第 47 步的 G = 1
第 46 步的 G = 1 + 0.99 × 1 = 1.99
第 45 步的 G = 1 + 0.99 × 1.99 = 2.97
...
第 1 步的 G  = 47（大约值 —— 因为它是从开始计算的，所以回报更高）
```

γ = 0.99 的**折扣因子 (discount factor)** 意味着遥远未来的奖励价值稍低。

**现实生活中的例子：** 开学第一天得到一颗金星奖章，比知道在第 100 天*可能*得到一颗更令人兴奋。未来的奖励被轻微地“打折”了。

**3. 更新策略**

对于采取的每个动作：
> 如果 G_t 很高（该动作导致了极好的结果）：**以后多做这个动作！**
> 如果 G_t 很低（该动作导致了糟糕的结果）：**以后少做这个动作！**

数学公式：`loss = -log_prob(action) × G_t`

获取梯度并更新策略就像是在告诉机器人：
*“你在第 20 步采取的那个动作？下次你应该多增加 5% 的频率去执行它！”*

## 什么是策略网络？

我们不使用表格，而是使用**神经网络**来表示策略。

```
观测值 (Observation)      策略网络 (Policy Network)       动作概率
[小车位置]  →   [128 个神经元]  →  →  [向左推: 30%]
[小车速度]  →   [128 个神经元]        [向右推: 70%]
[车杆角度]  →
[车杆角速度] →
```

网络输出每个动作的**概率**。然后我们进行采样：
> 掷骰子 → 1-30：向左推，31-100：向右推

**现实生活中的例子：** 天气预报说“降水概率 70%”。你并不确定一定会下雨——你是基于概率做出决定的。机器人也是如此！

---

## 归一化：为什么要减去均值

在使用 G_t 进行更新之前，我们会进行归一化：
```
G_normalized = (G - mean(G)) / std(G)
```

**为什么？** 想象所有的奖励都是正数（在 CartPole 中确实如此——每一步都是 +1）。如果没有归一化，每一个动作看起来都挺“好”，更新信号就会变得混乱。

归一化后，一些回报变为正数（高于平均水平 → 增加概率），一些变为负数（低于平均水平 → 减少概率）。信号变得清晰得多！

**现实生活中的例子：** 老师按比例评分。如果平均分是 70 分，你得了 85 分，那很棒！但如果平均分是 90 分，你得了 85 分，那就是低于平均水平。单凭原始分数并不能说明全部情况。

---

## 问题：高方差

REINFORCE 有一个巨大的弱点：**方差 (variance)**。回报 G_t 的噪声非常大。

**现实生活中的例子：** 想象仅通过每家餐厅的一道菜来评判厨师。有时厨师状态不佳，有时食材不新鲜。仅仅一道菜不足以可靠地判断这家餐厅是否优秀！

REINFORCE 会等待整个回合结束才进行更新。一个回合可能运气极好，另一个回合可能运气极差。梯度会四处跳动。

这就是为什么训练曲线（图表中）看起来坎坷不平：
- 有些运行得分 500（太棒了！）
- 有些运行跌至 50（太惨了！）

尽管有噪声，REINFORCE 最终还是能学会——但需要耐心。

## 实验结果

```
第 100 回合 | 最近 100 回合平均奖励:  43.1
第 200 回合 | 最近 100 回合平均奖励: 193.9
第 500 回合 | 最近 100 回合平均奖励: 408.4
第 1000 回合 | 最近 100 回合平均奖励: 500.0  ← 任务完成！
```

机器人学会了平衡车杆达到最高 500 步——成功解决！

尽管存在方差问题，REINFORCE 在 CartPole 上依然有效，因为：
1. 回合较短（所以我们在每次训练中能获得很多回合）
2. 最优策略相对简单（主要是朝车杆倾斜的方向推）

## 核心总结

| 概念 | 通俗解释 |
|---------|---------------|
| **策略 (Policy)** | 机器人选择动作的“秘诀” |
| **回合 (Episode)** | 从开始到结束的一局完整游戏 |
| **回报 G_t** | 从当前时刻开始的总未来奖励 |
| **折扣 γ** | 未来的奖励比眼前的奖励价值稍低 |
| **归一化 (Normalize)** | 减去平均值，使信号更清晰 |
| **方差 (Variance)** | 梯度估计值的波动程度 |

---

## 下一步是什么？

REINFORCE 的最大弱点是**方差**。在下一个脚本 (`reinforce_baseline.py`) 中，我们添加了**基准线 (baseline)**，它在不改变算法平均学习内容的前提下，极大地减少了这种噪声。

核心思想：与其问“这个动作好吗？”，不如问“这个动作是否**优于预期**？”这个微小的改变能让学习过程变得稳定得多。
