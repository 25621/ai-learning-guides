# 带基准线的 REINFORCE：穿透噪声

## 普通 REINFORCE 的问题

想象你是一名学生，正在试图判断你在考试中的答案是否正确。

**糟糕的反馈：** “你得了 7 分！”

7 分算好吗？如果总分是 10 分，那确实不错！如果其他人都得了 9 分，那就不行！如果没有背景信息，你无法判断是否应该改变你的答题风格。

这正是 REINFORCE 的问题所在：它使用**原始回报** (G_t) 来评估动作。200 分的总回报可能意味着表现极佳，也可能意味着表现糟糕，这完全取决于具体情况。

---

## 引入基准线 (Baseline)

**基准线** b(s) 是一个参考点：“在这种情况下，我**预期**能获得多少奖励？”

我们不再问“这个动作好吗？”，而是问：

> **“这个动作是否比我通常预期的要好？”**

```
旧信号：更新量 ∝ G_t
新信号：更新量 ∝ (G_t - b(s_t))
```

**现实生活中的例子：** 你在数学考试中得了 85 分。
- 如果班级平均分是 60 分 → 你的成绩**比平均分高出 25 分** → 太棒了！
- 如果班级平均分是 90 分 → 你的成绩**比平均分低了 5 分** → 还需要努力！

当你的表现优于预期时，**优势 (Advantage)** (G_t - b(s)) 为正；当表现差于预期时，优势为负。这是一个清晰得多的学习信号！

---

## 什么是基准线？

自然的基准线是**价值函数 V(s)**：

> V(s) = “如果我处于状态 s 并按照当前策略进行游戏，预期的总奖励是多少”

我们通过一个单独的**价值网络**（也称为基准网络或评论员/Critic）来学习它：

```
状态  →  [128 个神经元]  →  [128 个神经元]  →  V(s)   (单个数字)
```

对于智能体访问的每个状态，V(s) 都会预测预期的回报。如果实际回报 G_t 高于 V(s)，则说明该动作优于预期！

---

## 两个网络共同学习

```
回合发生
     ↓
计算实际回报 G_t
     ↓
         ┌─────────────────────────────┐
         │ 优势 (Advantage) = G_t - V(s_t) │
         │  +: 动作表现更好             │
         │  -: 动作表现较差             │
         └─────────────────────────────┘
              ↓                  ↓
       更新策略网络          更新价值网络
    (使好的动作出现几率    (使预测在下次
     更高/更低)             更加准确)
```

**现实生活中的例子：** 两个朋友一起去餐厅。

- 朋友 A（价值网络）：“我预测这道菜能得 7/10 分”
- 朋友 B（策略网络）：你尝试了这道菜并给出了 9/10 分的评价
- 优势 = 9 - 7 = +2 → “这比预期的要好！下次再点！”

下次光顾时，朋友 A 会将预测值更新为接近 9/10。
朋友 B 下次更有可能点这道菜。

---

## 为什么这能降低方差？

**数学直觉：**

不带基准线：`梯度 ∝ ∇log π(a|s) × G_t`

G_t 的值在不同回合之间波动很大：
```
回合 1: G = [45, 44, 43, ..., 1]   (中等局)
回合 2: G = [500, 499, ..., 1]      (超神局！)
回合 3: G = [12, 11, ..., 1]        (惨败局)
```

由于 G_t 数值大且充满噪声，梯度估计会剧烈跳动。

带基准线：`梯度 ∝ ∇log π(a|s) × (G_t - V(s_t))`

优势 (G_t - V(s_t)) 的数值要小得多，且集中在零附近：
```
回合 1: 优势 ≈ [-2, +1, -3, ..., 0]   (数值小，居中)
回合 2: 优势 ≈ [+10, +8, ..., +3]      (这局确实打得好)
回合 3: 优势 ≈ [-5, -6, ..., -2]       (这局确实打得烂)
```

**现实生活中的例子：** 测量你的跑步速度。
- 不带基准线：“我跑了 8 公里/小时”（没有背景信息，毫无意义）
- 带基准线：“我比我的平均速度快了 2 公里/小时”（显然很棒！）

优势始终是一个比较值——它自然更小且更稳定。

---

## 至关重要：无偏性！

基准线不会改变算法学习的**内容**——它只改变学习的**速度**和**稳定性**。

**为什么？** 因为从期望值来看，预期优势始终为 0：

> E[G_t - V(s_t)] = E[G_t] - V(s_t) = V(s_t) - V(s_t) = 0

任何不依赖于具体动作的 b(s) 都可以作为有效的基准线！

**现实生活中的例子：** 按比例评分（拉开档次）不会改变谁表现最好——它只是让分数更容易解释。排名保持不变，改变的只是衡量尺度。

---

## 实验结果

```
不带基准线  — 最终 100 回合平均分: 500.0, 梯度方差: 599.3
带基准线 — 最终 100 回合平均分: 491.4, 梯度方差: 578.8
```

两种方法在 CartPole 环境中都能达到近乎完美的表现，但请注意：
1. **梯度方差**是可以测量的（右侧图表显示了训练过程中的方差）
2. 使用基准线后，智能体能够**更可靠地**达到高性能——在训练过程中，因噪声导致的奖励暴跌情况更少发生。

在更复杂的环境（如 LunarLander, MuJoCo）中，方差缩减的效果会更加显著。

---

## 核心公式

```
基准线值更新：   V(s) ← V(s) + α(G_t - V(s))   [最小化均方误差 MSE]
策略梯度更新：   θ ← θ + α ∇log π(a_t|s_t) · (G_t - V(s_t))
优势：           A_t = G_t - V(s_t)
```

---

## 核心总结

| 概念 | 通俗解释 |
|---------|---------------|
| **基准线 b(s)** | 状态 s 的预期奖励——我们的参考点 |
| **优势 A_t** | “这个动作是否比预期的要好？” |
| **价值网络** | 一个学习预测预期回报的神经网络 |
| **方差缩减** | 梯度估计中的噪声减少 → 训练更稳定 |
| **无偏 (Unbiased)** | 基准线平均而言不会改变目标策略；它只让学习信号噪声更小、更稳定 |

---

## 下一步是什么？

基准线实际上是某种更强大方法的基础：**演员-评论员 (Actor-Critic)** 方法。

Actor-Critic 不是在回合结束时才计算 V(s)，而是在每一步都使用**时序差分 (Temporal Difference)** 学习来更新 V(s)。这使得更新速度更快，并允许智能体从不完整的环节中学习！

请参阅 `a2c_lunarlander.py` 获取完整的 Actor-Critic 实现。
