# A2C：演员与评论家的协同工作

## 核心思想

REINFORCE 算法必须等到游戏完全结束后才进行更新。这就像是一个教练在整场足球比赛中保持沉默，直到比赛结束才给出所有反馈。

**A2C (Advantage Actor-Critic)** 在游戏进行过程中给出反馈——每隔几步，教练就会暂停并说：“那次传球很棒！那次铲球很糟糕！”

这使得学习速度更快且更高效。

---

## 认识两个角色

> **什么是 LunarLander？** 在本文档中，我们使用 **LunarLander** 环境——这是一个物理模拟环境，你需要控制一艘小型航天器，使用三个引擎（左侧、主引擎和右侧）使其平稳降落在月球的目标着陆点上。它是强化学习中的一个标准基准，可在 Gymnasium 库中找到。

### 演员 (Actor) 🎭
**演员**就是策略（Policy）——它决定采取哪种行动。

> “我处于这种状态。我应该启动左引擎还是右引擎？”

**现实生活中的例子：** 汽车的 *驾驶员*，负责转动方向盘和踩踏板。

### 评论家 (Critic) 🎬
**评论家**负责估算当前情况的好坏——即价值 V(s)。

> “处于这种状态大约值 +150 分的未来总奖励。”

**现实生活中的例子：** 坐在驾驶员旁边的 *导航员*，会说：“我们走在一条好路上——预计 30 分钟后到达。” 或者 “我们要进入拥堵路段了——这会很慢。”

### 他们共享一个大脑
在我们的实现中，两者使用 **相同的神经网络主干**：

```
          状态 (LunarLander 为 8 个数字)
                       ↓
          ┌─────────────────────────┐
          │  共享层 (Shared Layers)  │
          │  [256 个神经元] → ReLU   │
          │  [256 个神经元] → ReLU   │
          └────────┬────────┬───────┘
                   ↓        ↓
          演员头 (Actor Head)  评论家头 (Critic Head)
          [4 个输出]            [1 个输出]
          (动作概率)            (V(s))
```

- **ReLU** (修正线性单元)：每层之后应用的激活函数——它输出 `max(0, x)`，保留正值并将负值置为零。这使得网络能够学习非线性模式。
- **动作概率 (action probs)**：采取 4 种动作中每一种的概率。演员每一步都会从这个分布中采样来选择一个动作。

**现实生活中的例子：** 一个大脑，两份工作——就像一个既能开车（演员）又知道路线好坏（评论家）的出租车司机。共享大脑意味着学习速度更快！

## 优势 (Advantage)：这比预期的好吗？

就像带有基准的 REINFORCE 一样，A2C 计算 **优势 (Advantage)**：

> A(s, a) = “实际结果” − “我们的预期”

但在 A2C 中，“实际结果”来自评论家的 **n 步自举 (n-step bootstrap)**（**自举 (bootstrapping)** = 使用评论家自己的预测 V(s) 来近似未来步数的价值，而不是等待实际回合结束——这就像在学期中利用当前的成绩来预估期末考试分数）：

```
实际 TD 回报: r_t + γ · r_{t+1} + γ² · r_{t+2} + ... + γⁿ · V(s_{t+n})
优势 A_t = TD 回报 - V(s_t)
```

**现实生活中的例子：** 你预期在这场比赛中进 3 个球 (V(s))。如果你进了 5 个球，你的优势就是 +2。如果你只进了 1 个球，你的优势就是 -2。

正优势 → “那个动作比预期的更有帮助 → 多做这个动作！”
负优势 → “那个动作的帮助比预期的少 → 少做这个动作！”

## 为什么使用多个并行环境？

我们的 A2C 使用 **8 个副本** 的 LunarLander 同时运行！

**为什么？** 因为来自单个环境的经历是 **相关的 (correlated)**——每一步都紧随前一步。这种相关性会误导神经网络，使其认为某些模式比实际更可靠。

通过 8 个环境，每一步都能提供来自完全不同情况的 8 个独立经历。这打破了相关性，并显著稳定了训练过程。

**现实生活中的例子：** 为了了解天气，哪种方式更好：
- 连续 8 小时观察一个城市（相关的——如果下午 2 点是晴天，下午 3 点可能也是晴天）
- 同时观察 8 个城市（去相关的——不同的天气模式，更多的信息！）

```
环境 1: [降落在月球上, 启动左引擎, 坠毁, 重置...]
环境 2: [下降太快, 同时启动两个引擎, 悬停, 着陆...]
环境 3: [向右倾斜, 启动右引擎, 稳定, 着陆...]
...
环境 8: [向左漂移, 启动左引擎, 平稳, ...]
```

所有 8 个环境同时更新网络——每次更新可获得 8 倍多样化的经历！

## N 步更新：不要等到游戏结束

REINFORCE 等待完整的一个回合（可能是 1000 步！）。

A2C 每隔 **n_steps = 128 步** 进行一次更新：

```
在 8 个环境中运行 128 步
    → 获得 128 × 8 = 1024 个经历元组
    → 计算优势和回报
    → 更新演员和评论家
    → 再运行 128 步...
```

**现实生活中的例子：** 一个学生在为考试复习。
- REINFORCE 风格：读完整个教科书，然后做模拟测试。
- A2C 风格：读 10 页，做个小测验，再读 10 页，再做个测验...

更频繁的反馈 = 更快的学习！

## 三种损失结合

A2C 同时使用三个损失项进行训练：

**损失 (Loss)** 是优化器试图最小化的数字。损失越小意味着网络当前的行为越接近训练目标。

### 1. 演员损失 (策略梯度)
使产生优势的动作可能性更大：
```
L_actor = -E[log π(a|s) · A(s,a)]
```
如果 A > 0：增加该动作的概率
如果 A < 0：减少该动作的概率

### 2. 评论家损失 (价值函数 MSE)
使价值预测更准确（**MSE** = 均方误差：对预测误差取平方并求平均值——平方会对大错误比小错误给予更重的惩罚）：
```
L_critic = E[(V(s) - return)²]
```
就像训练任何 **回归 (regression)** 模型一样（回归 = 预测一个连续的数字，这里是预期回报 V(s)）——最小化预测误差。

### 3. 熵奖励 (探索)
防止策略过快变得过于自信：
```
L_entropy = -H[π(·|s)] = E[log π(a|s)]
```
高熵 = 多样化的动作选择 = 探索
低熵 = 自信、狭窄的选择 = 利用

**现实生活中的例子：** 熵奖励就像一个老师说：“不要在每个多选题上都猜 A！尝试不同的答案，这样你才能学到什么是有效的。”

```
总损失 = L_actor + 0.5 × L_critic - 0.01 × 熵
```

## LunarLander：更艰巨的挑战

**LunarLander-v3** 是一个 Gymnasium（原 OpenAI Gym）环境——“v3”是版本号，表示该环境的第三次修订。智能体控制一艘必须安全降落在月球指定平台上的小型航天器。它比 CartPole 难得多：
- 8 维状态空间（位置、速度、角度、腿部接触、燃料）
- 4 种离散动作（什么都不做、启动左引擎、启动主引擎、启动右引擎）
- 奖励：成功着陆 +100，坠毁 -100，少量的燃料惩罚

训练曲线显示奖励从极负值逐渐向正值提升。在 LunarLander 上运行 A2C 在着陆器学会基本稳定性之前需要大量的经验。

---

## 关键方程

```
n 步回报:    G_t = r_t + γ·r_{t+1} + ... + γⁿ·V(s_{t+n})
优势:        A_t = G_t - V(s_t)
演员更新:    θ_π ← θ_π - α ∇ L_actor
评论家更新:  θ_V ← θ_V - α ∇ L_critic
```

---

## 关键要点

| 概念 | 通俗解释 |
|---------|---------------|
| **演员 (Actor)** | 策略——决定做什么 |
| **评论家 (Critic)** | 价值函数——判断情况有多好 |
| **优势 (Advantage)** | “这比预期的好吗？”（实际 - 预期） |
| **N 步回报** | 在使用 V(s) 进行自举之前，先向未来看 n 步 |
| **并行环境** | 多个环境以获得去相关、多样化的经验 |
| **熵奖励** | 鼓励演员不断尝试新事物 |

## 下一步

A2C 很好，但它有一个主要弱点：有时它更新策略过于激进。一次糟糕的更新可能会摧毁之前更新中学到的所有好成果。

**PPO (Proximal Policy Optimization)** 通过一个聪明的“安全裁剪”来解决这个问题，防止任何单次更新对策略改变太大。

查看 `ppo_scratch.py` 了解 PPO 的实现！
