# 连续控制的 PPO：让双足机器人行走 (BipedalWalker)

## 离散动作 vs. 连续动作

到目前为止，我们解决的每个环境都使用的是**离散 (discrete)** 动作：
- CartPole：向左推或向右推（2 个选择）
- LunarLander：不发动/左/主/右引擎（4 个选择）

但现实世界的机器人需要**连续 (continuous)** 动作：
- 人形机器人：“给每个关节施加多大的力”（从 -1 到 +1 的任何值）
- 汽车：“方向盘具体转动多少度”（从 -30° 到 +30° 的任何角度）
- 机械臂：“在这个方向上精确施加 2.3 牛顿的力”

**现实生活中的例子**：在键盘上打字 = 离散动作（按 A, B, C...）。用铅笔写字 = 连续动作（向右移动手 2.3 厘米，施加 40 克压力...）。

## 连续动作的高斯策略 (Gaussian Policy)

对于连续动作，我们不再使用分类分布（Categorical distribution，从 N 个类别中选择），而是使用**正态分布（高斯分布）**：

```
动作 ~ 正态分布(μ, σ)
```

其中：
- **μ (mu, 均值)**：分布的中心 —— 网络“瞄准”的动作值
- **σ (sigma, 标准差)**：离散程度 —— 要添加多少随机性/探索

```
          概率
             │
        0.4 ─┤      ██████
             │    ████████████
        0.2 ─┤  ██████████████████
             │████████████████████████
             └──────────────────────── 动作值
           -1  -0.5   0   0.5   1
                      ↑
                    均值 μ
```

**现实生活中的例子**：一名熟练的弓箭手瞄准靶心 (μ)。他们的箭不会全部落在完全相同的位置 —— 会有一定的分布 (σ)。随着他们的练习，他们在保持瞄准靶心的同时，准确度会提高（σ 减小）。

## 我们的高斯演员-评论家 (Gaussian Actor-Critic) 网络

```
状态 (24 个数值) → [256 个神经元] → [256 个神经元] →
    ├── 演员 (Actor): 4 个均值 (μ₁, μ₂, μ₃, μ₄)
    │                + 4 个 log_std 参数 (在所有状态间共享!)
    └── 评论家 (Critic): 1 个价值 (V(s))
```

`log_std`（**标准差**的对数 —— 衡量离散程度或不确定性）是一个**可学习的参数**，它不依赖于状态。这既保持了简单性，又允许探索在训练过程中发生变化。

**为什么用 log_std 而不是 std？** 标准差必须是正数。使用 `log_std` 允许网络输出任何实数（正数或负数），然后我们应用 `exp(log_std)`（指数函数，对数的逆运算）来恢复一个保证为正数的标准差。这防止了标准差变成负数或零。

## 计算连续动作的对数概率

对于离散动作：`log_prob = log(P(action=LEFT))`

对于连续动作，**正态分布**描述了均值周围的一条平滑的钟形曲线。在连续数学中，单个精确值的概率为零，因此我们使用该值处的曲线高度，称为 **pdf**（概率密度函数）：
```
log_prob = Σᵢ log[Normal(μᵢ, σᵢ).pdf(aᵢ)]
```

`log` 表示自然对数。它将极小的密度值转化为稳定的数值，更容易让神经网络进行优化。我们对所有动作维度（对于 BipedalWalker 是 4 个）求和，因为完整的动作是一个由 4 个数字组成的向量。

**现实生活中的例子**：明天温度恰好是 5.732...°C 的概率是多少？对于连续的天气，你会查看正态分布曲线，看那个精确点的高度。更可能的温度（靠近均值）具有更高的概率。

## BipedalWalker：一个行走的挑战

BipedalWalker-v3 是一个必须学会行走且不摔倒的 2D 机器人：

```
          O (头部)
         /│\
        / │ \
       /  │  \
      L   │   R   ← 两条腿，每条腿都有一个膝关节
     / \  │  / \
    ●   ● │ ●   ●  ← 4 个电机 (每条腿有臀部/膝盖)
```

**状态空间 (24 个数值)：**
- 躯干：角度、角速度、水平速度、垂直速度 (4 个)
- 关节：4 个电机（2 个臀部，2 个膝盖）各自的角度和速度，加上 2 个地面接触传感器（每条腿一个）(10 个)
- 10 个激光雷达 (LIDAR) 测距传感器（看到前方地面的距离读数）(10 个)

**动作空间 (4 个连续值，均在 [-1, 1] 范围内)：**
这些动作值控制正好 4 个关节的**扭矩**（电机施加的旋转力）（没有动作直接作用于躯干）：
- 腿 1 臀部扭矩，腿 1 膝盖扭矩，腿 2 臀部扭矩，腿 2 膝盖扭矩

**奖励：**
- 到达目标（右侧）获得 +300
- 摔倒（身体接触地面）获得 -100
- 每向前移动一步获得少量奖励
- 每次使用电机有少量惩罚（鼓励效率）

**解决标准**：100 个回合内的平均奖励 > 300

## 与离散 PPO 的主要区别

除了以下内容，其他都一样：

| | 离散 PPO | 连续 PPO |
|---|---|---|
| **策略** | 分类分布 Categorical(logits) | 正态分布 Normal(μ, σ) |
| **采样** | 动作 = 从 {0,1,...,N} 中采样 | 动作 = μ + σ × 噪声 |
| **log_prob** | log P(action=k) | Σ log Normal(μᵢ, σᵢ).pdf(aᵢ) |
| **钳制 (Clamp)** | 不需要 | 将动作钳制在 [-1, 1] |

**Logits** 是离散动作的原始、未归一化的分数。分类策略使用 **softmax** 将它们转换为概率 —— softmax 函数接收任何一组数字并将其压缩成有效的概率分布（所有值均为正，总和为 1）。例如，logits [2.0, 1.0, 0.5] 变成概率 [0.59, 0.24, 0.17]。连续 PPO 动作本身**不**使用 softmax，因为动作不是从固定菜单中选择的。相反，策略输出正态分布的均值和标准差，然后从中采样实值扭矩。

**钳制 (Clamp)** 意味着强制一个值在有效范围内。代码使用 `action.clamp(-1, 1)`，这样环境就不会收到超出其允许范围的电机指令。

**裁剪 (Clip)** 在 PPO 中有不同的含义：PPO 裁剪损失函数内部的概率比，如 [PPO 裁剪部分](./ppo_scratch_explained.md#the-clipping-trick) 所述。动作钳制保护环境接口；PPO 裁剪保护策略更新。

## 从零开始行走：智能体学到了什么

**训练早期（负奖励）**：机器人随机挥动双腿，立即摔倒。每个回合都在几秒钟内以碰撞结束。

**训练中期**：机器人发现交替移动双腿可以产生向前的进展。它开始迈出细小、别扭的步伐 —— 奖励变得不那么负了。

**训练后期**：出现了一种平滑、高效的行走**步态 (gait)**。步态是一种重复的运动模式，如左右交替迈步。机器人通过利用激光雷达传感器动态调整，实时适应不平整的地形。

**现实生活中的例子**：婴儿学走路：
1. 立即摔倒（负奖励）
2. 走一步，摔倒（负值稍小）
3. 走几步（少量正奖励）
4. 穿过房间（大量正奖励！）

## 为什么 BipedalWalker 需要 PPO (而不是 REINFORCE)

- **BipedalWalker 回合**可能长达 1600 步（比 CartPole 长得多！）
- **奖励是稀疏的** —— 每一步向前迈进的奖励微乎其微
- **REINFORCE 需要**数千个完整的回合才能获得有用的信号

PPO 配合 [GAE (广义优势估计)](./ppo_scratch_explained.md#gae-smarter-advantage-estimates) 进行 n 步更新，让机器人可以从不完整的回合中学习：
> “虽然我在 50 步后摔倒了，但这些步骤展示了一些向前的进展。
> 让我使用这 50 步的回报估计，而不是等待回合结束。”

## 结果

经过 500 次更新（约 100 万个环境步）：
- 机器人取得了明显进展，从随机挥动转向了一些向前的移动
- 学习曲线持续改善
- 完全收敛到奖励 > 300 需要更多的训练（500-1000 万步）

学习曲线展示了连续控制典型的“S 曲线”：
1. 初期进展缓慢（学习稳定性）
2. 快速提升（步态发现）
3. 逐渐精炼（步态优化）

## 关键要点

| 概念 | 通俗解释 |
|---------|---------------|
| **高斯策略** | 不再从菜单中选择，而是在一定数值范围内投掷飞镖 |
| **μ (均值)** | 策略“瞄准”的地方 |
| **σ (标准差)** | 策略使用的随机性/探索程度 |
| **log_std 作为可学习参数** | 一个由基于梯度的优化（对奖励进行梯度*上升*，或等效地对 PPO 损失进行梯度*下降*）更新的全局探索率 —— 就像任何其他网络权重一样 |
| **连续控制** | 控制实值输出（扭矩、力、角度） |

## 下一步是什么？

PPO 有许多**超参数 (hyperparameters)** —— 在训练开始前你选择的设置（与网络权重等通过自动学习获得的*参数*不同）。例子包括 `clip_eps`、学习率、轮数 (epochs) 和批次大小 (batch size)。

PPO 对这些选择有多敏感？`ppo_hyperparams.py` 通过系统地改变每个超参数进行实验，展示了它们对学习速度和稳定性的影响。
