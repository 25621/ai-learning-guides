# PPO：安全且稳定的策略更新

## A2C 存在的问题

想象你正在练习在手指上平衡一根扫帚。经过几周的练习，你可以让它保持平衡 30 秒！

现在你的教练给了你一个建议：“手腕稍微再向左倾斜一点。”

**好的建议 → 谨慎的改变 → 依然能保持平衡 30 秒 ✓**

但如果教练反应过度，大喊：“立即向左大幅度倾斜！”
你矫枉过正 → 扫帚倒了 → 你失去了几周练习积累的成果。

这就是 A2C 的问题：**大幅度的梯度更新可能会摧毁一个好的策略**。

**PPO (近端策略优化 Proximal Policy Optimization)** 就是一套防止这种情况发生的安全系统。

---

## 核心理念：靠近已经行之有效的方法

PPO 的关键约束：

> **“在单次更新中，不要过度改变策略。”**

在更新之前，我们有一个“旧”策略 π_old。
在更新之后，我们有一个“新”策略 π_new。

PPO 通过**概率比 (probability ratio)** 来衡量策略改变了多少：

```
r(θ) = π_new(a|s) / π_old(a|s)
```

- r = 1.0：策略未改变
- r = 1.5：新策略采取该动作的可能性增加了 50%
- r = 0.5：新策略采取该动作的可能性减少了 50%

**现实生活中的例子**：你是一名正在调整食谱的厨师。
- r = 1.0：盐的用量和以前一样
- r = 2.0：盐量翻倍 —— 太极端了！
- r = 0.9：少放 10% 的盐 —— 微小且安全的改变

## 裁剪技巧 (The Clipping Trick)

PPO 将概率比限制在 [1-ε, 1+ε] 范围内（通常 ε = 0.2）：

```
L_CLIP = E[min(r(θ) · A,  clip(r(θ), 1-ε, 1+ε) · A)]
```

让我们分解一下：

**情况 1：动作是好的 (A > 0)**

我们希望更多地执行这个动作 (r > 1)。但我们限制了增加的幅度：
```
如果 r > 1.2：裁剪到 1.2，不再有进一步推动的动力
```
这防止了我们在一个方向上偏移得太远。

**情况 2：动作是差的 (A < 0)**

我们希望减少执行这个动作 (r < 1)。同样地，我们也设置了限制：
```
如果 r < 0.8：裁剪到 0.8，不再有进一步惩罚的动力
```

**直观图示：**
```
ε = 0.2，因此安全的概率比窗口是 0.8 到 1.2。

好的动作 (A > 0)：增加动作概率，但在 1.2 之后停止奖励
概率比 r:       0.6      0.8      1.0      1.2      1.4
激励信号:        ↑        ↑        ↑        ↑        -
含义:         太低      可以      旧的     最大     被裁剪

差的动作 (A < 0)：减少动作概率，但在 0.8 以下停止奖励
概率比 r:       0.6      0.8      1.0      1.2      1.4
激励信号:        -        ↓        ↓        ↓        ↓
含义:         被裁剪     最大     旧的     可以      太高
```

`-` 标记了平坦的裁剪区域。在该区域内，即使让概率比变得更加极端也不会改善目标函数，因此 PPO 没有额外的动力去进一步推进。

**现实生活中的例子**：汽车的限速器。你可以加速，但一旦达到 120 km/h，限速器就会起作用，不让你开得更快。它在不阻止你移动的情况下保证了你的安全。

## 为什么这能防止灾难性的更新

**灾难性更新 (catastrophic update)** 是指一次巨大的策略改变完全摧毁了智能体已经学到的一切 —— 几个小时的训练成果在一次梯度更新中化为乌有。

如果没有裁剪：一个巨大的梯度步长可能会剧烈改变策略。
有了裁剪：在 [1-ε, 1+ε] 范围之外梯度为零，因此策略每步只能移动一点点。

**现实生活中的例子**：一名优秀的医生动手术时切口小而精确 —— 而不是大面积切割。PPO 就是强化学习优化器中那名“细心的医生”。

## GAE：更明智的优势估计

PPO 使用 **广义优势估计 (GAE, Generalized Advantage Estimation)** 来计算优势函数：

```
δ_t = r_t + γ · V(s_{t+1}) - V(s_t)          (TD 误差)
A_t = δ_t + γλ · δ_{t+1} + (γλ)² · δ_{t+2} + ...
```

GAE 有一个参数 λ (lambda)：
- λ = 0：仅使用一步 TD 误差（低方差，高偏差）
- λ = 1：使用完整的蒙特卡洛回报（高方差，低偏差）
- λ = 0.95：两者的良好平衡！

**现实生活中的例子**：规划公路旅行。
- λ=0：只看接下来的 5 英里（安全，但可能会错过后面的捷径）。
- λ=1：考虑整个 500 英里的旅程（信息更多，但非常不确定）。
- λ=0.95：看得远，但对附近的道路赋予更高的权重 ← 最佳平衡！

## 多轮迭代：高效复用数据

在收集了一批经验（展开 rollout）之后，REINFORCE 在一次更新后就会将其丢弃。

PPO 会将每批数据重复利用 **K 轮 (epochs)**（通常是对同一组数据进行 4-10 次遍历）：

```
收集 512 步 × 4 个环境 = 2048 个转换 (transitions)
第 1 轮：32 个小批次 (minibatches) × 分别更新
第 2 轮：打乱顺序，另外 32 个小批次 × 分别更新
第 3 轮：...
第 4 轮：...
```

**什么是“小批次 (minibatch)”？** 一次性使用所有 2048 个转换进行更新速度慢且耗内存；一次只更新一个转换则噪声太大。**小批次**是介于两者之间的一小块数据 —— 在这里，2048 ÷ 32 = **每个小批次 64 个转换**。我们为每个小批次计算一次梯度，因此每一轮执行 32 次微小且稳定的更新，而不是 1 次巨大的更新。（这与深度学习中到处使用的“小批次”概念相同 —— 参见 [小批次梯度下降](https://zh.wikipedia.org/wiki/随机梯度下降#小批次梯度下降)。）

裁剪功能确保了这些多次遍历不会导致过调 —— 如果没有裁剪，多轮迭代会因为推动策略过远而摧毁它！

**现实生活中的例子**：一个学生有 30 道练习题。
- REINFORCE：每道题做一遍，学到一点东西，然后扔掉。
- PPO：每道题做 4 遍（每次从不同角度思考），并限制你的改变，以免你记错模式。

## 完整的 PPO 损失函数

```
L = L_CLIP - c₁ · L_entropy + c₂ · L_critic

L_CLIP    = 裁剪后的策略梯度
L_entropy = 熵奖励（鼓励探索）
L_critic  = V(s) 与回报之间的均方误差 (MSE)
```

典型系数：c₁ = 0.01（熵），c₂ = 0.5（评论家）。

**有两个术语值得展开解释：**

- **策略梯度 (Policy gradient)** —— 损失函数的“演员”部分。它使用梯度信号将策略推向具有更高优势的动作，并偏离具有较低优势的动作。这与 REINFORCE 中引入的核心思想相同 —— 参见 [REINFORCE 详解](./reinforce_cartpole_explained.md#the-old-way-vs-the-new-way) 以获取直观理解。PPO 只是在它周围添加了裁剪外壳。
- **MSE (均方误差)** —— 损失函数的“评论家”部分。评论家 V(s) 预测状态的预期回报；我们将它的预测与实际回报进行比较，并对差值求平方：`MSE = mean((V(s) - return)²)`。求平方对大错误的惩罚比小错误更重，并为训练提供平滑、可导的信号。（标准的回归损失 —— 参见 [均方误差](https://zh.wikipedia.org/wiki/均方误差)。）

## 结果

```
更新 200 次 | 平均奖励: ~120
更新 400 次 | 平均奖励: ~280
更新 800 次 | 平均奖励: ~280-300
```

PPO 在 CartPole 环境下表现出稳步提升，但往往在 280-300 左右进入**平台期 (plateau)**。（平台期意味着学习曲线变平 —— 即使训练继续，奖励也不再提高。策略已经找到了一个局部良好的策略，但没有取得进一步进展。）这实际上是预料之中的 —— PPO 是为更困难、回合更长的环境设计的。

一个有趣的观察是：**REINFORCE 解决 CartPole 的速度更快！**（平均 500 对比 平均 300）

为什么？CartPole 的回合很短（≤500 步），因此 REINFORCE 的精确回报非常准确。PPO 的自举 (bootstrapped) 估计增加了不必要的复杂性。PPO 真正发光发热的地方是在那些无法等待完整回合结束的环境中（比如 BipedalWalker）。

**什么是 "BipedalWalker"？** BipedalWalker（具体指 [Gymnasium](https://gymnasium.farama.org/environments/box2d/bipedal_walker/) 中的 `BipedalWalker-v3`）是一个经典的强化学习基准环境：一个两足机器人必须学会跨越不平整的地形向前行走而不摔倒。与 CartPole 的两个离散动作（左/右）不同，BipedalWalker 具有**连续**动作 —— 四个扭矩值，每条腿的关节各一个，每个值都是 [-1, 1] 之间的实数。回合可能会持续数千步，这正是 PPO 的数据效率和稳定性大显身手的地方。

## 关键公式

```
概率比:      r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)
裁剪损失:    L_CLIP = E[min(r_t A_t, clip(r_t, 1-ε, 1+ε) · A_t)]
GAE:        A_t = Σ_{l=0}^{∞} (γλ)^l · δ_{t+l}
```

---

## 关键要点

| 概念 | 通俗解释 |
|---------|---------------|
| **概率比 r(θ)** | 策略在这个动作上改变了多少 |
| **裁剪 ε (Clip)** | 安全边界 —— 策略改变程度不要超过这个值 |
| **GAE** | 一种通过预见多个步长来估计优势函数的聪明方法 |
| **数据效率** | 每次展开都是从多个并行环境中收集的（去相关的、稳定的经验），然后复用进行 K 轮小批次更新 —— 裁剪保证了这些重复遍历的安全性 |

## 下一步是什么？

到目前为止，我们所有的环境都只有**离散**动作（向左或向右推）。

真正的机器人需要控制**连续**动作 —— 比如“精确施加 0.73 牛顿的力”。

`ppo_continuous.py` 使用**高斯策略 (Gaussian policy)** 将 PPO 扩展到连续动作，并在更困难的 BipedalWalker-v3 环境中进行测试！
