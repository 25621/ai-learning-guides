# 从零开始构建深度 Q 网络 (DQN) 🧠

## 线性模型的问题

还记得我们之前的线性公式吗？

> 分数 = w₁ × 小车位置 + w₂ × 小车速度 + w₃ × 木棒角度 + w₄ × 木棒速度

这在 CartPole 任务中表现尚可，但如果面对一个你能看到成千上万个像素的视频游戏呢？你无法写出一个如此简单的配方！

我们需要一种能够观察复杂情况并找出最佳动作的东西。那就是 **神经网络 (neural network)**。

---

## 什么是神经网络？

想想你的大脑。数百万个被称为神经元的小细胞相互交流。当你触摸到烫的东西时，神经元会发送信号：“烫！→ 立即移开手！”每个神经元传递信息，它们共同做出明智的决定。

计算机上的 **神经网络** 工作方式相同：

```
输入层 (Input Layer)   隐藏层 1           隐藏层 2           输出层 (Output Layer)
[小车位置]    →        [128 个神经元]  →  [128 个神经元]  →  [向左推的分数]
[小车速度]    →        [    ...     ]     [    ...     ]     [向右推的分数]
[木棒角度]    →
[木棒速度]    →
```

每个箭头都有一个 **权重 (weight)**（代表连接的强度）。网络中存在成千上万个这样的权重——而网络会学习所有这些权重！

**现实生活中的例子：** 餐厅的厨师品尝你的食物，并同时调整数百种配料。每个味蕾就像一个神经元，它们共同告诉厨师“多加点盐”或“少放点胡椒”。训练网络就像厨师通过成千上万次烹饪进行学习。

---

## DQN = 深度 Q 网络

**DQN** (Deep Q-Network) 由 DeepMind 在 2013 年发明。他们采用了旧的 Q 学习公式，并将 Q 表替换为了神经网络！

不再是：
> Q 表[状态][动作] = 分数

而是：
> Q 网络(状态) → [向左的分数, 向右的分数]

网络将状态作为输入，并一次性输出所有动作的 Q 值。这比分别计算它们要高效得多！

---

## 本脚本：原生版本 (Naive Version)

本脚本展示了 **没有任何** 特殊技巧的 DQN。它只是：
1. 观察状态
2. 询问网络“向左有多好？向右有多好？”
3. 执行分数较高的动作
4. 获得奖励，更新网络

**这种方式本质上是不稳定的！** 把它想象成一个每次学习新知识都会立即忘记之前课程的学生。网络在每一步之后都会更新，这导致了混乱。

**现实生活中的例子：** 想象一下通过每尝一口就改变整个菜谱来学习烹饪。你可能会从“太咸”变成“一点盐都没有”再到“太咸了”，永远无法稳定在合适的用量上。这就是这里发生的情况！

## 你将看到的结果

当你运行 `dqn_cartpole.py` 时：
- 分数可能会大幅跳动（不稳定的学习过程）。
- 有时智能体会变得非常出色，然后又忘掉了一切。
- 损失 (loss) 图表显示出剧烈的波动。

**这是预料之中的！** 它说明了为什么我们需要改进——经验回放 (experience replay) 和目标网络 (target network)。这些内容将在接下来的部分介绍！

## ε-贪婪技巧 (ε-Greedy Trick) 🎲

智能体并不总是选择最好的动作。有时它会随机选择！

为什么？因为如果它总是选择看起来最好的动作，它可能永远无法发现更好的选项。

> 以概率 ε (epsilon)：选择一个随机动作（探索 Exploration！）
> 以概率 1-ε：选择已知的最佳动作（利用 Exploitation！）

我们从 ε = 1.0（100% 随机）开始，并缓慢降低到 ε = 0.01（1% 随机）。这样，智能体在开始时会进行大量探索，然后专注于它学到的知识。

**现实生活中的例子：** 当你去一个新城市旅行时，你可能起初会随机尝试一些餐厅（探索）。过了一段时间后，你会回到你最喜欢的餐厅（利用）。但你仍然会偶尔尝试一些新餐厅，以防有隐藏的美味！

## 关键词汇

| 单词 | 含义 |
|------|---------|
| **神经网络** | 由数学神经元组成的层，通过数据进行学习 |
| **深度 (Deep)** | 拥有超过一个隐藏层（因此被称为“深度”学习） |
| **DQN** | 深度 Q 网络 —— 使用神经网络代替 Q 表 |
| **ε-贪婪 (ε-Greedy)** | 策略：有时随机探索，有时利用已有的最佳知识 |
| **不稳定性 (Instability)** | 网络不断“遗忘”，因为更新之间会相互干扰 |

---

## 缺失的部分（以及为什么它们很重要）

这种原生 DQN 有两个大问题：

1. **相关的更新**：每一个经历都是按顺序到来的（第 1 步，第 2 步，第 3 步……）。如果第 5 步很糟糕，附近的所有更新都会被混淆在一起。
   
2. **移动的目标**：每次更新后，网络都会改变。但下一次更新使用相同的网络来计算目标值应该是多少。这就像在射击一个移动的靶心！

这些问题由接下来的脚本中的 **经验回放 (Experience Replay)** 和 **目标网络 (Target Networks)** 解决。它们共同将 DQN 从一个摇摇欲坠的初学者变成一个冠军级的游戏玩家！
