# 经验回放 (Experience Replay)：教会机器人记住过去 🎒

## 问题所在：遗忘（与混淆）

还记得原生 DQN 是如何不稳定的吗？最大的原因是 **相关的学习 (correlated learning)**。

当机器人玩游戏时，它按顺序经历各种事情：
> 第 1 步 → 第 2 步 → 第 3 步 → 第 4 步 → ...

这些步骤是相连的！如果机器人在第 10 步向左倾斜，第 11 步通常也会向左倾斜。它们不是独立的——它们彼此依赖。

当我们使用这些相关的步骤来更新网络时，就像试图通过反复阅读同一个章节来学习历史。你会变得非常精通这一个章节，但却忘掉了其他所有内容！

**现实生活中的例子：** 想象一下你为了考试复习，却只练习昨天的家庭作业。你对那些题目变得非常拿手，但考试却有不同的题目！你需要练习各种不同题目的 **混合**。

---

## 解决方案：一个记忆盒 📦

**经验回放 (Experience Replay)** 为机器人增加了一个大型记忆盒（即 **回放缓冲区 replay buffer**）。

机器人不再仅仅从最新的经历中学习，而是：
1. **存储**：将每一次经历都存入记忆盒：(状态, 动作, 奖励, 下一个状态)
2. **随机抽取**：从盒子里随机抽取一小部分记忆
3. **混合学习**：从这些随机混合的记忆中学习，而不仅仅是学习最新的一步

```
游戏第 1 步 → [存入盒子]
游戏第 2 步 → [存入盒子]
游戏第 3 步 → [存入盒子]
...
游戏第 50 步 → [存入盒子] → 抽取 64 个随机记忆 → 更新网络
游戏第 51 步 → [存入盒子] → 抽取 64 个随机记忆 → 更新网络
```

**现实生活中的例子：** 想想相册。你不会仅仅通过看今天的照片来了解你的生活。你也会翻阅 **旧** 照片——一个包含了美好回忆和棘手时刻的混合。这能帮助你理解整个人生的规律，而不仅仅是今天。

---

## 为什么随机采样会有帮助

当我们随机选取记忆时，我们打破了相关性。机器人可能同时从以下记忆中学习：
- 一个木棒保持得非常完美的记忆（来自 500 步之前）
- 一个木棒即将倒下的记忆（来自 20 步之前）
- 一个运气很好的记忆（来自第 3 步）

这种随机混合意味着：
✅ 机器人能从各种各样的情况中学习
✅ 每个记忆可以被“回放”多次（高效利用经验）
✅ 网络不会过度拟合最近发生的事件

---

## 小批量 (Mini-Batch) 学习

我们不是一次只针对一个经历进行更新，而是一次针对 **64 个经历** 进行更新（一个“小批量”）。这就像：
- 旧方法：读一张抽认卡，考考自己
- 新方法：读 64 张不同的抽认卡，然后针对这一堆卡片考考自己

小批量学习使学习信号更加可靠，减少了噪声。

---

## 预热期 (Warmup Period)

我们不会立即开始学习！回放缓冲区首先需要一些记忆。在训练开始前，我们要等到盒子里至少有 **500 个经历**。

**现实生活中的例子：** 在你收集齐食材之前，你不会尝试去做一顿饭。预热期就像是做饭前的买菜过程！

---

## 比较结果显示了什么

当你运行 `dqn_experience_replay.py` 时，你会看到两条学习曲线：

| 原生 DQN (Naive DQN) | DQN + 经验回放 |
|-----------|-------------|
| 非常颠簸 | 更平滑 |
| 频繁崩溃（忘掉一切） | 进步更持久稳定 |
| 高方差 | 低方差 |

带有经验回放的版本通常：
- 更可靠地达到高分
- 不会频繁从 500 分跌回 30 分
- 显示出更稳定的学习进度

## 代码中的回放缓冲区

```
ReplayBuffer:
  - 容量 (capacity)：10,000 条记忆（存满后忘掉最旧的）
  - push(state, action, reward, next_state, done)：存入记忆
  - sample(batch_size=64) → 随机批量：抽取记忆
```

把它想象成一个有 10,000 行的笔记本。当它记满时，你会擦掉最旧的一行并写下最新的一行。你总是从随机的一页开始学习！

---

## 关键词汇

| 单词 | 含义 |
|------|---------|
| **经验回放 (Experience Replay)** | 存储并随机重复使用过去的经验进行训练 |
| **回放缓冲区 (Replay Buffer)** | 存储过去 (状态, 动作, 奖励, 下一状态) 元组的记忆盒 |
| **相关的更新 (Correlated updates)** | 当训练数据相互依赖时（不利于学习！） |
| **小批量 (Mini-batch)** | 用于一次更新步骤的一小部分随机记忆样本 |
| **去相关 (Decorrelation)** | 打破连续经历之间的联系 |

---

## 还有什么缺失？

即使有了回放缓冲区，还有一个问题：**移动的目标**。

每当我们更新网络时，Q 值就会改变。但这些更新后的 Q 值也被用来计算下一次更新的目标值。这是一个混乱的循环！

这由 **目标网络 (Target Network)** 解决——一个每隔 100 步才更新一次的网络冻结副本。这让“靶心”能在一段时间内保持不动，这样机器人就能可靠地瞄准它！
