# 探索 PettingZoo 环境 🦓

## 什么是 PettingZoo？

如果你曾经做过单智能体强化学习，你可能用过 **Gymnasium**（OpenAI Gym 的继承者）。每个环境看起来都一样：`env.reset()`，`env.step(action) → obs, reward, done, info` —— 一个对世界的新*观察*，一个标量*奖励*信号，一个表示“游戏结束”的*完成*标志，以及一个用于调试额外内容的*信息*字典。这种统一性正是强化学习库能够发挥作用的原因。

**PettingZoo** 的理念完全相同，但它是针对*多智能体*的。它是一个多智能体环境的动物园 —— 全部遵循一个定义良好的 API：
- **经典玩具问题**：像“石头剪刀布”这样的简单环境，用于测试基础算法。
- **协同网格世界**：智能体在网格中导航以实现共同目标。
- **雅达利多人游戏**：经典的竞技游戏，如 Pong。
- **MPE (多粒子环境)**：用于复杂协调和竞争的连续空间物理环境。

如果你能编写一段在某个 PettingZoo 环境下运行的代码，你几乎不需要任何修改就可以将其插入到任何其他环境中。

## 两种 API 风格

多智能体设置比单智能体设置更复杂，因为两个智能体可以同时采取行动，或者轮流采取行动，甚至按任意顺序采取行动。PettingZoo 通过两种并行的 API 解决了这个问题：

### 1) AEC (智能体-环境循环 Agent-Environment-Cycle)

一次只有一个智能体采取行动。环境按某种顺序循环遍历智能体，每个智能体都会得到：
- **观察 (observation)** —— 他们*现在*看到了什么，
- **奖励 (reward)** —— 在上一轮完整环节中通过*联合*动作赚取的收益（也就是说，这是*所有*智能体行动的结果，而不仅仅是你；例如在国际象棋比赛中，你的奖励反映的是你对手最后一步棋之后的棋盘状态，而不仅仅是你的），
- **终止标志 (termination flag)** —— 当回合*自然*结束（如：将军、有人获胜）时为 `True`，
- **截断标志 (truncation flag)** —— 当回合因时间限制而*提前*结束时为 `True`。

这对于像国际象棋、围棋、扑克这样的**回合制游戏**来说非常自然。

```python
env.reset()
for agent in env.agent_iter():
    obs, reward, term, trunc, info = env.last()
    if term or trunc:
        env.step(None)
        continue
    action = my_policy(obs, agent)
    env.step(action)
```

### 2) Parallel (并行)

所有智能体在每一步都同时观察并采取行动。`step()` 接收一个动作*字典*并返回观察字典和奖励字典。

这对于像 MPE（多粒子环境，所有点状智能体同时移动）或多智能体网格世界这样的**实时游戏**来说非常自然。

```python
obs, info = env.reset()
while env.agents:
    actions = {a: my_policy(obs[a]) for a in env.agents}
    obs, rewards, terms, truncs, info = env.step(actions)
```

这两种风格是**同构的 (isomorphic)** —— 结构上等价且可以相互转换：任何 AEC 环境都可以自动包装成并行环境，反之亦然。PettingZoo 提供了转换包装器，因此你只需要针对一种风格编写代码。

## 现实生活中的类比

- **AEC = 棋盘游戏之夜**。“Alice 行动。现在是 Bob。现在是 Carol。回到 Alice。”下一个行动的人会看到最新的棋盘状态。
- **Parallel = 多人视频游戏**。所有四名玩家同时按下按钮；游戏每秒更新世界 60 次。
- **为什么统一的 API 很重要**。想象一下，如果每款多人视频游戏都需要自己的摇杆。PettingZoo 就是多智能体强化学习 (MARL) 的“通用摇杆”。

## 我们的代码做了什么

我们从头开始构建了一个 **PettingZoo 风格**的环境：**迭代协同博弈 (Iterated Coordination Game)**。两个智能体反复选择通道 `0` 或 `1`：

- 选择相同 → 两者都获得 +1
- 选择不同 → 两者都获得 -1

每个智能体收到的**观察**是之前的*联合动作* —— 两个智能体上一轮选择的内容，并打包成一个整数。具体来说：每个智能体最后的动作是 `{start, 0, 1}`（3 种状态）之一，因此这对动作编码为 `3 × agent_1_state + agent_2_state`，产生 9 个可能的整数（0 – 8）。整数 0 是“开始”状态 —— 它表示尚未采取任何动作（一个回合的最开始）。
一个回合持续 25 步，因此每个智能体的最大总回报为 +25，最小总回报为 -25。**随机游戏的得分 ≈ 0**，因为在每一步，两个独立的随机智能体以相同的概率各选择 0 或 1：它们有 50% 的时间匹配 (+1)，50% 的时间不匹配 (-1)，预期的单步奖励为 0.5 × (+1) + 0.5 × (-1) = **0**。25 步累加后，预期的回合回报也是 0。

然后我们：

1. **通过随机展开演示 AEC 接口** —— 这确认了基本的 AEC 循环：`agent_iter()` 产生当前轮到的智能体，`last()` 读取该智能体当前的观察和累积奖励，`step()` 将他们选择的动作交还给环境。
2. **通过并行接口训练两个独立的 Q 学习器**。每个智能体都维护自己的 Q 表，以**联合动作观察**（编码了*两个*智能体上一轮行为的单个整数）作为键，因此它可以学习“当上次我们都选 0 时，我这次也应该选 0”。
3. **尝试导入真实的 `pettingzoo` 库**并使用随机策略展开其内置环境之一（石头剪刀布）。如果未安装 PettingZoo，我们将跳过此步骤并显示一条友好的消息。

### 你应该看到的结果

| 阶段 | 预期结果 |
|-------|----------|
| 随机展开 (AEC)            | 平均回合回报接近 **0** —— 随机智能体独立选择通道，匹配和不匹配的情况大致相等。 |
| 独立 Q 学习器 (并行) —— 前 100 个回合 | 约 **0** —— 智能体在探索时仍然表现出很大程度的随机性。 |
| 独立 Q 学习器 —— 最后 100 个回合             | 强正值，**+20 到 +25** —— **协同已经出现**：两个智能体都学会了在每一轮可靠地选择相同的通道。 |

图表 `outputs/pettingzoo_coordination.png` 显示了单个回合的回报（灰色）和滚动**均值**曲线（蓝色）。均值平滑了噪声较大的回合，因此你可以看到趋势：智能体从 ~0 附近的无协调随机游戏转向 ~+25 附近的稳定**协同**。绿色虚线标记了完美协同的上限。

## 为什么要先构建自定义环境？

因为 **API 本身就是课程。**（理解如何构建多个智能体与环境之间的交互结构比具体的游戏规则更重要。）多智能体强化学习有许多形式（回合制、实时、协同、竞技、混合），它们都符合 AEC / 并行模式。一旦你实现了这两个循环，每个 PettingZoo 环境都只是插入不同的环境构造函数的问题 —— 训练代码保持不变。

这正是 Gymnasium 改变单智能体强化学习的方式：通过将环境变成统一接口背后的黑盒。

---

## 独立 Q 学习在何处有利有弊

协同博弈是*宽容的* —— 智能体共享奖励符号，因此他们的利益是一致的。独立学习器可以愉快地解决这个问题，因为一个智能体的任何改进都会帮助另一个智能体。

在**对抗性**游戏 (RPS) 中，独立 Q 学习会永远振荡（当一个智能体适应时，另一个智能体会改变策略进行反击，导致无休止的追逐）。
在**部分可观测 (partially-observable)** 的游戏中，它根本无法学习，因为“观察”只是状态的一部分（一个智能体可能会因为一个好的动作而受到惩罚，仅仅是因为它看不到另一个智能体在做什么）。PettingZoo 包含这两种环境，因此你可以亲眼看到这些失败模式。

## 需要记住的关键术语

| 单词 | 含义 |
|------|---------|
| **PettingZoo**     | 多智能体强化学习的 Gymnasium —— 一个标准化多智能体强化学习环境库 |
| **AEC**            | 智能体-环境循环：每步一个智能体行动（回合制） |
| **Parallel API**   | 所有智能体在每步同时行动 |
| **MPE**            | 多粒子环境，PettingZoo 附带的一个流行的协同/竞争测试平台（通常涉及移动的点执行基于物理的任务）。 |
| **CTDE**           | 集中式训练，分布式执行 (Centralised Training, Decentralised Execution) —— 以全局视角（访问所有状态）进行训练，以局部观察（每个智能体根据其有限的视野行动）进行部署。 |
| **独立 Q 学习 (Independent Q-learning)** | 每个智能体运行普通的 Q 学习（标准、未经修改的 Q 学习算法），忽略其他学习者的存在。 |

## 一句话总结

> **PettingZoo 为每个多智能体环境提供了相同的形状 —— 因此你今天编写的代码在明天完全不同的游戏中仍然适用。**

一旦这两种 API 风格变成你的第二本能，你就可以进阶到 MADDPG（连续控制智能体的集中式评论家）、QMIX（协同团队的值混合）、MAPPO（多智能体 PPO）或任何其他现代多智能体强化学习算法 —— 你代码中的环境部分永远不需要改变。
