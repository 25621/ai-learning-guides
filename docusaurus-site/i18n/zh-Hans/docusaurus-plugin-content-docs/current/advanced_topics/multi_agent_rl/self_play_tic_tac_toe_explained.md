# 自博弈 (Self-Play)：通过让智能体自己与自己玩来教学 ♟️

## 什么是自博弈？

想象一个想要精通国际象棋的孩子，但她找不到对手。于是她自己和自己玩。左手对右手。每一局，*双方*都努力求胜。每一局，*双方*都从中学习成功的经验。

这就是**自博弈 (Self-Play)**：同一个智能体同时扮演两名玩家，每一次移动都成了对下一位移动者的教训。没有老师，没有专家对手。只有一个学习者，而它本身就是自己不断向上攀登的阶梯。

自博弈听起来像是个小花招——难道不需要真正的对手吗？——但它实际上是过去十年中许多著名强化学习里程碑背后的引擎：**AlphaGo Zero**、**AlphaZero**、**MuZero**、**OpenAI Five**。它们都使用了自博弈。原因很简单：随着你的进步，你的对手也以同样的速度在进步。挑战始终与你的技能水平相匹配。

---

## 为什么它有效

有三点让自博弈变得特别：

1. **无穷无尽的对手。** 你永远不会缺对手。对手随时随地都在，而且是免费的。
2. **随你共同成长的课程。** 初学者只能和初学者玩。随着你变强，你的“影子”也会自动变强。
3. **对称性。** 在零和游戏（一方的胜利就是另一方的失败）中，一套 Q 值可以描述双方；你只需要在轮到另一方时翻转符号即可。因此，*单一*的 Q 表就可以实现自我教学。

井字棋 (Tic-tac-toe) 是完美的测试场：它足够小，可以用字典存储，但也足够复杂，面对有策略的玩家，随机选择步法几乎总是会导致失败。

---

## 现实生活中的类比

- **对着墙练习网球。** 你不会输给墙，但你可以练习发球。自博弈就像是在两端都练习——你既是墙又是球员，并在两者之间来回切换。
- **辩论俱乐部练习。** 那些总是尝试站在自己观点的对立面进行辩护的人，往往会成为更优秀的辩论者。每一次辩论都在同时训练进攻和防守。
- **AlphaGo Zero。** 它在学习过程中没有使用任何人类对局数据。它从随机移动开始，与自己进行了数百万次对局；在短短几天内，它就超越了之前所有的围棋程序，包括击败了李世石的那一个版本。

## 我们的代码做了什么

我们为*当前轮到移动的玩家*学习一个 Q 表：

```
Q[(棋盘状态, 轮到谁走)][动作] = 该玩家的预期回报
```

训练循环如下：

1. 从空棋盘开始。`player = X`。
2. 双方玩家都由**同一个智能体**扮演，使用 ε-贪婪策略。
3. 每局游戏结束后，回溯历史记录中的每一个（棋盘、玩家、动作）三元组，并应用 Q-learning 更新。
4. 奖励在不同回合间正负号翻转：如果 X 获胜，X 做出的每一步都得到 +1（或从未来的获胜状态引导价值）；O 做出的每一步得到 -1。
5. 我们慢慢降低（衰减）我们的探索率 (ε)，从 0.2 → 0.02，以便智能体在训练后期致力于其最佳表现，而不是尝试随机动作。

每隔 2,500 个回合，我们就会让智能体与一个**随机对手**进行评估（我们冻结学习过程，评估期间不对 Q 表进行更新，且双方都采取贪婪策略）。在足够的自博弈之后，智能体应该能赢得或平掉约 100% 的对局。

### 你应该看到的结果

在进行 50,000 次自博弈回合后：

| 对阵双方 | 预期结果 |
|----------|-----------------|
| 训练后的智能体 vs 随机对手 (1000 局) | **~95-99% 胜或平**，几乎 0% 失败 |
| 训练后的智能体 vs 自身 (200 局贪婪对弈) | **全部 200 局平局**。井字棋是一个如果双方都玩得完美，最后总会以平局告终的游戏。自博弈产生全是平局的结果，正是收敛的标志。 |

图表 `outputs/self_play_tic_tac_toe.png` 显示了智能体相对于随机对手的胜/平/负比例随时间的变化：
- 胜率从约 60% 开始（当双方都随机玩时，先手 X 具有天然优势，因为它能放置更多的棋子，因此基准胜率约为 60%）。
- 攀升至 >90%。
- 败率降至接近 0%。

脚本最后还会打印一局示例对战的每一步，以便你观察智能体是如何玩的。

## 注意这些细节

- **符号翻转至关重要。** 一个常见的错误是：忘记了“对手最大化他们的价值”意味着在目标计算中“最小化我们的价值”。我们代码中的更新使用的是 `target = reward - gamma * max(Q[下一个状态, 对手])`。
- **此处未利用对称性。** 真正的实现会对棋盘进行规范化（意味着它们会将任何棋盘状态旋转或镜像为标准的、唯一的“规范形式”，以便智能体识别相同的局势），从而在 8 种对称性中共享 Q 值。我们跳过了这一步——状态空间足够小，可以进行暴力计算。
- **Q 表的增长。** 在 50k 次自博弈后，你会看到几千个“状态-玩家”键值对。这在这里没问题；但对于国际象棋或围棋，你需要神经网络来代替。这就是为什么 **AlphaZero 用 CNN + MCTS 替换了表格**。

## 自博弈在哪些情况下会失效

- **非零和博弈。** “双赢”与对称博弈是不相容的；你不能简单地翻转正负号。
- **非对称角色。** 如果“进攻方”和“防守方”的动作空间不同，你需要两个独立的网络。
- **策略循环。** 纯粹的自博弈可能会陷入类似“石头剪刀布”的循环中。AlphaStar 通过维护一个巨大的过往版本智能体池（或称“联赛”），并从中随机抽取对手来解决这个问题，从而使智能体学习击败多种不同的游戏风格，而不只是当前的这一种。
- **奖励劫持 (Reward hacking)。** 自博弈会让双方变聪明，但仅限于你*定义的*游戏。如果你的奖励系统有漏洞（例如奖励玩家存活更久而不是获胜），双方会共同利用该漏洞，导致产生怪异且无用的行为，而不是精通真正的游戏。

## 需要记住的核心词汇

| 词汇 | 含义 |
|------|---------|
| **自博弈 (Self-play)** | 同一个智能体扮演游戏的两方 |
| **零和 (Zero-sum)** | 一方的收益 = 另一方的损失 |
| **对称性 (Symmetry)** | 如果翻转正负号，一个 Q 表可以服务于双方 |
| **种群博弈 (Population play)**| 与*许多*过去版本的自己作为对手进行自博弈 (AlphaStar) |
| **课程学习 (Curriculum)** | 自然的难度递进——自博弈可以免费获得这一点 |
| **MCTS** | 蒙特卡洛树搜索——AlphaZero 与自博弈配合使用的规划算法 |

---

## 一句话总结

> **自博弈将进步变成了它自己的阶梯：每当你变强，你的对手也会自动变强。**

这一想法在结合了**神经网络**（受大脑启发的从数据中学习模式的数学函数）和树搜索后，在围棋、象棋、将棋、Dota 2 和星际争霸中击败了人类顶尖高手。
