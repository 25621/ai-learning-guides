# 矩阵游戏：最简单的多智能体世界 🎲

## 什么是矩阵游戏？

想象一下，你和你的朋友各选一个手势 —— **石头、剪刀或布** —— 且*同时*进行。你们各向对方选择保密。胜负由一个小表格决定：

|        | 石头 | 布 | 剪刀 |
|--------|:----:|:-----:|:--------:|
| 石头     |  0,0  | -1,+1 | +1,-1 |
| 布    | +1,-1 |  0,0  | -1,+1 |
| 剪刀 | -1,+1 | +1,-1 |  0,0  |

这个表格就是游戏的*整个世界*。没有移动，没有时间，没有地图。只有一个一次性的决策。我们称之为 **矩阵游戏 (matrix game)**，因为收益矩阵 (payoff matrix) 就是整个环境。

矩阵游戏是研究 **多智能体强化学习 (multi-agent RL)** 最纯净的场所，因为在训练过程中唯一可能改变的是每个玩家的*策略 (policy)* —— 即选择每个动作的概率。

---

## 为什么它是“多智能体”的

在单智能体强化学习中，环境是固定的：风总是以同样的方式吹，地板永远不会动。智能体不断改进，最终赢得比赛。

在矩阵游戏中，你的“环境”是*另一个正在学习的智能体*。随着他们变得越来越聪明，对你来说什么是“好招”也会*随之改变*。这被称为 **非平稳性 (non-stationarity)**，它是多智能体强化学习的核心问题。

> 如果你一直出石头，你的对手最终会开始一直出布。所以你切换到剪刀。所以他们切换到石头。所以你切换到布……以此类推。“最佳动作”永远不会停留在原地。

经典的解决方案是 **混合策略 (mixed strategies)**：不要确定性地选择任何一个动作，而是以一种对手无法利用的方式进行随机化。

## 我们玩的三个游戏

### 1) 石头剪刀布 (零和博弈)
- 一个玩家的收益就是另一个玩家的损失。
- **纳什均衡 (Nash equilibrium)** 是：每个玩家以 ⅓ 的概率选择每个动作。任何偏离都会被对方利用。
- 我们预期这两个 Q 学习器会在 ⅓-⅓-⅓ 附近摆动 —— 永远不会完全稳定，因为每当一个学习器发生偏移，另一个就会做出反应。

### 2) 囚徒困境 (非零和博弈/一般和博弈)
两个嫌疑人被分开审讯：

|           | 合作 (Cooperate) | 背叛 (Defect) |
|-----------|:---------:|:------:|
| 合作 |   3, 3    |  0, 5  |
| 背叛    |   5, 0    |  1, 1  |

- 无论对方做什么，“背叛”都优于“合作” —— 这是一项 **占优策略 (dominant strategy)**。
- 两个玩家都是理性的 → 两个都背叛 → 两个都得到 1，尽管（合作，合作）本可以各得 3。自私的最佳反应破坏了集体福利。
- 我们预期 Q 学习会清晰地收敛到（背叛，背叛）。

### 3) 猎鹿博弈 (协调博弈)
两名猎人可以合作捕获一只鹿（巨大的奖励），或者各自捕获一只兔子（奖励小但安全）：

|       | 猎鹿 | 捕兔 |
|-------|:----:|:----:|
| 猎鹿  | 4, 4 | 0, 3 |
| 捕兔  | 3, 0 | 2, 2 |

- （猎鹿，猎鹿）是 **收益占优 (payoff-dominant)** —— 对双方都最好。
- （捕兔，捕兔）是 **风险占优 (risk-dominant)** —— 如果你不信任你的伙伴，这是最安全的。
- 结果取决于初始条件：独立的 Q 学习器往往最终陷入*较差的*（捕兔，捕兔）均衡，因为捕兔对于学习来说更安全。

## 现实生活中的例子

- **双头垄断定价。** 同一条街上的两家咖啡馆每天早上各定一个价格。收益矩阵的形状决定了它们最终是处于一个高的“合作”价格（对它们有利，对顾客不利），还是一个低的“割喉式”竞争价格。
- **网络协议。** 路由器和发送器选择定时策略；网络的拥塞结果由类似于矩阵游戏的“成功通过”与“退避”的收益决定。
- **拍卖出价。** 每个出价者在不知道其他人的情况下选择一个出价；收益取决于整个出价向量。纳什均衡是一个*出价策略*，而不是一个单一的数字。

## 我们的代码做了什么

对于每个游戏，我们：
1. 创建两个无状态的 Q 学习器（Q 只是每个动作对应的一个数字 —— 在一次性游戏中没有状态）。
2. 循环运行 20,000 步。每一步：两个智能体同时选择一个 ε-greedy 动作，获得奖励，并更新它们的 Q 值。
3. 在滚动的 500 步窗口中跟踪每个智能体的 **经验动作频率 (empirical action frequency)**。我们不仅观察抽象的概率，还计算他们最近实际选择了哪些动作（例如：“在最近的 500 轮中，他们有 40% 的时间出了石头”）。这为我们展示了他们策略变化的实时、实际图像。
4. 绘制随时间变化的频率图，保存至 `outputs/<game>.png`，并打印最终的 Q 值。

### 你应该看到的现象

| 游戏 | 预期的图表结果 |
|------|------------------------------|
| **石头剪刀布** | 两个玩家都徘徊在 ⅓-⅓-⅓ 附近，但有明显的抖动。曲线相互追逐 —— 经典的循环行为。 |
| **囚徒困境** | 两个玩家的“背叛”频率迅速上升到 ~1.0。“合作”被彻底抛弃。 |
| **猎鹿博弈** | 大多数随机种子会稳定在（捕兔，捕兔）。一些幸运的种子会达到（猎鹿，猎鹿）—— 尝试更改脚本中的随机种子，观察它是如何反转的。 |

## 独立学习失效的地方

我们的智能体是*独立的* —— 它们只看到自己的奖励，从未看到对手的动作或 Q 值。这是最简单的基准，但它有局限性：

- 它 **无法保证在一般和博弈中收敛**。
- 它可能会陷入 **坏的均衡**（如猎鹿博弈）。
- 它 **无法对对手建模**。

真正的多智能体算法通过显式地推理其他学习器来解决这个问题。以下是常见的几种算法的核心思想：

| 算法 | 核心思想 | 现实生活类比 |
|-----------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------|
| **虚构游戏 (Fictitious play)** | 记录对手选择每个动作的频率。假设明天他们会做他们一直以来所做的事 —— 然后选择你对该信念的最佳反应。 | 在多场国际象棋比赛中观察对手的习惯，并据此调整你的开局。 |
| **CFR (逆事实遗憾最小化)** | 在每一轮之后，问：“如果我没选这个动作，而选了其他动作，我会多后悔？” 逐渐将概率转移到那些你后悔错过的动作上。常用于扑克，因为它能处理 **不完全信息** 游戏（你看不到对手的牌）。 | 在一把扑克牌局后进行复盘并思考：“我本该下更大的注 —— 下次我会那样做。” |
| **LOLA (学习对手的学习意识)** | 你的梯度步骤考虑了对手*也在*进行梯度更新这一事实。你在优化自己的更新时，预判对手的下一步更新 —— 提前预判两步而不是一步。 | 在谈判交易时思考：“如果我出价 X，他们会还价 Y，所以我应该先从 Z 开始。” |
| **MADDPG (多智能体深度确定性策略梯度)** | 每个智能体的*评论家 (critic)*（价值评估器）都以 **全局视野** 进行训练：它能看到每个人的观测和动作。而*执行者 (actor)*（部署的策略）仍然只使用局部信息 —— 这就是 CTDE（集中式训练，分布式执行）模式。 | 一名篮球教练观看全场比赛（集中式评论家），但教导每个球员仅根据他们能看到的情况做出反应（分布式执行者）。 |

但独立 Q 学习是正确的第一步。你能亲身感受到非平稳性问题带来的挑战，之后那些修复方案也就顺理成章了。

## 关键术语

| 单词 | 含义 |
|------|---------|
| **Payoff matrix (收益矩阵)** | 定义一次性多智能体游戏的表格 |
| **Nash equilibrium (纳什均衡)** | 一个策略组合，其中任何单个智能体都无法通过偏离策略来提高自身收益 |
| **Mixed strategy (混合策略)** | 在多个动作上进行随机化的策略 |
| **Non-stationarity (非平稳性)** | 环境（= 其他智能体）在学习过程中不断变化 |
| **Independent learner (独立学习器)** | 忽略其他学习器存在的智能体 |
| **Zero-sum (零和)** | 一个智能体的收益正好是另一个智能体的损失 |
| **General-sum (一般和/非零和)** | 双方可能共赢、共输，或处于两者之间的任何情况 |

## 一句话总结

> **在矩阵游戏中，“环境”是另一个学习者 —— 所以最佳动作一直在变。**
