# 探索策略比较 🔦

## 一句话问题

强化学习智能体必须做两件方向相反的事情：

- **利用 (Exploit)**：执行目前效果最好的动作。
- **探索 (Explore)**：尝试新事物，以防它更好。

过度偏向“利用”，你将永远安于平庸的现状。过度偏向“探索”，你将永远无法兑现收益。*如何* 探索——而不只是 *是否* 探索——是区分能解决《Montezuma's Revenge》的智能体与得分为零的智能体的关键。

本脚本将 **五种** 探索策略在相同的两个困难任务上进行正面交锋，以便你观察它们的特性。

## 现实生活中的比喻：挑选午餐地点

你刚搬到一个拥有 200 家餐厅的新城市。

- **ε-greedy** = “去我目前最喜欢的餐厅，但每隔十天掷一次骰子，*完全随机* 选一家餐厅。” 你会进行广泛但 *漫无目的* 的抽样——而且你会不断地再次光顾那些你已经讨厌的地方。
- **乐观初始化 (Optimistic initialisation)** = “假设我没去过的 *每一家* 餐厅都是城里最好的，直到事实证明并非如此。” 你会有条不紊地尝试所有 200 家，随着现实让你失望而将它们一一划掉——你会很快找到真正棒的餐厅。
- **UCB (置信上限)** = “偏好我最喜欢的餐厅，但给那些我几乎没去过的地方提供 *额外奖励 (bonus)* ——我了解得越少，奖励就越大。” 这在决定今天去尝试 *哪家* 陌生餐厅时很聪明，但每个决定都是局部的：它只挑选 *眼下* 看起来最好的选项，而不会规划一条穿越整个未探索区域的路线。它不会想“我应该穿过城市去东边，因为那里聚集了二十家没试过的地方”——每家餐厅都是一步一步被孤立地评估的。
- **基于计数的奖励加成 (Count-based reward bonus)** = 类似于 UCB，但你也 *享受新奇本身* ——去一家全新的餐厅吃饭在本质上是令人满足的，这种满足感塑造了你走进哪些社区进行探索的长期计划。
- **预测误差奖励加成 (Prediction-error reward bonus)** = “我从让我 *感到意外* 的餐饭中获得快感——一些我无法预见的东西。” 一家表现完全符合预期的新餐厅？平平无奇。一家与你的心理模型截然不同的餐厅？迷人，你会更新你的计划去寻找更多类似的地方。

## 五种策略（均在 `compare_exploration.py` 中）

### 1. ε-greedy —— 默认策略，它是 *犹豫不决*，而非探索

贪婪地行动，但以概率 ε 采取均匀随机的动作。它是 DQN 及其变体中的标准基准。它在困难任务上的致命缺陷是：**每一步都是独立的硬币翻转。** 要在一条包含 `N` 个正确动作的链条上走下去，你需要硬币连续 `N` 次都翻转正确——这在指数级上是不可能的。ε-greedy 是 *抖动*，而非 *探索*。

### 2. 乐观初始化 —— “在证明无聊之前是无辜的”

将 *每个* Q 值都设定为可能的最大回报，即 `R_max / (1 − γ)`。现在，智能体从未尝试过的动作看起来就像是世界上最棒的事情，因此 **贪婪** 策略被迫去尝试它；只有在访问过它之后，该值才会跌落至真相。关于 *未* 尝试区域的乐观态度会通过 **价值函数自动传播**（通过 Q-learning 的自举），因此智能体会被一步步拉向它未见过的世界部分。几乎不需要额外成本，不需要额外记账——而且正如你将看到的，它是小型表格世界中最强大的 *深度* 探索者。

### 3. UCB 风格动作选择 —— 奖励加成在 *选择* 中，而非 *奖励* 中

选择 `argmax_a [ Q(s,a) + c·√(ln t / N(s,a)) ]`：偏好高价值动作，但夸大那些你很少尝试的动作。在多臂老虎机问题中非常有名。缺点是：奖励加成仅存在于 **动作选择规则** 中，从不在奖励中——因此它 *不会* 流经价值函数。UCB 擅长“确保我已经尝试了 *这个* 状态下的每个动作”，但在“规划一条通往遥远未探索区域的路线”方面表现较弱。

### 4. 基于计数的 **奖励** 加成 —— 好奇心的经典版本

将 `1/√(N(s,a))` 添加到 **奖励** 中（权重 `beta` 随时间衰减）。因为是在奖励中，Q-learning *会* 传播它：通往新奇区域的状态变得更有价值。这是 MBIE-EB / 经典的“探索加成”思想——也是练习 1 的前半部分。

### 5. 预测误差 **奖励** 加成 —— 好奇心，ICM/RND 版本

将来自小型学习前向模型的 `−log P(s'|s,a)` 添加到奖励中（同样具有衰减的 `beta`）。这是五种策略中最敏锐的新奇信号：在确定性的世界中，一旦你见过一次转移，它的意外程度就会降至 ~0，而不是像 `1/√N` 那样缓慢消退。它是 ICM / RND 的表格版本——也是练习 1 的后半部分。

## 两个测试任务

- **任务 A —— MiniMontezuma**：钥匙→门→宝藏的网格世界，奖励仅在宝藏处（约 15 步完美移动）。测试“你是否能从长期的稀疏奖励链中幸存？”
- **任务 B —— DeepSea(N)**：教科书般的深度探索链，运行长度为 `N = 5, 8, 11, 14`。奖励隐藏在 `N` 个正确动作之后，每个动作都有微小的即时成本——因此短视的智能体会学会规避成本，永远找不到奖品。测试“随着链条变得 *更长*，你的策略是否仍然有效？”

## 实际发生的情况（运行并观察）

**任务 A —— MiniMontezuma：**

| 策略 | 首次发现宝藏 | 最终解决率 |
|----------|---------------:|-----------------:|
| ε-greedy | 从未 | 0.00 |
| 乐观初始化 | 约第 1 回合 | 1.00 |
| UCB 动作选择 | 约第 3 回合 | ~0.95 |
| 计数奖励加成 | 约第 82 回合 | ~0.41 |
| 预测奖励加成 | 约第 23 回合 | 1.00 |

**任务 B —— DeepSea，找到奖励的种子比例：**

| 策略 | N=5 | N=8 | N=11 | N=14 |
|----------|----:|----:|-----:|-----:|
| ε-greedy | 0 | 0 | 0 | 0 |
| 乐观初始化 | 1.0 | 1.0 | 1.0 | 1.0 |
| UCB 动作选择 | 1.0 | 1.0 | 0.0 | 0.0 |
| 计数奖励加成 | 1.0 | 1.0 | ~0.1 | 0.0 |
| 预测奖励加成 | ~0.9 | ~0.8 | ~0.9 | ~0.2 |

*(数字会随随机种子略有波动，但大趋势是非常稳固的。)*

## 启示

1. **ε-greedy 并不是探索。** 它从未解决过 *任何一个* 困难任务。随机的犹豫不决根本无法穿透长期的正确序列。（然而它仍然是很多代码中的默认设置——因为在 *简单* 任务上，它足够好且极其简单。）

2. **真正的探索意味着对未知保持乐观——无论以何种方式。** 无论你是将乐观情绪注入 *初始值*（策略 2）、*动作选择*（策略 3），还是 *自我生成的奖励*（策略 4–5），共同的主题都是：*让未探索的事物看起来很有吸引力*，然后让学习带你到达那里。

3. **在稀疏奖励的网格上，所有四种“真正的”策略都奏效——且预测误差加成最快到达目的地**，因为它产生了最清晰的“这是新的”信号。

4. **在需要长距离传播乐观情绪的 *深度* 链条上，简单的冠军是乐观初始化。** 它免费地通过价值函数传播乐观情绪。UCB 最先崩溃（它的奖励加成从未进入价值函数，因此它无法进行深度 *规划*）。奖励加成表现更好——它们 *确实* 传播——但普通的表格 Q-learning 在奖励加成衰减之前，将这种乐观情绪推向长链底部的速度较慢。

5. **最后一点正是为什么将深度探索扩展到像素级别需要额外的火力** —— 带自举的 DQN、使用真实神经网络的 RND（以便乐观情绪在相似状态间 *泛化*，而不是一次传播一个网格单元）、Go-Explore（字面意思上记住并返回有希望的状态）。这里的表格模型向你展示了 *原理*；真实的系统是这些原理加上一个可以泛化的网络。

## 关键术语

| 术语 | 含义 |
|------|---------|
| **探索-利用权衡** | 尝试新事物 vs. 兑现已知收益——强化学习的核心张力 |
| **犹豫不决 (Dithering)** | 通过向动作添加随机噪声来进行“探索”（ε-greedy、高斯噪声）——在困难任务上表现微弱 |
| **不确定情况下的乐观主义** | 伞式原则：在检查过之前，将未知事物视为很棒的事物 |
| **乐观初始化** | 通过将所有初始值设定为最大可能回报来实现该原则 |
| **UCB** | 置信上限：选择 `argmax (价值 + 随访问次数缩小的奖励加成)` |
| **深度探索 (Deep exploration)** | 需要一系列长期 *连贯* 的“非同寻常”动作的探索，而不只是一个动作 |
| **`beta` 退火 (annealing)** | 随时间缩小好奇心权重，以便智能体最终停止探索并开始利用 |

## 一句话总结

> **ε-greedy 只是噪声；每种真正的探索策略都通过使未探索的事物看起来具有吸引力来发挥作用——无论是通过乐观的初始值、动作选择奖励加成，还是自我生成的新奇奖励——而正确的选择取决于你的奖励仅仅是 *稀疏* 的（比如在平地上寻找一个隐藏的奖品），还是真正 *深层* 的（比如需要一长串精确动作才能破解的组合锁）。**
