# 在 Montezuma's Revenge（蒙特祖玛的复仇）上训练 🏛️🔑

## 为什么这款游戏如此出名（在强化学习领域）

2015 年，DeepMind 的 DQN 学会了从原始像素中以超越人类的水平玩几十款 Atari 游戏。这登上了新闻头条。但结果表中埋藏着一款 DQN 得分为 **0** 的游戏 —— 就像什么都没做一样：**Montezuma's Revenge**。

为什么？看看这款游戏在第一个房间里对你的要求：

1. 爬*下*梯子。
2. 穿过窗台。
3. 跳过一个滚动的头骨（时机不对 → 你就死了）。
4. 爬*上*另一个梯子。
5. 拿到钥匙。

这大约需要 **100 次精确的按键操作**，而在钥匙到手之前，游戏 **一分都不会给你**。在整个开场序列中，奖励信号是一个平坦、毫无特征的 **零**。

普通的强化学习智能体通过调整朝向它实际获得的奖励来学习。如果它能到达的任何地方奖励都是零，那就*没有学习的依据* —— 这就像试图通过感受下坡方向来寻找一个完全平坦的山谷底部。所以 DQN 只是在起始平台上永远地晃来晃去。Montezuma 成为了 **困难探索 (hard exploration)** 的基准：这款游戏你只有通过*聪明地*探索而非随机探索才能通关。

突破发生在 2018 年，随着 **随机网络蒸馏 (Random Network Distillation, RND)** 的出现 —— 其技巧正是工作项 1 的主题：添加一个 **内在好奇心奖励 (intrinsic curiosity bonus)**，让智能体在到达新画面时奖励*自己*，突然间就有了一个密集的信号引导它深入关卡。RND 在 Montezuma 上获得了超越人类的分数。（后来还有：Go-Explore, Agent57, ……）

## 现实生活中“Montezuma 式”稀疏奖励的例子

- **组合锁 / 带有隐晦线索的寻宝游戏。** 没有过程分。在突然拿到奖品之前，你的得分一直是零。
- **论文被接收，或初创公司实现盈利。** 几个月没有外部奖励，然后（可能）有一个巨大的奖励。
- **视频游戏速通路线。** 连续进行几十次帧精确的输入，在技巧成功或失败之前没有任何反馈。
- **密室逃脱。** 在你把几个发现串联起来之前，房间几乎不会告诉你任何信息。

在所有这些情况中，“随便试试”是毫无希望的。你需要*系统地*探索 —— 而内部的“噢，那是新东西，继续前进”信号正是让你保持系统性的关键。

## 为什么我们在这里不真的在像素级的 Montezuma 上训练

要妥善地完成*真实*的训练意味着：

- 使用卷积网络来查看 210×160 的 RGB 画面，
- 帧堆叠 (frame-stacking)（以便智能体可以辨别头骨移动的方向），
- 一个 RND 模块（另外两个网络：一个固定的随机“目标”网络和一个训练好的“预测”网络），
- 以及 **数千万帧的环境交互** —— 这需要耗费大量的 GPU 运行时间。

那是一个研究项目，而不是教学脚本。所以 `montezuma_revenge.py` 做了两件更实际的事情：

### 1. 它会*触碰*一下真实的游戏（如果安装了 `ale-py`）

它通过 Gymnasium 加载 `ALE/MontezumaRevenge-v5`，运行一个 **均匀随机智能体 2000 步**，并报告游戏总奖励。它打印出的数字几乎总是 **0.0** —— 将抽象的“稀疏奖励”一词变成了一个具体的、可以亲自运行的事实。如果未安装 Atari 软件包，它会打印单行 `pip install` 命令并继续。

### 2. 它在一个*比例模型* `MiniMontezumaEnv` 上训练表格智能体

这是一个微小的网格世界，其*骨架*与 Montezuma 的第一个房间相同：

```
###############
#S....#.......#
#.....#.......#
#.....#...T...#     S = 起点 (start)
#.....D.......#     K = 钥匙 (key)      D = 门 (door)（仅在携带钥匙时可通过）
#..K..#.......#     T = 宝藏 (treasure)（唯一能提供奖励的格子）
###############
```

要获胜，你必须：走到 **钥匙** 处（约 6 步），捡起它；走到 **门** 处（约 4 步） —— 门现在会打开；走过去到达 **宝藏** 处（约 5 步）。大约 **15 次完美的移动**，在拿到宝藏之前 **没有任何反馈**。`has_key` 标志是智能体状态的一部分，所以一旦你抓住了钥匙，就会有一整个新房间的状态等待发现 —— 就像在真实游戏中打开了新画面一样。

然后我们训练两次普通的 **表格 Q 学习器**：

| 智能体 | 在 MiniMontezuma 上的结果 |
|-------|--------------------------|
| **无好奇心 (epsilon-greedy)** | 在所有 1,500 个回合中，回报保持为 **0**。它甚至从未到达钥匙处。（听起来耳熟吗？这就是真实游戏中的 DQN。） |
| **带预测误差好奇心奖励** | 在约 20-25 个回合内到达宝藏，然后学会了 **最优的 15 步路线**。（这就是缩小到 Q 表大小的 RND 思想。） |

图表并排显示了两条学习曲线，以及好奇智能体学会的实际路线，并在网格上绘制出来（起点 → 钥匙 → 门 → 宝藏）。脚本还会以 ASCII 帧的形式打印该路线。

## 课程总结

> **“稀疏奖励”不仅仅是一个奇怪的 Atari 游戏的怪癖 —— 它是任何成功需要长距离、特定序列动作的世界中的默认状态。** 一个仅靠奖励驱动的智能体（如原生 DQN）简直无法起步：没有可以遵循的梯度。好奇心奖励制造了一个梯度 —— 一个密集的、自我生成的“这是新东西，继续前进”信号 —— 正是这个信号带着智能体穿过零奖励的荒漠，找到第一个真正的奖励。之后的一切（RND, Go-Explore, Agent57）都是同一招式的扩展、神经网络版本。

## 关键术语

| 单词 | 含义 |
|------|---------|
| **Hard exploration (困难探索)** | 只有通过聪明地探索才能成功的问题；随机探索会失败 |
| **Sparse reward (稀疏奖励)** | 奖励在几乎所有地方都是零；只有在完成一段很长的正确序列后才能获得 |
| **Montezuma's Revenge** | 经典深度强化学习智能体（DQN, A3C）得分为 0 的 Atari 游戏 —— 公认的困难探索基准 |
| **RND (随机网络蒸馏)** | 2018 年利用预测误差好奇心奖励击败 Montezuma 的方法 |
| **Go-Explore** | “记住有前途的状态，回到那里，然后从那里开始探索” —— 另一个 Montezuma 攻克者 |
| **Scale model (比例模型)** | 一个保留了难题*结构*的小型、廉价环境，以便你可以快速研究 |

---

## 一句话总结

> **Montezuma's Revenge 是一款让强化学习界明白“从未收到的奖励无法教给你任何东西”的游戏 —— 而当时的以及现在的解决方法都是提供好奇心奖励，让智能体在找到真正的奖品之前通过探索来奖励自己。**
