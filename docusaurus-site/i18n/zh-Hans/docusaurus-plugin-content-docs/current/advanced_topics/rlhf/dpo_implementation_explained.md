# DPO：跳过裁判，直接寻找源头

## 核心思想

经典的 RLHF 分为两个阶段：首先训练一个奖励模型，然后使用 PPO 来追逐它的分数。DPO (Direct Preference Optimization，直接偏好优化) 提出了一个聪明的疑问：

*如果奖励模型只是一个中间步骤，我们能否跳过它？*

事实证明：可以。DPO 直接从偏好对中训练语言模型，无需独立的裁判，无需 PPO 采样循环，也无需调整 KL 系数。它使用一个优雅的公式，运行起来就像监督学习一样。

这使得 DPO 运行更简单、更稳定、更快速——这也是为什么它迅速成为许多开源对齐模型的默认选择。

## 现实生活中的比喻

假设你正在指导一名学生写论文。

PPO 的方法是：雇佣一名老师为论文打分，然后让学生写一篇又一篇的论文，并根据老师的分数进行调整。

DPO 的方法是：一次向学生展示两篇论文，并说：“这一篇更好——倾向于像这一篇这样写，远离那一篇。” 中间没有老师。学生直接从比较中进行调整。

两种方法都有效。DPO 通常完成得更快，因为没有人需要训练和维护一个单独的老师。

## 学习原理 (直观理解)

DPO 使用与奖励建模相同的偏好对：提示词 (prompt)、选定回答 (chosen)、拒绝回答 (rejected)。对于每一对，它会问两个问题：

1. 与参考模型相比，模型产生选定回答的 **可能性是否增加了**？
2. 与参考模型相比，模型产生拒绝回答的 **可能性是否降低了**？

训练过程会同时推动这两个数字向正确的方向发展。至关重要的是，参考模型在比较中始终存在——它扮演着与 PPO 中 KL 惩罚相同的角色。允许模型发生偏移，但这种偏移始终是 *相对于* 起点的。

DPO 论文中一个微妙而精妙的结论是，这个单一的损失函数在数学上等同于“训练一个奖励模型，然后运行带有 KL 惩罚的 PPO”。目的地相同，但旅程更简单。

## 实验显示了什么

我们在 2,000 个偏好对上直接训练了一个策略，进行了 300 个 epoch。

![DPO 训练](outputs/dpo_implementation.png)

- **左图** - 随着模型学会偏好选定回答而非拒绝回答，DPO 损失下降。
- **中图** - 偏好准确度（策略为选定回答分配更高隐式奖励的频率）上升到约 99%。
- **右图** - 隐式奖励差距增长。DPO 从不明确给出一个“奖励”，但选定回答与拒绝回答之间的对数概率差距（经 beta 缩放）可以被视为奖励。它稳步扩大，意味着模型对其偏好变得更加自信。

注意这与 PPO 相比看起来是多么整洁。没有采样循环，没有探索噪声，也没有单独的奖励模型在运行。每个 epoch 都是在偏好数据集上进行的纯监督式更新。

## 它在 RLHF 流水线中的位置

DPO 是经典流水线第二步和第三步的 *替代方案*：

- **经典流程：** 偏好数据 → 奖励模型 → PPO → 对齐后的模型。
- **DPO 流程：** 偏好数据 → 对齐后的模型。(完成。)

需要注意的是，DPO 是在固定的偏好数据集上进行训练的。而 PPO 因为每一轮都会采样新的回答，原则上可以探索得更远。在实践中，对于大多数“在策划好的偏好数据集上进行对齐”的使用案例，DPO 都能胜出。

## 为什么这在实验室之外也很重要

“跳过中间测量”的模式无处不在：

- 教练通过并排演示来纠正游泳者的姿势，而不是用秒表为每一圈打分。
- 摄影师一次编辑两张照片，挑选更好的一张，而不是建立一个“好照片”评分准则。
- 招聘经理比较两份简历，而不是根据 30 项清单为每份简历打分。

当你只需要进行 *排序* 时，你不需要一个绝对的量表。DPO 就是将这一见解应用于语言模型的结果。

## 一句话总结

**DPO 将偏好对直接转化为更好的模型，中间没有奖励模型——比 PPO 更简单，且通常同样出色。**
