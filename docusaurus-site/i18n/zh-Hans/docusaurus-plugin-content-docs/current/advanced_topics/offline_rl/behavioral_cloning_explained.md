# 行为克隆 (Behavioral Cloning, BC) 🐒

## 什么是行为克隆？

想象一下你想学习打网球。你观看了数百小时温布尔登网球公开赛的录像，并简单地 **模仿球员的动作**。你并不去思考他们的击球是否是 *最佳* 击球——你只是让自己的身体位置与他们保持一致，并以同样的方式挥拍。

这就是行为克隆。**没有奖励，没有规划，只有模仿。**

在强化学习术语中：获取 `(状态, 动作)` 对的数据集，并训练一个神经网络根据状态预测动作，就像图像分类模型预测猫 vs 狗一样。这里的“标签”就是数据采集者所采取的任何行动。

---

## 它与“真正的”离线强化学习有何不同

| 方法 | 是否使用奖励？ | 能否超越数据？ |
|----------|---------------|---------------------|
| **BC**   | ❌ 否         | ❌ 否——最好的情况下，也只能达到数据的平均水平 |
| **CQL** (及类似方法) | ✅ 是 | ✅ 是——能从混合质量的数据中组合出优秀的决策 |

BC 是强化学习的“监督学习视角”。它非常简单，通常表现得惊人地强劲，是通用的基准。**如果一个离线强化学习算法在相同的数据集上不能击败 BC，那么它就毫无意义。**

## 现实生活中的例子

- **从行车记录仪视频中学习驾驶。** 观察道路，预测人类使用的方向盘角度。两个具有里程碑意义的例子：
  - **ALVINN (1989)** —— 有史以来第一个神经网络驾驶员；一个微小的 3 层网络，通过摄像头和激光输入进行训练，在高速公路上驾驶面包车。
  - **NVIDIA PilotNet (2016)** —— 一个现代深度 CNN，在行车记录仪素材上进行端到端训练；纯粹通过模仿人类驾驶员学习车道保持和基本转向，无需手工设计的规则。
- **学徒模仿名厨。** “厨师做什么，我就做什么。” 如果厨师水平很高，这种方法效果很好；如果厨师水平很差，那学徒也会变成一个平庸的厨师。
- **GitHub Copilot。** 自动补全通过训练来预测“人类下一步会输入什么代码？”——纯粹是对源代码日志的模仿。
- **模仿你的哥哥姐姐。** 孩子们在开始思考哥哥姐姐 *为什么* 这样做之前，会模仿他们很多年。

## 数学表达 (一行代码)

对于数据集中的每个 `(s, a)`，最小化：

```
loss = -log π(a | s)        (交叉熵)
```

就是这样。策略 `π` 只是一个输出动作 Logits 的 MLP（多层感知机）；训练过程与 MNIST 完全相同。让我们拆解一下这些专业术语：
- **`π` (Pi)：** “策略”的标准符号——决定做什么的规则或神经网络。
- **MLP (多层感知机)：** 一种基础的标准神经网络。
- **Logits：** 网络在将其转化为概率之前直接输出的原始、未经归一化的分数。
- **交叉熵 (Cross-entropy)：** 当模型为正确答案分配低概率时对其进行惩罚的标准公式。
- **MNIST：** 著名的手写数字入门数据集。

通过 BC 训练一个智能体玩游戏，在本质上与训练一个网络识别 MNIST 中的手写数字完全相同。在 MNIST 中，输入是图像，输出是数字 (0-9)。在 BC 中，输入是游戏状态，输出是动作（例如，“向左移动”）。

## 我们的代码做了什么

脚本 `behavioral_cloning.py` 执行以下操作：

1. **加载由 `d4rl_dataset.py` 构建的所有四个数据集** (`random`, `medium`, `expert`, `medium-replay`)。
2. 为每个数据集 **训练一个独立的 BC 策略**，进行 10,000 步交叉熵梯度下降。完全忽略奖励列。
3. 每隔 2,500 步，通过在真实的 CartPole-v1 环境中以贪婪方式运行当前策略来进行 **评估**（运行 20 个回合，取平均值）。
4. 绘制以下图表：
   - 柱状图：每个数据集的最终 BC 回报。
   - 学习曲线图：每种 BC 变体在训练过程中的提升情况。

## 你应该看到的结果

典型运行结果如下：

```
Final evaluation returns:
  BC on random          ->    ~20  ± 少量   (≈ 随机游走)
  BC on medium          ->   ~150  ± 波动   (≈ 中等水平策略)
  BC on expert          ->   ~480  ± 少量   (≈ 专家水平策略)
  BC on medium-replay   ->    ~60  ± 波动   (≈ 混合数据的平均水平)
```

柱状图清晰地揭示了事实：**BC 的回报遵循数据集的平均回报。** 它无法超越这个上限，因为它无法在混合数据集中区分“好的”部分和“坏的”部分——两者都被视为同等有效的模仿目标。

结论是：**BC 继承了数据的上限。**

## BC vs CQL —— 最直观的比较

在 **medium-replay** 数据集上（最现实、质量参差不齐的情况）：

| 方法 | 约最终回报 | 为什么？ |
|--------|--------------------:|------|
| BC     | ~60   | 模仿了失败的早期运行和后期良好运行的 *平均水平* |
| CQL    | ~400+ | 使用奖励来偏好高 Q 值的转移；从混合数据中拼凑出一个优秀的策略 |

因此，CQL **超越了数据**，而 BC **匹配了数据**。这就是为什么离线强化学习是一个研究领域，而不仅仅是“做模仿学习”的原因。当数据质量参差不齐（真实的日志通常如此）时，奖励感知方法能恢复出更多价值。

在 **expert** 数据上，比较结果发生了逆转：BC 匹配了专家水平 (~480)。你可能会奇怪为什么 CQL 在这里只是“打平”而不是输掉。因为 CQL 被设计为 *保守* 的，并惩罚数据集中未见过的动作，它最终做的正是专家所做的。它无法击败专家（因为已经达到了最高可能的分数），但它也不会主动破坏专家的策略。它只是与 BC 的表现持平。

这就是著名的“数据质量 vs 算法”权衡：

```
                            专家 (EXPERT) 数据  →  BC 胜出, CQL 打平
   算法复杂程度              ↑         
                            混合 (MIXED) 数据  →  CQL 明显击败 BC
                            
                            随机 (RANDOM) 数据  →  全部失败; 需要探索
```

## BC 在现代强化学习中的应用

- **在线强化学习的预训练。** 许多现代系统（如 RT-1, Voyager, 游戏机器人）先在演示数据上使用 BC 进行预训练，然后使用 PPO/SAC 进行在线微调。
- **RLHF。** InstructGPT 的第一步是监督微调——纯粹是对人类撰写回答的 BC。PPO + 奖励模型是后续步骤。
- **DAgger (Ross et al., 2011)。** 一个巧妙的扩展，用于解决 **复合误差 (compounding-error)** 问题。
  *为什么即使 BC 克隆得很完美，复合误差仍然是个问题？* 即使 BC 模型有 99% 的准确率，那 1% 的错误最终也会发生。一旦发生，智能体就会进入完美驱动数据集中从未见过的状态。因为它感到困惑，所以会犯更大的错误，从而进一步偏离已知数据，最终导致彻底失败（比如开车掉下悬崖）。
  *解决方法：* 我们可以一直让专家开车，但专家的时间很昂贵。相反，DAgger 让 BC 策略去驾驶。当策略犯错并漂移到怪异状态时，我们暂停，询问专家“你 *在这里* 会怎么做？”，并将此添加到数据集中。我们只“在 BC 策略访问的状态上再次查询专家”，因为我们只需要专家教我们如何从特定的错误中恢复，而不是一直查询他们。
- **Decision Transformer (Chen et al., 2021)。** 一种“聪明”的 BC，它将动作预测建立在所需的 *待收回报 (return-to-go)* 之上，实质上将离线强化学习转回为“预测下一个 Token”的问题。

## 关键术语

| 术语 | 含义 |
|------|---------|
| **模仿学习 (Imitation learning)** | “模仿演示者”的统称；BC 是其中最简单的成员 |
| **复合误差 (Compounding error)** | 一个微小的 BC 错误带你进入数据集未见过的状态，错误在此不断累积 |
| **演示数据 (Demonstration data)** | 由专家生成的轨迹，用作 BC 训练集 |
| **数据上限 (Data ceiling)** | BC 的回报受限于数据集中平均回报的上限 |
| **DAgger** | 一种交互式解决复合误差的方法 |

---

## 一句话总结

> **行为克隆只是在（状态, 动作）对上进行的监督学习——当数据质量良好时表现强劲，当数据质量参差不齐时则无能为力。**
