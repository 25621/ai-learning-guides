# 保守 Q 学习 (Conservative Q-Learning, CQL) 🛡️

## 什么是 CQL？

想象一下，你正在通过阅读一本包含他人过去股票交易记录的大型账本来学习如何投资。账本里记录了买入、卖出和持有——但 **没有记录任何没有人真正做过的交易**。

现在想象一个过度自信的学生看着账本说：*“如果有人每个周一都买彩票会怎样？那肯定是一笔惊人的交易！”*

问题在于：**账本里没有关于周一买彩票的数据**，所以这个学生只是在幻想。然而，那种幻想出来的交易在纸面上看起来很棒，所以学生的“策略”一直想要执行它。

这种幻想问题就是 **分布偏移 (distribution shift)**：离线学习者会偏爱数据集从未测试过的动作，因为没有数据可以反驳这种乐观态度。CQL 就是针对这一问题的良方。

---

## 为什么 Q 学习在离线状态下会出错

普通的 Q 学习目标是：

```
target(s, a) = r + γ · max_{a'} Q(s', a')
```

那个 `max_{a'}` 就是危险所在。当数据集从未在状态 `s'` 下记录过动作 `a'` 时，网络只会 *猜测* 一个 Q 值——而神经网络倾向于对未见过的输入 **高估** Q 值。目标值继承了这种高估，网络学会预测那个更大的数字，然后在下一步我们又会 **外推 (extrapolate)** 到更高的值（投影到远超数据支持的范围）。策略因此追逐着一个幻象。

如果你能继续收集更多数据，这将会自我修正（幻象动作在现实中被证明是糟糕的）。但 **在离线强化学习中，你无法收集更多数据。** 幻象将永远存在。

---

## CQL 的技巧

CQL (Kumar et al., 2020) 在损失函数中添加了一个惩罚项：

```
cql_loss(s)  =  log Σ_a exp Q(s, a)   -   Q(s, a_dataset)
```

它包含两个部分：

1. **`log Σ_a exp Q(s, a)`**（读作：*“所有动作的 log-sum-exp”*）是所有动作的 **软最大值 (soft maximum)** —— 这是一个平滑且可微的 `max` 近似值，它同时考虑所有动作，而不是硬性选择一个胜出者。惩罚这一项会 **全面** 降低 Q 值（统一压低所有预测值）——特别是对于那些具有 *最高* Q 值的动作，而这正是幻象所在。
2. **`- Q(s, a_dataset)`** 奖励数据集实际记录动作的高 Q 值——保护分布内 (in-distribution) 的 Q 值不受上述压低的影响。

最终效果：**未见动作的 Q 值被压低，已见动作的 Q 值被拉高。** 学习到的 Q 值成为了真实 Q 值的 *下界*。**`argmax`** 策略（即简单选择具有最高 Q 值的动作的规则）不再追逐幻象。

总损失函数为：

```
L  =  Bellman_MSE   +   α · cql_loss
```

（其中 **`Bellman_MSE`** 是普通 Q 学习的标准误差，衡量网络当前的猜测与其自身未来猜测之间的差异）。

`α` 是保守系数调节旋钮。太小 → 分布偏移会再次潜入。太大 → 智能体会过于保守，永远无法超越数据的水平。

## 现实生活中的例子

- **保守的象棋教练。** 你只能从已经下过的棋局中学习。鲁莽的教练会说：“这一步没有先例的假设性着法可能是天才之举！” CQL 则是那位会说“我们没有关于那一着的数据——让我们坚持现实棋手尝试过的着法”的教练。
- **餐厅菜单选择。** Yelp 评论从未涵盖菜单外的隐藏菜品。一个天真的策略会根据幻想出来的五星评分推荐隐藏菜品。CQL 则只推荐那些被点过足够多次、值得信赖的菜品。
- **从日志中学习机器人抓取。** 机器人有抓取杯子、瓶子和书本的视频——但从未抓取过刀。CQL 会拒绝自信地推荐“抓住刀刃”。

## 我们的代码做了什么

脚本 `cql.py` 执行以下操作：

1. **加载由 `d4rl_dataset.py` 构建的四个数据集**。
2. **选择 `medium-replay`** 作为训练集——这是最现实的（质量参差不齐），也对天真方法最具破坏性。
3. **在完全相同的条件下训练三个纯离线智能体**，唯一的区别是 `α`：
   - `α = 0`   →  天真的离线 DQN (无惩罚——那个有缺陷的基准)
   - `α = 1.0` →  轻度 CQL
   - `α = 5.0` →  强力 CQL
4. **每隔 2,500 步梯度下降进行一次评估**，通过在真实环境中以贪婪方式运行策略（10 个回合）。这是 *唯一* 与环境接触的地方；训练本身从未见过环境。
5. **将学习曲线绘制** 到 `outputs/cql.png`。

## 你应该看到的结果

典型运行结果打印如下：

```
Final evaluation returns (avg over 10 episodes, greedy):
  naive offline DQN (alpha=0)         ->  ~30-150  (不稳定; 经常崩溃)
  CQL (alpha=1.0)                     ->  ~300-450
  CQL (alpha=5.0)                     ->  ~450-500
```

在学习曲线图中：

- **红色曲线** (`α = 0`) 早期上升，然后经常 **坠入深渊**，一旦分布偏移的幻象感染了 **贝尔曼目标 (Bellman target)**（我们在训练 Q 网络时用作“正确答案”的数字：`r + γ · max Q(s', ·)`）。当幻象 Q 值污染了那个目标，每一步梯度下降都会让情况变得更糟。**贝尔曼损失 (Bellman loss)**（Q 网络的预测与贝尔曼目标之间的 MSE）看起来很正常——这就是问题的 **诡诈之处**：网络与其自身的错误信念完美一致，因此损失函数不会发出警告。
- **橙色曲线** (`α = 1.0`) 上升较慢，但 **保持在高位**。
- **绿色曲线** (`α = 5.0`) 最稳定，通常也是最好的。

贝尔曼损失面板显示了另一个迹象：天真的 DQN 损失可以保持很小，但其策略却极其糟糕，因为网络内部与其自身的幻象是一致的。

## CQL 在该领域的位置

CQL 意义重大，因为它为分布偏移提供了一个有原则且简单的修复方法。演进路线如下：

```
DQN (在线)
   │
   ▼
天真离线 DQN  ── 因分布偏移而失效
   │
   ▼
CQL (Kumar 2020)    ── 添加保守惩罚：Q 是一个下界
   │
   ▼
IQL (Kostrikov 2021)  ── 根本不查询未见动作的 Q 值
   │
   ▼
Decision Transformer (Chen 2021)  ── 完全跳过 Q 值，将强化学习视为序列建模
                                      （在给定过去状态和所需总回报的情况下
                                       预测 *下一个动作*，就像 LLM 预测
                                       下一个词一样）
```

这条演进路线上的每一步都是对同一个问题的不同回答：**我如何避免向我的 Q 网络询问它从未见过的事情？**

---

## 关键术语

| 术语 | 含义 |
|------|---------|
| **分布偏移 (Distribution shift)** | 训练好的策略想要的动作超出了数据的范围 |
| **分布外 (Out-of-distribution, OOD)** | 数据集从未记录过的 (s, a) 对 |
| **真实 Q 值** | 如果我们能完美测量，在状态 `s` 下采取动作 `a` 的真实预期未来回报 |
| **保守 Q 值** | 一个学习到的 Q 函数，试图保持在真实 Q 值之下，而不是过度承诺 |
| **Logsumexp** | 一个平滑、可微的 `max` 近似函数 |
| **Alpha (α)** | CQL 的保守系数——压低 OOD 动作 Q 值的力度 |

---

## 一句话总结

> **CQL 添加了一个“悲观惩罚”，惩罚数据集从未尝试过的动作上的高 Q 值——这样策略就不会迷恋上幻想。**
