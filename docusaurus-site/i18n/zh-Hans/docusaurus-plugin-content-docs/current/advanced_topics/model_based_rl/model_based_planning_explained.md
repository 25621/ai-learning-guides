# 使用学习到的模型进行规划 (MPC) 🔮

## 核心理念

你已经有了一个 **世界模型 (world model)**（一个可以预测未来的神经网络）。现在该怎么办？

最直接的用途是 **规划 (planning)**：在每一时刻，询问模型：“如果我尝试*这个*计划会发生什么？*那个*计划呢？或者是*另外一个*计划？” 然后执行看起来最好的那个计划 —— 但 **只执行其中的第一步**。因为模型并不完美，我们只执行一个动作，观察真实环境反馈的实际新状态，然后从头开始重新规划。

这个技巧有一个专门的名字：**模型预测控制 (Model Predictive Control, MPC)**。

---

## 现实生活类比

你在餐厅看菜单。你不会当场就定死五道菜判的顺序 —— 你先点第一道菜，看看自己感觉有多饱，然后再决定甜点。

或者：你在一条弯曲的道路上开车。你不会锁定接下来 30 秒的转向输入 —— 你不断向前看，规划未来几秒钟的路线，执行下一个转向动作，然后重新规划。

这种 **长远规划 / 近期执行 / 重新规划** 的循环就是 MPC。

---

## “随机射击 (Random Shooting)” 是如何工作的

还有更高级的规划器 —— 例如：
- **CEM (交叉熵方法)**：通过每轮只保留前 k 个最佳计划，迭代地改进计划的分布。
- **MCTS (蒙特卡洛树搜索)**：由模拟统计数据引导构建搜索树，被 AlphaGo 和 MuZero 使用。
- **基于梯度的规划器**：对模型关于动作的预测进行求导，直接遵循梯度。

我们使用的是最简单但有效的方法：**随机射击 (random shooting)**。

```
给定当前状态 s：
    1. 采样 N=200 条长度为 H=15 的随机动作序列。
    2. 对于每条序列，通过世界模型从 s 开始模拟，累加每一步的塑形奖励（并行进行 200 个梦想 —— 速度极快！）。
    3. 找到预测总奖励最高的序列。
    4. 在真实环境中执行该序列的“第一步”动作。
    5. 观察真实环境的下一个状态。丢弃计划的剩余部分。
    6. 回到步骤 1 —— 从头开始重新规划。
```

200 个计划 × 15 步 = 每一个真实步骤对应 3,000 次想象的转移。世界模型在单次神经网络批量前向传递中运行所有这些 —— 通常只需几毫秒。

---

## 为什么要每一步都重新规划？

因为模型是不完美的。误差会在展开 (rollout) 过程中累积（正如 `world_model.py` 生成的图表 `outputs/world_model.png` 所显示的）。在第 0 步制定的计划仅在前几步是可靠的；到第 15 步时，模型可能就在“产生幻觉”了。因此，我们只信任 **第一步动作**，然后根据最新的真实状态刷新计划。

这也是人类不会制定一个 100 步的国际象棋计划并死守不放的原因 —— 情况在变，你预测得越远，它与现实的契合度就越低。

---

## 一个小细节：奖励必须告诉规划器一些信息

在 CartPole 中，真实的奖励是每一步 `+1`，直到杆子倒下。模型会对几乎每个计划都忠实地预测 `+1, +1, +1, ...`，因为随机计划在模型内部很少会很快结束 —— 这样每个计划的得分都一样。规划器无法从中做出选择。

解决方法：在规划过程中将真实奖励替换为 **平滑的代理奖励 (smooth proxy)**：

```python
reward_proxy(state) = 1
                    - |杆子角度| / 0.21          # 杆子竖直吗？ (1=是)
                    - 0.1 * |小车位置| / 2.4     # 小车在中心吗？ (1=是)
```

现在，*可能*导致杆子倒下的计划得分会明显低于保持竖直的计划。规划器就可以对它们进行排名。

> **现实生活的教训：** 扁平的奖励信号 —— “你又多活了一秒” —— 对于短期规划毫无用处。密集的、塑形过的信号更有帮助。

## 我们的代码做了什么

`model_based_planning.py`：

1. **加载** `world_model.py` 保存的世界模型权重。（如果缺失，它会现场重新训练一个。）
2. 在真实的 CartPole-v1 上 **运行 20 个回合** 的 MPC。
3. 同时 **运行 20 个回合** 的均匀随机策略作为基准。
4. 将两者并排 **绘图** 并打印平均值。

### 运行后你应该看到的结果

| 策略 | 平均奖励 (生存步数) |
|--------|-------------------------------:|
| 随机 (Random) | ~22 (CartPole 的典型值 —— 杆子很快倒下) |
| MPC (我们的) | ~150–500 (随种子变化；许多回合接近 500) |
| 最大可能值 | 500 |

这种 **5–25 倍的提升** 是在没有策略网络、没有价值函数且没有进一步训练的情况下实现的。仅仅是一个世界模型 + 每步 200 个梦想。

图表 `outputs/model_based_planning.png` 显示了每个回合的两个彩色条 —— MPC 几乎总是比随机策略高，且许多回合达到了 500 步的上限。

## 基于模型的规划的优势

- **样本效率高。** 所有的学习都是从一批随机转移中完成的。不需要进一步的环境交互即可得出有用的策略。
- **易于重新设定目标。** 想以不同的方式控制智能体？更改奖励代理即可 —— 无需重新训练。（为了好玩，可以尝试最大化小车速度。）
- **可解释性强。** 你可以检查智能体考虑过的计划、预测的轨迹以及得分。

## 局限性（以及人们如何应对）

- **随机射击很笨。** 它是盲目采样计划。对于更高维度的空间，可以切换到 **CEM**（见上文）或 **iLQR**（迭代线性二次型调节器，一种经典的优化控制方法，将模型近似为局部线性并解析求解），或者使用 **基于梯度的规划器** 通过可微分模型遵循梯度来改进动作。
- **复合模型误差。** 长视野会产生漂移。人们使用 **概率系综 (probabilistic ensembles)**（在相同数据上训练的多个模型，如 PETS，Chua 等人，2018），这样规划器可以注意到模型之间的分歧，并惩罚模型不确定的计划。
- **我们最终想要的是真实奖励。** 奖励塑形有所帮助，但对于更复杂的任务，人们会在世界模型*内部*训练一个 **价值函数 (value function)** —— 一个学习过的评论家，用于估算任何状态的长期回报，而无需手工构建代理奖励。**Dreamer**（完全在潜空间想象中训练执行者-评论家）和 **MuZero**（将 MCTS 与学习到的价值网络结合）都使用了这一思想。

## 这与现代系统的联系

你刚刚运行的精确配方 —— **学习到的动力学 + 规划** —— 是现代人工智能研究中一些最强强化学习系统的支柱：

- **MuZero** (DeepMind)：结合了学习到的世界模型和蒙特卡洛树搜索。它在不需要预先知道规则的情况下精通了围棋、象棋、将棋和 Atari 游戏。
- **Dreamer / DreamerV3** (Hafner 等人)：在学习到的 **潜空间 (latent-space)** 世界模型中训练策略（这意味着模型在预测未来之前将原始图像或状态压缩成紧凑、抽象的表示）。它在 100 多个基准测试中达到了最先进的 (SOTA) 性能。
- **PETS / PlaNet / TD-MPC**：这些算法家族（研究路线）将这一想法扩展到复杂的连续控制任务，如机器人。

你已经用几百行代码构建了这个家族中最袖珍的一员。

## 关键术语

| 术语 | 通俗解释 |
|------|---------------|
| **MPC** | 模型预测控制 —— 提前计划，执行一步，重新计划 |
| **Random shooting (随机射击)** | 对大量随机计划打分，选最好的那个 |
| **Horizon (H, 视野)** | 计划向前看多少步 |
| **N samples (N 个采样)** | 每步考虑多少个备选计划 |
| **Receding horizon (退避视野)** | 每一步都重新规划，而不是死守一个计划 |
| **Reward proxy / shaping (奖励代理/塑形)** | 一个平滑的替代奖励，为规划器提供有用的优化信号 |

## 一句话总结

> **一旦你有了世界模型，规划就只是“做一百个关于未来的梦，选最好的第一步，然后重复执行”。**
