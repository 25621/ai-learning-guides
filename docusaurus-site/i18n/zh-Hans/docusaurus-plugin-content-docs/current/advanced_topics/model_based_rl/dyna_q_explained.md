# Dyna-Q：通过想象学得更快 🧠

## 什么是 Dyna-Q？

想象一个名叫 Mia 的孩子正在熟悉她的新学校。每天她在走廊里走动并发现新事物：“图书馆在食堂后面”，“史密斯先生的房间在楼梯口附近”。

一个 **普通的 Q 学习** 学生只从她 *今天* 所做的事情中学习。如果她今天只是从教室走到食堂，她就只更新关于那一条路径的记忆。

而一个 **Dyna-Q** 学生则不同。每次在真实的走动之后，她都会坐下来花一分钟 **在脑海中回放** 她记住的几次过去的走动。每一次回放都会加强她的心理地图。几周后，她对学校了如指掌——不是因为她走得更多，而是因为她 **对自己已经看到的东西思考得更多**。

这就是 Dyna-Q 对强化学习智能体所做的：它既从真实的经验中学习，也从它在过程中建立的模型所产生的 **想象经验** 中学习。

---

## 三个要素

Dyna-Q 是 “Q 学习 + 模型 + 规划”。一个真实的步骤完成了 **三项** 工作：

1. **直接强化学习 (Direct RL)** —— 从 `(s, a, r, s')` 进行常规的 Q 学习更新。
2. **模型学习 (Model learning)** —— 记录下来：“当我在 `s` 下执行 `a` 时，我得到了 `r` 并到达了 `s'`。”
3. **规划 (Planning)** —— 从模型的记忆中随机选取 *n* 个 `(s, a)` 对，并进行 *n* 次额外的 Q 学习更新，**假装** 这些步骤刚刚发生过。

第三步就是魔法所在。如果 `n = 50`，世界上每一个真实的步骤都会导致对 Q 表的 **51 次更新**。智能体的学习速度比纯粹的 Q 学习者快约 50 倍（按真实步骤衡量）。

---

## 循环图解

```
                   ┌────────────────────────────────────┐
                   │                                    │
   真实世界  ──► 执行动作 a ──► 观察 (r, s')             │
                            │                           │
              ┌─────────────┼──────────────┐            │
              ▼             ▼              ▼            │
        Q 学习更新      Model[s,a] ← (r,s')    规划 ────┘
                                           (n 次想象更新)
```

模型只是一个查找表：
`(状态, 动作) → (奖励, 下一状态)`。构建成本低，查询成本也低。

## 现实生活中的例子

- **国际象棋研究**：特级大师们花数小时在脑海中回放他们自己的对局和大师对局。每一次回放都是“规划” —— 从已经发生的经历中额外学习。
- **音乐家练习音阶**：在演奏完一个棘手的小节后，他们在继续之前会在脑海中默演十次。手指没有动，但大脑正在更新。
- **自动驾驶汽车**：在等红灯闲置时，它会在模拟中回放最后一百次并线，以便在不磨损轮胎的情况下微调其策略。

## 我们的代码做了什么

我们使用经典的 **Dyna 迷宫 (Dyna Maze)**（[Sutton & Barto, 图 8.2](http://incompleteideas.net/book/the-book.html)）：一个带有围墙的 6×9 网格，起点 `S` 在左中位置，终点 `G` 在右上角。

我们运行了三个变体，每个变体在 30 个随机种子上取平均值：

| 设置 | 每步真实动作对应的规划步数 | 含义 |
|---------|------------------------------|---------|
| `n = 0` | 0 | 普通 Q 学习 |
| `n = 5` | 5 | 少量的想象练习 |
| `n = 50` | 50 | 大量的想象练习 |

脚本记录了随训练进行 **每回合平均真实步数**。步数越少意味着智能体学会了通往目标的更直接路径。

### 运行代码时你应该看到的结果

迷宫中最短路径约为 9 步；通过 ε-贪婪探索，训练良好的智能体平均每回合约 10 步。运行 50 个回合后，所有三种设置都会收敛到此结果 —— 区别在于 **速度**：

| 设置 | 每回合步数 (最后 10 回合) | 意味着什么 |
|---------|--------------------------------:|---------------|
| `n = 0`   | ~10 | 已收敛 —— 但它花费了约 30-50 个回合的徘徊才到达这里 |
| `n = 5`   | ~10 | 在约 10 个回合内收敛 |
| `n = 50`  | ~10 | 在约 3-5 个回合内收敛 |

有趣的信号是 *学习曲线*，而不是最终的数字。保存在 `outputs/dyna_q.png` 中的图表显示了三条以非常不同速率下降的曲线：`n = 50` 在寥寥几个回合内就达到了底部，而 `n = 0` 在运行了很久之后仍在缓慢下降。（在这样一个微小的确定性迷宫中，普通 Q 学习最终确实能达到目标 —— Dyna-Q 只是需要少得多的真实回合，这在真实步骤成本高昂的环境中非常有意义。）

## 为什么它在这个迷宫中效果这么好

有两个原因：

1. **环境是确定性的**：每个 `(s, a)` 总是给出相同的 `(r, s')`，所以模型在访问一次后就是准确的。想象的经验和真实的经验一样好。
2. **真实步骤很昂贵，想象步骤是免费的**：每一次想象的更新只是几次查表，而真实的步骤需要智能体去行走。当真实交互成本高昂时（想想：真实的机器人，真实的游戏），Dyna-Q 具有极高的样本效率。

---

## Dyna-Q 的局限性

- **随机性环境**：如果 `(s, a)` 可以导致许多不同的 `s'` 值，那么“记住最后结果”的模型就会误导你。解决方法：存储访问计数或训练概率模型。
- **非平稳环境**：如果世界发生了变化 —— 例如，原本开启的门口突然关闭，或者出现了捷径 —— 模型就会过时并给出错误的预测。**Dyna-Q+** 通过添加 **探索奖励** 解决了这个问题：长时间未访问的状态会获得少量额外奖励，促使智能体回去检查世界是否发生了变化。
- **大型状态空间**：以 `(s, a)` 为键的字典无法扩展到数百万个状态或连续状态。这正是 **学习到的（神经网络）世界模型** 所填补的空白 —— 参见接下来的 `world_model.py`。

## 关键术语

| 单词 | 含义 |
|------|---------|
| **模型 (Model)** | 对 `(状态, 动作) → (奖励, 下一状态)` 的记忆 |
| **规划步 (Planning step)** | 使用来自模型的想象数据进行 Q 更新 |
| **直接强化学习 (Direct RL)** | 使用真实数据进行 Q 更新 |
| **样本效率 (Sample efficiency)** | 衡量 AI 模型或算法有效利用训练数据以达到特定性能水平的能力 |
| **Dyna** | Sutton 提出的将学习和规划交织在一起的架构 |

## 一句话总结

> **Dyna-Q 既从实践中学习，也从想象中学习 —— 而想象是免费的。**

这一思想在其现代神经形式中，驱动了一些有史以来最强大的强化学习智能体（MuZero, Dreamer, World Models）。
