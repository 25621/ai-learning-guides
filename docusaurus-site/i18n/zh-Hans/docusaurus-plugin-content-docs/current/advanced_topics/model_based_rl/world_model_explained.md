# 训练世界模型：教会智能体“做梦” 🌍

## 什么是“世界模型”？

**世界模型 (World Model)** 是智能体对*宇宙的内部拷贝*。给它一个状态和一个动作，它就能预测下一步会发生什么：

```
(状态, 动作)  ──►  神经网络  ──►  (下一个状态, 奖励)
```

它不是真实世界 —— 它是智能体通过观察现实并学习模仿现实，为自己构建的一个**模拟器**。

一旦训练完成，模型就能让智能体在不采取任何实际行动的情况下询问“如果……会怎样”的问题：

> *“如果我现在向左推，然后连续向右推两次，我会到哪里？杆子会倒吗？”*

智能体在模型内部思考一百个计划的时间，可能还不够在现实中做出一次移动。这就是其核心意义所在。

## 现实生活中的类比

想想*你*是如何解谜题的。你不会体力活似地把每一块拼图尝试塞进每一个槽位。你会**想象**如果拼图 A 放在这里会发生什么。如果脑海中的模拟看起来不对，你在动手之前就会否定它。

你的大脑拥有一套习得的世界模型 —— 建立在多年对物体行为观察的基础上 —— 这让你在付出行动前可以模拟结果。

其他例子：

- **棋手**会在脑海中预演后面几步。
- **司机**会想：“如果我现在刹车，后面的车是否有足够的空间。”
- **孩子**堆积木：“如果我把大的放在上面，塔会晃动。”（他们通过以前撞倒积木塔的经验学会了这一模型。）

在任何情况下，**心理模型 + 想象力 = 更少的风险、更好的决策**。

## 智能体如何构建它的模型？

它只是通过**观察**。具体来说：

1. **收集数据**。让任何策略（甚至是随机策略）与真实环境交互一段时间。保存每一次转换：
   ```
   (状态, 动作, 奖励, 下一个状态)
   ```
2. **训练神经网络**，根据 `(状态, 动作)` 预测 `下一个状态` 和 `奖励`。这是监督学习：每一个保存的转换都是一个带标签的示例，输入是“智能体看到的和做的”，标签是“接下来实际发生的”。
3. **验证**。预留 10% 的数据，检查模型的预测与真实情况的差异。误差低意味着模型已经掌握了环境的**动力学 (dynamics)**：即状态在动作后如何变化。

我们使用的一个技巧：与其直接预测 `下一个状态`，不如预测**增量 (delta)** `下一个状态 − 当前状态`。大多数物理现象是增量式的（“小车移动了一点点”），小目标对神经网络更友好。

## 我们的设置

| 选择 | 数值 | 理由 |
|--------|-------|-----|
| 环境 | `CartPole-v1` | 4 维状态，2 种动作 —— 易于建模 |
| 数据 | 来自随机策略的 20,000 次转换 | 对状态空间有广泛的覆盖 |
| 网络 | MLP, 2 × 128 ReLU 隐藏层 | MLP = 多层感知机（标准“原生”神经网络）。两个 128 神经元的隐藏层，使用 ReLU 激活函数。容量足够且训练快。 |
| 损失函数 | `(状态增量, 奖励)` 的 MSE | MSE = 均方误差（预测误差平方的平均值）。标准的回归损失函数。 |
| 优化器 | Adam, lr = 1e-3, 30 个 epochs | Adam = 自适应优化器（自动调整每个参数的学习率）。现成可用，无需特殊调优。 |

整个训练在 CPU 上几秒钟内即可完成。

## “好”模型是什么样的？

有两个评估指标至关重要：

### 1. 单步准确率（验证集 MSE）

这是指“模型预测未来一局的表现如何？”在 30 个 epochs 之后，你应该能看到验证集 MSE 在 **1e-4 到 1e-3** 范围内。这非常微小 —— 意味着杆的角度和小车的位置准确到了小数点后几位。

### 2. k 步展开中的**复合误差 (Compounding error)**

这是*真正的*测试。获取一个状态，将其输入模型，然后获取其预测值并再次输入模型 —— 连续进行 `k` 步。误差会增长，因为每一步都会在前一次预测的基础上增加一点噪声。

```
第 1 步： L2 误差 ≈ 0.01 （几乎完美）
第 5 步： L2 误差 ≈ 0.05
第 10 步：L2 误差 ≈ 0.15
第 20 步：L2 误差 ≈ 0.40 （明显偏离）
```

*（L2 误差 = 预测的下一个状态与真实状态之间的欧几里得距离 —— 可以理解为“模型在 4 维状态空间中的猜测偏离了多少？”）*

**为什么这很重要**：如果我们用模型规划未来 15 步，第 15 步的*确切*状态可能是错的 —— 但如果“好计划 vs 坏计划”的相对排名得以保持，规划依然有效。（这就是 `model_based_planning.py` 所利用的特性。）

`outputs/world_model.png` 中的图表并排显示了这两个诊断结果：训练损失曲线在对数坐标下漂亮地下降，而展开误差曲线则稳步上升。

## 为什么要预测“增量 (Delta)”？

对比向网络描述同一个问题的两种方式：

| 目标 | 典型量级 | 容易还是困难？ |
|--------|------------------:|--------------|
| `下一个状态`        | 0–2.4 (小车位置) | 网络必须复现位置**以及**微小的变化 |
| `下一个状态 - 状态` | ~0.02            | 网络只需学习微小的变化 |

预测增量还意味着：如果网络输出全零（就像未经训练的初始网络经常做的那样），预测结果仅仅是“没有移动” —— 这对于单一步伐来说是一个合理且安全的默认值。相比之下，直接预测绝对的 `下一个状态` 最初会输出完全随机的垃圾数值，导致早期训练高度不稳定。

## 这能带给我们什么？

训练好的世界模型是以下各项的基础：

- **规划 (Planning)** —— 在想象的动作序列中进行搜索（参见 `model_based_planning.py`）。
- **Dyna 风格的增强** —— 在想象的数据上训练 Q 网络，成倍提高样本效率。
- **好奇心/探索** —— 访问模型无法很好预测的状态。
- **Dreamer / World-Models 论文** —— 除了最初的数据收集外，完全在模型内部训练*策略*，与现实世界零交互。

## 局限性与注意事项

- **分布外漂移 (Out-of-distribution drift)**：模型只了解它见过的世界。如果规划得太激进，你可能会进入模型从未见过的区域 —— 那里的预测纯属幻想。
- **复合误差**：正如图表所示，由于误差累积，在长**时界 (horizons)**（预测未来的许多步）内进行规划是不可靠的。现代系统通过使用**概率集成 (probabilistic ensembles)**（训练多个模型并检查它们是否达成一致，如 PETS 或 Dreamer）来解决这个问题，这样规划器就能准确知道模型在每一步的*不确定程度*，从而避开风险较大的未知路径。
- **随机环境**：标准的确定性回归器只能预测*平均*结果，完全遗漏了可能结果的分布。复杂的现实环境需要概率模型（如具有高斯输出的模型，或**隐随机模型 (latent stochastic models)** —— 将世界状态编码为压缩空间中的概率分布的网络，使它们能够捕捉真正的随机性而非将其平均掉）来准确表示不确定性和随机性。

## 核心词汇

| 术语 | 通俗解释 |
|------|---------------|
| **世界模型 (World model)** | 模仿环境的神经网络 |
| **动力学 (Dynamics)** | 函数 `(s, a) → s'` |
| **奖励模型 (Reward model)** | 函数 `(s, a) → r`（通常绑定在一起） |
| **单步预测** | 模型根据真实状态输出的结果 |
| **展开 (Rollout)** | 连续的单步预测，将输出反馈作为输入 |
| **复合误差 (Compounding error)** | 在展开过程中不断增长的微小误差 |
