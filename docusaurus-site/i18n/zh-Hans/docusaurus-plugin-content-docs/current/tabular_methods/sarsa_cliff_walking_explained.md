# Cliff Walking 的 SARSA 算法 🏔️

## 什么是 SARSA？

想象一条**非常长的走廊**，边缘是一个**可怕的悬崖**。如果你从悬崖上掉下去，你就必须回到起点重新开始！你的目标是尽可能快地从走廊的一端走到另一端，且不掉下去。

**SARSA** 是一个通过练习来学习穿越这条走廊的机器人。它学习选择一条**安全路径**来避开悬崖——即使这条路稍微长一点——因为它知道自己在探索时可能会不小心滑到边缘！

---

## 核心思想：从你的实际行动中学习

SARSA 代表：**S**tate（状态）→ **A**ction（动作）→ **R**eward（奖励）→ **S**tate（状态）→ **A**ction（动作）

这是 SARSA 用来学习的五条信息：

1. **S** —— 我现在在哪？（当前状态）
2. **A** —— 我实际采取了什么动作？
3. **R** —— 我得到了什么奖励？
4. **S** —— 我最终到了哪里？（下一个状态）
5. **A** —— 我**下一步实际上打算采取**什么动作？

最后一个 "A" 是 SARSA 的特别之处！它使用它*下一步实际上会采取*的动作来进行更新（即使那是一个随机的探索性动作），而不是完美的理想动作。

**现实生活中的例子：** 想想学习骑自行车。如果你知道自己有时会随机摇晃（探索），你就会离停放的车辆稍微远一点——因为你知道摇晃的自己可能会撞上去！SARSA 也是这样：它学习一条安全路径，因为它考虑到了自己可能犯的随机错误。

```

[ ][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ]
[S][C][C][C][C][C][C][C][C][C][C][G]
   ← ← ← ← 悬崖 (CLIFF) ← ← ← ← ←
```

- **S** = 起点 Start (左下角)
- **G** = 终点 Goal (右下角)
- **C** = 悬崖 Cliff —— 踩到这里 = -100 奖励，重开！
- 其他每一步 = -1 奖励

## 我们的代码发现了什么

在对 SARSA 进行 500 个回合 (episodes) 的训练后：

| 结果 | 数值 |
|--------|-------|
| 最终 50 回合平均奖励 | **-21.6** |
| 最优（冒险）路径奖励 | -13 |

SARSA 学习到的策略是**沿着网格的顶部**行进——这是一条安全的迂回路线！它虽然多花了几步（-21 而不是 -13），但在训练过程中几乎从未掉下悬崖。

## 现实生活中的例子

- **护士给药**：即使存在稍微快一点的方法，也会采取经过验证的安全规程（安全路径），因为微小的错误（探索）可能是危险的。
- **航空公司飞行员**：即使快捷方式看起来更快，也会严格遵守检查清单（安全路径），以考虑到人为错误。
- **学习烹饪**：从经过充分测试的食谱（安全）开始，而不是尝试冒险的捷径。

## 需要记住的核心词汇

- **同策 (On-policy)**：学习它实际正在使用的策略（包括其随机错误）
- **SARSA 更新**：使用*实际*的下一步动作，而不是理论上的最佳动作
- **安全路径**：避开危险的长路径，考虑到了探索时可能犯的错误
- **TD (时序差分) 控制**：在每一步之后都更新数值（而不是等待整个回合结束）

核心思想：**SARSA 是诚实的——它从它实际所做的动作中学习，而不是从它希望自己所做的动作中学习。这让它在危险面前变得谨慎和安全！**
