# Frozen Lake 的 Q-Learning 智能體 🧊

## 什麼是 Q-Learning？

想像一個冰凍的池塘，冰面非常滑。這裡有一個**起點 (Start)** 和一個**終點 (Goal)**，中間還有一些**坑 (Holes)**。如果你掉進坑裡，就得重新開始！

由於冰面很滑，即使你想向右走，也可能向上或向下滑動。**Q-Learning 智能體**是一個通過不斷嘗試來學習的機器人——它會一遍又一遍地嘗試，學習如何從起點到達終點而不掉進坑裡！

---

## Q-Learning 中的 "Q" 代表什麼？

**"Q"** 代表 **"Quality"（質量）**——具體來說，是指在特定情況下採取特定動作的*質量*。

你可以把它想像成餐廳評分：“在這家餐廳點披薩有多好（質量如何）？”Q(s, a) 問的是：“當我在狀態 **s** 時，採取動作 **a** 有多好？”

高 Q 值意味著：“絕佳選擇！這個動作會帶來豐厚的獎勵。”
低 Q 值意味著：“餿主意！這個動作通常會導致麻煩。”

**現實生活中的例子：** 想像你是一個正在決定是否要在晚飯前吃糖的孩子。你現在“吃糖”的 Q 值可能很高（它嚐起來很棒！），但總體而言卻很低（媽媽會生氣，晚些時候你會覺得噁心）。Q-Learning 學習如何權衡這些未來的後果——而不僅僅是眼前的感受！

---

## 核心思想：神奇的分數表

Q-Learning 建構了一個大表，稱為 **Q 表 (Q-table)**。每一行代表冰面上的一個方格，每一列代表一個動作（左、右、上、下）。表中的數字就是**分數**：“從這個方格採取這個動作有多好？”

每當機器人嘗試移動時：
1. 它會得到回饋（是掉下去了？還是到達了終點？）
2. 它會使用以下公式更新表中的分數：

> **新分數 = 舊分數 + 學習率 × (實際發生的情況 − 我的預期)**

機器人基本上是在問：“這次移動比我想象的更好還是更差？”

**現實生活中的例子：** 想想嬰兒學習走路。每當他們嘗試邁步卻摔倒時，他們就會學到“那一步很糟糕”。每當他們成功時，他們就會記住“那行得通！”經過多次嘗試，他們就學會了走路。Q-Learning 也是在做同樣的事情，只不過是用一個表來記錄！

---

## Q-Learning 的特別之處：離策 (Off-Policy)！

這裡有一個巧妙的地方：當 Q-Learning 更新它的表時，它*總是假設它下次會做出完美的動作*，即使在訓練期間它有時會嘗試隨機動作。

這使得 Q-Learning 成為**離策 (Off-Policy)** 的：它*學習*到的策略（始終選擇已知的最佳動作）與其在訓練期間*遵循*的策略（有時選擇隨機動作進行探索）是分開的。具體來說，Q 表的更新使用的是下一個狀態的*最大* Q 值——理論上的最佳值——即使機器人的實際下一步可能是隨機的。

簡單來說：機器人可能會為了探索而隨機向左遊蕩，但它的學習過程仍然按照它下一步會採取*最佳*動作來計算。這種分離讓 Q-Learning 無論探索程度如何，都能收斂到最優策略。

---

## 我們的代碼發現了什麼

我們在 4×4 的滑冰面 Frozen Lake 上訓練了 **50,000 個回合 (episodes)**：

| 指標 | 結果 |
|--------|--------|
| 貪婪評估成功率 | **73.1%** |
| 階段性目標 (>70%) | ✓ **通過** |

冰面非常滑，所以即使是最優策略也無法 100% 獲勝！

學習到的 Q 表顯示，智能體弄清楚了：在避開坑的同時，向右和向下移動。

## 現實生活中的例子

- **自動駕駛汽車**：通過試運行學習在十字路口該走哪條車道。
- **推薦系統**：根據使用者是否喜歡之前的建議來學習推薦哪些電影。
- **視訊遊戲 AI**：一個通過嘗試多條路徑來學習穿越迷宮的角色。

## 需要記住的核心詞彙

- **Q 表 (Q-table)**：“每個狀態下每個動作有多好”的表格
- **Q(s, a)**：在狀態 s 採取動作 a 的分數
- **獎勵 (Reward)**：智能體採取動作後得到的東西（到達終點 +1，否則為 0）
- **離策 (Off-Policy)**：即使在隨機探索時也能學習最優策略
- **ε-貪婪 (ε-greedy)**（ε = epsilon）：大多數時候執行已知的最佳動作；有時進行隨機探索
- **折扣因子 γ (γ = gamma)**：未來獎勵的價值（就像更傾向於現在拿錢而不是以後拿錢）

核心思想：**Q-Learning 為每種情況建立了一份“小抄”，並不斷完善它，直到它知道在任何地方該如何邁出最佳一步。**
