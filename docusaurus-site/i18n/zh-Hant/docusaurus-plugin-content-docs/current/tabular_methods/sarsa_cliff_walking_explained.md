# Cliff Walking 的 SARSA 演算法 🏔️

## 什麼是 SARSA？

想像一條**非常長的走廊**，邊緣是一個**可怕的懸崖**。如果你從懸崖上掉下去，你就必須回到起點重新開始！你的目標是儘可能快地從走廊的一端走到另一端，且不掉下去。

**SARSA** 是一個透過練習來學習穿越這條走廊的機器人。它學習選擇一條**安全路徑**來避開懸崖——即使這條路稍微長一點——因為它知道自己在探索時可能會不小心滑到邊緣！

---

## 核心思想：從你的實際行動中學習

SARSA 代表：**S**tate（狀態）→ **A**ction（動作）→ **R**eward（獎勵）→ **S**tate（狀態）→ **A**ction（動作）

這是 SARSA 用來學習的五條資訊：

1. **S** —— 我現在在哪？（當前狀態）
2. **A** —— 我實際採取了什麼動作？
3. **R** —— 我得到了什麼獎勵？
4. **S** —— 我最終到了哪裡？（下一個狀態）
5. **A** —— 我**下一步實際上打算採取**什麼動作？

最後一個 "A" 是 SARSA 的特別之處！它使用它*下一步實際上會採取*的動作來進行更新（即使那是一個隨機的探索性動作），而不是完美的理想動作。

**現實生活中的例子：** 想想學習騎自行車。如果你知道自己有時會隨機搖晃（探索），你就會離停放的車輛稍微遠一點——因為你知道搖晃的自己可能會撞上去！SARSA 也是這樣：它學習一條安全路徑，因為它考慮到了自己可能犯的隨機錯誤。

```
   ← ← ← ← 懸崖 (CLIFF) ← ← ← ← ←
```

- **S** = 起點 Start (左下角)
- **G** = 終點 Goal (右下角)
- **C** = 懸崖 Cliff —— 踩到這裡 = -100 獎勵，重開！
- 其他每一步 = -1 獎勵

## 我們的程式碼發現了什麼

在對 SARSA 進行 500 個回合 (episodes) 的訓練後：

| 結果 | 數值 |
|--------|-------|
| 最終 50 回合平均獎勵 | **-21.6** |
| 最佳（冒險）路徑獎勵 | -13 |

SARSA 學習到的策略是**沿著網格的頂部**行進——這是一條安全的迂迴路線！它雖然多花幾步（-21 而不是 -13），但在訓練過程中幾乎從未掉下懸崖。

## 現實生活中的例子

- **護士給藥**：即使存在稍微快一點的方法，也會採取經過驗證的安全規程（安全路徑），因為微小的錯誤（探索）可能是危險的。
- **航空公司飛行員**：即使捷徑看起來更快，也會嚴格遵守檢查清單（安全路徑），以考慮到人為錯誤。
- **學習烹飪**：從經過充分測試的食譜（安全）開始，而不是嘗試冒險的捷徑。

## 需要記住的核心詞彙

- **同策 (On-policy)**：學習它實際正在使用的策略（包括其隨機錯誤）
- **SARSA 更新**：使用*實際*的下一步動作，而不是理論上的最佳動作
- **安全路徑**：避開危險的長路徑，考慮到了探索時可能犯的錯誤
- **TD (時序差分) 控制**：在每一步之後都更新數值（而不是等待整個回合結束）

核心思想：**SARSA 是誠實的——它從它實際所做的動作中學習，而不是從它希望自己所做的動作中學習。這讓它在危險面前變得謹慎和安全！**
