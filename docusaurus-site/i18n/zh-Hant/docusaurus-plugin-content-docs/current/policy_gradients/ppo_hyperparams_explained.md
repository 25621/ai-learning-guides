# PPO 超參數敏感性：什麼最重要？

## 為什麼超參數很重要

想像一下烤巧克力蛋糕。食譜要求：
- 2 個雞蛋
- 200 克麵粉
- 1 茶匙發粉
- 180°C 烘烤 35 分鐘

如果你用 10 個雞蛋，蛋糕會爆炸。如果你只用 0.1 茶匙發粉，蛋糕就發不起來。如果你在 300°C 下烘烤 10 分鐘，蛋糕外面焦了，裡面還是生的。

**PPO 中的超參數就像食材和烤箱設定。** 正確的組合效果極佳；錯誤的設定可能會導致完全無法學習。

這個腳本透過一次只改變一個超參數，在 3 個不同的隨機種子下運行每個設定，並比較結果，系統地測試了 3 個關鍵超參數。

---

## 三個實驗

### 實驗 1：裁剪 ε (Clip Epsilon)

```
ε = 0.05   (非常保守 —— 僅允許極小的策略變化)
ε = 0.2    (標準 —— 平衡了安全性和速度)
ε = 0.4    (激進 —— 允許較大的策略變化)
```

**ε 控制什麼？**

ε 是舊策略周圍「安全窗口」的大小：
```
機率比必須保持在 [1 - ε,  1 + ε] 之間
ε=0.05: 機率比在 [0.95, 1.05] 之間 ← 極小的變化
ε=0.2:  機率比在 [0.80, 1.20] 之間 ← 標準  
ε=0.4:  機率比在 [0.60, 1.40] 之間 ← 較大的變化
```

**現實生活中的例子**：把 ε 想像成「一次允許你轉動方向盤多遠」。
- ε=0.05：就像在冰上行駛 —— 只能做微小的調整。
- ε=0.2： 正常駕駛 —— 合理的轉彎。
- ε=0.4： 賽車手 —— 激進的轉向，有**甩尾/打滑 (spinning out)** 的風險（因為變化太劇烈而失去控制）。

**預期結果**：
- ε=0.05：學習緩慢但穩定（過於謹慎）。
- ε=0.2： 良好的平衡（**「金髮姑娘」原則 (Goldilocks value)** —— 恰到好處，源自童話故事，指不冷不熱，剛剛好）。
- ε=0.4： 學習可能很快，但可能會**超調和振盪 (overshoot and oscillate)**（超調 = 越過了最佳策略；振盪 = 在最佳策略周圍來回擺動而無法穩定）。

---

### 實驗 2：學習率 (Learning Rate)

```
lr = 1e-4  (慢但穩定)
lr = 3e-4  (標準)
lr = 1e-3  (快但冒險)
```

**學習率控制什麼？**

學習率就像爬山時的「步長」（每一步 = 對神經網路權重的一次更新，向提高獎勵的方向移動）：
- 太小：需要永遠的時間才能到達山頂（收斂緩慢）。
- 太大：你會越過山峰掉到另一邊（**發散 diverges** —— 訓練獎勵崩潰或劇烈波動，而不是穩定提高）。
- 恰到好處：穩步向頂峰邁進。

**現實生活中的例子**：給吉他調音。
- lr=1e-4：微調**弦鈕 (peg)** —— 耗時很長但很精確。
- lr=3e-4：正常調音 —— 轉幾圈就能找到正確的音高。
- lr=1e-3：猛力**拽 (yanks)** 弦鈕 —— 可能會**繃斷 (snap)** 琴弦（就像過大的更新會不可逆地破壞訓練一樣）！

**預期結果**：
- lr=1e-4：最終效果不錯，但非常緩慢。
- lr=3e-4：整體表現最佳。
- lr=1e-3：初期進展飛快，隨後出現不穩定。

---

### 實驗 3：更新輪數 (Update Epochs, K)

```
K = 3   (保守 —— 每個批次數據只跑幾遍)
K = 10  (標準)
K = 20  (激進 —— 每個批次數據跑很多遍)
```

**更新輪數控制什麼？**

在收集了一段**展開 (rollout)**（= 玩一段時間遊戲以收集新經驗）後，PPO 將這些經驗打包成一個**批次 (batch)**。然後，它對同一組數據運行 K **輪 (passes)**（每輪遍歷整個批次，每次更新一次網路）。
更多的輪數 = 從每個批次中壓榨出更多的學習成果，但有**對陳舊數據過擬合 (overfitting to stale data)** 的風險（= 記住了在舊策略下成立但在策略更新後不再有效的模式）。

**現實生活中的例子**：一個學生練習 20 道數學題。
- K=3： 每道題做 3 遍 → 仍在學習，不會死記硬背練習集。
- K=10：每道題做 10 遍 → 熟練掌握了這些特定的題目。
- K=20：每道題做 20 遍 → **背下了答案，但並沒有真正理解數學**（= 模型完美擬合了特定批次，但失去了泛化能力）！

> ⚠️ **「但是 K=20 的結果看起來還行 —— 為什麼我要在意？」**
> PPO 的裁剪技巧限制了每輪策略的變化程度，因此 K=20 不會導致突然的崩潰。
> 然而，代理人仍然在悄悄地過度適應那些不再反映當前策略實際體驗的數據。
> 這會**減慢長期的學習速度**：每次展開教給代理人的東西比它應得的要少，因為後面的輪次在回收利用越來越陳舊的資訊。
> 這種損害是逐漸的，而不是戲劇性的 —— 這正是為什麼在短期實驗中很容易被忽視的原因。

裁剪防止了災難性的過擬合，但過多的輪數仍然會減慢整體學習速度。

**預期結果**：
- K=3： 效率較低（每批次浪費了一些學習潛力）。
- K=10：良好的平衡。
- K=20：策略有對**陳舊數據過度自信**的風險（= 網路的更新是由不再符合當前策略遇到的體驗驅動的，從而悄悄侵蝕了樣本效率）。

## 如何閱讀結果

圖表顯示了三個子圖，每個子圖改變一個超參數：

```
左圖：   裁剪 ε —— 哪個 ε 學習最快？
中間圖： 學習率 —— 哪個 lr 最穩定？
右圖：   更新輪數 —— 哪個 K 找到了最好的策略？
```

每條線都是 **3 個隨機種子的平均獎勵**（以減少偶然性）。

**尋找什麼**：
1. **學習速度**：哪條線最快達到高獎勵？
2. **最終性能**：哪條線達到了最高的最終獎勵？
3. **穩定性**：哪條線的振盪最少？

一個好的超參數可以平衡這三點！

## 方法論：科學實驗

本實驗採用**消融研究 (ablation study)** 設計（= 一次只改變或移除一個組件以衡量其單獨影響的方法）：
1. 選擇預設值：ε=0.2, lr=3e-4, K=10
2. 一次只改變一個參數。
3. 保持其他所有參數不變。
4. 比較結果。

這告訴了我們每個參數孤立存在時的影響。

**現實生活中的例子**：測試一種新肥料是否對植物有幫助。
- 改變肥料，保持其他一切相同（相同的土壤、水、陽光）。
- 如果植物長得更好 → 肥料起作用了！

## 實踐中的常見發現

| 超參數 | 太小 | 黃金甜點位 | 太大 |
|----------------|-----------|------------|-----------|
| **ε (clip)** | 收斂緩慢 | ε ≈ 0.2 | 不穩定 |
| **lr (學習率)** | 太慢 | 2.5e-4 到 3e-4 | 發散 |
| **K (輪數)** | **浪費數據** (未提取完整訊號就丟棄) | K = 4-10 | 對陳舊數據過擬合 |
| **n_steps** | 雜訊太大 | 128-2048 | **記憶體溢位 (OOM)** |
| **batch_size** | 雜訊太大 | 32-256 | **記憶體溢位 (OOM)** |

這些「甜點位」可能會根據環境的不同而發生變化！

## 核心洞察：PPO 相對穩健

與早期的演算法（如沒有目標網路的 DQN）相比，PPO 對超參數選擇相對穩健。裁剪機制提供了一個自然的「安全網」。

**現實生活中的例子**：帶有 **ABS**（防鎖死煞車系統 —— 一種安全功能，防止車輪在猛烈煞車時鎖死，保持駕駛員的控制）的汽車與沒有 ABS 的汽車相比：
- 沒有 ABS (DQN)：一個錯誤的動作（差的超參數）就會讓你甩尾失控。
- 有 ABS (PPO)：汽車會自我糾正 —— 合理的超參數都能正常工作。

這種魯棒性是 PPO 在實踐中成為最受歡迎的強化學習演算法的主要原因！

## 關鍵要點

| 概念 | 通俗解釋 |
|---------|---------------|
| **消融研究** | 一次只改變一件事，看看它的效果 |
| **裁剪 ε (Clip epsilon)** | 安全邊界 —— 0.2 通常是最好的 |
| **學習率 (Learning rate)** | **步長** —— 每次更新後網路權重調整的幅度。**2.5e-4 到 3e-4** 是 0.00025 到 0.0003 的科學計數法 —— 它們是無因次的乘數，不是時間值 |
| **更新輪數 K** | 每個批次數據重複利用多少次 —— 標準是 4-10 次 |
| **隨機種子 (Random Seeds)** | 每個實驗都使用不同的隨機種子重複運行。使用多個種子可以揭示結果是連貫一致的，還是僅僅靠運氣 |

```
REINFORCE              A2C                    PPO
     │                  │                      │
完整回合更新           N 步更新               N 步 + 裁剪
簡單但雜訊大           更快但不穩定           穩定且高效
適合簡單環境           中等難度環境           困難環境
                      (工業標準)
```
