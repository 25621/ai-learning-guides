# REINFORCE：教會機器人做出更好的選擇

## 我們想要做什麼？

想像你有一個正在玩影片遊戲的機器人。每一秒，機器人都必須做出選擇：
**「我是應該按下按鈕，還是不按？」**

我們不希望機器人像 Q-learning 那樣在表中記住每種情況，而是希望它學習一套**策略 (Policy)**——一套直接說明「在這種情況下，執行這個動作」的規則。

這套規則被稱為**策略** (π, pi)。在強化學習中，π 的意思是「選擇動作的規則」。

---

## 舊方法 vs 新方法 {#the-old-way-vs-the-new-way}

**舊方法 (Q-learning / DQN)：** 學習每個動作有多「好」（Q 值），然後選擇最好的。
> 「向左推得 7 分，向右推得 5 分 → 向左推！」

**新方法 (策略梯度 - Policy Gradient)：** 直接學習應該「選擇」哪個動作。
> 「當桿子向右傾斜時，以 80% 的機率向右推，以 20% 的機率向左推。」
*（**梯度 Gradient** 這個詞指的是我們為了讓這些機率慢慢向正確方向調整而採取的數學「步驟」。）*

**現實生活中的例子：** 學習騎自行車。
- 舊方法：計算「向左傾斜 5 度」與「向左傾斜 7 度」的*精確分數*。
- 新方法：不斷練習，直到你的**身體**學會——自然而然地做出感覺正確的動作！

---

## REINFORCE 是如何工作的？

REINFORCE 觀察機器人從頭到尾玩完一局遊戲（一個**回合 episode**），然後問：「哪些動作導致了高分？讓我們多做些那樣的動作！」

### 分步解析

**1. 運行一個回合**

機器人做出選擇並收集經驗：
```
第 1 步：狀態 = [桿子向右傾斜] → 動作 = 向右推 → 獎勵 = +1
第 2 步：狀態 = [桿子接近平衡] → 動作 = 向右推 → 獎勵 = +1
第 3 步：狀態 = [桿子向左傾斜] → 動作 = 向左推 → 獎勵 = +1
...
第 47 步：狀態 = [桿子倒了！] → 回合結束
```

**2. 計算回報**

對於每一步，計算 G_t —— **從該點開始的總獎勵**：
```
第 47 步的 G = 1
第 46 步的 G = 1 + 0.99 × 1 = 1.99
第 45 步的 G = 1 + 0.99 × 1.99 = 2.97
...
第 1 步的 G  = 47（大約值 —— 因為它是從開始計算的，所以回報更高）
```

γ = 0.99 的**折扣因子 (discount factor)** 意味著遙遠未來的獎勵價值稍低。

**現實生活中的例子：** 開學第一天得到一顆金星獎章，比知道在第 100 天*可能*得到一顆更令人興奮。未來的獎勵被輕微地「打折」了。

**3. 更新策略**

對於採取的每個動作：
> 如果 G_t 很高（該動作導致了極好的結果）：**以後多做這個動作！**
> 如果 G_t 很低（該動作導致了糟糕的結果）：**以後少做這個動作！**

數學公式：`loss = -log_prob(action) × G_t`

獲取梯度並更新策略就像是在告訴機器人：
*「你在第 20 步採取的那個動作？下次你應該多增加 5% 的頻率去執行它！」*

## 什麼是策略網路？

我們不使用表格，而是使用**神經網路**來表示策略。

```
觀測值 (Observation)      策略網路 (Policy Network)       動作機率
[小車位置]  →   [128 個神經元]  →  →  [向左推: 30%]
[小車速度]  →   [128 個神經元]        [向右推: 70%]
[車桿角度]  →
[車桿角速度] →
```

網路輸出每個動作的**機率**。然後我們進行採樣：
> 擲骰子 → 1-30：向左推，31-100：向右推

**現實生活中的例子：** 天氣預報說「降水機率 70%」。你並不確定一定會下雨——你是基於機率做出決定的。機器人也是如此！

## 正規化：為什麼要減去均值

在使用 G_t 進行更新之前，我們會進行正規化：
```
G_normalized = (G - mean(G)) / std(G)
```

**為什麼？** 想像所有的獎勵都是正數（在 CartPole 中確實如此——每一步都是 +1）。如果沒有正規化，每一個動作看起來都挺「好」，更新訊號就會變得混亂。

正規化後，一些回報變為正數（高於平均水準 → 增加機率），一些變為負數（低於平均水準 → 減少機率）。訊號變得清晰得多！

**現實生活中的例子：** 老師按比例評分。如果平均分是 70 分，你得了 85 分，那很棒！但如果平均分是 90 分，你得了 85 分，那就是低於平均水準。單憑原始分數並不能說明全部情況。

## 問題：高變異數

REINFORCE 有一個巨大的弱點：**變異數 (variance)**。回報 G_t 的雜訊非常大。

**現實生活中的例子：** 想像僅透過每家餐廳的一道菜來評判廚師。有時廚師狀態不佳，有時食材不新鮮。僅僅一道菜不足以可靠地判斷這家餐廳是否優秀！

REINFORCE 會等待整個回合結束才進行更新。一個回合可能運氣極好，另一個回合可能運氣極差。梯度會四處跳動。

這就是為什麼訓練曲線（圖表中）看起來坎坷不平：
- 有些運行得分 500（太棒了！）
- 有些運行跌至 50（太慘了！）

儘管有雜訊，REINFORCE 最終還是能學會——但需要耐心。

## 實驗結果

```
第 100 回合 | 最近 100 回合平均獎勵:  43.1
第 200 回合 | 最近 100 回合平均獎勵: 193.9
第 500 回合 | 最近 100 回合平均獎勵: 408.4
第 1000 回合 | 最近 100 回合平均獎勵: 500.0  ← 任務完成！
```

機器人學會了平衡車桿達到最高 500 步——成功解決！

儘管存在變異數問題，REINFORCE 在 CartPole 上依然有效，因為：
1. 回合較短（所以我們在每次訓練中能獲得很多回合）
2. 最佳策略相對簡單（主要是朝車桿傾斜的方向推）

## 核心總結

| 概念 | 通俗解釋 |
|---------|---------------|
| **策略 (Policy)** | 機器人選擇動作的「秘訣」 |
| **回合 (Episode)** | 從開始到結束的一局完整遊戲 |
| **回報 G_t** | 從當前時刻開始的總未來獎勵 |
| **折扣 γ** | 未來的獎勵比眼前的獎勵價值稍低 |
| **正規化 (Normalize)** | 減去平均值，使訊號更清晰 |
| **變異數 (Variance)** | 梯度估計值的波動程度 |

---

## 下一步是什麼？

REINFORCE 的最大弱點是**變異數**。在下一個腳本 (`reinforce_baseline.py`) 中，我們添加了**基準線 (baseline)**，它在不改變演算法平均學習內容的前提下，極大地減少了這種雜訊。

核心思想：與其問「這個動作好嗎？」，不如問「這個動作是否**優於預期**？」這個微小的改變能讓學習過程變得穩定得多。
