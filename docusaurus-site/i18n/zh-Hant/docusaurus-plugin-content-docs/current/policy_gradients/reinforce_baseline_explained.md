# 帶基準線的 REINFORCE：穿透雜訊

## 普通 REINFORCE 的問題

想像你是一名學生，正在試圖判斷你在考試中的答案是否正確。

**糟糕的回饋：** 「你得了 7 分！」

7 分算好嗎？如果總分是 10 分，那確實不錯！如果其他人都得了 9 分，那就不行！如果沒有背景資訊，你無法判斷是否應該改變你的答題風格。

這正是 REINFORCE 的問題所在：它使用**原始回報** (G_t) 來評估動作。200 分的總回報可能意味著表現極佳，也可能意味著表現糟糕，這完全取決於具體情況。

---

## 引入基準線 (Baseline)

**基準線** b(s) 是一個參考點：「在這種情況下，我**預期**能獲得多少獎勵？」

我們不再問「這個動作好嗎？」，而是問：

> **「這個動作是否比我通常預期的要好？」**

```
舊訊號：更新量 ∝ G_t
新訊號：更新量 ∝ (G_t - b(s_t))
```

**現實生活中的例子：** 你在數學考試中得了 85 分。
- 如果班級平均分是 60 分 → 你的成績**比平均分高出 25 分** → 太棒了！
- 如果班級平均分是 90 分 → 你的成績**比平均分低了 5 分** → 還需要努力！

當你的表現優於預期時，**優勢 (Advantage)** (G_t - b(s)) 為正；當表現差於預期時，優勢為負。這是一個清晰得多的學習訊號！

---

## 什麼是基準線？

自然的基準線是**價值函數 V(s)**：

> V(s) = 「如果我處於狀態 s 並按照當前策略進行遊戲，預期的總獎勵是多少」

我們透過一個單獨的**價值網路**（也稱為基準網路或評論員/Critic）來學習它：

```
狀態  →  [128 個神經元]  →  [128 個神經元]  →  V(s)   (單個數字)
```

對於代理人訪問的每個狀態，V(s) 都會預測預期的回報。如果實際回報 G_t 高於 V(s)，則說明該動作優於預期！

---

## 兩個網路共同學習

```
回合發生
     ↓
計算實際回報 G_t
     ↓
         ┌─────────────────────────────┐
         │ 優勢 (Advantage) = G_t - V(s_t) │
         │  +: 動作表現更好             │
         │  -: 動作表現較差             │
         └─────────────────────────────┘
              ↓                  ↓
       更新策略網路          更新價值網路
    (使好的動作出現幾率    (使預測在下次
     更高/更低)             更加準確)
```

**現實生活中的例子：** 兩個朋友一起去餐廳。

- 朋友 A（價值網路）：「我預測這道菜能得 7/10 分」
- 朋友 B（策略網路）：你嘗試了這道菜並給出了 9/10 分的評價
- 優勢 = 9 - 7 = +2 → 「這比預期的要好！下次再點！」

下次光顧時，朋友 A 會將預測值更新為接近 9/10。
朋友 B 下次更有可能點這道菜。

---

## 為什麼這能降低變異數？

**數學直覺：**

不帶基準線：`梯度 ∝ ∇log π(a|s) × G_t`

G_t 的值在不同回合之間波動很大：
```
回合 1: G = [45, 44, 43, ..., 1]   (中等局)
回合 2: G = [500, 499, ..., 1]      (超神局！)
回合 3: G = [12, 11, ..., 1]        (慘敗局)
```

由於 G_t 數值大且充滿雜訊，梯度估計會劇烈跳動。

帶基準線：`梯度 ∝ ∇log π(a|s) × (G_t - V(s_t))`

優勢 (G_t - V(s_t)) 的數值要小得多，且集中在零附近：
```
回合 1: 優勢 ≈ [-2, +1, -3, ..., 0]   (數值小，居中)
回合 2: 優勢 ≈ [+10, +8, ..., +3]      (這局確實打得好)
回合 3: 優勢 ≈ [-5, -6, ..., -2]       (這局確實打得爛)
```

**現實生活中的例子：** 測量你的跑步速度。
- 不帶基準線：「我跑了 8 公里/小時」（沒有背景資訊，毫無意義）
- 帶基準線：「我比我的平均速度快了 2 公里/小時」（顯然很棒！）

優勢始終是一個比較值——它自然更小且更穩定。

---

## 至關重要：無偏性！

基準線不會改變演算法學習的**內容**——它只改變學習的**速度**和**穩定性**。

**為什麼？** 因為從期望值來看，預期優勢始終為 0：

> E[G_t - V(s_t)] = E[G_t] - V(s_t) = V(s_t) - V(s_t) = 0

任何不依賴於具體動作的 b(s) 都可以作為有效的基準線！

**現實生活中的例子：** 按比例評分（拉開檔次）不會改變誰表現最好——它只是讓分數更容易解釋。排名保持不變，改變的只是衡量尺度。

---

## 實驗結果

```
不帶基準線  — 最終 100 回合平均分: 500.0, 梯度變異數: 599.3
帶基準線 — 最終 100 回合平均分: 491.4, 梯度變異數: 578.8
```

兩種方法在 CartPole 環境中都能達到近乎完美的表現，但請注意：
1. **梯度變異數**是可以測量的（右側圖表顯示了訓練過程中的變異數）
2. 使用基準線後，代理人能夠**更可靠地**達到高性能——在訓練過程中，因雜訊導致的獎勵暴跌情況更少發生。

在更複雜的環境（如 LunarLander, MuJoCo）中，變異數縮減的效果會更加顯著。

---

## 核心公式

```
基準線值更新：   V(s) ← V(s) + α(G_t - V(s))   [最小化均方誤差 MSE]
策略梯度更新：   θ ← θ + α ∇log π(a_t|s_t) · (G_t - V(s_t))
優勢：           A_t = G_t - V(s_t)
```

---

## 核心總結

| 概念 | 通俗解釋 |
|---------|---------------|
| **基準線 b(s)** | 狀態 s 的預期獎勵——我們的參考點 |
| **優勢 A_t** | 「這個動作是否比預期的要好？」 |
| **價值網路** | 一個學習預測預期回報的神經網路 |
| **變異數縮減** | 梯度估計中的雜訊減少 → 訓練更穩定 |
| **無偏 (Unbiased)** | 基準線平均而言不會改變目標策略；它只讓學習訊號雜訊更小、更穩定 |

---

## 下一步是什麼？

基準線實際上是某種更強大方法的基礎：**演員-評論家 (Actor-Critic)** 方法。

Actor-Critic 不是在回合結束時才計算 V(s)，而是在每一步都使用**時序差分 (Temporal Difference)** 學習來更新 V(s)。這使得更新速度更快，並允許代理人從不完整的環節中學習！

請參閱 `a2c_lunarlander.py` 獲取完整的 Actor-Critic 實現。
