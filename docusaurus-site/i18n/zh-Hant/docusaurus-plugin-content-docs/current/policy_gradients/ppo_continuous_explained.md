# 連續控制的 PPO：讓雙足機器人行走 (BipedalWalker)

## 離散動作 vs. 連續動作

到目前為止，我們解決的每個環境都使用的是**離散 (discrete)** 動作：
- CartPole：向左推或向右推（2 個選擇）
- LunarLander：不發動/左/主/右引擎（4 個選擇）

但現實世界的機器人需要**連續 (continuous)** 動作：
- 人形機器人：「給每個關節施加多大的力」（從 -1 到 +1 的任何值）
- 汽車：「方向盤具體轉動多少度」（從 -30° 到 +30° 的任何角度）
- 機械臂：「在這個方向上精確施加 2.3 牛頓的力」

**現實生活中的例子**：在鍵盤上打字 = 離散動作（按 A, B, C...）。用鉛筆寫字 = 連續動作（向右移動手 2.3 公分，施加 40 克壓力...）。

## 連續動作的高斯策略 (Gaussian Policy)

對於連續動作，我們不再使用分類分佈（Categorical distribution，從 N 個類別中選擇），而是使用**正態分佈（常態分佈）**：

```
動作 ~ 常態分佈(μ, σ)
```

其中：
- **μ (mu, 均值)**：分佈的中心 —— 網路「瞄準」的動作值
- **σ (sigma, 標準差)**：離散程度 —— 要添加多少隨機性/探索

```
          機率
             │
        0.4 ─┤      ██████
             │    ████████████
        0.2 ─┤  ██████████████████
             │████████████████████████
             └──────────────────────── 動作值
           -1  -0.5   0   0.5   1
                      ↑
                    均值 μ
```

**現實生活中的例子**：一名熟練的弓箭手瞄準靶心 (μ)。他們的箭不會全部落在完全相同的位置 —— 會有一定的分佈 (σ)。隨著他們的練習，他們在保持瞄準靶心的同時，準確度會提高（σ 減小）。

## 我們的高斯演員-評論家 (Gaussian Actor-Critic) 網路

```
狀態 (24 個數值) → [256 個神經元] → [256 個神經元] →
    ├── 演員 (Actor): 4 個均值 (μ₁, μ₂, μ₃, μ₄)
    │                + 4 個 log_std 參數 (在所有狀態間共享!)
    └── 評論家 (Critic): 1 個價值 (V(s))
```

`log_std`（**標準差**的對數 —— 衡量離散程度或不確定性）是一個**可學習的參數**，它不依賴於狀態。這既保持了簡單性，又允許探索在訓練過程中發生變化。

**為什麼用 log_std 而不是 std？** 標準差必須是正數。使用 `log_std` 允許網路輸出任何實數（正數或負數），然後我們應用 `exp(log_std)`（指數函數，對數的逆運算）來恢復一個保證為正數的標準差。這防止了標準差變成負數或零。

## 計算連續動作的對數機率

對於離散動作：`log_prob = log(P(action=LEFT))`

對於連續動作，**常態分佈**描述了均值周圍的一條平滑的鐘形曲線。在連續數學中，單個精確值的機率為零，因此我們使用該值處的曲線高度，稱為 **pdf**（機率密度函數）：
```
log_prob = Σᵢ log[Normal(μᵢ, σᵢ).pdf(aᵢ)]
```

`log` 表示自然對數。它將極小的密度值轉化為穩定的數值，更容易讓神經網路進行優化。我們對所有動作維度（對於 BipedalWalker 是 4 個）求和，因為完整的動作是一個由 4 個數字組成的向量。

**現實生活中的例子**：明天溫度恰好是 5.732...°C 的機率是多少？對於連續的天氣，你會查看常態分佈曲線，看那個精確點的高度。更可能的溫度（靠近均值）具有更高的機率。

## BipedalWalker：一個行走的挑戰

BipedalWalker-v3 是一個必須學會行走且不摔倒的 2D 機器人：

```
          O (頭部)
         /│\
        / │ \
       /  │  \
      L   │   R   ← 兩條腿，每條腿都有一個膝關節
     / \  │  / \
    ●   ● │ ●   ●  ← 4 個馬達 (每條腿有臀部/膝蓋)
```

**狀態空間 (24 個數值)：**
- 軀幹：角度、角速度、水平速度、垂直速度 (4 個)
- 關節：4 個馬達（2 個臀部，2 個膝蓋）各自的角度和速度，加上 2 個地面接觸感測器（每條腿一個）(10 個)
- 10 個雷射雷達 (LIDAR) 測距感測器（看到前方地面的距離讀數）(10 個)

**動作空間 (4 個連續值，均在 [-1, 1] 範圍內)：**
這些動作值控制正好 4 個關節的**轉矩**（馬達施加的旋轉力）（沒有動作直接作用於軀幹）：
- 腿 1 臀部轉矩，腿 1 膝蓋轉矩，腿 2 臀部轉矩，腿 2 膝蓋轉矩

**獎勵：**
- 到達目標（右側）獲得 +300
- 摔倒（身體接觸地面）獲得 -100
- 每向前移動一步獲得少量獎勵
- 每次使用馬達有少量懲罰（鼓勵效率）

**解決標準**：100 個回合內的平均獎勵 > 300

## 與離散 PPO 的主要區別

除了以下內容，其他都一樣：

| | 離散 PPO | 連續 PPO |
|---|---|---|
| **策略** | 分類分佈 Categorical(logits) | 常態分佈 Normal(μ, σ) |
| **採樣** | 動作 = 從 {0,1,...,N} 中採樣 | 動作 = μ + σ × 雜訊 |
| **log_prob** | log P(action=k) | Σ log Normal(μᵢ, σᵢ).pdf(aᵢ) |
| **鉗制 (Clamp)** | 不需要 | 將動作鉗制在 [-1, 1] |

**Logits** 是離散動作的原始、未正規化的分數。分類策略使用 **softmax** 將它們轉換為機率 —— softmax 函數接收任何一組數字並將其壓縮成有效的機率分佈（所有值均為正，總和為 1）。例如，logits [2.0, 1.0, 0.5] 變成機率 [0.59, 0.24, 0.17]。連續 PPO 動作本身**不**使用 softmax，因為動作不是從固定菜單中選擇的。相反，策略輸出常態分佈的均值和標準差，然後從中採樣實值轉矩。

**鉗制 (Clamp)** 意味著強制一個值在有效範圍內。程式碼使用 `action.clamp(-1, 1)`，這樣環境就不會收到超出其允許範圍的馬達指令。

**裁剪 (Clip)** 在 PPO 中有不同的含義：PPO 裁剪損失函數內部的機率比，如 [PPO 裁剪部分](./ppo_scratch_explained.md#the-clipping-trick) 所述。動作鉗制保護環境介面；PPO 裁剪保護策略更新。

## 從零開始行走：代理人學到了什麼

**訓練早期（負獎勵）**：機器人隨機揮動雙腿，立即摔倒。每個回合都在幾秒鐘內以碰撞結束。

**訓練中期**：機器人發現交替移動雙腿可以產生向前的進展。它開始邁出細小、別扭的步伐 —— 獎勵變得不那麼負了。

**訓練後期**：出現了一種平滑、高效的行走**步態 (gait)**。步態是一種重複的運動模式，如左右交替邁步。機器人透過利用雷射雷達感測器動態調整，即時適應不平整的地形。

**現實生活中的例子**：嬰兒學走路：
1. 立即摔倒（負獎勵）
2. 走一步，摔倒（負值稍小）
3. 走幾步（少量正獎勵）
4. 穿過房間（大量正獎勵！）

## 為什麼 BipedalWalker 需要 PPO (而不是 REINFORCE)

- **BipedalWalker 回合**可能長達 1600 步（比 CartPole 長得多！）
- **獎勵是稀疏的** —— 每一步向前邁進的獎勵微乎其微
- **REINFORCE 需要**數千個完整的回合才能獲得有用的訊號

PPO 配合 [GAE (廣義優勢估計)](./ppo_scratch_explained.md#gae-smarter-advantage-estimates) 進行 n 步更新，讓機器人可以從不完整的回合中學習：
> 「雖然我在 50 步後摔倒了，但這些步驟展示了一些向前的進展。
> 讓我使用這 50 步的回報估計，而不是等待回合結束。」

---

## 結果

經過 500 次更新（約 100 萬個環境步）：
- 機器人取得了明顯進展，從隨機揮動轉向了一些向前的移動
- 學習曲線持續改善
- 完全收斂到獎勵 > 300 需要更多的訓練（500-1000 萬步）

學習曲線展示了連續控制典型的「S 曲線」：
1. 初期進展緩慢（學習穩定性）
2. 快速提升（步態發現）
3. 逐漸精煉（步態優化）

## 關鍵要點

| 概念 | 通俗解釋 |
|---------|---------------|
| **高斯策略** | 不再從菜單中選擇，而是在一定數值範圍內投擲飛鏢 |
| **μ (均值)** | 策略「瞄準」的地方 |
| **σ (標準差)** | 策略使用的隨機性/探索程度 |
| **log_std 作為可學習參數** | 一個由基於梯度的優化（對獎勵進行梯度*上升*，或等效地對 PPO 損失進行梯度*下降*）更新的全局探索率 —— 就像任何其他網路權重一樣 |
| **連續控制** | 控制實值輸出（轉矩、力、角度） |

## 下一步是什麼？

PPO 有許多**超參數 (hyperparameters)** —— 在訓練開始前你選擇的設定（與網路權重等透過自動學習獲得的*參數*不同）。例子包括 `clip_eps`、學習率、輪數 (epochs) 和批次大小 (batch size)。

PPO 對這些選擇有多敏感？`ppo_hyperparams.py` 透過系統地改變每個超參數進行實驗，展示了它們對學習速度和穩定性的影響。
