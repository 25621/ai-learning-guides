# PPO：安全且穩定的策略更新

## A2C 存在的問題

想像你正在練習在手指上平衡一根掃帚。經過幾周的練習，你可以讓它保持平衡 30 秒！

現在你的教練給了你一個建議：「手腕稍微再向左傾斜一點。」

**好的建議 → 謹慎的改變 → 依然能保持平衡 30 秒 ✓**

但如果教練反應過度，大喊：「立即向左大幅度傾斜！」
你矯枉過正 → 掃帚倒了 → 你失去了幾周練習累積的成果。

這就是 A2C 的問題：**大幅度的梯度更新可能會摧毀一個好的策略**。

**PPO (近端策略優化 Proximal Policy Optimization)** 就是一套防止這種情況發生安全系統。

---

## 核心理念：靠近已經行之有效的方法

PPO 的關鍵約束：

> **「在單次更新中，不要過度改變策略。」**

在更新之前，我們有一個「舊」策略 π_old。
在更新之後，我們有一個「新」策略 π_new。

PPO 透過**機率比 (probability ratio)** 來衡量策略改變了多少：

```
r(θ) = π_new(a|s) / π_old(a|s)
```

- r = 1.0：策略未改變
- r = 1.5：新策略採取該動作的可能性增加了 50%
- r = 0.5：新策略採取該動作的可能性減少了 50%

**現實生活中的例子**：你是一名正在調整食譜的廚師。
- r = 1.0：鹽的用量和以前一樣
- r = 2.0：鹽量翻倍 —— 太極端了！
- r = 0.9：少放 10% 的鹽 —— 微小且安全的改變

## 裁剪技巧 (The Clipping Trick) {#the-clipping-trick}

PPO 將機率比限制在 [1-ε, 1+ε] 範圍內（通常 ε = 0.2）：

```
L_CLIP = E[min(r(θ) · A,  clip(r(θ), 1-ε, 1+ε) · A)]
```

讓我們分解一下：

**情況 1：動作是好的 (A > 0)**

我們希望更多地執行這個動作 (r > 1)。但我們限制了增加的幅度：
```
如果 r > 1.2：裁剪到 1.2，不再有進一步推動的動力
```
這防止了我們在一個方向上偏移得太遠。

**情況 2：動作是差的 (A < 0)**

我們希望減少執行這個動作 (r < 1)。同樣地，我們也設置了限制：
```
如果 r < 0.8：裁剪到 0.8，不再有進一步懲罰的動力
```

**直觀圖示：**
```
ε = 0.2，因此安全的機率比窗口是 0.8 到 1.2。

好的動作 (A > 0)：增加動作機率，但在 1.2 之後停止獎勵
機率比 r:       0.6      0.8      1.0      1.2      1.4
激勵訊號:        ↑        ↑        ↑        ↑        -
含義:         太低      可以      舊的     最大     被裁剪

差的動作 (A < 0)：減少動作機率，但在 0.8 以下停止獎勵
機率比 r:       0.6      0.8      1.0      1.2      1.4
激勵訊號:        -        ↓        ↓        ↓        ↓
含義:         被裁剪     最大     舊的     可以      太高
```

`-` 標記了平坦的裁剪區域。在該區域內，即使讓機率比變得更加極端也不會改善目標函數，因此 PPO 沒有額外的動力去進一步推進。

**現實生活中的例子**：汽車的限速器。你可以加速，但一旦達到 120 km/h，限速器就會起作用，不讓你開得更快。它在不阻止你移動的情況下保證了你的安全。

## 為什麼這能防止災難性的更新

**災難性更新 (catastrophic update)** 是指一次巨大的策略改變完全摧毀了代理人已經學到的一切 —— 幾個小時的訓練成果在一次梯度更新中化為烏有。

如果沒有裁剪：一個巨大的梯度步長可能會劇烈改變策略。
有了裁剪：在 [1-ε, 1+ε] 範圍之外梯度為零，因此策略每步只能移動一點點。

**現實生活中的例子**：一名優秀的醫生動手術時切口小而精確 —— 而不是大面積切割。PPO 就是強化學習優化器中那名「細心的醫生」。

## GAE：更明智的優勢估計 {#gae-smarter-advantage-estimates}

PPO 使用 **廣義優勢估計 (GAE, Generalized Advantage Estimation)** 來計算優勢函數：

```
δ_t = r_t + γ · V(s_{t+1}) - V(s_t)          (TD 誤差)
A_t = δ_t + γλ · δ_{t+1} + (γλ)² · δ_{t+2} + ...
```

GAE 有一個參數 λ (lambda)：
- λ = 0：僅使用一步 TD 誤差（低變異數，高偏差）
- λ = 1：使用完整的蒙地卡羅回報（高變異數，低偏差）
- λ = 0.95：兩者的良好平衡！

**現實生活中的例子**：規劃公路旅行。
- λ=0：只看接下來的 5 英里（安全，但可能會錯過後面的捷徑）。
- λ=1：考慮整個 500 英里的旅程（資訊更多，但非常不確定）。
- λ=0.95：看得遠，但對附近的道路賦予更高的權重 ← 最佳平衡！

## 多輪迭代：高效複用數據

在收集了一批經驗（展開 rollout）之後，REINFORCE 在一次更新後就會將其丟棄。

PPO 會將每批數據重複利用 **K 輪 (epochs)**（通常是對同一組數據進行 4-10 次遍歷）：

```
收集 512 步 × 4 個環境 = 2048 個轉換 (transitions)
第 1 輪：32 個小批次 (minibatches) × 分別更新
第 2 輪：打亂順序，另外 32 個小批次 × 分別更新
第 3 輪：...
第 4 輪：...
```

**什麼是「小批次 (minibatch)」？** 一次性使用所有 2048 個轉換進行更新速度慢且耗記憶體；一次只更新一個轉換則雜訊太大。**小批次**是介於兩者之間的一小塊數據 —— 在這裡，2048 ÷ 32 = **每個小批次 64 個轉換**。我們為每個小批次計算一次梯度，因此每一輪執行 32 次微小且穩定的更新，而不是 1 次巨大的更新。（這與深度學習中到處使用的「小批次」概念相同 —— 參見 [小批次梯度下降](https://zh.wikipedia.org/wiki/随机梯度下降#小批次梯度下降)。）

裁剪功能確保了這些多次遍歷不會導致過調 —— 如果沒有裁剪，多輪迭代會因為推動策略過遠而摧毀它！

**現實生活中的例子**：一個學生有 30 道練習題。
- REINFORCE：每道題做一遍，學到一點東西，然後扔掉。
- PPO：每道題做 4 遍（每次從不同角度思考），並限制你的改變，以免你記錯模式。

## 完整的 PPO 損失函數

```
L = L_CLIP - c₁ · L_entropy + c₂ · L_critic

L_CLIP    = 裁剪後的策略梯度
L_entropy = 熵獎勵（鼓勵探索）
L_critic  = V(s) 與回報之間的均方誤差 (MSE)
```

典型係數：c₁ = 0.01（熵），c₂ = 0.5（評論家）。

**有兩個術語值得展開解釋：**

- **策略梯度 (Policy gradient)** —— 損失函數的「演員」部分。它使用梯度訊號將策略推向具有更高優勢的動作，並偏離具有較低優勢的動作。這與 REINFORCE 中引入的核心思想相同 —— 參見 [REINFORCE 詳解](./reinforce_cartpole_explained.md#the-old-way-vs-the-new-way) 以獲取直觀理解。PPO 只是在它周圍添加了裁剪外殼。
- **MSE (均方誤差)** —— 損失函數的「評論家」部分。評論家 V(s) 預測狀態的預期回報；我們將它的預測與實際回報進行比較，並對差值求平方：`MSE = mean((V(s) - return)²)`。求平方對大錯誤的懲罰比小錯誤更重，並為訓練提供平滑、可導的訊號。（標準的回歸損失 —— 參見 [均方誤差](https://zh.wikipedia.org/wiki/均方誤差)。）

## 結果

```
更新 200 次 | 平均獎勵: ~120
更新 400 次 | 平均獎勵: ~280
更新 800 次 | 平均獎勵: ~280-300
```

PPO 在 CartPole 環境下表現出穩步提升，但往往在 280-300 左右進入**平台期 (plateau)**。（平台期意味著學習曲線變平 —— 即使訓練繼續，獎勵也不再提高。策略已經找到了一個局部良好的策略，但沒有取得進一步進展。）這實際上是預料之中的 —— PPO 是為更困難、回合更長的環境設計的。

一個有趣的觀察是：**REINFORCE 解決 CartPole 的速度更快！**（平均 500 對比 平均 300）

為什麼？CartPole 的回合很短（≤500 步），所以 REINFORCE 的精確回報非常準確。PPO 的自舉 (bootstrapped) 估計增加了不必要的複雜性。PPO 真正發光發熱的地方是在那些無法等待完整回合結束的環境中（比如 BipedalWalker）。

**什麼是 "BipedalWalker"？** BipedalWalker（具體指 [Gymnasium](https://gymnasium.farama.org/environments/box2d/bipedal_walker/) 中的 `BipedalWalker-v3`）是一個經典的強化學習基準環境：一個兩足機器人必須學會跨越不平整的地形向前行走而不摔倒。與 CartPole 的兩個離散動作（左/右）不同，BipedalWalker 具有**連續**動作 —— 四個轉矩值，每條腿的關節各一個，每個值都是 [-1, 1] 之間的實數。回合可能會持續數千步，這正是 PPO 的數據效率和穩定性大顯身手的地方。

## 關鍵公式

```
機率比:      r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)
裁剪損失:    L_CLIP = E[min(r_t A_t, clip(r_t, 1-ε, 1+ε) · A_t)]
GAE:        A_t = Σ_{l=0}^{∞} (γλ)^l · δ_{t+l}
```

---

## 關鍵要點

| 概念 | 通俗解釋 |
|---------|---------------|
| **機率比 r(θ)** | 策略在這個動作上改變了多少 |
| **裁剪 ε (Clip)** | 安全邊界 —— 策略改變程度不要超過這個值 |
| **GAE** | 一種透過預見多個步長來估計優勢函數的聰明方法 |
| **數據效率** | 每次展開都是從多個平行環境中收集的（去相關的、穩定的經驗），然後複用進行 K 輪小批次更新 —— 裁剪保證了這些重複遍歷的安全性 |

## 下一步是什麼？

到目前為止，我們所有的環境都只有**離散**動作（向左或向右推）。

真正的機器人需要控制**連續**動作 —— 比如「精確施加 0.73 牛頓的力」。

`ppo_continuous.py` 使用**高斯策略 (Gaussian policy)** 將 PPO 擴展到連續動作，並在更困難的 BipedalWalker-v3 環境中進行測試！
