# 從零開始構建深度 Q 網路 (DQN) 🧠

## 線性模型的問題

還記得我們之前的線性公式嗎？

> 分數 = w₁ × 小車位置 + w₂ × 小車速度 + w₃ × 木棒角度 + w₄ × 木棒速度

這在 CartPole 任務中表現尚可，但如果面對一個你能看到成千上萬個像素的影片遊戲呢？你無法寫出一個如此簡單的配方！

我們需要一種能夠觀察複雜情況並找出最佳動作的東西。那就是 **神經網路 (neural network)**。

---

## 什麼是神經網路？

想想你的大腦。數百萬個被稱為神經元的小細胞相互交流。當你觸摸到燙的東西時，神經元會發送訊號：「燙！→ 立即移開手！」每個神經元傳遞資訊，它們共同做出明智的決定。

電腦上的 **神經網路** 工作方式相同：

```
輸入層 (Input Layer)   隱藏層 1           隱藏層 2           輸出層 (Output Layer)
[小車位置]    →        [128 個神經元]  →  [128 個神經元]  →  [向左推的分數]
[小車速度]    →        [    ...     ]     [    ...     ]     [向右推的分數]
[木棒角度]    →
[木棒速度]    →
```

每個箭頭都有一個 **權重 (weight)**（代表連接的強度）。網路中存在成千上萬個這樣的權重——而網路會學習所有這些權重！

**現實生活中的例子：** 餐廳的廚師品嚐你的食物，並同時調整數百種配料。每個味蕾就像一個神經元，它們共同告訴廚師「多加點鹽」或「少放點胡椒」。訓練網路就像廚師透過成千上萬次烹飪進行學習。

---

## DQN = 深度 Q 網路

**DQN** (Deep Q-Network) 由 DeepMind 在 2013 年發明。他們採用了舊的 Q 學習公式，並將 Q 表替換為了神經網路！

不再是：
> Q 表[狀態][動作] = 分數

而是：
> Q 網路(狀態) → [向左的分數, 向右的分數]

網路將狀態作為輸入，並一次性輸出所有動作的 Q 值。這比分別計算它們要高效得多！

---

## 本腳本：原生版本 (Naive Version)

本腳本展示了 **沒有任何** 特殊技巧的 DQN。它只是：
1. 觀察狀態
2. 詢問網路「向左有多好？向右有多好？」
3. 執行分數較高的動作
4. 獲得獎勵，更新網路

**這種方式本質上是不穩定的！** 把它想像成一個每次學習新知識都會立即忘記之前課程的學生。網路在每一步之後都會更新，這導致了混亂。

**現實生活中的例子：** 想像一下透過每嚐一口就改變整個菜譜來學習烹飪。你可能會從「太鹹」變成「一點鹽都沒有」再到「太鹹了」，永遠無法穩定在合適的用量上。這就是這裡發生的情況！

## 你將看到的結果

當你執行 `dqn_cartpole.py` 時：
- 分數可能會大幅跳動（不穩定的學習過程）。
- 有時代理人會變得非常出色，然後又忘掉了一切。
- 損失 (loss) 圖表顯示出劇烈的波動。

**這是預料之中的！** 它說明了為什麼我們需要改進——經驗回放 (experience replay) 和目標網路 (target network)。這些內容將在接下來的部分介紹！

## ε-貪婪技巧 (ε-Greedy Trick) 🎲

代理人並不總是選擇最好的動作。有時它會隨機選擇！

為什麼？因為如果它總是選擇看起來最好的動作，它可能永遠無法發現更好的選項。

> 以機率 ε (epsilon)：選擇一個隨機動作（探索 Exploration！）
> 以機率 1-ε：選擇已知的最佳動作（利用 Exploitation！）

我們從 ε = 1.0（100% 隨機）開始，並緩慢降低到 ε = 0.01（1% 隨機）。這樣，代理人在開始時會進行大量探索，然後專注於它學到的知識。

**現實生活中的例子：** 當你去一個新城市旅行時，你可能起初會隨機嘗試一些餐廳（探索）。過了一段時間後，你會回到你最喜歡的餐廳（利用）。但你仍然會偶爾嘗試一些新餐廳，以防有隱藏的美味！

## 關鍵詞彙

| 單字 | 含義 |
|------|---------|
| **神經網路** | 由數學神經元組成的層，透過數據進行學習 |
| **深度 (Deep)** | 擁有超過一個隱藏層（因此被稱為「深度」學習） |
| **DQN** | 深度 Q 網路 —— 使用神經網路代替 Q 表 |
| **ε-貪婪 (ε-Greedy)** | 策略：有時隨機探索，有時利用已有的最佳知識 |
| **不穩定性 (Instability)** | 網路不斷「遺忘」，因為更新之間會相互干擾 |

---

## 缺失的部分（以及為什麼它們很重要）

這種原生 DQN 有兩個大問題：

1. **相關的更新**：每一個經歷都是按順序到來的（第 1 步，第 2 步，第 3 步……）。如果第 5 步很糟糕，附近的所有更新都會被混淆在一起。
   
2. **移動的目標**：每次更新後，網路都會改變。但下一次更新使用相同的網路來計算目標值應該是多少。這就像在射擊一個移動的靶心！

這些問題由接下來的腳本中的 **經驗回放 (Experience Replay)** 和 **目標網路 (Target Networks)** 解決。它們共同將 DQN 從一個搖搖欲墜的初學者變成一個冠軍級的遊戲玩家！
