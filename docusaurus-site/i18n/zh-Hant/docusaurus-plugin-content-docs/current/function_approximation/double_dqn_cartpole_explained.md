# Double DQN：解決過度自信問題 🤔

## 問題所在：DQN 覺得自己比實際更強

想像一下有人問你：「城裡最好的餐廳是哪家？」

你可能會說：「披薩宮非常棒——絕對是 10/10 分！」但你其實只去過兩次。你並不真正知道它是否 *真的* 值 10/10 分。你可能會高估它，因為你那兩次去的時候恰好運氣好，吃到很棒的披薩。

同樣的問題也發生在 DQN 身上：代理人 (Agent) 會 **高估 Q 值**。

---

## 為什麼 DQN 會產生高估？

當 DQN 計算目標值 (Target) 時，它會執行：
> target = reward + γ × **max** Q(next_state)

這裡的 `max` 操作就是問題所在！當你從幾個帶有雜訊的估計值中選取最大值時，你幾乎總是會選到那個帶有最大隨機誤差（正向偏差）的估值。

**現實生活中的例子：** 你讓 5 個朋友猜測一棟建築的高度。他們的猜測分別是：40m, 38m, 45m (運氣好的猜測!), 39m, 41m。真實高度是 40m。如果你使用 `max(guesses)` = 45m，你就偏差太大了！帶雜訊猜測的最大值幾乎總是高估。

經過成千上萬次的更新，DQN 不斷向這些過度膨脹的目標值訓練，認為情況比實際更好。這會減慢學習速度，或者導致代理人做出過度自信且糟糕的決策。

---

## Double DQN 的解決方法

**Double DQN** (Hasselt et al., 2016) 將 `max` 操作分成了兩個步驟：

**第 1 步 —— 選哪個動作？** 使用 **在線網路 (online network)** 來挑選最好的動作：
> best_action = argmax Q_online(next_state)

**第 2 步 —— 它的價值是多少？** 使用 **目標網路 (target network)** 來評估該動作的價值：
> target = reward + γ × Q_target(next_state, best_action)

```
普通 DQN (Vanilla DQN):  target = r + γ × max_a Q_target(s', a)
                                 ↑ 同一個網路既負責選擇又負責評估 → 產生偏差

Double DQN:              best_a = argmax_a Q_online(s', a)   ← 在線網路負責選擇
                         target = r + γ × Q_target(s', best_a) ← 目標網路負責評估
                                 ↑ 不同的網路負責不同的任務 → 減少偏差
```

**現實生活中的例子：** 在一次求職面試中，你不會讓應聘者自己給自己的表現評分（這就是普通 DQN 的問題！）。相反，應聘者 *提交* 他們最好的作品，而由一個 *獨立的* 考官進行評估。兩個不同的人 = 更公正的評定！

---

## 為什麼分離會有幫助？

由於目標網路更新頻率較低，在線網路和目標網路擁有不同的權重。它們對「哪個動作最好」會有不同的「看法」。

當它們不一致時：
- 在線網路說：「動作 A 看起來很棒！」
- 目標網路說：「實際上，動作 A 表現一般——大約值 7 分，而不是 10 分。」

透過使用目標網路對在線網路所選動作的價值估算，我們得到了一個更誠實、通膨更低的數字。

---

## 程式碼差異：僅一行！

從普通 DQN 到 Double DQN，唯一的程式碼變化在於目標值的計算：

```python
# 普通 DQN:
q_next = target_net(s_next).max(dim=1).values

# Double DQN:
best_actions = online_net(s_next).argmax(dim=1, keepdim=True)   # 使用在線網路選擇動作
q_next = target_net(s_next).gather(1, best_actions)              # 使用目標網路評估價值
```

僅僅是兩行程式碼的改變——但對穩定性和準確性的影響是巨大的！

當你執行 `double_dqn_cartpole.py` 時，你會看到兩張圖表：

**圖 1：學習曲線**
- 普通 DQN 和 Double DQN 最終都能解決 CartPole 任務。
- Double DQN 通常收斂得更快且更穩定。
- CartPole 任務足夠簡單，因此差異適中；在 Atari 遊戲上這種差異會更顯著。

**圖 2：Q 值估計**
- 普通 DQN：Q 值隨時間不斷向上漂移（高估）。
- Double DQN：Q 值保持得更加適中且準確。

Q 值高估圖是關鍵的見解——它展示了普通 DQN 學到了虛高的價值，而這最終會損害性能。

## Double DQN 到底強多少？

| 指標 | 普通 DQN | Double DQN |
|--------|------------|------------|
| Q 值準確度 | 高估 | 更準確 |
| 學習穩定性 | 方差更大 | 方差更小 |
| CartPole 表現 | 良好 | 略好 |
| Atari 表現 (50 款遊戲) | 基準 | 接近人類水準的遊戲數量增加 2.6 倍 |

在複雜的 Atari 遊戲上，Double DQN 帶來的提升比在 CartPole 上顯著得多（因為 Atari 遊戲的 Q 值估計雜訊大得多）。

## DQN 改進家族

Double DQN 只是對普通 DQN 的眾多改進之一。2017 年的「Rainbow」論文結合了 6 項改進：

1. **Double DQN** (修正高估) ← 本腳本！
2. **優先經驗回放 (Prioritized Replay)** (從令人驚訝的經歷中學習更多)
3. **決鬥網路 (Dueling Networks)** (將「狀態有多好？」與「哪個動作最好？」分開)
4. **多步回報 (Multi-step returns)** (向更遠的未來看)
5. **分佈強化學習 (Distributional RL)** (學習回報的完整分佈，而不只是平均值)
6. **嘈雜網路 (NoisyNets)** (學習到的探索，而非 [ε-greedy](../foundations/multi_armed_bandit_explained.md#the-epsilon-greedy-strategy))

Rainbow 結合了所有這些改進，並實現了當時最好的 Atari 遊戲性能！

## 關鍵詞彙

| 單字 | 含義 |
|------|---------|
| **高估 (Overestimation)** | Q 值高於真實值（過度樂觀） |
| **Double DQN** | 使用在線網路選擇動作，使用目標網路進行評估 |
| **解耦 (Decoupling)** | 將原本由同一個網路完成的兩項任務分開 |
| **偏差 (Bias)** | 某個方向上的系統性誤差（總是太高或總是太低） |
| **Rainbow** | 一種結合了 6 項改進以獲得最高性能的 DQN 變體 |

你現在已經完成了階段 3 的完整進階：

| 演算法 | 增加了什麼 | 為什麼有幫助 |
|-----------|-------------|-------------|

| + 目標網路 | 用於目標的凍結副本 | 穩定「靶心」 |
| Atari DQN | CNN + 影格疊加 | 從像素中學習！ |
| Double DQN | 分離 選擇/評估 | 減少高估 |

每一步都解決了一個特定的問題。這就是真實研究的進行方式——一次進行一項細緻的改進！
