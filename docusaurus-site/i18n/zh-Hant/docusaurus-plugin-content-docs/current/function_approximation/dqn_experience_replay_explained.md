# 經驗回放 (Experience Replay)：教會機器人記住過去 🎒

## 問題所在：遺忘（與混淆）

還記得原生 DQN 是如何不穩定的嗎？最大的原因是 **相關的學習 (correlated learning)**。

當機器人玩遊戲時，它按順序經歷各種事情：
> 第 1 步 → 第 2 步 → 第 3 步 → 第 4 步 → ...

這些步驟是相連的！如果機器人在第 10 步向左傾斜，第 11 步通常也會向左傾斜。它們不是獨立的——它們彼此依賴。

當我們使用這些相關的步驟來更新網路時，就像試圖透過反覆閱讀同一個章節來學習歷史。你會變得非常精通這一個章節，但卻忘掉了其他所有內容！

**現實生活中的例子：** 想像一下你為了考試複習，卻只練習昨天的家庭作業。你對那些題目變得非常拿手，但考試卻有不同的題目！你需要練習各種不同題目的 **混合**。

---

## 解決方案：一個記憶盒 📦

**經驗回放 (Experience Replay)** 為機器人增加了一個大型記憶盒（即 **回放緩衝區 replay buffer**）。

機器人不再僅僅從最新的經歷中學習，而是：
1. **儲存**：將每一次經歷都存入記憶盒：(狀態, 動作, 獎勵, 下一個狀態)
2. **隨機抽取**：從盒子裡隨機抽取一小部分記憶
3. **混合學習**：從這些隨機混合的記憶中學習，而不僅僅是學習最新的一步

```
遊戲第 1 步 → [存入盒子]
遊戲第 2 步 → [存入盒子]
遊戲第 3 步 → [存入盒子]
...
遊戲第 50 步 → [存入盒子] → 抽取 64 個隨機記憶 → 更新網路
遊戲第 51 步 → [存入盒子] → 抽取 64 個隨機記憶 → 更新網路
```

**現實生活中的例子：** 想想相簿。你不會僅僅透過看今天的照片來了解你的生活。你也會翻閱 **舊** 照片——一個包含了美好回憶和棘手時刻的混合。這能幫助你理解整個人生的規律，而不僅僅是今天。

---

## 為什麼隨機採樣會有幫助

當我們隨機選取記憶時，我們打破了相關性。機器人可能同時從以下記憶中學習：
- 一個木棒保持得非常完美的記憶（來自 500 步之前）
- 一個木棒即將倒下的記憶（來自 20 步之前）
- 一個運氣很好的記憶（來自第 3 步）

這種隨機混合意味著：
✅ 機器人能從各種各樣的情況中學習
✅ 每個記憶可以被「回放」多次（高效利用經驗）
✅ 網路不會過度擬合最近發生的事件

---

## 小批次 (Mini-Batch) 學習

我們不是一次只針對一個經歷進行更新，而是一次針對 **64 個經歷** 進行更新（一個「小批次」）。這就像：
- 舊方法：讀一張抽認卡，考考自己
- 新方法：讀 64 張不同的抽認卡，然後針對這一堆卡片考考自己

小批次學習使學習訊號更加可靠，減少了雜訊。

---

## 預熱期 (Warmup Period)

我們不會立即開始學習！回放緩衝區首先需要一些記憶。在訓練開始前，我們要等到盒子裡至少有 **500 個經歷**。

**現實生活中的例子：** 在你收集齊食材之前，你不會嘗試去做一頓飯。預熱期就像是做飯前的買菜過程！

---

## 比較結果顯示了什麼

當你執行 `dqn_experience_replay.py` 時，你會看到兩條學習曲線：

| 原生 DQN (Naive DQN) | DQN + 經驗回放 |
|-----------|-------------|
| 非常顛簸 | 更平滑 |
| 頻繁崩潰（忘掉一切） | 進步更持久穩定 |
| 高方差 | 低方差 |

帶有經驗回放的版本通常：
- 更可靠地達到高分
- 不會頻繁從 500 分跌回 30 分
- 顯示出更穩定的學習進度

---

## 程式碼中的回放緩衝區

```
ReplayBuffer:
  - 容量 (capacity)：10,000 條記憶（存滿後忘掉最舊的）
  - push(state, action, reward, next_state, done)：存入記憶
  - sample(batch_size=64) → 隨機批量：抽取記憶
```

把它想像成一個有 10,000 行的筆記本。當它記滿時，你會擦掉最舊的一行並寫下最新的一行。你總是從隨機的一頁開始學習！

---

## 關鍵詞彙

| 單字 | 含義 |
|------|---------|
| **經驗回放 (Experience Replay)** | 儲存並隨機重複使用過去的經驗進行訓練 |
| **回放緩衝區 (Replay Buffer)** | 儲存過去 (狀態, 動作, 獎勵, 下一狀態) 元組的記憶盒 |
| **相關的更新 (Correlated updates)** | 當訓練資料相互依賴時（不利於學習！） |
| **小批次 (Mini-batch)** | 用於一次更新步驟的一小部分隨機記憶樣本 |
| **去相關 (Decorrelation)** | 打破連續經歷之間的聯繫 |

## 還有什麼缺失？

即使有了回放緩衝區，還有一個問題：**移動的目標**。

每當我們更新網路時，Q 值就會改變。但這些更新後的 Q 值也被用來計算下一次更新的目標值。這是一個混亂的循環！

這由 **目標網路 (Target Network)** 解決——一個每隔 100 步才更新一次的網路凍結副本。這讓「靶心」能在一段時間內保持不動，這樣機器人就能可靠地瞄準它！
