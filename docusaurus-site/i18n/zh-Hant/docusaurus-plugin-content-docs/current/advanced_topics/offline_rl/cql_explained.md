# 保守 Q 學習 (Conservative Q-Learning, CQL) 🛡️

## 什麼是 CQL？

想像一下，你正在透過閱讀一本包含他人過去股票交易記錄的大型帳本來學習如何投資。帳本裡記錄了買入、賣出和持有 —— 但 **沒有記錄任何沒有人真正做過的交易**。

現在想像一個過度自信的學生看著帳本說：*「如果有人每個週一都買彩券會怎樣？那肯定是一筆驚人的交易！」*

問題在於：**帳本裡沒有關於週一買彩券的數據**，所以這個學生只是在幻想。然而，那種幻想出來的交易在紙面上看起來很棒，所以學生的「策略」一直想要執行它。

這種幻想問題就是 **分布偏移 (distribution shift)**：離線學習者會偏愛數據集從未測試過的動作，因為沒有數據可以反駁這種樂觀態度。CQL 就是針對這一問題的良方。

---

## 為什麼 Q 學習在離線狀態下會出錯

普通的 Q 學習目標是：

```
target(s, a) = r + γ · max_{a'} Q(s', a')
```

那個 `max_{a'}` 就是危險所在。當數據集從未在狀態 `s'` 下記錄過動作 `a'` 時，網絡只會 *猜測* 一個 Q 值 —— 而神經網路傾向於對未見過的輸入 **高估** Q 值。目標值繼承了這種高估，網絡學會預測那個更大的數字，然後在下一步我們又會 **外推 (extrapolate)** 到更高的值（投影到遠超數據支持的範圍）。策略因此追逐著一個幻象。

如果你能繼續收集更多數據，這將會自我修正（幻象動作在現實中被證明是糟糕的）。但 **在離線強化學習中，你無法收集更多數據。** 幻象將永遠存在。

---

## CQL 的技巧

CQL (Kumar et al., 2020) 在損失函數中添加了一個懲罰項：

```
cql_loss(s)  =  log Σ_a exp Q(s, a)   -   Q(s, a_dataset)
```

它包含兩個部分：

1. **`log Σ_a exp Q(s, a)`**（讀作：*「所有動作的 log-sum-exp」*）是所有動作的 **軟最大值 (soft maximum)** —— 這是一個平滑且可微的 `max` 近似值，它同時考慮所有動作，而不是硬性選擇一個勝出者。懲罰這一項會 **全面** 降低 Q 值（統一壓低所有預測值） —— 特別是對於那些具有 *最高* Q 值的動作，而這正是幻象所在。
2. **`- Q(s, a_dataset)`** 獎勵數據集實際記錄動作的高 Q 值 —— 保護分布內 (in-distribution) 的 Q 值不受上述壓低的影響。

最終效果：**未見動作的 Q 值被壓低，已見動作的 Q 值被拉高。** 學習到的 Q 值成為了真實 Q 值的 *下界*。**`argmax`** 策略（即簡單選擇具有最高 Q 值的動作的規則）不再追逐幻象。

總損失函數為：

```
L  =  Bellman_MSE   +   α · cql_loss
```

（其中 **`Bellman_MSE`** 是普通 Q 學習的標準誤差，衡量網路當前的猜測與其自身未來猜測之間的差異）。

`α` 是保守係數調節旋鈕。太小 → 分布偏移會再次潛入。太大 → 智能體會過於保守，永遠無法超越數據的水平。

## 現實生活中的類比

- **保守的象棋教練。** 你只能從已經下過的棋局中學習。魯莽的教練會說：「這一步沒有先例的假設性著法可能是天才之舉！」 CQL 則是那位會說「我們沒有關於那一著的數據 —— 讓我們堅持現實棋手嘗試過的著法」的教練。
- **餐廳菜單選擇。** Yelp 評論從未涵蓋菜單外的隱藏菜品。一個天真的策略會根據幻想出來的五星評分推薦隱藏菜品。CQL 則只推薦那些被點過足夠多次、值得信賴的菜品。
- **從日誌中學習機器人抓取。** 機器人有抓取杯子、瓶子和書本的影片 —— 但從未抓取過刀。CQL 會拒絕自信地推薦「抓住刀刃」。

## 我們的代碼做了什麼

腳本 `cql.py` 執行以下操作：

1. **加載由 `d4rl_dataset.py` 構建的四個數據集**。
2. **選擇 `medium-replay`** 作為訓練集 —— 這是最現實的（質量參差不齊），也對天真方法最具破壞性。
3. **在完全相同的條件下訓練三個純離線智能體**，唯一的區別是 `α`：
   - `α = 0`   →  天真的離線 DQN (無懲罰 —— 那個有缺陷的基準)
   - `α = 1.0` →  輕度 CQL
   - `α = 5.0` →  強力 CQL
4. **每隔 2,500 步梯度下降進行一次評估**，透過在真實環境中以貪婪方式運行策略（10 個回合）。這是 *唯一* 與環境接觸的地方；訓練本身從未見過環境。
5. **將學習曲線繪製** 到 `outputs/cql.png`。

## 你應該看到的結果

典型運行結果打印如下：

```
Final evaluation returns (avg over 10 episodes, greedy):
  naive offline DQN (alpha=0)         ->  ~30-150  (不穩定; 經常崩潰)
  CQL (alpha=1.0)                     ->  ~300-450
  CQL (alpha=5.0)                     ->  ~450-500
```

在學習曲線圖中：

- **紅色曲線** (`α = 0`) 早期上升，然後經常 **墜入深淵**，一旦分布偏移的幻象感染了 **貝爾曼目標 (Bellman target)**（我們在訓練 Q 網絡時用作「正確答案」的數字：`r + γ · max Q(s', ·)`）。當幻象 Q 值污染了那個目標，每一步梯度下降都會讓情況變得更糟。**貝爾曼損失 (Bellman loss)**（Q 網絡的預測與貝爾曼目標之間的 MSE）看起來很正常 —— 這就是問題的 **詭詐之處**：網絡與其自身的錯誤信念完美一致，因此損失函數不會發出警告。
- **橙色曲線** (`α = 1.0`) 上升較慢，但 **保持在高位**。
- **綠色曲線** (`α = 5.0`) 最穩定，通常也是最好的。

貝爾曼損失面板顯示了另一個跡象：天真的 DQN 損失可以保持很小，但其策略卻極其糟糕，因為網絡內部與其自身的幻象是一致的。

## CQL 在該領域的位置

CQL 意義重大，因為它為分布偏移提供了一個有原則且簡單的修復方法。演進路線如下：

```
DQN (在線)
   │
   ▼
天真離線 DQN  ── 因分布偏移而失效
   │
   ▼
CQL (Kumar 2020)    ── 添加保守懲罰：Q 是一個下界
   │
   ▼
IQL (Kostrikov 2021)  ── 根本不查詢未見動作的 Q 值
   │
   ▼
Decision Transformer (Chen 2021)  ── 完全跳過 Q 值，將強化學習視為序列建模
                                      （在給定過去狀態和所需總回報的情況下
                                       預測 *下一個動作*，就像 LLM 預測
                                       下一個詞一樣）
```

這條演進路線上的每一步都是對同一個問題的不同回答：**我如何避免向我的 Q 網路詢問它從未見過的事情？**

---

## 關鍵術語

| 術語 | 含義 |
|------|---------|
| **分布偏移 (Distribution shift)** | 訓練好的策略想要的動作超出了數據的範圍 |
| **分布外 (Out-of-distribution, OOD)** | 數據集從未記錄過的 (s, a) 對 |
| **真實 Q 值** | 如果我們能完美測量，在狀態 `s` 下採取動作 `a` 的真實預期未來回報 |
| **保守 Q 值** | 一個學習到的 Q 函數，試圖保持在真實 Q 值之下，而不是過度承諾 |
| **Logsumexp** | 一個平滑、可微的 `max` 近似函數 |
| **Alpha (α)** | CQL 的保守係數 —— 壓低 OOD 動作 Q 值的力度 |

---

## 一句話總結

> **CQL 添加了一個「悲觀懲罰」，懲罰數據集從未嘗試過的動作上的高 Q 值 —— 這樣策略就不會迷戀上幻想。**
