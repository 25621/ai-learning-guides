# 獎勵建模：教會計算機人類的偏好

## 核心思想

獎勵模型就像一個小法官。你給它看同一個問題的兩個答案，告訴它人更喜歡哪一個。隨著時間的推移，它就會學會給人們更喜歡的答案打更高的分數。

為什麼我們需要這樣一個法官？因為我們對語言模型的大多數期望（如「有幫助」、「禮貌」或「文筆好」）很難用數學公式寫下來。沒有一個單一的等式可以定義這些概念。但人類幾乎總能從兩個選項中指出更好的那個。獎勵模型將這些簡單的「這個更好」的投票轉化為學習演算法可以使用的分數。

## 現實生活中的類比

想像你在教朋友做布朗尼蛋糕。

你不會扔給他們一本 50 頁的《如何做出好布朗尼》規則手冊。相反，你會品嘗他們做的兩批蛋糕，然後說：

「這一批更好吃。」

經過幾輪這樣的嘗試，你的朋友開始注意到規律。也許口感更軟糯的總是獲勝，也許烤過頭的總是失敗。你的朋友通過你的比較建立了一套心理評分系統。

獎勵模型也是如此，只不過使用的是數字。它不需要知道*為什麼*選中的答案更好，它只需要大量的「A 勝過 B」的例子，就能逐漸學習到一個與偏好一致的分數。

## 學習原理（僅直覺層面）

每個例子都是一個三元組：一個提示詞 (prompt)、一個**被選中的 (chosen)** 回答和一個**被拒絕的 (rejected)** 回答。我們希望模型給被選中的回答打的分數比被拒絕的高——無論高出多少。

這種訓練「推力」在精神上非常簡單：

- 被選中的分數太低？往上推。
- 被拒絕的分數太高？往下壓。
- 已經排好序且有明顯差距？那就別管它們。

這種推力被稱為 Bradley-Terry 損失函數，它是現代 RLHF 系統的標準配方。

## 實驗結果展示

我們在 2,000 對合成的偏好資料上訓練了一個獎勵模型。下圖展示了同一次訓練運行的三個視角。

![獎勵模型訓練](outputs/reward_modeling.png)

- **左圖**：損失函數 (Loss) 迅速下降。模型對自己的排名越來越有信心。
- **中圖**：偏好準確率攀升至近 100%。在幾乎每一對資料中，被選中的回答得分都高於被拒絕的。
- **右圖**：被選中與被拒絕回答的分數分布逐漸拉開。開始時它們相互重疊；訓練後，被選中的回答明顯位於右側。

這種分離就是我們要達到的目的。後續步驟（PPO 或 DPO）現在可以將這個分數作為優化的目標。

## 在 RLHF 流程中的位置

路線圖將 RLHF 描述為「使模型與人類偏好對齊」。獎勵模型是三個步驟中的第一步：

1. **獎勵模型（本文件）**：將偏好投票轉化為分數。
2. **PPO 微調**：在保持模型原有行為的同時，推動語言模型獲得更高分數。
3. **DPO**：一種更新、更快捷的方法，完全跳過了獎勵模型。

因此，獎勵建模是*人類判斷*與*機器優化*之間的橋梁。如果這座橋搭錯了，下游的每一步都會偏離航線。

## 為什麼這在實驗室之外也很重要

同樣的想法出現在很多地方：

- **推薦系統**通過點擊、跳過和觀看時長來學習你喜歡什麼。
- **搜索引擎**通過「你點擊了哪個結果」來學習排名。
- **餐廳**通過重複訂單來了解受歡迎的菜品，而不是讓顧客寫文章描述他們喜歡什麼。

只要**比較**比**評分**更容易，獎勵模型就是正確的工具。

## 一句話總結

**獎勵模型是一個經過學習的「法官」，它將「這個更好」的偏好轉化為數字分數，供 RLHF 的其餘部分進行優化。**
