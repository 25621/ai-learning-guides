# 矩陣遊戲：最簡單的多智能體世界 🎲

## 什麼是矩陣遊戲？

想像一下，你和你的朋友各選一個手勢 —— **石頭、剪刀或布** —— 且*同時*進行。你們各向對方選擇保密。勝負由一個小表格決定：

|        | 石頭 | 布 | 剪刀 |
|--------|:----:|:-----:|:--------:|
| 石頭     |  0,0  | -1,+1 | +1,-1 |
| 布    | +1,-1 |  0,0  | -1,+1 |
| 剪刀 | -1,+1 | +1,-1 |  0,0  |

這個表格就是遊戲的*整個世界*。沒有移動，沒有時間，沒有地圖。只有一個一次性的決策。我們稱之為 **矩陣遊戲 (matrix game)**，因為收益矩陣 (payoff matrix) 就是整個環境。

矩陣遊戲是研究 **多智能體強化學習 (multi-agent RL)** 最純淨的場所，因為在訓練過程中唯一可能改變的是每個玩家的*策略 (policy)* —— 即選擇每個動作的概率。

---

## 為什麼它是「多智能體」的

在單智能體強化學習中，環境是固定的：風總是以同樣的方式吹，地板永遠不會動。智能體不斷改進，最終贏得比賽。

在矩陣遊戲中，你的「環境」是*另一個正在學習的智能體*。隨著他們變得越來越聰明，對你來說什麼是「好招」也會*隨之改變*。這被稱為 **非平穩性 (non-stationarity)**，它是多智能體強化學習的核心問題。

> 如果你一直出石頭，你的對手最終會開始一直出布。所以你切換到剪刀。所以他們切換到石頭。所以你切換到布……以此類推。「最佳動作」永遠不會停留在原地。

經典的解決方案是 **混合策略 (mixed strategies)**：不要確定性地選擇任何一個動作，而是以一種對手無法利用的方式進行隨機化。

## 我們玩的三個遊戲

### 1) 石頭剪刀布 (零和博弈)
- 一個玩家的收益就是另一個玩家的損失。
- **納許均衡 (Nash equilibrium)** 是：每個玩家以 ⅓ 的概率選擇每個動作。任何偏離都會被對方利用。
- 我們預期這兩個 Q 學習器會在 ⅓-⅓-⅓ 附近擺動 —— 永遠不會完全穩定，因為每當一個學習器發生偏移，另一個就會做出反應。

### 2) 囚徒困境 (非零和博弈/一般和博弈)
兩個嫌疑人被分開審訊：

|           | 合作 (Cooperate) | 背叛 (Defect) |
|-----------|:---------:|:------:|
| 合作 |   3, 3    |  0, 5  |
| 背叛    |   5, 0    |  1, 1  |

- 無論對方做什麼，「背叛」都優於「合作」 —— 這是一項 **佔優策略 (dominant strategy)**。
- 兩個玩家都是理性的 → 兩個都背叛 → 兩個都得到 1，儘管（合作，合作）本可以各得 3。自私的最佳反應破壞了集體福利。
- 我們預期 Q 學習會清晰地收斂到（背叛，背叛）。

### 3) 獵鹿博弈 (協調博弈)
兩名獵人可以合作捕獲一隻鹿（巨大的獎勵），或者各自捕獲一隻兔子（獎勵小但安全）：

|       | 獵鹿 | 捕兔 |
|-------|:----:|:----:|
| 獵鹿  | 4, 4 | 0, 3 |
| 捕兔  | 3, 0 | 2, 2 |

- （獵鹿，獵鹿）是 **收益佔優 (payoff-dominant)** —— 對雙方都最好。
- （捕兔，捕兔）是 **風險佔優 (risk-dominant)** —— 如果你不信任你的夥伴，這是最安全的。
- 結果取決於初始條件：獨立的 Q 學習器往往最終陷入*較差的*（捕兔，捕兔）均衡，因為捕兔對於學習來說更安全。

## 現實生活中的例子

- **雙頭壟斷定價。** 同一條街上的兩家咖啡館每天早上各定一個價格。收益矩陣的形狀決定了它們最終是處於一個高的「合作」價格（對它們有利，對顧客不利），還是一個低的「割喉式」競爭價格。
- **網路協定。** 路由器和發送器選擇定時策略；網路的擁塞結果由類似於矩陣遊戲的「成功通過」與「退避」的收益決定。
- **拍賣出價。** 每個出價者在不知道其他人的情況下選擇一個出價；收益取決於整個出價向量。納許均衡是一個*出價策略*，而不是一個單一的數字。

## 我們的代碼做了什麼

對於每個遊戲，我們：
1. 創建兩個無狀態的 Q 學習器（Q 只是每個動作對應的一個數字 —— 在一次性遊戲中沒有狀態）。
2. 循環運行 20,000 步。每一步：兩個智能體同時選擇一個 ε-greedy 動作，獲得獎勵，並更新它們的 Q 值。
3. 在滾動的 500 步窗口中跟蹤每個智能體的 **經驗動作頻率 (empirical action frequency)**。我們不僅觀察抽象的概率，還計算他們最近實際選擇了哪些動作（例如：「在最近的 500 輪中，他們有 40% 的時間出了石頭」）。這為我們展示了他們策略變化的實時、實際圖像。
4. 繪製隨時間變化的頻率圖，保存至 `outputs/<game>.png`，並打印最終的 Q 值。

### 你應該看到的現象

| 遊戲 | 預期的圖表結果 |
|------|------------------------------|
| **石頭剪刀布** | 兩個玩家都徘徊在 ⅓-⅓-⅓ 附近，但有明顯的抖動。曲線相互追逐 —— 經典的循環行為。 |
| **囚徒困境** | 兩個玩家的「背叛」頻率迅速上升到 ~1.0。「合作」被徹底拋棄。 |
| **獵鹿博弈** | 大多數隨機種子會穩定在（捕兔，捕兔）。一些幸運的種子會達到（獵鹿，獵鹿）—— 嘗試更改腳本中的隨機種子，觀察它是如何反轉的。 |

## 獨立學習失效的地方

我們的智能體是*獨立的* —— 它們只看到自己的獎勵，從未看到對手的動作或 Q 值。這是最簡單的基準，但它有局限性：

- 它 **無法保證在一般和博弈中收斂**。
- 它可能會陷入 **壞的均衡**（如獵鹿博弈）。
- 它 **無法對對手建模**。

真正的多智能體算法透過顯式地推理其他學習器來解決這個問題。以下是常見的幾種算法的核心思想：

| 算法 | 核心思想 | 現實生活類比 |
|-----------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------|
| **虛構遊戲 (Fictitious play)** | 記錄對手選擇每個動作的頻率。假設明天他們會做他們一直以來所做的事 —— 然後選擇你對該信念的最佳反應。 | 在多場國際象棋比賽中觀察對手的習慣，並據此調整你的開局。 |
| **CFR (逆事實遺憾最小化)** | 在每一輪之後，問：「如果我沒選這個動作，而選了其他動作，我會多後悔？」 逐漸將概率轉移到那些你後悔錯過的動作上。常用於撲克，因為它能處理 **不完全信息** 遊戲（你看不到對手的牌）。 | 在一把撲克牌局後進行復盤並思考：「我本該下更大的注 —— 下次我會那樣做。」 |
| **LOLA (學習對手的學習意識)** | 你的梯度步驟考慮了對手*也在*進行梯度更新這一事實。你在優化自己的更新時，預判對手的下一步更新 —— 提前預判兩步而不是一步。 | 在談判交易時思考：「如果我出價 X，他們會還價 Y，所以我應該先從 Z 開始。」 |
| **MADDPG (多智能體深度確定性策略梯度)** | 每個智能體的*評論家 (critic)*（價值評估器）都以 **全局視野** 進行訓練：它能看到每個人的觀測和動作。而*執行者 (actor)*（部署的策略）仍然只使用局部信息 —— 這就是 CTDE（集中式訓練，分佈式執行）模式。 | 一名籃球教練觀看全場比賽（集中式評論家），但教導每個球員僅根據他們能看到的情況做出反應（分佈式執行者）。 |

但獨立 Q 學習是正確的第一步。你能親身感受到非平穩性問題帶來的挑戰，之後那些修復方案也就順理成章了。

## 關鍵術語

| 單詞 | 含義 |
|------|---------|
| **Payoff matrix (收益矩陣)** | 定義一次性多智能體遊戲的表格 |
| **Nash equilibrium (納許均衡)** | 一個策略組合，其中任何單個智能體都無法透過偏離策略來提高自身收益 |
| **Mixed strategy (混合策略)** | 在多個動作上進行隨機化的策略 |
| **Non-stationarity (非平穩性)** | 環境（= 其他智能體）在學習過程中不斷變化 |
| **Independent learner (獨立學習器)** | 忽略其他學習器存在的智能體 |
| **Zero-sum (零和)** | 一個智能體的收益正好是另一個智能體的損失 |
| **General-sum (一般和/非零和)** | 雙方可能共贏、共輸，或處於兩者之間的任何情況 |
