# 探索策略比較 🔦

## 一句話問題 {#one-sentence-problem}

強化學習智能體必須做兩件方向相反的事情：

- **利用 (Exploit)**：執行目前效果最好的動作。
- **探索 (Explore)**：嘗試新事物，以防它更好。

過度偏向「利用」，你將永遠安於平庸的現狀。過度偏向「探索」，你將永遠無法兌現收益。*如何* 探索——而不只是 *是否* 探索——是區分能解決《Montezuma's Revenge》的智能體與得分為零的智能體的關鍵。

本腳本將 **五種** 探索策略在相同的兩個困難任務上進行正面交鋒，以便你觀察它們的特性。

## 現實生活中的比喻：挑選午餐地點 {#real-life-analogy}

你剛搬到一個擁有 200 家餐廳的新城市。

- **ε-greedy** = 「去我目前最喜歡的餐廳，但每隔十天擲一次骰子，*完全隨機* 選一家餐廳。」 你會進行廣泛但 *漫無目的* 的取樣——而且你會不斷地再次光顧那些你已經討厭的地方。
- **樂觀初始化 (Optimistic initialisation)** = 「假設我沒去過的 *每一家* 餐廳都是城裡最好的，直到事實證明並非如此。」 你會有條不紊地嘗試所有 200 家，隨著現實讓你失望而將它們一一劃掉——你會很快找到真正棒的餐廳。
- **UCB (置信上限)** = 「偏好我最喜歡的餐廳，但給那些我幾乎沒去過的地方提供 *額外獎勵 (bonus)* ——我了解得越少，獎勵就越大。」 這在決定今天去嘗試 *哪家* 陌生餐廳時很聰明，但每個決定都是局部的：它只挑選 *眼下* 看起來最好的選項，而不會規劃一條穿越整個未探索區域的路線。它不會想「我應該穿過城市去東邊，因為那裡聚集了二十家沒試過的地方」——每家餐廳都是一步一步被孤立地評估的。
- **基於計數的獎勵加成 (Count-based reward bonus)** = 類似於 UCB，但你也 *享受新奇本身* ——去一家全新的餐廳吃飯在本質上是令人滿足的，這種滿足感塑造了你走進哪些社區進行探索的長期計劃。
- **預測誤差獎勵加成 (Prediction-error reward bonus)** = 「我從讓我 *感到意外* 的餐點中獲得快感——一些我無法預見的東西。」 一家表現完全符合預期的新餐廳？平平無奇。一家與你的心理模型截然不同的餐廳？迷人，你會更新你的計劃去尋找更多類似的地方。

## 五種策略（均在 `compare_exploration.py` 中） {#five-strategies}

### 1. ε-greedy —— 預設策略，它是 *猶豫不決*，而非探索 {#epsilon-greedy}

貪婪地行動，但以機率 ε 採取均勻隨機的動作。它是 DQN 及其變體中的標準基準。它在困難任務上的致命缺陷是：**每一步都是獨立的硬幣翻轉。** 要在一條包含 `N` 個正確動作的鏈條上走下去，你需要硬幣連續 `N` 次都翻轉正確——這在指數級上是不可能的。ε-greedy 是 *抖動*，而非 *探索*。

### 2. 樂觀初始化 —— 「在證明無聊之前是無辜的」 {#optimistic-initialization}

將 *每個* Q 值都設定為可能的最大回報，即 `R_max / (1 − γ)`。現在，智能體從未嘗試過的動作看起來就像是世界上最棒的事情，因此 **貪婪** 策略被迫去嘗試它；只有在訪問過它之後，該值才會跌落至真相。關於 *未* 嘗試區域的樂觀態度會通過 **價值函數自動傳播**（通過 Q-learning 的自舉），因此智能體會被一步步拉向它未見過的世界部分。幾乎不需要額外成本，不需要額外記帳——而且正如你將看到的，它是小型表格世界中最強大的 *深度* 探索者。

### 3. UCB 風格動作選擇 —— 獎勵加成在 *選擇* 中，而非 *獎勵* 中 {#ucb-selection}

選擇 `argmax_a [ Q(s,a) + c·√(ln t / N(s,a)) ]`：偏好高價值動作，但誇大那些你很少嘗試的動作。在多臂拉霸機問題中非常有名。缺點是：獎勵加成僅存在於 **動作選擇規則** 中，從不在獎勵中——因此它 *不會* 流經價值函數。UCB 擅長「確保我已經嘗試了 *這個* 狀態下的每個動作」，但在「規劃一條通往遙遠未探索區域的路線」方面表現較弱。

### 4. 基於計數的 **獎勵** 加成 —— 好奇心的經典版本 {#count-based-reward}

將 `1/√(N(s,a))` 添加到 **獎勵** 中（權重 `beta` 隨時間衰減）。因為是在獎勵中，Q-learning *會* 傳播它：通往新奇區域的狀態變得更有價值。這是 MBIE-EB / 經典的「探索加成」思想——也是練習 1 的前半部分。

### 5. 預測誤差 **獎勵** 加成 —— 好奇心，ICM/RND 版本 {#prediction-error-reward}

將來自小型學習前向模型的 `−log P(s'|s,a)` 添加到獎勵中（同樣具有衰減的 `beta`）。這是五種策略中最敏銳的新奇信號：在確定性的世界中，一旦你見過一次轉移，它的意外程度就會降至 ~0，而不是像 `1/√N` 那樣緩慢消退。它是 ICM / RND 的表格版本——也是練習 1 的後半部分。

## 兩個測試任務 {#two-test-tasks}

- **任務 A —— MiniMontezuma**：鑰匙→門→寶藏的網格世界，獎勵僅在寶藏處（約 15 步完美移動）。測試「你是否能從長期的稀疏獎勵鏈中倖存？」
- **任務 B —— DeepSea(N)**：教科書般的深度探索鏈，執行長度為 `N = 5, 8, 11, 14`。獎勵隱藏在 `N` 個正確動作之後，每個動作都有微小的即時成本——因此短視的智能體會學會規避成本，永遠找不到獎品。測試「隨著鏈條變得 *更長*，你的策略是否仍然有效？」

## 實際發生的情況（執行並觀察） {#what-happens}

**任務 A —— MiniMontezuma：**

| 策略 | 首次發現寶藏 | 最終解決率 |
|----------|---------------:|-----------------:|
| ε-greedy | 從未 | 0.00 |
| 樂觀初始化 | 約第 1 回合 | 1.00 |
| UCB 動作選擇 | 約第 3 回合 | ~0.95 |
| 計數獎勵加成 | 約第 82 回合 | ~0.41 |
| 預測獎勵加成 | 約第 23 回合 | 1.00 |

**任務 B —— DeepSea，找到獎勵的種子比例：**

| 策略 | N=5 | N=8 | N=11 | N=14 |
|----------|----:|----:|-----:|-----:|
| ε-greedy | 0 | 0 | 0 | 0 |
| 樂觀初始化 | 1.0 | 1.0 | 1.0 | 1.0 |
| UCB 動作選擇 | 1.0 | 1.0 | 0.0 | 0.0 |
| 計數獎勵加成 | 1.0 | 1.0 | ~0.1 | 0.0 |

*(數字會隨隨機種子略有波動，但大趨勢是非常穩固的。)*

## 啟發 {#takeaways}

1. **ε-greedy 並不是探索。** 它從未解決過 *任何一個* 困難任務。隨機的猶豫不決根本無法穿透長期的正確序列。（然而它仍然是很多程式碼中的預設設置——因為在 *簡單* 任務上，它足夠好且極其簡單。）

2. **真正的探索意味著對未知保持樂觀——無論以何種方式。** 無論你是將樂觀情緒注入 *初始值*（策略 2）、*動作選擇*（策略 3），還是 *自我生成的獎勵*（策略 4–5），共同的主題都是：*讓未探索的事物看起來很有吸引力*，然後讓學習帶你到達那裡。

3. **在稀疏獎勵的網格上，所有四種「真正的」策略都奏效——且預測誤差加成最快到達目的地**，因為它產生了最清晰的「這是新的」信號。

4. **在需要長距離傳播樂觀情緒的 *深度* 鏈條上，簡單的冠軍是樂觀初始化。** 它免費地通過價值函數傳播樂觀情緒。UCB 最先崩潰（它的獎勵加成從未進入價值函數，因此它無法進行深度 *規劃*）。獎勵加成表現更好——它們 *確實* 傳播——但普通的表格 Q-learning 在獎勵加成衰減之前，將這種樂觀情緒推向長鏈底部的速度較慢。

5. **最後一點正是為什麼將深度探索擴展到像素級別需要額外的火力** —— 帶自舉的 DQN、使用真實神經網路的 RND（以便樂觀情緒在相似狀態間 *泛化*，而不是一次傳播一個網格單元）、Go-Explore（字面意思上記住並返回有希望的狀態）。這裡的表格模型向你展示了 *原理*；真實的系統是這些原理加上一個可以泛化的網路。

## 關鍵術語 {#key-terms}

| 術語 | 含義 |
|------|---------|
| **探索-利用權衡** | 嘗試新事物 vs. 兌現已知收益——強化學習的核心張力 |
| **猶豫不決 (Dithering)** | 通過向動作添加隨機雜訊來進行「探索」（ε-greedy、高斯雜訊）——在困難任務上表現微弱 |
| **不確定情況下的樂觀主義** | 傘式原則：在檢查過之前，將未知事物視為很棒的事物 |
| **樂觀初始化** | 通過將所有初始值設定為最大可能回報來實現該原則 |
| **UCB** | 置信上限：選擇 `argmax (價值 + 隨訪問次數縮小的獎勵加成)` |
| **深度探索 (Deep exploration)** | 需要一系列長期 *連貫* 的「非同尋常」動作的探索，而不只是一個動作 |
| **`beta` 退火 (annealing)** | 隨時間縮小好奇心權重，以便智能體最終停止探索並開始利用 |

## 一句話總結 {#summary}

> **ε-greedy 只是雜訊；每種真正的探索策略都通過使未探索的事物看起來具有吸引力來發揮作用——無論是通過樂觀的初始值、動作選擇獎勵加成，還是自我生成的新奇獎勵——而正確的選擇取決於你的獎勵僅僅是 *稀疏* 的（比如在平地上尋找一個隱藏的獎品），還是真正 *深層* 的（比如需要一長串精確動作才能破解的組合鎖）。**
