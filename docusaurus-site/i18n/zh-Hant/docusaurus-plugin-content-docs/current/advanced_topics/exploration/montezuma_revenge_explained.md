# 在 Montezuma's Revenge（蒙特祖馬的復仇）上訓練 🏛️🔑

## 為什麼這款遊戲如此出名（在強化學習領域） {#why-this-game-is-famous}

2015 年，DeepMind 的 DQN 學會了從原始像素中以超越人類的水平玩幾十款 Atari 遊戲。這登上了新聞頭條。但結果表中埋藏著一款 DQN 得分為 **0** 的遊戲 —— 就像什麼都沒做一樣：**Montezuma's Revenge**。

為什麼？看看這款遊戲在第一個房間裡對你的要求：

1. 爬*下*梯子。
2. 穿過窗台。
3. 跳過一個滾動的頭骨（時機不對 → 你就死了）。
4. 爬*上*另一個梯子。
5. 拿到鑰匙。

這大約需要 **100 次精確的按鍵操作**，而在鑰匙到手之前，遊戲 **一分都不會給你**。在整個開場序列中，獎勵信號是一個平坦、毫無特徵的 **零**。

普通的強化學習智能體通過調整朝向它實際獲得的獎勵來學習。如果它能到達的任何地方獎勵都是零，那就*沒有學習的依據* —— 這就像試圖通過感受下坡方向來尋找一個完全平坦的山谷底部。所以 DQN 只是在起始平台上永遠地晃來晃去。Montezuma 成為了 **困難探索 (hard exploration)** 的基準：這款遊戲你只有通過*聰明地*探索而非隨機探索才能通關。

突破發生在 2018 年，隨著 **隨機網路蒸餾 (Random Network Distillation, RND)** 的出現 —— 其技巧正是工作項 1 的主題：添加一個 **內在好奇心獎勵 (intrinsic curiosity bonus)**，讓智能體在到達新畫面時獎勵*自己*，突然間就有了一個密集的信號引導它深入關卡。RND 在 Montezuma 上獲得了超越人類的分數。（後來還有：Go-Explore, Agent57, ……）

## 現實生活中「Montezuma 式」稀疏獎勵的例子 {#real-life-examples}

- **組合鎖 / 帶有隱晦線索的尋寶遊戲。** 沒有過程分。在突然拿到獎品之前，你的得分一直是零。
- **論文被接收，或新創公司實現獲利。** 幾個月沒有外部獎勵，然後（可能）有一個巨大的獎勵。
- **影片遊戲速通路線。** 連續進行幾十次影格精確的輸入，在技巧成功或失敗之前沒有任何回饋。
- **密室逃脫。** 在你把幾個發現串聯起來之前，房間幾乎不會告訴你任何資訊。

在所有這些情況中，「隨便試試」是毫無希望的。你需要*系統地*探索 —— 而內部的「噢，那是新東西，繼續前進」信號正是讓你保持系統性的關鍵。

## 為什麼我們在這裡不真的在像素級的 Montezuma 上訓練 {#why-not-pixel-level}

要妥善地完成*真實*的訓練意味著：

- 使用卷積網路來查看 210×160 的 RGB 畫面，
- 影格堆疊 (frame-stacking)（以便智能體可以辨別頭骨移動的方向），
- 一個 RND 模組（另外兩個網路：一個固定的隨機「目標」網路和一個訓練好的「預測」網路），
- 以及 **數千萬影格的環境交互** —— 這需要耗費大量的 GPU 執行時間。

那是一個研究專案，而不是教學腳本。所以 `montezuma_revenge.py` 做了兩件更實際的事情：

### 1. 它會*觸碰*一下真實的遊戲（如果安裝了 `ale-py`） {#touching-real-game}

它通過 Gymnasium 加載 `ALE/MontezumaRevenge-v5`，執行一個 **均勻隨機智能體 2000 步**，並報告遊戲總獎勵。它列印出的數字幾乎總是 **0.0** —— 將抽象的「稀疏獎勵」一詞變成了一個具體的、可以親自執行的事實。如果未安裝 Atari 套件，它會列印單行 `pip install` 命令並繼續。

### 2. 它在一個*比例模型* `MiniMontezumaEnv` 上訓練表格智能體 {#scale-model-training}

這是一個微小的網格世界，其*骨架*與 Montezuma 的第一個房間相同：

```
###############
#S....#.......#
#.....#.......#
#.....#...T...#     S = 起點 (start)
#.....D.......#     K = 鑰匙 (key)      D = 門 (door)（僅在攜帶鑰匙時可通過）
#..K..#.......#     T = 寶藏 (treasure)（唯一能提供獎勵的格子）
###############
```

要獲勝，你必須：走到 **鑰匙** 處（約 6 步），撿起它；走到 **門** 處（約 4 步） —— 門現在會打開；走過去到達 **寶藏** 處（約 5 步）。大約 **15 次完美的移動**，在拿到寶藏之前 **沒有任何回饋**。`has_key` 標誌是智能體狀態的一部分，所以一旦你抓住了鑰匙，就會有一整個新房間的狀態等待發現 —— 就像在真實遊戲中打開了新畫面一樣。

然後我們訓練兩次普通的 **表格 Q 學習器**：

| 智能體 | 在 MiniMontezuma 上的結果 |
|-------|--------------------------|
| **無好奇心 (epsilon-greedy)** | 在所有 1,500 個回合中，回報保持為 **0**。它甚至從未到達鑰匙處。（聽起來耳熟嗎？這就是真實遊戲中的 DQN。） |
| **帶預測誤差好奇心獎勵** | 在約 20-25 個回合內到達寶藏，然後學會了 **最佳的 15 步路線**。（這就是縮小到 Q 表大小的 RND 思想。） |

圖表並排顯示了兩條學習曲線，以及好奇智能體學會的實際路線，並在網格上繪製出來（起點 → 鑰匙 → 門 → 寶藏）。腳本還會以 ASCII 影格的形式列印該路線。

## 課程總結 {#lesson-summary}

> **「稀疏獎勵」不僅僅是一個奇怪的 Atari 遊戲的怪癖 —— 它是任何成功需要長距離、特定序列動作的世界中的預設狀態。** 一個僅靠獎勵驅動的智能體（如原生 DQN）簡直無法起步：沒有可以遵循的梯度。好奇心獎勵製造了一個梯度 —— 一個密集的、自我生成的「這是新東西，繼續前進」信號 —— 正是這個信號帶著智能體穿過零獎勵的荒漠，找到第一個真正的獎勵。之後的一切（RND, Go-Explore, Agent57）都是同一招式的擴展、神經網路版本。

## 關鍵術語 {#key-terms}

| 單字 | 含義 |
|------|---------|
| **Hard exploration (困難探索)** | 只有通過聰明地探索才能成功的問題；隨機探索會失敗 |
| **Sparse reward (稀疏獎勵)** | 獎勵在幾乎所有地方都是零；只有在完成一段很長的正確序列後才能獲得 |
| **Montezuma's Revenge** | 經典深度強化學習智能體（DQN, A3C）得分為 0 的 Atari 遊戲 —— 公認的困難探索基準 |
| **RND (隨機網路蒸餾)** | 2018 年利用預測誤差好奇心獎勵擊敗 Montezuma 的方法 |
| **Go-Explore** | 「記住有前途的狀態，回到那裡，然後從那裡開始探索」 —— 另一個 Montezuma 攻克者 |
| **Scale model (比例模型)** | 一個保留了難題*結構*的小型、廉價環境，以便你可以快速研究 |

---

## 一句話總結 {#summary}

> **Montezuma's Revenge 是一款讓強化學習界明白「從未收到的獎勵無法教給你任何東西」的遊戲 —— 而當時的以及現在的解決方法都是提供好奇心獎勵，讓智能體在找到真正的獎品之前通過探索來獎勵自己。**
