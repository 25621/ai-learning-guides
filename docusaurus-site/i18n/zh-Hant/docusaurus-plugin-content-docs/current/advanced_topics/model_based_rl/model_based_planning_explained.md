# 使用學習到的模型進行規劃 (MPC) 🔮

## 核心理念

你已經有了一個 **世界模型 (world model)**（一個可以預測未來的神經網路）。現在該怎麼辦？

最直接的用途是 **規劃 (planning)**：在每一時刻，詢問模型：「如果我嘗試*這個*計劃會發生什麼？*那個*計劃呢？或者是*另外一個*計劃？」 然後執行看起來最好的那個計劃 —— 但 **只執行其中的第一步**。因為模型並不完美，我們只執行一個動作，觀察真實環境回饋的實際新狀態，然後從頭開始重新規劃。

這個技巧有一個專門的名字：**模型預測控制 (Model Predictive Control, MPC)**。

---

## 現實生活類比

你在餐廳看菜單。你不會當場就定死五道菜判的順序 —— 你先點第一道菜，看看自己感覺有多飽，然後再決定甜點。

或者：你在一條彎曲的道路上開車。你不會鎖定接下來 30 秒的轉向輸入 —— 你不斷向前看，規劃未來幾秒鐘的路線，執行下一個轉向動作，然後重新規劃。

這種 **長遠規劃 / 近期執行 / 重新規劃** 的循環就是 MPC。

---

## 「隨機射擊 (Random Shooting)」 是如何運作的

還有更高級的規劃器 —— 例如：
- **CEM (交叉熵方法)**：通過每輪只保留前 k 個最佳計劃，迭代地改進計劃的分佈。
- **MCTS (蒙特卡洛樹搜索)**：由模擬統計資料引導構建搜索樹，被 AlphaGo 和 MuZero 使用。
- **基於梯度的規劃器**：對模型關於動作的預測進行求導，直接遵循梯度。

我們使用的是最簡單但有效的方法：**隨機射擊 (random shooting)**。

```
給定當前狀態 s：
    1. 取樣 N=200 條長度為 H=15 的隨機動作序列。
    2. 對於每條序列，通過世界模型從 s 開始模擬，累加每一步的塑形獎勵（並行進行 200 個夢想 —— 速度極快！）。
    3. 找到預測總獎勵最高的序列。
    4. 在真實環境中執行該序列的「第一步」動作。
    5. 觀察真實環境的下一個狀態。丟棄計劃的剩餘部分。
    6. 回到步驟 1 —— 從頭開始重新規劃。
```

200 個計劃 × 15 步 = 每一個真實步驟對應 3,000 次想像的轉移。世界模型在單次神經網路批次前向傳遞中執行所有這些 —— 通常只需幾毫秒。

---

## 為什麼要每一步都重新規劃？

因為模型是不完美的。誤差會在展開 (rollout) 過程中累積（正如 `world_model.py` 生成的圖表 `outputs/world_model.png` 所顯示的）。在第 0 步制定的計劃僅在前幾步是可靠的；到第 15 步時，模型可能就在「產生幻覺」了。因此，我們只信任 **第一步動作**，然後根據最新的真實狀態刷新計劃。

這也是人類不會制定一個 100 步的西洋棋計劃並死守不放的原因 —— 情況在變，你預測得越遠，它與現實的契合度就越低。

## 一個小細節：獎勵必須告訴規劃器一些資訊

在 CartPole 中，真實的獎勵是每一步 `+1`，直到桿子倒下。模型會對幾乎每個計劃都忠實地預測 `+1, +1, +1, ...`，因為隨機計劃在模型內部很少會很快結束 —— 這樣每個計劃的得分都一樣。規劃器無法從中做出選擇。

解決方法：在規劃過程中將真實獎勵替換為 **平滑的代理獎勵 (smooth proxy)**：

```python
reward_proxy(state) = 1
                    - |桿子角度| / 0.21          # 桿子豎直嗎？ (1=是)
                    - 0.1 * |小車位置| / 2.4     # 小車在中心嗎？ (1=是)
```

現在，*可能*導致桿子倒下的計劃得分會明顯低於保持豎直的計劃。規劃器就可以對它們進行排名。

> **現實生活的教訓：** 扁平的獎勵信號 —— 「你又多活了一秒」 —— 對於短期規劃毫無用處。密集的、塑形過的信號更有幫助。

## 我們的程式碼做了什麼

`model_based_planning.py`：

1. **載入** `world_model.py` 保存的世界模型權重。（如果缺失，它會現場重新訓練一個。）
2. 在真實的 CartPole-v1 上 **執行 20 個回合** 的 MPC。
3. 同時 **執行 20 個回合** 的均勻隨機策略作為基準。
4. 將兩者並排 **繪圖** 並列印平均值。

### 執行後你應該看到的結果

| 策略 | 平均獎勵 (生存步數) |
|--------|-------------------------------:|
| 隨機 (Random) | ~22 (CartPole 的典型值 —— 桿子很快倒下) |
| MPC (我們的) | ~150–500 (隨種子變化；許多回合接近 500) |
| 最大可能值 | 500 |

這種 **5–25 倍的提升** 是在沒有策略網路、沒有價值函數且沒有進一步訓練的情況下實現的。僅僅是一個世界模型 + 每步 200 個夢想。

圖表 `outputs/model_based_planning.png` 顯示了每個回合的兩個彩色條 —— MPC 幾乎總是比隨機策略高，且許多回合達到了 500 步的上限。

## 基於模型的規劃的優勢

- **樣本效率高。** 所有的學習都是從一批隨機轉移中完成的。不需要進一步的環境交互即可得出有用的策略。
- **易於重新設定目標。** 想以不同的方式控制代理人？更改獎勵代理即可 —— 無需重新訓練。（為了好玩，可以嘗試最大化小車速度。）
- **可解釋性強。** 你可以檢查代理人考慮過的計劃、預測的軌跡以及得分。

## 局限性（以及人們如何應對）

- **隨機射擊很笨。** 它是盲目取樣計劃。對於更高維度的空間，可以切換到 **CEM**（見上文）或 **iLQR**（迭代線性二次型調節器，一種經典的優化控制方法，將模型近似為局部線性並解析求解），或者使用 **基於梯度的規劃器** 通過可微分模型遵循梯度來改進動作。
- **複合模型誤差。** 長視野會產生漂移。人們使用 **概率系綜 (probabilistic ensembles)**（在相同資料上訓練的多個模型，如 PETS，Chua 等人，2018），這樣規劃器可以注意到模型之間的分歧，並懲罰模型不確定的計劃。
- **我們最終想要的是真實獎勵。** 獎勵塑形有所幫助，但對於更複雜的任務，人們會在世界模型*內部*訓練一個 **價值函數 (value function)** —— 一個學習過的評論家，用於估算任何狀態的長期回報，而無需手工構建代理獎勵。**Dreamer**（完全在潛空間想像中訓練執行者-評論家）和 **MuZero**（將 MCTS 與學習到的價值網路結合）都使用了這一思想。

## 這與現代系統的聯繫

你剛剛執行的精確配方 —— **學習到的動力學 + 規劃** —— 是現代人工智慧研究中一些最強強化學習系統的支柱：

- **MuZero** (DeepMind)：結合了學習到的世界模型和蒙特卡洛樹搜索。它在不需要預先知道規則的情況下精通了圍棋、象棋、將棋和 Atari 遊戲。
- **Dreamer / DreamerV3** (Hafner 等人)：在學習到的 **潛空間 (latent-space)** 世界模型中訓練策略（這意味著模型在預測未來之前將原始圖像或狀態壓縮成緊湊、抽象的表示）。它在 100 多個基準測試中達到了最先進的 (SOTA) 性能。
- **PETS / PlaNet / TD-MPC**：這些演算法家族（研究路線）將這一想法擴展到複雜的連續控制任務，如機器人。

你已經用幾百行程式碼構建了這個家族中最袖珍的一員。

## 關鍵術語

| 術語 | 通俗解釋 |
|------|---------------|
| **MPC** | 模型預測控制 —— 提前計劃，執行一步，重新計劃 |
| **Random shooting (隨機射擊)** | 對大量隨機計劃打分，選最好的那個 |
| **Horizon (H, 視野)** | 計劃向前看多少步 |
| **N samples (N 個取樣)** | 每步考慮多少個備選計劃 |
| **Receding horizon (退避視野)** | 每一步都重新規劃，而不是死守一個計劃 |
| **Reward proxy / shaping (獎勵代理/塑形)** | 一個平滑的替代獎勵，為規劃器提供有用的優化信號 |

## 一句話總結

> **一旦你有了世界模型，規劃就只是「做一百個關於未來的夢，選最好的第一步，然後重複執行」。**
