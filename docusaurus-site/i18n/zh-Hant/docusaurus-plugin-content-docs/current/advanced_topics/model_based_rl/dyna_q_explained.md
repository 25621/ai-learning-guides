# Dyna-Q：通過想像學得更快 🧠

## 什麼是 Dyna-Q？

想像一個名叫 Mia 的孩子正在熟悉她的新學校。每天她在走廊裡走動並發現新事物：「圖書館在食堂後面」，「史密斯先生的房間在樓梯口附近」。

一個 **普通的 Q 學習** 學生只從她 *今天* 所做的事情中學習。如果她今天只是從教室走到食堂，她就只更新關於那一條路徑的記憶。

而一個 **Dyna-Q** 學生則不同。每次在真實的走動之後，她都會坐下來花一分鐘 **在腦海中回放** 她記住的幾次過去的走動。每一次回放都會加強她的心理地圖。幾週後，她對學校了如指掌——不是因為她走得更多，而是因為她 **對自己已經看到的東西思考得更多**。

這就是 Dyna-Q 對強化學習代理人所做的：它既從真實的經驗中學習，也從它在過程中建立的模型所產生的 **想像經驗** 中學習。

---

## 三個要素

Dyna-Q 是 「Q 學習 + 模型 + 規劃」。一個真實的步驟完成了 **三項** 工作：

1. **直接強化學習 (Direct RL)** —— 從 `(s, a, r, s')` 進行常規的 Q 學習更新。
2. **模型學習 (Model learning)** —— 記錄下來：「當我在 `s` 下執行 `a` 時，我得到了 `r` 並到達了 `s'`。」
3. **規劃 (Planning)** —— 從模型的記憶中隨機選取 *n* 個 `(s, a)` 對，並進行 *n* 次額外的 Q 學習更新，**假裝** 這些步驟剛剛發生過。

第三步就是魔法所在。如果 `n = 50`，世界上每一個真實的步驟都會導致對 Q 表的 **51 次更新**。代理人的學習速度比純粹的 Q 學習者快約 50 倍（按真實步驟衡量）。

## 循環圖解

```
                   ┌────────────────────────────────────┐
                   │                                    │
   真實世界  ──► 執行動作 a ──► 觀察 (r, s')             │
                            │                           │
              ┌─────────────┼──────────────┐            │
              ▼             ▼              ▼            │
        Q 學習更新      Model[s,a] ← (r,s')    規劃 ────┘
                                           (n 次想像更新)
```

模型只是一個查找表：
`(狀態, 動作) → (獎勵, 下一狀態)`。構建成本低，查詢成本也低。

## 現實生活中的例子

- **西洋棋研究**：特級大師們花數小時在腦海中回放他們自己的對局和大師對局。每一次回放都是「規劃」 —— 從已經發生的經歷中額外學習。
- **音樂家練習音階**：在演奏完一個棘手的小節後，他們在繼續之前會在腦海中默演十次。手指沒有動，但大腦正在更新。
- **自動駕駛汽車**：在等紅燈閒置時，它會在模擬中回放最後一百次切換車道，以便在不磨損輪胎的情況下微調其策略。

## 我們的程式碼做了什麼

我們使用經典的 **Dyna 迷宮 (Dyna Maze)**（[Sutton & Barto, 圖 8.2](http://incompleteideas.net/book/the-book.html)）：一個帶有圍牆的 6×9 網格，起點 `S` 在左中位置，終點 `G` 在右上角。

我們執行了三個變體，每個變體在 30 個隨機種子上取平均值：

| 設置 | 每步真實動作對應的規劃步數 | 含義 |
|---------|------------------------------|---------|
| `n = 0` | 0 | 普通 Q 學習 |
| `n = 5` | 5 | 少量的想像練習 |
| `n = 50` | 50 | 大量的想像練習 |

腳本記錄了隨訓練進行 **每回合平均真實步數**。步數越少意味著代理人學會了通往目標的更直接路徑。

### 執行程式碼時你應該看到的結果

迷宮中最短路徑約為 9 步；通過 ε-貪婪探索，訓練良好的代理人平均每回合約 10 步。執行 50 個回合後，所有三種設置都會收斂到此結果 —— 區別在於 **速度**：

| 設置 | 每回合步數 (最後 10 回合) | 意味著什麼 |
|---------|--------------------------------:|---------------|
| `n = 0`   | ~10 | 已收斂 —— 但它花費了約 30-50 個回合的徘徊才到達這裡 |
| `n = 5`   | ~10 | 在約 10 個回合內收斂 |
| `n = 50`  | ~10 | 在約 3-5 個回合內收斂 |

有趣的信號是 *學習曲線*，而不是最終的數字。保存在 `outputs/dyna_q.png` 中的圖表顯示了三條以非常不同速率下降的曲線：`n = 50` 在寥寥幾個回合內就達到了底部，而 `n = 0` 在執行了很久之後仍在緩慢下降。（在這樣一個微小的確定性迷宮中，普通 Q 學習最終確實能達到目標 —— Dyna-Q 只是需要少得多的真實回合，這在真實步驟成本高昂的環境中非常有意義。）

## 為什麼它在這個迷宮中效果這麼好

有兩個原因：

1. **環境是確定性的**：每個 `(s, a)` 總是給出相同的 `(r, s')`，所以模型在訪問一次後就是準確的。想像的經驗和真實的經驗一樣好。
2. **真實步驟很昂貴，想像步驟是免費的**：每一次想像的更新只是幾次查表，而真實的步驟需要代理人去行走。當真實交互成本高昂時（想想：真實的機器人，真實的遊戲），Dyna-Q 具有極高的樣本效率。

---

## Dyna-Q 的局限性

- **隨機性環境**：如果 `(s, a)` 可以導致許多不同的 `s'` 值，那麼「記住最後結果」的模型就會誤導你。解決方法：存儲訪問計數或訓練概率模型。
- **非平穩環境**：如果世界發生了變化 —— 例如，原本開啟的門口突然關閉，或者出現了捷徑 —— 模型就會過時並給出錯誤的預測。**Dyna-Q+** 通過添加 **探索獎勵** 解決了這個問題：長時間未訪問的狀態會獲得少量額外獎勵，促使代理人回去檢查世界是否發生了變化。
- **大型狀態空間**：以 `(s, a)` 為鍵的字典無法擴展到數百萬個狀態或連續狀態。這正是 **學習到的（神經網路）世界模型** 所填補的空白 —— 參見接下來的 `world_model.py`。

## 關鍵術語

| 單字 | 含義 |
|------|---------|
| **模型 (Model)** | 對 `(狀態, 動作) → (獎勵, 下一狀態)` 的記憶 |
| **規劃步 (Planning step)** | 使用來自模型的想像資料進行 Q 更新 |
| **直接強化學習 (Direct RL)** | 使用真實資料進行 Q 更新 |
| **樣本效率 (Sample efficiency)** | 衡量 AI 模型或演算法有效利用訓練資料以達到特定性能水平的能力 |
| **Dyna** | Sutton 提出的將學習和規劃交織在一起的架構 |

## 一句話總結

> **Dyna-Q 既從實踐中學習，也從想像中學習 —— 而想像是免費的。**

這一思想在其現代神經形式中，驅動了一些有史以來最強大的強化學習代理人 (MuZero, Dreamer, World Models)。
