# 訓練世界模型：教會智能體「做夢」 🌍

## 什麼是「世界模型」？

**世界模型 (World Model)** 是智能體對*宇宙的內部拷貝*。給它一個狀態和一個動作，它就能預測下一步會發生什麼：

```
(狀態, 動作)  ──►  神經網路  ──►  (下一個狀態, 獎勵)
```

它不是真實世界 —— 它是智能體透過觀察現實並學習模仿現實，為自己構建的一個**模擬器**。

一旦訓練完成，模型就能讓智能體在不採取任何實際行動的情況下詢問「如果……會怎樣」的問題：

> *「如果我現在向左推，然後連續向右推兩次，我會到哪裡？桿子會倒嗎？」*

智能體在模型內部思考一百個計劃的時間，可能還不夠在現實中做出一次移動。這就是其核心意義所在。

---

## 現實生活中的類比

想想*你*是如何解謎題的。你不會體力活似地把每一塊拼圖嘗試塞進每一個槽位。你會**想像**如果拼圖 A 放在這裡會發生什麼。如果腦海中的模擬看起來不對，你在動手之前就會否定它。

你的大腦擁有一套習得的世界模型 —— 建立在多年對物體行為觀察的基礎上 —— 這讓你在付出行動前可以模擬結果。

其他例子：

- **棋手**會在腦海中預演後面幾步。
- **司機**會想：「如果我現在煞車，後面的車是否有足夠的空間。」
- **孩子**堆積木：「如果我把大的放在上面，塔會晃動。」（他們透過以前撞倒積木塔的經驗學會了這一模型。）

在任何情況下，**心理模型 + 想像力 = 更少的風險、更好的決策**。

## 智能體如何構建它的模型？

它只是透過**觀察**。具體來說：

1. **收集數據**。讓任何策略（甚至是隨機策略）與真實環境交互一段時間。保存每一次轉換：
   ```
   (狀態, 動作, 獎勵, 下一個狀態)
   ```
2. **訓練神經網路**，根據 `(狀態, 動作)` 預測 `下一個狀態` 和 `獎勵`。這是監督式學習：每一個保存的轉換都是一個帶標籤的範例，輸入是「智能體看到的和做的」，標籤是「接下來實際發生的」。
3. **驗證**。預留 10% 的數據，檢查模型的預測與真實情況的差異。誤差低意味著模型已經掌握了環境的**動力學 (dynamics)**：即狀態在動作後如何變化。

我們使用的一個技巧：與其直接預測 `下一個狀態`，不如預測**增量 (delta)** `下一個狀態 − 當前狀態`。大多數物理現象是增量式的（「小車移動了一點點」），小目標對神經網路更友好。

## 我們的設置

| 選擇 | 數值 | 理由 |
|--------|-------|-----|
| 環境 | `CartPole-v1` | 4 維狀態，2 種動作 —— 易於建模 |
| 數據 | 來自隨機策略的 20,000 次轉換 | 對狀態空間有廣泛的覆蓋 |
| 網絡 | MLP, 2 × 128 ReLU 隱藏層 | MLP = 多層感知機（標準「原生」神經網路）。兩個 128 神經元的隱藏層，使用 ReLU 激活函數。容量足夠且訓練快。 |
| 損失函數 | `(狀態增量, 獎勵)` 的 MSE | MSE = 均方誤差（預測誤差平方的平均值）。標準的回歸損失函數。 |
| 優化器 | Adam, lr = 1e-3, 30 個 epochs | Adam = 自適應優化器（自動調整每個參數的學習率）。現成可用，無需特殊調優。 |

整個訓練在 CPU 上幾秒鐘內即可完成。

## 「好」模型是什麼樣的？

有兩個評估指標至關重要：

### 1. 單步準確率（驗證集 MSE）

這是指「模型預測未來一局的表現如何？」在 30 個 epochs 之後，你應該能看到驗證集 MSE 在 **1e-4 到 1e-3** 範圍內。這非常微小 —— 意味著桿的角度和小車的位置準確到了小數點後幾位。

### 2. k 步展開中的**複合誤差 (Compounding error)**

這是*真正的*測試。獲取一個狀態，將其輸入模型，然後獲取其預測值並再次輸入模型 —— 連續進行 `k` 步。誤差會增長，因為每一步都會在前一次預測的基礎上增加一點噪聲。

```
第 1 步： L2 誤差 ≈ 0.01 （幾乎完美）
第 5 步： L2 誤差 ≈ 0.05
第 10 步：L2 誤差 ≈ 0.15
第 20 步：L2 誤差 ≈ 0.40 （明顯偏離）
```

*(L2 誤差 = 預測的下一個狀態與真實狀態之間的歐幾里得距離 —— 可以理解為「模型在 4 維狀態空間中的猜測偏離了多少？」)*

**為什麼這很重要**：如果我們用模型規劃未來 15 步，第 15 步的*確切*狀態可能是錯的 —— 但如果「好計劃 vs 壞計劃」的相對排名得以保持，規劃依然有效。（這就是 `model_based_planning.py` 所利用的特性。）

`outputs/world_model.png` 中的圖表並排顯示了這兩個診斷結果：訓練損失曲線在對數座標下漂亮地下降，而展開誤差曲線則穩步上升。

## 為什麼要預測「增量 (Delta)」？

對比向網絡描述同一個問題的兩種方式：

| 目標 | 典型量級 | 容易還是困難？ |
|--------|------------------:|--------------|
| `下一個狀態`        | 0–2.4 (小車位置) | 網絡必須復現位置**以及**微小的變化 |
| `下一個狀態 - 狀態` | ~0.02            | 網絡只需學習微小的變化 |

預測增量還意味著：如果網絡輸出全零（就像未經訓練的初始網絡經常做的那樣），預測結果僅僅是「沒有移動」 —— 這對於單一步伐來說是一個合理且安全的默認值。相比之下，直接預測絕對的 `下一個狀態` 最初會輸出完全隨機的垃圾數值，導致早期訓練高度不穩定。

## 這能帶給我們什麼？

訓練好的世界模型是以下各項的基礎：

- **規劃 (Planning)** —— 在想像的動作序列中進行搜索（參見 `model_based_planning.py`）。
- **Dyna 風格的增強** —— 在想像的數據上訓練 Q 網絡，成倍提高樣本效率。
- **好奇心/探索** —— 訪問模型無法很好預測的狀態。
- **Dreamer / World-Models 論文** —— 除了最初的數據收集外，完全在模型內部訓練*策略*，與現實世界零交互。

## 局限性與注意事項

- **分佈外漂移 (Out-of-distribution drift)**：模型只了解它見過的世界。如果規劃得太激進，你可能會進入模型從未見過的區域 —— 那裡的預測純屬幻想。
- **複合誤差**：正圖表所示，由於誤差累積，在長**時界 (horizons)**（預測未來的許多步）內進行規劃是不可靠的。現代系統透過使用**概率集成 (probabilistic ensembles)**（訓練多個模型並檢查它們是否達成一致，如 PETS 或 Dreamer）來解決這個問題，這樣規劃器就能準確知道模型在每一步的*不確定程度*，從而避開風險較大的未知路徑。
- **隨機環境**：標準的確定性回歸器只能預測*平均*結果，完全遺漏了可能結果的分佈。複雜的現實環境需要概率模型（如具有高斯輸出的模型，或**隱隨機模型 (latent stochastic models)** —— 將世界狀態編碼為壓縮空間中的概率分佈的網絡，使它們能夠捕捉真正的隨機性而非將其平均掉）來準確表示不確定性和隨機性。

## 核心詞彙

| 術語 | 通俗解釋 |
|------|---------------|
| **世界模型 (World model)** | 模仿環境的神經網路 |
| **動力學 (Dynamics)** | 函數 `(s, a) → s'` |
| **獎勵模型 (Reward model)** | 函數 `(s, a) → r`（通常綁定在一起） |
| **單步預測** | 模型根據真實狀態輸出的結果 |
| **展開 (Rollout)** | 連續的單步預測，將輸出反饋作為輸入 |
| **複合誤差 (Compounding error)** | 在展開過程中不斷增長的微小誤差 |
