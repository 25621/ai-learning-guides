# Чувствительность гиперпараметров PPO: что важнее всего?

## Почему важны гиперпараметры

Представьте, что вы печете шоколадный торт. В рецепте указано:
- 2 яйца
- 200 г муки
- 1 чайная ложка разрыхлителя
- 35 минут при 180°C

Если вы используете 10 яиц, торт «взорвется». Если 0.1 чайной ложки разрыхлителя — он не поднимется. Если печь при 300°C 10 минут — он сгорит снаружи и останется сырым внутри.

**Гиперпараметры в PPO похожи на ингредиенты и настройки духовки.** Правильное сочетание работает прекрасно; неверные настройки могут полностью сорвать обучение.

Этот скрипт систематически тестирует 3 ключевых гиперпараметра, меняя только ОДИН за раз, запуская каждую настройку с 3 разными случайными семенами (seeds) и сравнивая результаты.

---

## Три эксперимента

### Эксперимент 1: Clip Epsilon (ε)

```
ε = 0.05   (очень консервативно — разрешены только крошечные изменения политики)
ε = 0.2    (стандарт — баланс безопасности и скорости)
ε = 0.4    (агрессивно — допускаются большие изменения политики)
```

**Что контролирует ε?**

ε — это размер «окна безопасности» вокруг старой политики:
```
отношение должно оставаться в [1 - ε,  1 + ε]
ε=0.05: отношение в [0.95, 1.05]  ← крошечные изменения
ε=0.2:  отношение в [0.80, 1.20]  ← стандарт  
ε=0.4:  отношение в [0.60, 1.40]  ← большие изменения
```

**Пример из жизни:** Представьте ε как «насколько далеко вам разрешено повернуть руль за один раз».
- ε=0.05: Как езда по льду — только крошечные корректировки.
- ε=0.2:  Обычное вождение — разумные повороты.
- ε=0.4:  Гонщик — агрессивное руление, риск **заноса** (потери контроля из-за слишком резкого изменения).

**Ожидаемые результаты:**
- ε=0.05: Медленное, но стабильное обучение (слишком осторожно).
- ε=0.2:  Хороший баланс (значение **«Златовласки»** — не слишком маленькое, не слишком большое, а в самый раз).
- ε=0.4:  Может учиться быстро, но может **проскочить цель и начать колебаться** (overshoot — проскочить оптимальную политику; oscillate — колебаться вокруг нее, не останавливаясь).

---

### Эксперимент 2: Скорость обучения (Learning Rate)

```
lr = 1e-4  (медленно, но стабильно)
lr = 3e-4  (стандарт)
lr = 1e-3  (быстро, но рискованно)
```

**Что контролирует скорость обучения?**

Скорость обучения подобна «размеру шага» при подъеме на гору (каждый шаг — одно обновление весов нейронной сети в направлении улучшения награды):
- Слишком маленькая: целая вечность, чтобы добраться до вершины (медленная сходимость).
- Слишком большая: вы перепрыгнете пик и упадете с другой стороны (**расходимость** — награда падает или дико колеблется).
- В самый раз: уверенный прогресс к вершине.

**Пример из жизни:** Настройка гитарной струны.
- lr=1e-4: Крошечные повороты **колка** (ручки настройки) — очень долго, но точно.
- lr=3e-4: Обычная настройка — находим нужную высоту за несколько поворотов.
- lr=1e-3: Сильные **рывки** колка — можно **порвать** струну (необратимо сломать процесс обучения)!

**Ожидаемые результаты:**
- lr=1e-4: В конечном итоге хороший результат, но очень медленно.
- lr=3e-4: Лучшая производительность в целом.
- lr=1e-3: Быстрый начальный прогресс, затем нестабильность.

---

### Эксперимент 3: Эпохи обновления (K)

```
K = 3   (консервативно — мало проходов по каждому пакету)
K = 10  (стандарт)
K = 20  (агрессивно — много проходов по каждому пакету)
```

**Что контролируют эпохи обновления?**

После сбора опыта (**rollout** — игра в течение некоторого времени для сбора новых данных), PPO упаковывает этот опыт в **пакет** (batch). Затем он делает K **проходов** (полных циклов по пакету, обновляя сеть при каждом проходе) по этим данным.
Больше эпох = выжать больше обучения из каждого пакета, но риск **переобучения под устаревшие данные** (overfitting — запоминание паттернов, которые были верны для старой политики, но уже не актуальны для новой).

**Пример из жизни:** Студент тренируется на наборе из 20 задач по математике.
- K=3:  Решить каждую задачу 3 раза → обучение продолжается, нет переобучения под конкретные примеры.
- K=10: Решить 10 раз → прочное освоение этих конкретных задач.
- K=20: Решить 20 раз → **зубрежка решений без понимания самой математики** (модель идеально подстраивается под конкретный пакет, но теряет способность к обобщению)!

> ⚠️ **«Но для K=20 результаты выглядят нормально — почему это должно меня волновать?»**
> Прием PPO с ограничением (clipping) сдерживает изменение политики за один проход, поэтому K=20 не вызовет внезапного краха. Однако агент начинает незаметно перестраиваться под данные, которые больше не отражают реальный опыт текущей политики. Это **замедляет долгосрочное обучение**: каждый сбор данных учит агента меньшему, чем мог бы, потому что последние проходы используют все более устаревшую информацию. Ущерб постепенный, а не резкий — именно поэтому его легко не заметить в коротких экспериментах.

**Ожидаемые результаты:**
- K=3:  Менее эффективно (часть потенциала обучения теряется).
- K=10: Хороший баланс.
- K=20: Риск того, что политика станет **слишком уверенной на основе устаревших данных** (эффективность выборки снижается).

## Как читать результаты

График показывает три панели, на каждой из которых меняется один гиперпараметр:

```
Левый график:    Clip Epsilon — какой ε учится быстрее всего?
Средний график:  Learning Rate — какой lr самый стабильный?
Правый график:   Update Epochs — какой K находит лучшую политику?
```

Каждая линия — это **средняя награда по 3 семенам** (для уменьшения влияния случайности).

**На что обращать внимание:**
1. **Скорость обучения:** Какая линия быстрее достигает высокой награды?
2. **Конечный результат:** Какая линия достигает наивысшей итоговой награды?
3. **Стабильность:** У какой линии меньше всего колебаний?

Хороший гиперпараметр балансирует все три показателя!

## Методология: научный эксперимент

Этот эксперимент использует дизайн **абляционного исследования** (ablation study — метод, при котором вы удаляете или меняете один компонент за раз, чтобы измерить его индивидуальное влияние):
1. Выбираем значения по умолчанию: ε=0.2, lr=3e-4, K=10
2. Меняем только ОДИН параметр за раз
3. Оставляем все остальное неизменным
4. Сравниваем результаты

Это говорит нам об эффекте КАЖДОГО параметра в отдельности.

**Пример из жизни:** Проверка, помогает ли новое удобрение растениям:
- Меняем удобрение, оставляем все остальное прежним (та же почва, полив, свет).
- Если растения растут лучше → удобрение помогло!

## Типичные выводы на практике

| Гиперпараметр | Слишком мало | Оптимально | Слишком много |
|---------------|--------------|------------|---------------|
| **ε (clip)** | Медленная сходимость | ε ≈ 0.2 | Нестабильность |
| **lr** | Слишком медленно | 2.5e-4 - 3e-4 | Расходимость (Divergence) |
| **K (эпохи)** | Потеря потенциала данных | K = 4-10 | Переобучение на старых данных |
| **n_steps** | Слишком шумно | 128-2048 | Ошибки памяти (OOM) |
| **batch_size** | Слишком шумно | 32-256 | Ошибки памяти (OOM) |

## Ключевой инсайт: PPO относительно надежен

По сравнению с более ранними алгоритмами (такими как DQN без целевых сетей), PPO относительно устойчив к выбору гиперпараметров. Механизм ограничения (clipping) обеспечивает естественную «сетку безопасности».

**Пример из жизни:** Автомобиль с **ABS** (антиблокировочной системой тормозов) и без нее:
- Без ABS (DQN): Один неверный поворот (плохой гиперпараметр) — и вас заносит.
- С ABS (PPO): Машина корректирует себя — разумные гиперпараметры в целом работают нормально.

Эта надежность — главная причина, по которой PPO является самым популярным алгоритмом RL на практике!

## Ключевые термины

| Термин | Простыми словами |
|---------|---------------|
| **Ablation study** | Изменение одной вещи за раз для оценки её влияния |
| **Clip epsilon ε** | Граница безопасности — 0.2 обычно лучше всего |
| **Learning rate** | **Размер шага** — насколько сильно корректируются веса сети |
| **Update epochs K** | Сколько раз повторно использовать каждый пакет данных |
| **Random Seeds** | Повторение эксперимента с разными **случайными числами** для проверки стабильности |

## Резюме: Методы градиента политики

```
REINFORCE              A2C                    PPO
     │                  │                      │
Полные эпизоды     N-шаговые обновления   N-шаговое + clipping
Просто, но шумно   Быстрее, но нестабильно Стабильно + эффективно
Для простых сред   Средняя сложность      Сложные среды
                                          (стандарт индустрии)
```

**Если вы изучите только ОДИН алгоритм на этом этапе, пусть это будет PPO.** Он лежит в основе ChatGPT, современных роботов и игровых ИИ.
