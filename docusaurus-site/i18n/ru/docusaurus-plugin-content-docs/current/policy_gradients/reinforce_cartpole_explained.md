# REINFORCE: Обучение робота принимать лучшие решения

## Что мы пытаемся сделать?

Представьте, что у вас есть робот, играющий в видеоигру. Каждую секунду робот должен выбирать:
**«Должен ли я нажать кнопку или нет?»**

Вместо того чтобы запоминать каждую ситуацию в таблице (как в Q-learning), мы хотим, чтобы робот выучил **рецепт** — набор правил, который прямо говорит: «В этой ситуации делай это действие».

Этот рецепт называется **политикой** (π, пи). В обучении с подкреплением π означает «правило выбора действий».

---

## Старый путь против нового

**Старый путь (Q-learning / DQN):** Изучить, насколько ХОРОШО каждое действие (Q-значения), а затем выбрать лучшее.
> «Нажатие ВЛЕВО имеет оценку 7, нажатие ВПРАВО — 5 → жмем ВЛЕВО!»

**Новый путь (Градиент политики):** Напрямую изучать, какое действие ВЫБРАТЬ.
> «Когда шест наклоняется вправо, жми ВПРАВО с вероятностью 80%, жми ВЛЕВО с вероятностью 20%».
*(Слово **Градиент** относится к математическому «шагу», который мы предпринимаем для постепенной корректировки этих вероятностей в нужном направлении).*

**Пример из реальной жизни:** Обучение езде на велосипеде.
- Старый путь: рассчитать *точную оценку* для «наклон влево на 5 градусов» против «наклона влево на 7 градусов».
- Новый путь: просто практиковаться, пока ваше **тело** не научится — нажимать на ту педаль, которая кажется правильной в этот момент!

## Как работает REINFORCE?

REINFORCE наблюдает за тем, как робот играет полную игру от начала до конца (один **эпизод**), а затем спрашивает: «Какие действия привели к хорошему результату? Давайте делать их чаще!»

### Шаг за шагом

**1. Играем эпизод**

Робот делает выбор и собирает опыт:
```
Шаг 1: Состояние = [шест наклоняется вправо] → Действие = жмем ВПРАВО → Награда = +1
Шаг 2: Состояние = [шест почти сбалансирован] → Действие = жмем ВПРАВО → Награда = +1
Шаг 3: Состояние = [шест наклоняется влево]   → Действие = жмем ВЛЕВО  → Награда = +1
...
Шаг 47: Состояние = [шест упал!] → Эпизод окончен
```

**2. Вычисляем доходы (Returns)**

Для каждого шага вычисляем G_t — **общую награду с этого момента и до конца игры**:
```
G на шаге 47 = 1
G на шаге 46 = 1 + 0.99 × 1 = 1.99
G на шаге 45 = 1 + 0.99 × 1.99 = 2.97
...
G на шаге 1  = 47 (примерно — более высокий доход, так как это начало игры)
```

**Коэффициент дисконтирования** γ = 0.99 означает, что награды в далеком будущем ценятся чуть меньше.

**Пример из реальной жизни:** Получить золотую звезду в первый день занятий в школе приятнее, чем знать, что вы *можете* получить её на сотый день. Будущие награды немного «дисконтируются».

**3. Обновляем политику**

Для каждого предпринятого действия:
> Если G_t был ВЫСОКИМ (это действие привело к отличному результату): **делай это чаще!**
> Если G_t был НИЗКИМ (это действие привело к плохому результату): **делай это реже!**

Математически: `потери (loss) = -log_prob(action) × G_t`

Вычисление градиента и обновление политики подобно тому, как если бы мы сказали роботу:
*«То действие, которое ты совершил на шаге 20? В следующий раз делай его на 5% чаще!»*

## Что такое сеть политики (Policy Network)?

Вместо таблицы мы используем **нейронную сеть** для представления политики.

```
Наблюдение          Сеть политики         Вероятности действий
[поз. тележки]  →   [128 нейронов]  →  →  [жать ВЛЕВО:  30%]
[скор. тележки] →   [128 нейронов]        [жать ВПРАВО: 70%]
[угол шеста]    →
[скор. шеста]   →
```

Сеть выдает **вероятности** для каждого действия. Затем мы делаем выбор:
> Бросаем кубик → 1-30: жмем ВЛЕВО, 31-100: жмем ВПРАВО

**Пример из реальной жизни:** Погодное приложение говорит: «Вероятность дождя 70%». Вы не ЗНАЕТЕ наверняка, пойдет ли дождь, — вы принимаете решение на основе вероятности. Робот делает то же самое!

## Нормализация: почему мы вычитаем среднее

Перед использованием G_t для обновления мы проводим нормализацию:
```
G_normalized = (G - mean(G)) / std(G)
```

**Почему?** Представьте, что все награды положительные (как в CartPole — всегда +1 за шаг). Без нормализации КАЖДОЕ действие кажется «хорошим», и сигнал обновления становится запутанным.

После нормализации некоторые доходы становятся положительными (выше среднего → жмем чаще), а некоторые — отрицательными (ниже среднего → жмем реже). Сигнал становится гораздо чище!

**Пример из реальной жизни:** Ваш учитель выставляет оценки по кривой распределения. Если средний балл — 70, а вы получили 85 — это отлично! Но если средний балл — 90, а вы получили 85 — это ниже среднего. Сама по себе оценка не дает полной картины.

---

## Проблема: Высокая дисперсия (Variance)

У REINFORCE есть серьезная слабость: **дисперсия**. Значения доходов G_t очень зашумлены.

**Пример из реальной жизни:** Представьте, что вы судите о шеф-поваре, попробовав всего ОДНО блюдо из каждого его ресторана. Иногда у повара был плохой день, иногда продукты подвели. Одной трапезы недостаточно, чтобы надежно узнать, хорош ли ресторан!

REINFORCE ждет завершения ПОЛНОГО эпизода перед обновлением. Один эпизод может быть очень удачным, другой — очень неудачным. Градиенты «скачут» во все стороны.

Вот почему кривая обучения (на графике) выглядит зубчатой:
- Некоторые запуски дают 500 (потрясающе!)
- Некоторые запуски падают до 50 (ужасно!)

Несмотря на шум, REINFORCE в конечном итоге обучается — но это требует терпения.

## Результаты

```
Эпизод  100 | Ср. награда (последние 100):  43.1
Эпизод  200 | Ср. награда (последние 100): 193.9
Эпизод  500 | Ср. награда (последние 100): 408.4
Эпизод 1000 | Ср. награда (последние 100): 500.0  ← Решено!
```

Робот научился балансировать шест максимальные 500 шагов — РЕШЕНО!

Несмотря на проблемы с дисперсией, REINFORCE эффективен в CartPole, потому что:
1. Эпизоды короткие (мы получаем их много за один цикл обучения).
2. Оптимальная политика проста (в основном нужно толкать тележку в ту сторону, куда наклоняется шест).

## Основные выводы

| Концепция | Простыми словами |
|-----------|------------------|
| **Политика** | Рецепт робота для выбора действий |
| **Эпизод** | Одна полная игра от начала до конца |
| **Доход G_t** | Общая будущая награда с текущего момента |
| **Дисконт γ** | Будущие награды ценятся чуть меньше, чем немедленные |
| **Нормализация** | Вычитание среднего для чистоты сигнала |
| **Дисперсия** | Насколько сильно скачут оценки градиента |

## Что дальше?

Главная слабость REINFORCE — это **дисперсия**. В следующем сценарии (`reinforce_baseline.py`) мы добавим **базовую линию** (baseline), которая радикально снижает этот шум, не меняя того, чему алгоритм обучается в среднем.

Ключевая идея: вместо вопроса «было ли это действие хорошим?», мы спрашиваем «было ли это действие **лучше ожидаемого**?». Это небольшое изменение делает обучение гораздо более стабильным.
