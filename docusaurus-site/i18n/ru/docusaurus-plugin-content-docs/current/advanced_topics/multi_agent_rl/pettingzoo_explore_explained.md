# Изучение сред PettingZoo 🦓

## Что такое PettingZoo?

Если вы занимались одноагентным RL, вы наверняка использовали **Gymnasium** (преемник OpenAI Gym). Там все среды выглядят одинаково: `env.reset()`, `env.step(action) → obs, reward, done, info`. Это единообразие позволяет работать библиотекам RL.

**PettingZoo** — это та же идея, но для *нескольких агентов*. Это «зоопарк» многоагентных сред, объединенных одним четким API:
- **Классические задачи**: простые среды вроде «Камень-Ножницы-Бумага» для тестирования базовых алгоритмов.
- **Кооперативные миры**: агенты, перемещающиеся по сетке для достижения общей цели.
- **Мультиплеер Atari**: классические соревновательные игры, например, Pong.
- **MPE (Multi-Particle Environment)**: физические среды в непрерывном пространстве для сложной координации и конкуренции.

Если вы напишете код, который работает в одной среде PettingZoo, вы сможете подключить любую другую почти без изменений.

---

## Два стиля API

Многоагентные настройки сложнее одноагентных, потому что агенты могут действовать одновременно, по очереди или в произвольном порядке. PettingZoo решает это с помощью двух параллельных API:

### 1) AEC (Agent-Environment-Cycle)

Агенты действуют по одному. Среда перебирает агентов по очереди, и каждый получает:
- **наблюдение** — то, что он видит *прямо сейчас*;
- **награду** — выигрыш, полученный в результате *совместных* действий в последнем полном раунде;
- **флаг завершения (termination)** — `True`, когда эпизод заканчивается *естественно* (например, мат в шахматах);
- **флаг усечения (truncation)** — `True`, когда эпизод прерывается по лимиту времени.

Это естественно для **пошаговых игр**, таких как шахматы, Го или покер.

```python
env.reset()
for agent in env.agent_iter():
    obs, reward, term, trunc, info = env.last()
    if term or trunc:
        env.step(None)
        continue
    action = my_policy(obs, agent)
    env.step(action)
```

### 2) Parallel

Все агенты наблюдают за миром и действуют одновременно на каждом шаге. Функция `step()` принимает *словарь* действий и возвращает словари наблюдений и наград.

Это естественно для **игр в реальном времени**, таких как MPE или многоагентные миры на сетке.

```python
obs, info = env.reset()
while env.agents:
    actions = {a: my_policy(obs[a]) for a in env.agents}
    obs, rewards, terms, truncs, info = env.step(actions)
```

Эти два стиля **изоморфны** — они структурно эквивалентны. Любую среду AEC можно автоматически «обернуть», чтобы она выглядела как Parallel, и наоборот. В PettingZoo есть готовые обертки для конвертации.

---

## Аналогия из реальной жизни

- **AEC = вечер настольных игр.** «Ход Алисы. Теперь Боба. Теперь Кэрол. Снова Алисы». Тот, кто ходит следующим, видит актуальное состояние доски.
- **Parallel = многопользовательская видеоигра.** Все четыре игрока нажимают кнопки одновременно; игра обновляет мир 60 раз в секунду.
- **Почему важен единый API.** Представьте, если бы для каждой видеоигры нужен был свой уникальный джойстик. PettingZoo — это «универсальный джойстик» для многоагентного RL (MARL).

## Что делает наш код

Мы создаем с нуля среду в стиле **PettingZoo**: **Повторяющаяся игра на координацию**. Два агента многократно выбирают канал `0` или `1`:

- Одинаковый выбор → оба получают +1.
- Разный выбор → оба получают -1.

**Наблюдение**, которое получает каждый агент — это предыдущее *совместное действие* (выбор обоих агентов в прошлом раунде, упакованный в одно число). Число 0 означает начало эпизода (действий еще не было). Эпизод длится 25 шагов, так что максимум можно набрать +25, а минимум -25. **Случайная игра дает в среднем 0**, так как агенты будут совпадать в половине случаев.

Далее мы:

1. **Демонстрируем интерфейс AEC** на примере случайных действий. Это подтверждает работу цикла `agent_iter()`, `last()` и `step()`.
2. **Обучаем двух независимых Q-обучающихся через интерфейс Parallel**. Каждый агент ведет свою Q-таблицу, используя совместное действие как состояние. Так он может выучить: «Если в прошлый раз мы оба выбрали 0, мне стоит снова выбрать 0».
3. **Пытаемся импортировать реальную библиотеку `pettingzoo`** и запустить одну из ее встроенных сред (Rock-Paper-Scissors).

### Что вы должны увидеть

| Этап | Ожидаемый результат |
|-------|----------|
| Случайная игра (AEC) | Средняя награда за эпизод около **0**. |
| Q-обучающиеся (Parallel) — первые 100 эп. | Около **0** — агенты еще исследуют мир. |
| Q-обучающиеся — последние 100 эп. | Сильно положительная, **от +20 до +25**. **Возникла координация**: оба агента научились выбирать один и тот же канал. |

График `outputs/pettingzoo_coordination.png` показывает награды за эпизоды (серый цвет) и сглаженную синюю кривую среднего значения. Видно, как агенты переходят от случайной игры (~0) к стабильной координации (~+25).

```bash

```

## Зачем сначала создавать свою среду?

Потому что **API и есть главный урок.** Многоагентный RL бывает разным (пошаговым, одновременным, кооперативным), но все виды вписываются в паттерн AEC / Parallel. Как только вы реализовали эти два цикла, любая среда PettingZoo становится лишь вопросом подключения другого конструктора — код обучения остается тем же.

Это именно то, как Gymnasium изменил одноагентный RL: превратил среду в «черный ящик» с единым интерфейсом.

---

## Где независимое Q-обучение помогает, а где вредит

Игры на координацию «великодушны» — у агентов общие интересы. Независимые обучающиеся легко справляются с ними, так как любое улучшение одного помогает и другому.

В **соревновательных** играх (RPS) независимое Q-обучение будет бесконечно колебаться (один адаптируется, другой тут же меняет стратегию, чтобы его обыграть).
В играх с **частичной наблюдаемостью** оно может вообще не сработать, так как «наблюдение» — это лишь часть состояния. PettingZoo включает оба типа сред, чтобы вы могли увидеть эти проблемы своими глазами.

## Ключевые слова

| Термин | Значение |
|--------|----------|
| **PettingZoo** | Библиотека стандартизированных сред для многоагентного RL |
| **AEC** | Agent-Environment-Cycle: агенты ходят по очереди |
| **Parallel API** | Все агенты действуют одновременно на каждом шаге |
| **MPE** | Популярный набор кооперативных и соревновательных сред в PettingZoo |
| **CTDE** | Centralised Training, Decentralised Execution — учимся, видя всё, действуем, видя только своё |
| **Независимое Q-обучение** | Каждый агент запускает обычное Q-обучение, игнорируя других |
