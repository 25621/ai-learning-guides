# Self-Play: Обучение агента через игру с самим собой ♟️

## Что такое Self-Play?

Представьте ребенка, который хочет научиться играть в шахматы, но ему не с кем играть. Тогда он начинает играть сам с собой. Левая рука против правой. В каждой партии *обе* стороны стараются победить. В каждой партии *обе* стороны учатся тому, что сработало.

Это и есть **self-play** (игра с самим собой): один и тот же агент выступает за обоих игроков. Каждый ход становится уроком для того, кто ходит следующим. Никакого учителя, никакого экспертного оппонента. Только ученик, который сам себе служит лестницей.

Self-play может показаться трюком — разве не нужен настоящий противник? Но именно этот метод стоит за самыми громкими достижениями RL последнего десятилетия: **AlphaGo Zero**, **AlphaZero**, **MuZero**, **OpenAI Five**. Все они используют self-play. Причина проста: по мере вашего совершенствования ваш оппонент становится лучше ровно на столько же. Уровень сложности всегда соответствует вашим навыкам.

---

## Почему это работает

Три вещи делают self-play особенным:

1. **Бесконечное количество оппонентов.** У вас никогда не закончатся игры. Оппонент всегда рядом и не требует затрат.
2. **Учебная программа (curriculum), которая растет вместе с вами.** Новичок может играть только с новичками. По мере вашего роста ваша «тень» растет автоматически.
3. **Симметрия.** В игре с нулевой суммой (выигрыш одного — проигрыш другого) один набор Q-значений описывает обе стороны; вам просто нужно сменить знак, когда наступает ход другого игрока. Таким образом, *одна* Q-таблица может учить саму себя.

Крестики-нолики — идеальный полигон: игра достаточно мала, чтобы поместиться в словарь, но достаточно сложна, чтобы случайные ходы почти всегда приводили к проигрышу против стратега.

## Аналогия из реальной жизни

- **Тренировка в теннисе со стенкой.** Стенке нельзя проиграть, но на ней можно отрабатывать подачу. Self-play — это то же самое, но на обоих концах: вы и стенка, и игрок, и вы постоянно меняетесь ролями.
- **Дискуссионный клуб, где аргументируют обе стороны.** Лучшие ораторы получаются из тех, кто всегда готов защищать позицию, противоположную их личным убеждениям. Каждый аргумент тренирует и нападение, и защиту.
- **AlphaGo Zero.** Она не видела ни одной игры человека. Начав со случайных ходов, она сыграла миллионы партий сама с собой; через несколько дней она играла лучше любой предыдущей программы, включая ту, что победила Ли Седоля.

## Что делает наш код

Мы обучаем одну Q-таблицу для *игрока, чей сейчас ход*:

```
Q[(доска, текущий_игрок)][действие] = ожидаемая награда для этого игрока
```

Цикл обучения выглядит так:

1. Начинаем с пустой доски. `игрок = X`.
2. Оба игрока ходят, используя **одного и того же агента** с ε-жадной стратегией.
3. После каждой игры мы проходим назад по всей истории (доска, игрок, действие) и применяем обновление Q-обучения.
4. Награда меняет знак при переходе хода: если X выиграл, каждый ход X получает +1; каждый ход O получает -1.
5. Мы постепенно снижаем уровень исследования (ε) с 0.2 до 0.02, чтобы к концу обучения агент больше доверял своим знаниям, а не случайным ходам.

Каждые 2500 эпизодов мы проверяем агента против **случайного оппонента** (замораживая обучение). После достаточного количества игр агент должен выигрывать или сводить вничью ~100% матчей.

### Что вы должны увидеть

После 50 000 эпизодов self-play:

| Матч | Ожидаемый результат |
|----------|-----------------|
| Обученный агент против Случайного (1000 игр) | **~95-99% побед или ничьих**, почти 0% поражений. |
| Обученный агент против Самого себя (200 игр) | **Все 200 ничьих**. Крестики-нолики — это игра, которая всегда заканчивается вничью при идеальной игре обоих. То, что агент играет сам с собой вничью — признак сходимости. |

График `outputs/self_play_tic_tac_toe.png` показывает доли побед/ничьих/поражений против случайного оппонента:
- Доля побед начинается с ~60% (у первого игрока в крестиках-ноликах есть врожденное преимущество при случайной игре).
- Поднимается выше 90%.
- Доля поражений падает почти до нуля.

В конце скрипт выводит пример игры ход за ходом.

## Обратите внимание на тонкости

- **Смена знака важна.** Распространенная ошибка: забыть, что «максимизация выгоды оппонента» означает *минимизацию нашей* в формуле обновления.
- **Симметрия доски.** В реальных задачах доски канонизируют (поворачивают и отражают), чтобы объединить Q-значения для симметричных состояний. Мы этого не делаем, так как пространство состояний достаточно мало для прямого перебора.
- **Рост Q-таблицы.** После 50к игр в таблице будет несколько тысяч записей. Это нормально. Для шахмат или Го потребовалась бы нейросеть, поэтому **AlphaZero заменяет таблицу на CNN + MCTS**.

---

## Где Self-Play ломается

- **Игры с ненулевой суммой.** «Оба довольны» несовместимо с симметричной игрой; нельзя просто перевернуть знак.
- **Асимметричные роли.** Если у «нападающего» и «защитника» разные наборы действий, нужны две отдельные сети.
- **Цикличность стратегий.** Чистый self-play может застрять в циклах типа «камень-ножницы-бумага». AlphaStar решила это, сохраняя «лигу» прошлых версий агента и выбирая оппонентов из нее, чтобы агент умел побеждать разные стили игры.
- **Взлом награды (Reward hacking).** Self-play делает обе стороны умнее, но только в рамках той игры, которую вы определили. Если в системе наград есть лазейка, обе стороны будут ее эксплуатировать.

## Ключевые слова

| Термин | Значение |
|--------|----------|
| **Self-play** | Один и тот же агент играет за обе стороны |
| **Нулевая сумма** | Выигрыш одного = проигрышу другого |
| **Симметрия** | Одна Q-таблица для обеих сторон при смене знака |
| **Population play** | Self-play с использованием пула прошлых версий агента в качестве оппонентов |
| **Curriculum** | Естественная прогрессия сложности (в self-play она бесплатна) |
| **MCTS** | Поиск по дереву Монте-Карло — алгоритм планирования, который AlphaZero сочетает с self-play |
