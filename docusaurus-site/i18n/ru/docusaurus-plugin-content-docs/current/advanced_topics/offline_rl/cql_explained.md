# Консервативное Q-обучение (Conservative Q-Learning, CQL) 🛡️

## Что это такое?

Представьте, что вы учитесь инвестировать деньги, читая гигантскую книгу учета прошлых сделок с акциями, совершенных другими людьми. В книге есть записи о покупках, продажах и удержании активов, но **нет никаких записей о сделках, которые никто на самом деле не совершал**.

Теперь представьте слишком самоуверенного студента, который смотрит в книгу и говорит:
*«А что, если бы кто-то покупал лотерейные билеты каждый понедельник? Это была бы потрясающая сделка!»*

Проблема: **в книге нет данных о покупке лотерейных билетов по понедельникам**, поэтому студент просто галлюцинирует. Тем не менее, эта вымышленная сделка выглядит на бумаге великолепно, поэтому «политика» студента продолжает стремиться к ней.

Эта проблема галлюцинаций называется **сдвигом распределения** (distribution shift): обучающийся в офлайн-режиме агент обожает действия, которые не были проверены в наборе данных, потому что нет данных, которые могли бы опровергнуть этот оптимизм. CQL — это решение.

---

## Как Q-обучение ломается в офлайне

Цель (target) обычного Q-обучения:

```
target(s, a) = r + γ · max_{a'} Q(s', a')
```

Этот `max_{a'}` и таит в себе опасность. Когда в наборе данных не записано действие `a'` в состоянии `s'`, сеть просто *угадывает* Q-значение — а нейронные сети склонны **завышать** Q для незнакомых входных данных. Цель (target) наследует это завышение, сеть учится предсказывать это большее число, и на следующем шаге мы **экстраполируем** еще выше (проецируем результат еще дальше за пределы того, что подтверждают данные). Политика гонится за призраком.

Если бы вы могли продолжать собирать данные, это бы исправилось само собой (призрачное действие на деле оказалось бы плохим). Но **в офлайн-RL вы не можете собирать больше данных.** Призрак остается навсегда.

## Трюк CQL

CQL (Kumar et al., 2020) добавляет штрафное слагаемое к функции потерь:

```
cql_loss(s)  =  log Σ_a exp Q(s, a)   -   Q(s, a_dataset)
```

Две части:

1. **`log Σ_a exp Q(s, a)`** (читается как *«log-sum-exp по всем действиям»*) — это **мягкий максимум** (soft maximum) по всем действиям; гладкая дифференцируемая аппроксимация `max`, которая учитывает все действия сразу, а не выбирает жестко одного победителя. Штрафование этого слагаемого уменьшает Q-значения **по всем направлениям** (равномерно толкая все предсказания вниз) — особенно для действий с *самым высоким* Q, где как раз и живут галлюцинации.
2. **`- Q(s, a_dataset)`** поощряет высокие Q для действия, которое на самом деле было записано в наборе данных, защищая Q-значения внутри распределения от вышеупомянутого сокращения.

Чистый эффект: **Q снижается для невидимых действий и повышается для увиденных.** Выученная Q становится *нижней границей* истинной Q. Политика **`argmax`** (правило, которое просто выбирает действие с самым высоким Q) перестает гоняться за призраками.

Полная функция потерь:

```
L  =  Bellman_MSE   +   α · cql_loss
```

(Где **`Bellman_MSE`** — стандартная ошибка обычного Q-обучения, измеряющая, насколько текущее предположение сети расходится с её собственным будущим предположением).

`α` — это «ручка» консервативности. Слишком маленькая → сдвиг распределения возвращается. Слишком большая → агент настолько консервативен, что никогда не прогрессирует дальше данных.

## Примеры из реальной жизни

- **Консервативный шахматный тренер.** Вы можете учиться только на уже сыгранных партиях. Безрассудный тренер скажет: «Этот гипотетический ход, не имеющий прецедентов, мог бы быть блестящим!» CQL — это тренер, который говорит: «У нас нет данных по этому ходу — давайте придерживаться тех ходов, которые пробовали реальные игроки».
- **Выбор блюд в меню ресторана.** Отзывы на Yelp никогда не охватывают блюда, которых нет в меню. Наивная политика рекомендовала бы блюда вне меню, основываясь на галлюцинируемых пятизвездочных рейтингах. CQL рекомендует только то, что заказывали достаточное количество раз, чтобы этому можно было доверять.
- **Захват предметов роботом по логам.** У робота есть видео захвата чашек, бутылок и книг, но никогда — ножа. CQL отказывается уверенно рекомендовать «схватить нож за лезвие».

## Что делает наш код

Скрипт `cql.py`:

1. **Загружает четыре набора данных**, созданных `d4rl_dataset.py`.
2. **Выбирает `medium-replay`** в качестве обучающего набора — он наиболее реалистичен (смешанное качество) и наиболее разрушителен для наивных методов.
3. **Обучает трех агентов чисто в офлайн-режиме** в идентичных условиях, за исключением `α`:
   - `α = 0`   → наивная офлайн DQN (без штрафа — сломанный базовый метод)
   - `α = 1.0` → умеренная CQL
   - `α = 5.0` → сильная CQL
4. **Оценивает каждого агента каждые 2500 шагов градиента**, запуская его жадным образом в реальной среде (10 эпизодов). Это *единственный* контакт со средой; само обучение никогда её не видит.
5. **Строит графики кривых обучения** в `outputs/cql.png`.

## Что вы должны увидеть

Типичный запуск выводит что-то вроде:

```
Итоговые доходы оценки (среднее по 10 эпизодам, жадно):
  наивная офлайн DQN (alpha=0)         ->  ~30-150  (нестабильно; часто падает)
  CQL (alpha=1.0)                     ->  ~300-450
  CQL (alpha=5.0)                     ->  ~450-500
```

На графике кривых обучения:

- **Красная кривая** (`α = 0`) сначала растет, а затем часто **срывается в обрыв**, как только галлюцинации сдвига распределения заражают **таргет Беллмана** (число, которое мы используем как «правильный ответ» при обучении Q-сети: `r + γ · max Q(s', ·)`). Когда призрачные Q-значения загрязняют эту цель, каждый шаг градиента только ухудшает ситуацию. **Ошибка Беллмана** (MSE между предсказанием Q-сети и таргетом Беллмана) выглядит нормально — в этом и заключается **коварство** проблемы: сеть идеально согласуется со своими собственными неверными убеждениями, поэтому ошибка не дает никаких предупреждений.
- **Оранжевая кривая** (`α = 1.0`) поднимается медленнее, но **остается наверху**.
- **Зеленая кривая** (`α = 5.0`) — самая стабильная и обычно лучшая.

Панель ошибки Беллмана показывает еще один признак: ошибка наивной DQN может оставаться маленькой, пока её политика ужасна, потому что сеть внутренне непротиворечива в своих галлюцинациях.

## Место CQL в области исследований

CQL стал очень важным событием, потому что предложил теоретически обоснованное и простое решение проблемы сдвига распределения. Родословная методов:

```
DQN (онлайн)
   │
   ▼
Наивная офлайн DQN ── ломается из-за сдвига распределения
   │
   ▼
CQL (Kumar 2020)    ── добавление консервативного штрафа: Q как нижняя граница
   │
   ▼
IQL (Kostrikov 2021)  ── вообще не запрашивать Q для невидимых действий
   │
   ▼
Decision Transformer (Chen 2021) ── пропустить Q вовсе, трактовать RL как моделирование последовательностей
                                     (предсказывать *следующее действие* на основе прошлых состояний и
                                      желаемого общего дохода, точно так же, как LLM
                                      предсказывает следующее слово)
```

Каждый шаг в этой родословной — это разный ответ на один и тот же вопрос: **как мне избежать расспросов моей Q-сети о вещах, которые она никогда не видела?**

## Ключевые слова для запоминания

| Слово | Значение |
|------|---------|
| **Distribution shift** | Сдвиг распределения. Обученная политика хочет совершать действия за пределами данных |
| **Out-of-distribution (OOD)** | Вне распределения. Пара (s, a), которая не была записана в наборе данных |
| **Истинная Q** | Реальный ожидаемый будущий доход от совершения действия `a` в состоянии `s`, если бы мы могли измерить его идеально |
| **Консервативная Q** | Выученная Q-функция, которая старается оставаться ниже истинной Q вместо того, чтобы давать завышенные обещания |
| **Logsumexp** | Гладкая дифференцируемая аппроксимация `max` |
| **Альфа (α)** | Ручка консервативности CQL — насколько сильно давить Q вниз для действий OOD |

## Резюме одной строкой

> **CQL добавляет «штраф за пессимизм», который наказывает за высокие Q-значения для действий, которые не пробовались в наборе данных — так политика не может «влюбиться» в галлюцинации.**
