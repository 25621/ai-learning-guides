# Double DQN：過大評価問題の解決 🤔

## 問題：DQN は自分を実際より優れていると思い込む

「この街で一番美味しいレストランはどこ？」と聞かれたところを想像してみてください。

あなたは「ピザ・パレスが最高だよ。間違いなく 10点満点だね！」と言うかもしれません。しかし、あなたはそこへ2回しか行ったことがありません。本当に 10点満点かどうかは、実は分かっていないのです。たまたまその2回の訪問で美味しいピザに当たっただけで、過大評価している可能性があります。

これと同じ問題が DQN でも起こります。エージェントが **Q値を過大評価してしまう** のです。

---

## なぜ DQN は過大評価するのか？

DQN がターゲットを計算するとき、以下の式を使います：
> ターゲット = 報酬 + γ × **max** Q(次の状態)

この `max` が問題なのです！ノイズを含んだいくつかの推定値の中から最大値を選ぶとき、ほとんどの場合、たまたまプラスの誤差（上向きのバイアス）が最も大きくなったものを選んでしまいます。

**現実世界の例：** 5人の友人に建物の高さを予想してもらいます。予想値は 40m, 38m, 45m（たまたま高く見積もった！）, 39m, 41m でした。本当の高さは 40m です。もしここで `max(予想値)` = 45m を採用したら、大きく外れてしまいます！ノイズのある予想値の最大値は、ほぼ常に過大評価になります。

何千回もの更新を通じて、DQN はこのように膨れ上がったターゲットに向かって訓練を続けてしまい、「物事は実際よりも良い状態にある」と学習してしまいます。これにより学習が遅れたり、自信過剰で不適切な決定を下したりする原因になります。

---

## Double DQN による解決策

**Double DQN** (Hasselt et al., 2016) は、この `max` を2つのステップに分割します：

**ステップ 1 — どの行動を選ぶか？** **オンラインネットワーク** を使って、最善の行動を選びます：
> 最善の行動 = argmax Q_online(次の状態)

**ステップ 2 — その価値はいくらか？** **ターゲットネットワーク** を使って、その行動を評価します：
> ターゲット = 報酬 + γ × Q_target(次の状態, 最善の行動)

```
通常の DQN:     ターゲット = r + γ × max_a Q_target(s', a)
                                 ↑ 同じネットワークが選択と評価の両方を行う → バイアスが発生

Double DQN:    最善の行動 = argmax_a Q_online(s', a)   ← オンラインが選択
               ターゲット = r + γ × Q_target(s', 最善の行動) ← ターゲットが評価
                                 ↑ 異なるネットワークを使用 → バイアスが減少
```

**現実世界の例：** 就職の面接で、応募者本人に自分のパフォーマンス・テストを採点させることはしません（これが通常の DQN の問題です！）。代わりに、候補者は自分の最高の作品を *提示* し、*別の* 試験官がそれを評価します。2人の異なる人物が関わることで、より公正な評価が可能になります！

---

## なぜ分割が役立つのか？

2つのネットワーク（オンラインとターゲット）は、ターゲットネットワークの更新頻度が低いため、異なる重みを持っています。そのため、どの行動が最善かについて異なる「意見」を持っています。

意見が食い違うとき：
- オンライン：「行動 A が良さそうだ！」
- ターゲット：「いや、実際には行動 A はまあまあだ。10点ではなく 7点くらいの価値しかないよ」

オンラインネットワークが *選んだ* 行動に対して、ターゲットネットワークの *価値* 推定値を使用することで、より正直で、膨れ上がっていない数値を得ることができます。

---

## コードの違い：わずか数行！

通常の DQN から Double DQN へのコードの変更点は、ターゲットの計算部分だけです：

```python
# 通常の DQN:
q_next = target_net(s_next).max(dim=1).values

# Double DQN:
best_actions = online_net(s_next).argmax(dim=1, keepdim=True)   # オンラインで選択
q_next = target_net(s_next).gather(1, best_actions)              # ターゲットで評価
```

わずか数行の変更ですが、安定性と正確性への影響は非常に大きいです！

## 比較から分かること

`double_dqn_cartpole.py` を実行すると、2つのプロットが表示されます：

**プロット 1：学習曲線**
- 通常の DQN と Double DQN の両方が CartPole を解決できるはずです。
- 多くの場合、Double DQN の方がより速く、より安定して収束します。
- CartPole は単純なタスクなので違いはわずかですが、Atari ではより劇的な差が出ます。

**プロット 2：Q値の推定**
- 通常の DQN：時間の経過とともに Q値が上向きに漂流（過大評価）します。
- Double DQN：Q値はより控えめで正確な値にとどまります。

Q値の過大評価のプロットは重要な洞察を与えてくれます。これは、通常の DQN が膨れ上がった価値を学習し、それが最終的にパフォーマンスを低下させていることを示しています。

## Double DQN はどれくらい優れているか？

| 指標 | 通常の DQN | Double DQN |
|--------|------------|------------|
| Q値の正確性 | 過大評価する | より正確 |
| 学習の安定性 | 分散が大きい | 分散が小さい |
| CartPole での性能 | 良い | わずかに向上 |
| Atari での性能 (50ゲーム) | ベースライン | 人間レベルに近いゲーム数が +2.6倍に増加 |

複雑な Atari ゲームでは、Double DQN は CartPole よりもはるかに大きな違いをもたらしました（Atari は Q値の推定に非常に多くのノイズが含まれるためです）。

## DQN 改善の系譜

Double DQN は、通常の DQN に対する多くの改善策の1つに過ぎません。「Rainbow」という論文 (2017) では、6つの改善策が組み合わされました：

1. **Double DQN** (過大評価の修正) ← 本教材の内容！
2. **優先度付き経験再生 (Prioritized Replay)** (驚きの大きい経験からより多く学ぶ)
3. **Dueling Networks** (「この状態はどれくらい良いか？」と「最善の行動は何か？」を分離する)
4. **マルチステップ収益 (Multi-step returns)** (より遠い未来まで考慮する)
5. **分布型強化学習 (Distributional RL)** (平均だけでなく収益の分布全体を学習する)
6. **NoisyNets** ( [ε-greedy](../foundations/multi_armed_bandit_explained.md#ε-グリーディ-イプシロングリーディ-戦略) の代わりに学習によって探索を行う)

Rainbow はこれらすべてを組み合わせ、当時の Atari で最高のパフォーマンスを達成しました！

## 覚えておくべきキーワード

| 用語 | 定義 |
|------|---------|
| **過大評価 (Overestimation)** | Q値が真の値よりも高くなっている状態（楽観的すぎる） |
| **Double DQN** | 行動選択にオンラインネットワーク、評価にターゲットネットワークを使用する手法 |
| **デカップリング (Decoupling)** | 同じネットワークが行っていた2つのタスクを切り離すこと |
| **バイアス (Bias)** | 一方向への系統的な誤差（常に高すぎる、あるいは常に低すぎる） |
| **Rainbow** | 最高の性能を得るために 6つの改善策を組み合わせた DQN の変種 |
