# 経験再生 (Experience Replay)：記憶することを教える 🎒

## 問題：忘却（と混乱）

単純な DQN が不安定だったのを覚えていますか？最大の理由は **相関のある学習 (correlated learning)** です。

ロボットがゲームをプレイするとき、物事を順番に経験します：
> ステップ 1 → ステップ 2 → ステップ 3 → ステップ 4 → ...

これらのステップはつながっています！もしロボットがステップ 10 で左に傾いたら、ステップ 11 も左に傾いた状態から始まります。これらは独立しているわけではなく、互いに依存しています。

これらの関連し合ったステップを使ってネットワークを更新するのは、歴史を学ぶのに同じ章を何度も繰り返し読むようなものです。その章については非常に詳しくなりますが、それ以外のことはすべて忘れてしまいます！

**現実世界の例：** 昨日の宿題だけを練習してテストの勉強をしているところを想像してみてください。その特定の問題については完璧になりますが、テストには別の問題が出ます！さまざまな問題を混ぜて練習する必要があります。

---

## 解決策：メモリボックス 📦

**経験再生 (Experience Replay)** は、ロボットに大きなメモリボックス（ **経験再生バッファ** ）を追加します。

最新の経験からすぐに学習する代わりに、ロボットは以下のことを行います：
1. すべての経験をメモリボックスに **保存** します：(状態, 行動, 報酬, 次の状態)
2. ボックスから経験を **ランダムにいくつか選び出します**
3. 最新のステップだけでなく、その **ランダムなミックス** から学習します

```
ゲームのステップ 1 → [ボックスに保存]
ゲームのステップ 2 → [ボックスに保存]
ゲームのステップ 3 → [ボックスに保存]
...
ゲームのステップ 50 → [ボックスに保存] → ランダムに 64個の思い出を選ぶ → ネットワークを更新
ゲームのステップ 51 → [ボックスに保存] → ランダムに 64個の思い出を選ぶ → ネットワークを更新
```

**現実世界の例：** 写真アルバムを思い浮かべてください。今日の写真だけを見て人生を学ぶわけではありません。古い写真もめくってみるはずです。良い思い出も、苦労した瞬間も混ざっています。これによって、今日のことだけでなく、人生全体のパターンを理解できるようになります。

---

## なぜランダムサンプリングが役立つのか

思い出をランダムに選ぶことで、相関関係を断ち切ることができます。ロボットは以下のような組み合わせから学習できます：
- 棒が完璧に立っていた時の思い出（500ステップ前）
- 棒が倒れそうだった時の思い出（20ステップ前）
- たまたまうまくいった時の思い出（ステップ 3）

このランダムなミックスによって：
✅ ロボットは多様な状況から学習できる
✅ 各思い出を何度も「再生」できる（経験の効率的な利用）
✅ ネットワークが直近の出来事だけに過剰に適合（過学習）するのを防げる

---

## ミニバッチ学習

一度に1つの経験から更新するのではなく、**一度に 64個の経験** （「ミニバッチ」）を使って更新します。これは以下のような違いです：
- 旧方式：単語カードを1枚読んで、自分でテストする
- 新方式：異なる 64枚の単語カードを読んでから、それらを混ぜてテストする

ミニバッチによって、学習のシグナルははるかに信頼性が高く、ノイズの少ないものになります。

---

## ウォームアップ期間

すぐに学習を始めるわけではありません！経験再生バッファにはまず、ある程度の思い出が必要です。ボックスに少なくとも **500個の経験** が貯まるまで、訓練は開始されません。

**現実世界の例：** 材料が揃う前に料理を始めようとはしませんよね。ウォームアップ期間は、料理を始める前の買い出しのようなものです！

---

## 比較から分かること

`dqn_experience_replay.py` を実行すると、2つの学習曲線が表示されます：

| 単純な DQN | DQN + 経験再生 |
|-----------|-------------|
| 非常にでこぼこしている | より滑らか |
| 頻繁にクラッシュする（すべてを忘れる） | より一貫した向上が見られる |
| 分散が大きい | 分散が小さい |

経験再生バージョンは通常：
- 良いスコアにより確実に到達する
- スコアが 500 から 30 に急落する頻度が低い
- 学習の進捗がより安定している

## コードでの経験再生バッファ

```
ReplayBuffer (経験再生バッファ):
  - 容量：10,000個の思い出（満杯になると古いものから忘れられる）
  - push(状態, 行動, 報酬, 次の状態, 終了フラグ)
  - sample(batch_size=64) → ランダムなバッチを抽出
```

10,000行あるノートのようなものだと考えてください。満杯になったら、一番古い行を消して新しいことを書きます。そして常に、ランダムに開いたページから勉強するのです！

---

## 覚えておくべきキーワード

| 用語 | 定義 |
|------|---------|
| **経験再生 (Experience Replay)** | 過去の経験を保存し、訓練のためにランダムに再利用すること |
| **経験再生バッファ (Replay Buffer)** | 過去の（状態, 行動, 報酬, 次の状態）のセットを保存するメモリボックス |
| **相関のある更新** | 訓練データが自分自身に依存している状態（学習には良くありません！） |
| **ミニバッチ** | 1回の更新ステップで使用される、思い出の小さなランダムサンプル |
| **無相関化 (Decorrelation)** | 連続する経験の間のつながりを断ち切ること |

---

## まだ足りないものは？

経験再生を使っても、まだ別の問題があります。それが **動く標的 (moving target)** です。

ネットワークを更新するたびに、Q値が変化します。しかし、その更新された Q値は、*次* の更新のターゲットを計算するためにも使われます。混乱のループです！

これは、**ターゲットネットワーク** （100ステップごとにしか更新されないネットワークのコピー）によって解決されます。これによって、標的（ブルズアイ）がしばらくの間固定されるため、ロボットは確実にそれを狙うことができるようになります！
