# CartPoleのための線形Q学習 🎪

## CartPoleとは？

指の上でほうきを垂直に立ててバランスを取っているところを想像してください。指を左右に少し動かすだけで、ほうきが倒れないようにすることができます。それが**CartPole**です！

小さなロボットがカート（車輪付きの箱）の上に座っており、その上にポールが立っています。ロボットはカートを**左**または**右**に押すことしかできません。ロボットは、あなたがほうきでバランスを取るのと同じように、できるだけ長くそのポールを直立させておくことを学ばなければなりません。

ロボットは世界について4つのことを見ることができます：
1. カートの位置
2. カートの移動速度
3. ポールの傾き
4. ポールの傾く速度

---

## 大きな問題：状態が多すぎる！

フェーズ2のQ学習を覚えていますか？それは、各状況（状態）において各行動がどれだけ良いかを記憶するために大きなテーブルを使用していました。Frozen Lakeでは、氷の上に16個のマスしかなかったので、それでうまくいきました。

しかし、CartPoleは違います！カートは**あらゆる位置**に存在し、**あらゆる速度**で移動し、ポールは**あらゆる角度**に傾く可能性があります。基本的には**無限の可能な状態**が存在します！無限の行を持つテーブルを作ることはできません。宇宙サイズのノートが必要になってしまいます！

**実生活の例：** 自転車の乗り方を学んでいるところを想像してください。考えられるすべてのふらつきを暗記することはできません。多すぎるからです。代わりに、あなたは**ルール**を学びます：「左に傾いたら右に押し、右に傾いたら左に押す」。シンプルなルールがすべてのふらつきに対して機能します。

---

## 解決策：魔法の数式

**線形関数近似（Linear function approximation）**は、巨大なテーブルを**小さな数式**に置き換えます：

> **スコア(状況, 行動) = w₁ × カート位置 + w₂ × カート速度 + w₃ × ポール角度 + w₄ × ポール速度**

- `w`の数字は**重み（weights）**と呼ばれます。これらはひねることができるノブのようなものです。
- 各行動（左に押す、右に押す）に対して**異なる重み**を学習します。
- この数式は、今どの行動がどれだけ良いかというスコアを算出します。

**実生活の例：** シンプルなレシピを考えてみましょう：「小麦粉1カップ + 卵2個 + バター1/2カップ」。重み（1, 2, 1/2）は、各材料がどれだけ重要かを教えてくれます。私たちは良い決断のためのレシピを学んでいるのです！

---

## どのように学習するのか？

ロボットはいろいろ試してフィードバックを得て、重みを調整します：

1. **ロボットがカートを押す**（スコアが最も高い行動を選択する）
2. **物理現象が起こる**（ポールが少し傾き、カートが動く）
3. **ロボットが報酬を得る**（ポールが立っているステップごとに+1、倒れたら0）
4. **ロボットが問う：** 「実際の結果は自分の予測より良かったか、悪かったか？」
5. **ロボットが重みを調整する**：次回の予測がより現実に近くなるようにします。

これは**Semi-Gradient TD Update（準勾配TD更新）**と呼ばれます。「驚きに基づいてレシピを少しずつ修正する」ための大げさな名前です。

> **新しい重み = 古い重み + 学習率 × (実際に起きたこと − 予測したこと) × 特徴量**

## コードの実行結果

`linear_q_cartpole.py`を実行すると、ロボットは：

- 最初はひどい状態から始まります（10〜30ステップでポールが倒れます）。
- 3,000回の試行を経て、徐々に良い重みを学習します。
- 最終的には、100〜400ステップ以上もポールのバランスを保てるようになります！

グラフは**学習曲線（learning curve）**を示しています。スコアが時間の経過とともにどのように向上するかを表しています。凸凹していることもありますが（学習は決してスムーズではありません！）、トレンドは上向きになるはずです。

---

## なぜこれがすごいのか（そして限界は？）

**すごい点：** わずか8つの数字（4つの重み × 2つの行動）を持つ小さな数式だけで、ポールのバランスを取ることができます！巨大なテーブルは必要ありません。

**限界：** この数式は複雑なタスクには単純すぎます。数字が大きければ常に影響も大きくなると仮定していますが、これは必ずしも真実ではありません。Atariのようなより難しいゲームでは、**ニューラルネットワーク**が必要です。それがDQNの役割です！

---

## 重要な用語集

| 単語 | 意味 |
|------|---------|
| **特徴量 (Feature)** | 世界について測定可能な1つのこと（例：ポールの角度） |
| **重み (Weight)** | 特徴量が意思決定にどれだけ影響を与えるか |
| **線形 (Linear)** | 数式が掛け算と足し算だけで構成されている（複雑な曲線がない） |
| **準勾配 (Semi-gradient)** | 誤差が少なくなる方向に重みを更新する手法 |
| **関数近似 (Function approximation)** | テーブルの代わりに数式を使用すること |
