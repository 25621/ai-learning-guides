# ゼロからの Deep Q-Network (DQN) 🧠

## 線形手法の問題点

以前に扱った線形の式を覚えていますか？

> スコア = w₁ × カートの位置 + w₂ × カートの速度 + w₃ × 棒の角度 + w₄ × 棒の速度

これは CartPole にはうまく機能しますが、何千ものピクセルが見えるビデオゲームの場合はどうでしょうか？そんな単純なレシピを書くことはできません！

複雑な状況を見て、最善の行動を判断できるものが必要です。それが **ニューラルネットワーク (neural network)** です。

---

## ニューラルネットワークとは？

自分の脳について考えてみてください。ニューロンと呼ばれる何百万もの小さな細胞が互いに会話しています。熱いものに触れたとき、ニューロンが信号を送ります。「熱い！ → 今すぐ手を離せ！」各ニューロンが情報を伝達し、それらが集まって賢い決断を下します。

**コンピュータ上のニューラルネットワーク** も同じように機能します：

```
入力層           中間層 1          中間層 2          出力層
[カートの位置]  →  [128個のニューロン]  →  [128個のニューロン]  →  [「左へ押す」スコア]
[カートの速度]  →  [      ...      ]      [      ...      ]      [「右へ押す」スコア]
[棒の角度]      →
[棒の速度]      →
```

各矢印には **重み (weight)** （そのつながりの強さ）があります。何千もの重みがあり、ネットワークはそれらすべてを学習します！

**現実世界の例：** レストランのシェフが料理を味見して、何百もの材料を一度に調整するようなものです。一つ一つの味蕾（みらい）がニューロンのようなもので、それらが集まってシェフに「塩をもっと足して」や「胡椒を減らして」と伝えます。ネットワークの訓練は、シェフが何千もの料理を通じて学んでいく過程に似ています。

---

## DQN = Deep Q-Network

**DQN** (Deep Q-Network) は、2013年に DeepMind によって発明されました。彼らは古い Q学習の式を採用し、Qテーブルをニューラルネットワークに置き換えたのです！

これまでは：
> Qテーブル[状態][行動] = スコア

これからは：
> Qネットワーク(状態) → [左のスコア, 右のスコア]

ネットワークは状態を入力として受け取り、すべての行動に対する Q値を一度に出力します。これは個別に計算するよりもはるかに効率的です！

## このスクリプト：「単純な」バージョン

このスクリプトは、特別な工夫を **一切加えていない** DQN を示しています。ただ以下を行うだけです：
1. 状態を見る
2. ネットワークに「左はどれくらい良い？右は？」と聞く
3. スコアが高い方の行動をとる
4. 報酬を得て、ネットワークを更新する

**これは意図的に不安定にしてあります！** 新しいことを学ぶたびに、以前の教訓をすぐに忘れてしまう学生を想像してみてください。ネットワークは一歩進むたびに更新されるため、混乱が生じます。

**現実世界の例：** 一口食べるごとにレシピを全部変えながら料理を学んでいるところを想像してみてください。「塩辛すぎる」から「塩を全くなし」へ、そして「塩辛すぎ」へと極端に振れてしまい、ちょうど良い加減に落ち着くことができません。それがここで起きていることです！

## 見られる現象

`dqn_cartpole.py` を実行すると：
- スコアが大きく上下します（不安定な学習）
- エージェントが非常に上手くなったかと思えば、突然すべてを忘れてしまうことがあります
- 損失 (loss) プロットが激しく変動します

**これは予想通りの結果です！** これによって、なぜ「経験再生 (experience replay)」や「ターゲットネットワーク (target networks)」といった改善策が必要なのかが分かります。これらは次のステップで扱います！

## ε-グリーディ (イプシロン・グリーディ) の工夫 🎲

ロボットは常に最善の行動をとるわけではありません。時々、ランダムに行動を選びます！

なぜでしょうか？もし常に最善に見える行動ばかりとっていたら、もっと良い選択肢を見つけることができないかもしれないからです。

> 確率 ε (イプシロン) で：ランダムに行動を選ぶ（探索！）
> 確率 1-ε で：既知の最善の行動を選ぶ（利用！）

最初は ε = 1.0 (100% ランダム) から始め、徐々に ε = 0.01 (1% ランダム) まで下げていきます。こうすることで、ロボットは最初はたくさん探索し、その後、学んだことに集中するようになります。

**現実世界の例：** 新しい街を訪れたとき、最初はランダムにレストランを試してみるかもしれません（探索）。しばらくして、お気に入りの店に戻るようになります（利用）。でも、もしかしたら隠れた名店があるかもしれないので、たまには新しい店を試してみますよね！

## 覚えておくべきキーワード

| 用語 | 定義 |
|------|---------|
| **ニューラルネットワーク** | データから学習する、つながり合った数学的なニューロンの層 |
| **Deep (深い)** | 中間層が複数あること（ここから「深層学習/ディープラーニング」と呼ばれます） |
| **DQN** | Deep Q-Network。Qテーブルの代わりにニューラルネットを使用する手法 |
| **ε-グリーディ** | 戦略：時々ランダムに探索し、それ以外は最善の知識を利用する |
| **不安定性 (Instability)** | 更新が互いに干渉し合うため、ネットワークが「忘れ」続けてしまうこと |

---

## 何が足りないのか（そしてなぜ重要か）

この単純な DQN には2つの大きな問題があります：

1. **相関のある更新**：すべての経験が順番に（ステップ1, 2, 3...）入ってきます。もしステップ5が悪い結果だったら、その近くの更新はすべて混乱してしまいます。
   
2. **動く標的**：更新のたびにネットワークは変化します。しかし、次の更新では、その *同じ* ネットワークを使ってターゲット（目標値）が何であるべきかを計算します。これは、動いている的を射ようとするようなものです！

これらの問題は、次のスクリプトで扱う **経験再生** と **ターゲットネットワーク** によって解決されます。これらを組み合わせることで、DQN はよろよろと歩く初心者から、ゲームのチャンピオンへと変貌を遂げます！
