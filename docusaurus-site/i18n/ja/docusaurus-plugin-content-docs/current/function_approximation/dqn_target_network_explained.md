# ターゲットネットワーク：標的（ブルズアイ）を安定させる 🎯

## 動くゴールポスト問題

弓矢で的（ブルズアイ）を射ようとしているところを想像してみてください。矢を放ち、どこに当たったかを確認し、次はもっとうまく当たるように狙いを調整します。簡単ですよね？

では、矢を放つたびに的が動いてしまったらどうでしょう。矢を放つたびに、次のショットのための的の位置がわずかに変わってしまうのです。これでは上達するはずがありません。常に逃げ回るターゲットを追いかけ続けることになります。

それが、ターゲットネットワークのない DQN が直面している問題です！

---

## なぜ Qターゲットは動き続けるのか

DQN において、各更新のターゲット（目標値）は以下の通りです：
> ターゲット = 報酬 + γ × max(Q(次の状態))

ここで **γ (ガンマ)** は **割引率** です。0 から 1 の間の数値（通常は 0.99）で、エージェントが *将来* の報酬を *今すぐ* の報酬と比べてどれくらい重視するかを制御します。

**現実世界の例：** 今すぐクッキーを1枚もらうか、明日2枚もらうかの選択を想像してください。今すぐクッキーが欲しくてたまらないなら、あなたの γ は低くなります（将来を大幅に割り引いています）。我慢強く待てるなら、あなたの γ は高くなります（将来の報酬が今と同じくらい重要です）。強化学習において γ = 0.99 は、「次のステップの報酬は、今の 99% の価値がある」ことを意味します。

この式の右側にある Q値は... 私たちが訓練しているまさにそのネットワークから来ています！

つまり、ネットワークを更新して Q値を良くしようとするたびに、ターゲットも変えてしまっているのです。フィードバック・ループの発生です：

1. ネットワークを更新 → Q値が変化する
2. Q値が変化する → ターゲットが変化する
3. ターゲットが変化する → ネットワークがこれまでとは違う形で更新される
4. これが永遠に続く — 不安定！

**現実世界の例：** 乗るたびに目盛りが変わる体重計で体重を測ろうとするようなものです。本当の体重がいつまで経っても分かりません！

---

## 解決策：標的を凍らせる！ ❄️

**ターゲットネットワーク** は、メインの Qネットワークのコピーであり、その場に固定（フリーズ）されたものです。

- **オンラインネットワーク** (`qnet`)：毎訓練ステップで更新される。素早く学習する。
- **ターゲットネットワーク** (`target_net`)：固定されたコピー。100ステップごとにしか更新されない。

ターゲットの計算には、この固定された（フリーズされた）ネットワークを使用します：
> ターゲット = 報酬 + γ × max(Q_TARGET(次の状態))

標的は 100ステップの間、動きません！これにより、オンラインネットワークに狙うべき安定した目標が与えられます。そして 100ステップごとにオンラインネットワークの重みをターゲットネットワークにコピーし、再び固定し、これを繰り返します。

**現実世界の例：** 生徒と先生を思い浮かべてください。先生は宿題（ターゲット）を出します。生徒は学習し、向上します。100回のレッスンの後、先生は宿題をより難しいものに更新します。先生が毎分毎秒変わってしまったら、生徒は混乱してしまいますよね！

## DQN のフルレシピ 🍕

完全な DQN アルゴリズム（経験再生 + ターゲットネットワーク）は以下の通りです：

```
1. オンラインネットワーク Q とターゲットネットワーク Q_target を初期化（重みは同じ）
2. 経験再生バッファ（メモリボックス）を作成

環境の各ステップにおいて：
  a. Q を使った ε-グリーディ方策で行動を選択
  b. (状態, 行動, 報酬, 次の状態) をバッファに保存

4ステップごとに：
  c. バッファからランダムなミニバッチを抽出
  d. 固定された Q_TARGET を使ってターゲットを計算
  e. 損失を最小化するように Q を更新

100ステップごとに：
  f. Q の重みを Q_TARGET にコピー（ターゲットの同期）
```

これは 2015年の DeepMind の DQN 論文にあるアルゴリズムそのものです！

## 比較から分かること

`dqn_target_network.py` を実行すると、以下のことが分かります：

**ターゲットネットワークなし（経験再生のみの DQN）：**
- 訓練自体は「それなり」に進むかもしれませんが、定期的にパフォーマンスが崩壊します。
- Q値が発散（爆発、または激しく振動）することがあります。
- 学習の予測が困難です。

**完全な DQN（経験再生 + ターゲットネットワーク）：**
- 学習が一貫して向上します。
- Q値が妥当な範囲にとどまります。
- 解決の閾値（CartPole で 195点以上）への収束が早くなります。

## 「死の三つ組 (Deadly Triad)」 ☠️

強化学習において、以下の3つを組み合わせると不安定性が生じます：

1. **関数近似** (テーブルの代わりにニューラルネットを使う) ← 使用中
2. **ブートストラップ** (Q値の推定に Q値を使う) ← 使用中
3. **オフポリス学習** (Q学習は実際の行動ではなく最大値を使う) ← 使用中

これら3つが揃うと「死の三つ組」と呼ばれます。DQN は以下の手法でこれをなだめています：
- 経験再生 → 相関関係を断ち切る
- ターゲットネットワーク → フィードバック・ループを断ち切る

完全に解決するわけではありませんが、管理可能なレベルにまで抑えることができます！

## 覚えておくべきキーワード

| 用語 | 定義 |
|------|---------|
| **ターゲットネットワーク** | ターゲットの計算にのみ使用される、Qネットワークの固定されたコピー |
| **オンラインネットワーク** | アクティブに訓練されている Qネットワーク |
| **同期 (Sync)** | オンラインネットワークの重みをターゲットネットワークにコピーすること |
| **フィードバック・ループ** | システムの出力が入力に跳ね返って変化させてしまうこと（不安定の原因） |
| **死の三つ組** | 関数近似 + ブートストラップ + オフポリス学習の組み合わせ |

## 現実世界への影響

2015年、DeepMind はこれら2つの工夫（経験再生 + ターゲットネットワーク）だけで、49種類の Atari ゲームを人間を超えるレベルでプレイできる AI を示した論文を発表しました。

それまで、人々は強化学習でニューラルネットワークを訓練するのは不安定すぎて不可能だと考えていました。DeepMind はそれが間違いであることを証明し、深層強化学習革命を巻き起こしたのです！

次は、この完全な DQN レシピを、生のピクセルを入力とする本物のビデオゲーム、Atari Pong に適用します！
