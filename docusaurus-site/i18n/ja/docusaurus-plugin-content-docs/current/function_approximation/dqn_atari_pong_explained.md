# Atari Pong での DQN 🏓

## Atari Pong とは？

Pong は、これまでに作られたビデオゲームの中で最も古いものの一つです。デジタル卓球のようなものです！2つのパドルがボールを打ち合います。相手がボールを打ち返せなかったらポイントを獲得します。誰かが 21 ポイントに達するとゲーム終了です。

私たちのバージョンでは、AI が一方のパドルを操作します。対戦相手（コンピュータ）がもう一方を操作します。ゲームは常にスコア -21（最悪の状態）から始まります。優れたエージェントは、スコア 0 または +21 に到達します。

---

## なぜ AI にとって Pong は難しいのか？

CartPole では、ロボットは数値を直接見ることができました（棒の角度、カートの速度など）。
Pong では、見えるのは **生のピクセル** 、つまり画面上の何万もの小さな色の点だけです！

```
CartPole の入力: [0.02, -0.14, 0.01, -0.23]   ← 4つの数字、簡単！
Pong の入力:     [ピクセルグリッド: 210×160×3]    ← 100,800個の数字、はるかに難しい！
```

ロボットはピクセルから以下のことを理解しなければなりません：
- 自分のパドルはどこか？
- ボールはどこか？
- ボールは左と右のどちらに動いているか？
- 速度はどれくらいか？

人間はこれを無意識に行っています（私たちには素晴らしい視覚があるからです！）。AI にとって、これは非常に大きな挑戦です。

---

## 動きを見る：フレーム・スタッキング 🎬

1つのフレーム（スクリーンショット）だけでは、ボールが左と右のどちらに動いているかは分かりません。動きを理解するには、パラパラ漫画のように、複数のフレームを同時に見る必要があります。

**フレーム・スタッキング (Frame Stacking)：** 直近の 4フレームをネットワークに同時に入力します。

```
フレーム 1: ボールが位置 40 にある
フレーム 2: ボールが位置 43 にある    → これら 4つのフレームを重ねる → ネットワークは「動き」を認識！
フレーム 3: ボールが位置 46 にある
フレーム 4: ボールが位置 49 にある
```

これにより、ネットワークは「ボールは速度 3 で右に動いている」と推論できるようになります。

**現実世界の例：** 映画を観るのと、1枚の写真を見るのを比較してみてください。カーレースの写真は単なるブレた画像に過ぎませんが、4枚の連続した写真を見れば、どの車が速いか分かります！

---

## CNN で「見る」 🔍

ピクセル入力に対しては、**畳み込みニューラルネットワーク (CNN: Convolutional Neural Network)** と呼ばれる特別なネットワークを使用します。CNN は、画像を一度にすべて見るのではなく、スライディングウィンドウ（小さな窓）を使ってパターンを検出します。これは、目が画像をスキャンするような仕組みです。

```
生のピクセル (84×84×4フレーム)
       ↓
畳み込み層 1 (8×8 フィルタ, ストライド 4) → エッジや形状を見つける
       ↓
畳み込み層 2 (4×4 フィルタ, ストライド 2) → オブジェクト（パドル、ボール）を見つける
       ↓
畳み込み層 3 (3×3 フィルタ, ストライド 1) → 関係性を見つける
       ↓
平坦化 (Flatten) → 512 個のニューロン → Q値 (各行動に対して1つ)
```

**現実世界の例：** 人混みの中で友達を探すとき、脳はまず形（人であること）に気づき、次に特徴（髪の色）、そして詳細（顔）を確認します。CNN も同じように、単純なパターンから複雑なパターンへと段階的に認識していきます！

---

## 前処理：世界を縮小する

Pong のフレームは 210×160 ピクセルのカラー画像です。これでは大きすぎます！そこで、各フレームに前処理を施します：

1. **グレースケール化** — Pong では色は重要ではありません（ボールは常に白です）。
2. **84×84 にリサイズ** — 小さくすることで訓練を高速化します。それでも十分に視認可能です。
3. **[0, 1] に正規化** — ピクセル値を 255 で割り、小さな数値にします。

**現実世界の例：** 50% のサイズでコピーをとるようなものです。重要な詳細（ボール、パドル）は依然として見えますが、単にサイズが小さくなるだけです。コピー機にとっても色は関係ありませんよね！

---

## 報酬クリッピング：すべてのゲームを平等に扱う ✂️

Pong では、得点すると +1、失点すると -1 の報酬が得られます。
他の Atari ゲームの中には、スコアが何千点にもなるものもあります！

私たちは **報酬を [-1, +1] の範囲にクリッピング** します。これにより、ネットワークは報酬の規模を気にする必要がなくなります。この同じコードを使って、報酬のスケールを調整することなく、あらゆる Atari ゲームを訓練することができます。

---

## 訓練にはどれくらいの時間がかかるか？

| 訓練のステップ数 | エージェントが学習すること |
|---|---|
| 10万ステップ | ほとんどランダム、かろうじて反応する程度 |
| 100万ステップ | 時々ボールの方へ動き始める |
| 500万ステップ | いくつかのショットを打ち返せるようになる |
| 1000万ステップ | 対等にプレイできるようになり、勝てることもある |
| 2000万ステップ〜 | コンピュータの対戦相手に頻繁に勝てるようになる |

私たちのデモは **30万ステップ** 実行します。これは、訓練のアーキテクチャが機能していることを確認し、初期の学習の様子を観察するには十分ですが、ゲームをマスターするには足りません。

**現実世界の例：** ピアノを習得するには何ヶ月もかかります。10分間の練習は、正しい方法で練習できていることを示してくれますが、すぐにコンサートで演奏できるようになると期待してはいけません！

---

## コードから分かったこと

Pong で 30万ステップ訓練した後：
- エージェントはスコア -20 前後から始まります（ほとんどボールを打ち返せません）。
- 最終的には、通常 -15 から -10 前後まで改善します。
- 学習曲線は、ランダムなプレイから徐々に向上していることを示します。

本気で対戦できるレベルの Pong の性能を見るには、GPU を使って 1000万ステップ以上実行する必要があります。実装は完全で正確です。あとは時間が必要なだけです！

## 覚えておくべきキーワード

| 用語 | 定義 |
|------|---------|
| **CNN** | 畳み込みニューラルネットワーク。画像入力に特化したネットワーク |
| **フレーム・スタッキング** | 動きを捉えるために、複数の連続したフレームを入力すること |
| **前処理 (Preprocessing)** | 生のフレームをネットワークに入力する前に変換すること（グレースケール化、リサイズ、正規化） |
| **報酬クリッピング** | 異なるゲーム間で動作するように、報酬を [-1, +1] の範囲に制限すること |
| **ALE** | Arcade Learning Environment。Atari ゲームを実行するためのライブラリ |

## 歴史的快挙

2015年に DeepMind が DQN を発表したとき、世界は驚愕しました。たった一つのアルゴリズム、同じアーキテクチャが、生のピクセルとスコアだけから 49種類もの異なる Atari ゲーム（多くは人間を超えるレベル）の遊び方を学習したからです！

DQN が登場する前、人々はゲームごとに戦略を手書きでコード化する必要があると考えていました。DQN は、汎用的な学習器が自力ですべてを理解できることを証明しました。それは AI の歴史における記念碑的な瞬間でした！
