# ベースライン付き REINFORCE：ノイズを切り裂く

## 素の REINFORCE の問題点

あなたがテストの自分の答えが良かったかどうかを判断しようとしている学生だと想像してください。

**悪いフィードバック：** 「7点でした！」

7点は良いのでしょうか？ 満点が10点なら、良いです！ 他の全員が9点なら、悪いです！ 文脈がなければ、自分の答え方を改善すべきかどうか判断できません。

これが REINFORCE のまさに問題点です：行動を評価するために**生のリターン** (G_t) をそのまま使っています。200点という合計リターンは、状況によって素晴らしくも、ひどくもあるのです。

---

## ベースラインの登場

**ベースライン (baseline)** b(s) とは、「この状況で通常**期待される**報酬はどれくらいか？」という基準点のことです。

「この行動は良かったか？」と聞く代わりに、こう聞きます：

> **「この行動は、普段の期待よりも良かったか？」**

```
古い信号：更新 ∝ G_t
新しい信号：更新 ∝ (G_t - b(s_t))
```

**実生活の例：** 数学のテストで85点を取りました。
- クラスの平均が60点なら → あなたの答えは**平均より25点高い** → 素晴らしい！
- クラスの平均が90点なら → あなたの答えは**平均より5点低い** → もっと頑張ろう！

**アドバンテージ (advantage)** (G_t - b(s)) は、期待より良かったときに正になり、悪かったときに負になります。これははるかに明確な学習信号です！

---

## ベースラインとは何か？

自然なベースラインは、**価値関数 V(s)** です：

> V(s) = 「現在の状態 s にいて、今の方策でプレイした場合に期待される合計報酬」

これを別の **価値ネットワーク（Value Network）** （ベースラインネットワーク、または Critic とも呼ばれます）で学習します：

```
状態  →  [128 ユニット]  →  [128 ユニット]  →  V(s)   (1つの数字)
```

エージェントが訪れる各状態について、V(s) が期待されるリターンを予測します。もし実際のリターン G_t が V(s) よりも高ければ、その行動は期待よりも良かったことになります！

---

## 2つのネットワークが共に学ぶ

```
エピソードが発生
     ↓
実際のリターン G_t を計算
     ↓
         ┌─────────────────────────────┐
         │ アドバンテージ = G_t - V(s_t)│
         │  +：行動は期待以上だった      │
         │  -：行動は期待以下だった      │
         └─────────────────────────────┘
              ↓                  ↓
    方策ネットワークを更新     価値ネットワークを更新
    （良い行動を選ばれやすく、  （次回の予測をより正確
     悪い行動を避けやすくする）  にする）
```

**実生活の例：** 2人の友人がレストランに行きます。

- 友人 1 (価値ネットワーク): 「この料理は 7/10 点くらいだと思うよ」
- 友人 2 (方策ネットワーク): 実際に料理を食べてみて、9/10 点と評価します
- アドバンテージ = 9 - 7 = +2 → 「期待以上だった！また注文しよう！」

次回の訪問時、友人1は予測を 9/10 に近づけます。
友人2は、次回その料理を注文する可能性が高くなります。

---

## なぜこれが分散を減らすのか？

**数学的証明（直感的理解）：**

ベースラインなし：`勾配 ∝ ∇log π(a|s) × G_t`

G_t の値はエピソードごとに大きく変動します：
```
エピソード 1: G = [45, 44, 43, ..., 1]   (中程度のゲーム)
エピソード 2: G = [500, 499, ..., 1]      (最高のゲーム！)
エピソード 3: G = [12, 11, ..., 1]        (最悪のゲーム)
```

G_t が大きくノイズが多いため、勾配の推定値が激しく跳ね上がります。

ベースラインあり：`勾配 ∝ ∇log π(a|s) × (G_t - V(s_t))`

アドバンテージ (G_t - V(s_t)) ははるかに小さく、ゼロ付近に集中します：
```
エピソード 1: アドバンテージ ≈ [-2, +1, -3, ..., 0]   (小さく、中心寄り)
エピソード 2: アドバンテージ ≈ [+10, +8, ..., +3]      (このゲームは最高だった)
エピソード 3: アドバンテージ ≈ [-5, -6, ..., -2]       (このゲームは最悪だった)
```

**実生活の例：** 自分の走る速度を測定する。
- ベースラインなし： 「時速 8km で走った」（文脈がなければ意味が薄い）
- ベースラインあり： 「平均より時速 2km 速く走った」（明らかに良い！）

アドバンテージは常に比較です。そのため、自然と小さく安定したものになります。

---

## 重要なこと：バイアス（偏り）がない！

ベースラインは、アルゴリズムが「何を」学ぶかを変えるのではなく、どれだけ「速く安定して」学ぶかを変えるだけです。

**なぜか？** アドバンテージの期待値は常に 0 だからです：

> E[G_t - V(s_t)] = E[G_t] - V(s_t) = V(s_t) - V(s_t) = 0

行動に依存しないいかなる b(s) も、有効なベースラインとして機能します！

**実生活の例：** 相対評価（偏差値など）で成績をつけても、誰が一番優秀かは変わりません。ただ、スコアの解釈が容易になるだけです。順位は変わらず、尺度だけが変わるのです。

---

## 結果

```
ベースラインなし — 最終100エピソード平均：500.0, 勾配分散：599.3
ベースラインあり — 最終100エピソード平均：491.4, 勾配分散：578.8
```

どちらの手法も CartPole でほぼ完璧なパフォーマンスに達しますが、以下の点に注目してください：
1. **勾配分散 (gradient variance)** は測定可能です（プロットの右側はトレーニング中の分散を示しています）。
2. ベースラインがあると、エージェントはより**確実に**高いパフォーマンスに到達します。トレーニング中に低い報酬へと転落してしまうクラッシュが少なくなります。

分散の削減効果は、より困難な環境（LunarLander、MuJoCoなど）でさらに劇的になります。

---

## 重要な数式

```
ベースラインの価値: V(s) ← V(s) + α(G_t - V(s))   [MSEを最小化]
方策勾配:          θ ← θ + α ∇log π(a_t|s_t) · (G_t - V(s_t))
アドバンテージ:     A_t = G_t - V(s_t)
```

---

## 重要なポイント

| コンセプト | 平易な説明 |
|---------|---------------|
| **ベースライン b(s)** | 状態 s における期待報酬。私たちの基準点 |
| **アドバンテージ A_t** | 「この行動は期待より良かったか？」 |
| **価値ネットワーク** | 期待されるリターンを予測することを学ぶニューラルネット |
| **分散の削減** | 勾配推定のノイズが減り、学習が安定する |
| **不偏 (Unbiased)** | ベースラインは平均的な目標方策を変えず、学習信号のノイズを減らして安定させるだけ |

---

## 次は？

ベースラインは、実はより強力な手法である **Actor-Critic (アクター・クリティック)** 法の始まりに過ぎません。

エピソードの最後になって初めて V(s) を計算する代わりに、Actor-Critic は **時間差 (Temporal Difference)** 学習を使用して、すべてのステップで V(s) を更新します。これにより、更新がはるかに速くなり、不完全なエピソードからも学習できるようになります！

詳細は `a2c_lunarlander.py` の Actor-Critic 実装を参照してください。
