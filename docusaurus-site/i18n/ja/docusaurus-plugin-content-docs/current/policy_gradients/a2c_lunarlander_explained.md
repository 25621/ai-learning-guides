# A2C: ActorとCriticが協力して働く

## 大きなアイデア

REINFORCEは、ゲームが完全に終わるまで更新を待ちます。それは、フットボールの試合中ずっと黙って見ていて、最後にすべてのフィードバックを与えるコーチのようなものです。

**A2C (Advantage Actor-Critic)** は、ゲーム中にフィードバックを与えます。数ステップごとにコーチが中断して言います。「今のパスは素晴らしかった！今のタックルはダメだ！」

これははるかに速く、効率的です。

---

## 2つのキャラクターの紹介

> **LunarLanderとは？** このドキュメント全体を通して、**LunarLander**環境を使用します。これは物理シミュレーションで、小さな宇宙船を制御し、3つのエンジン（左、メイン、右）を使用して月面のターゲットパッドにソフトランディングさせなければなりません。これは強化学習における標準的なベンチマークであり、Gymnasiumライブラリで利用可能です。

### Actor (俳優) 🎭
**Actor**は方策（Policy）であり、どの行動を取るかを決定します。

> 「私はこの状態にいる。左のエンジンを噴射すべきか、それとも右か？」

**実生活の例：** ハンドルを回し、ペダルを踏む車の「運転手」。

### Critic (批評家) 🎬
**Critic**は、現在の状況がどれだけ良いかを推定します。つまり、価値 V(s) です。

> 「この状態にいることは、将来得られる報酬の合計として約+150ポイントの価値がある。」

**実生活の例：** 運転手の隣に座っている「ナビゲーター」。「良い道にいるね。30分で着きそうだ」とか「渋滞に突っ込んでいるよ。これは遅くなりそうだ」と言います。

### 彼らは脳を共有している
私たちの実装では、両者が**同じニューラルネットワークのバックボーン**を使用します：

```
          状態 (LunarLanderの場合は8つの数字)
                       ↓
          ┌─────────────────────────┐
          │  共有レイヤー            │
          │  [256 ユニット] → ReLU   │
          │  [256 ユニット] → ReLU   │
          └────────┬────────┬───────┘
                   ↓        ↓
          Actor Head    Critic Head
          [4つの出力]    [1つの出力]
          (行動確率)     (V(s))
```

- **ReLU** (Rectified Linear Unit): 各レイヤーの後に適用される活性化関数です。`max(0, x)`を出力し、正の値を維持して負の値をゼロにします。これにより、ネットワークは非線形なパターンを学習できるようになります。
- **行動確率 (action probs)**: 4つの各行動を取る確率です。Actorはこの分布からサンプリングして、各ステップの行動を選択します。

**実生活の例：** 1つの脳で2つの仕事。運転（Actor）もし、そのルートが良いかどうかも知っている（Critic）タクシー運転手のようなものです。脳を共有することで、より速く学習できます！

## アドバンテージ：期待より良かったか？

ベースライン付きのREINFORCEと同様に、A2Cは**アドバンテージ（Advantage）**を計算します：

> A(s, a) = 「実際の結果」 − 「期待していたこと」

しかし、ここでの「実際の結果」は、Criticの**nステップ・ブートストラップ（n-step bootstrap）**から得られます（**ブートストラッピング**とは、エピソードが終わるのを待つ代わりに、現在の成績を使って学期末の試験結果を推定するように、Critic自身の予測 V(s) を使って将来のステップの価値を近似することです）：

```
実際のTDリターン: r_t + γ · r_{t+1} + γ² · r_{t+2} + ... + γⁿ · V(s_{t+n})
アドバンテージ A_t = TDリターン - V(s_t)
```

**実生活の例：** この試合で3ゴール決めると予想していたとします (V(s))。もし5ゴール決めたら、あなたのアドバンテージは+2です。もし1ゴールしか決められなかったら、アドバンテージは-2です。

正のアドバンテージ → 「その行動は予想以上に役立った → もっとやろう！」
負のアドバンテージ → 「その行動は予想ほど役立たなかった → 控えよう！」

## なぜ複数の並列環境を使用するのか？

私たちのA2Cは、同時に実行されるLunarLanderの**8つのコピー**を使用します！

**なぜか？** 単一の環境からの経験は**相関（correlated）**しているからです。あるステップは前のステップと密接に関連しています。この相関はニューラルネットワークを騙し、実際よりもパターンが信頼できるものだと思い込ませてしまいます。

8つの環境を使用すると、各ステップで非常に異なる状況から8つの独立した経験が得られます。これにより相関が断ち切られ、トレーニングが劇的に安定します。

**実生活の例：** 天気について学ぶには、どちらが良いでしょうか？
- 1つの都市を8時間連続で観察する（相関がある。午後2時に晴れていたら、3時も晴れている可能性が高い）
- 8つの都市を同時に観察する（相関がない。異なる気象パターン、より多くの情報！）

```
環境 1: [月に着陸、左噴射、クラッシュ、リセット...]
環境 2: [落下が速すぎる、両方噴射、ホバリング、着陸...]
環境 3: [右に傾く、右噴射、安定、着陸...]
...
環境 8: [左に漂う、左噴射、安定、...]
```

これら8つすべてが同時にネットワークを更新します。1回の更新で8倍の多様な経験が得られます！

## Nステップ更新：ゲームが終わるまで待たない

REINFORCEはエピソード全体（1000ステップになることもあります！）を待ちます。

A2Cは **n_steps = 128 ステップ**ごとに更新します：

```
8つの環境で128ステッププレイする
    → 128 × 8 = 1024 個の経験タプルを取得
    → アドバンテージとリターンを計算
    → ActorとCriticを更新
    → さらに128ステッププレイする...
```

**実生活の例：** 試験勉強をしている学生。
- REINFORCEスタイル：教科書を全部読んでから、模擬試験を受ける。
- A2Cスタイル：10ページ読んでクイズを受け、また10ページ読んでクイズを受ける...

頻繁なフィードバック = より速い学習！

## 3つの損失の組み合わせ

A2Cは、同時に3つの損失項（Loss terms）を使用してトレーニングします。

**損失（Loss）**とは、オプティマイザが最小化しようとする数値です。損失が小さいほど、ネットワークの現在の動作がトレーニングの目的に近いことを意味します。

### 1. Actor Loss (方策勾配)
アドバンテージのある行動が選ばれやすくなるようにします：
```
L_actor = -E[log π(a|s) · A(s,a)]
```
もし A > 0 なら、その行動の確率を上げます。
もし A < 0 なら、その行動の確率を下げます。

### 2. Critic Loss (価値関数 MSE)
価値の予測をより正確にします（**MSE** = Mean Squared Error：予測誤差を2乗して平均したもの。2乗することで、小さなミスよりも大きなミスを重く罰します）：
```
L_critic = E[(V(s) - return)²]
```
あらゆる**回帰（regression）**モデル（回帰 = 連続的な数値を予測すること。ここでは期待リターン V(s)）のトレーニングと同様に、予測誤差を最小化します。

### 3. Entropy Bonus (探索)
方策が早すぎる段階で自信を持ちすぎないようにします：
```
L_entropy = -H[π(·|s)] = E[log π(a|s)]
```
高いエントロピー = 多様な行動選択 = 探索
低いエントロピー = 自信に満ちた狭い選択 = 活用

**実生活の例：** エントロピーボーナスは、先生が「選択問題で全部Aを選ぶな！何がうまくいくか学ぶために、違う答えも試してみなさい」と言うようなものです。

```
合計損失 = L_actor + 0.5 × L_critic - 0.01 × エントロピー
```

## LunarLander：より難しい挑戦

**LunarLander-v3** は Gymnasium（旧OpenAI Gym）の環境です。「v3」は第3版であることを示すバージョン番号です。エージェントは小さな宇宙船を制御し、月面の指定されたパッドに安全に着陸させなければなりません。これはCartPoleよりもはるかに困難です：
- 8次元の状態空間（位置、速度、角度、脚の接地、燃料）
- 4つの離散的な行動（何もしない、左噴射、メイン噴射、右噴射）
- 報酬：着陸で+100、クラッシュで-100、燃料の使用による小さなペナルティ

学習曲線は、大きなマイナスの報酬からプラスの報酬へと徐々に改善していく様子を示しています。LunarLander上のA2Cが基本的な安定性を学ぶには、かなりの経験が必要です。

## 重要な数式

```
nステップリターン: G_t = r_t + γ·r_{t+1} + ... + γⁿ·V(s_{t+n})
アドバンテージ:     A_t = G_t - V(s_t)
Actorの更新:      θ_π ← θ_π - α ∇ L_actor
Criticの更新:     θ_V ← θ_V - α ∇ L_critic
```

## 重要なポイント

| コンセプト | 平易な説明 |
|---------|---------------|
| **Actor** | 方策。何をすべきかを決定する |
| **Critic** | 価値関数。状況がどれだけ良いかを判断する |
| **Advantage** | 「これは期待より良かったか？」（実際 - 期待） |
| **Nステップリターン** | V(s) でブートストラップする前に n ステップ先を見る |
| **並列環境** | 相関のない多様な経験を得るための複数の環境 |
| **エントロピーボーナス** | Actorが新しいことを試し続けるように促す |

## 次は？

A2Cは素晴らしいですが、1つの大きな弱点があります。時々、方策をあまりに急激に更新しすぎることがあります。1つの悪い更新が、それまでの学習で得た良い成果をすべて台無しにしてしまう可能性があります。

**PPO (Proximal Policy Optimization)** は、一度の更新で方策が変わりすぎないようにする賢い「セーフティクリップ」を使用して、これを修正します。

PPOの実装については、`ppo_scratch.py`を参照してください！
