# PPOのハイパーパラメータ感度：何が最も重要か？

## なぜハイパーパラメータが重要なのか

チョコレートケーキを焼くことを想像してみてください。レシピにはこうあります：
- 卵 2個
- 小麦粉 200g
- ベーキングパウダー 小さじ1
- 180℃で35分

卵を10個使うと、ケーキは爆発してしまいます。ベーキングパウダーが小さじ0.1だと、膨らみません。
300℃で10分焼くと、外側は焦げて中は生のままです。

**PPOにおけるハイパーパラメータは、材料やオーブンの設定のようなものです。** 正しい組み合わせなら素晴らしく機能しますが、間違った設定では学習が全く進まないこともあります。

このスクリプトでは、一度に1つのパラメータだけを変更し、それぞれ3つの異なるランダムシードで実行して結果を比較することで、3つの主要なハイパーパラメータを系統的にテストします。

---

## 3つの実験

### 実験1：クリップ・イプシロン (ε)

```
ε = 0.05   (非常に保守的 — 方策の極めて小さな変更のみを許可)
ε = 0.2    (標準的 — 安全性とスピードのバランスが良い)
ε = 0.4    (攻撃的 — 方策の大きな変更を許可)
```

**ε は何を制御するのか？**

ε は、古い方策の周囲にある「安全窓（safety window）」のサイズです：
```
確率は [1 - ε, 1 + ε] の範囲内に留まらなければなりません。
ε=0.05: 比率が [0.95, 1.05] 内  ← 極小の変更
ε=0.2:  比率が [0.80, 1.20] 内  ← 標準
ε=0.4:  比率が [0.60, 1.40] 内  ← 大きな変更
```

**実生活の例：** ε を「一度の操作でどれだけハンドルを切ってよいか」と考えてみてください。
- ε=0.05: 氷の上を運転しているようなもの。極めて小さな調整のみ。
- ε=0.2:  通常の運転。適切な転換。
- ε=0.4:  レーシングドライバー。攻撃的なステアリング。**スピン（制御不能）**のリスクがあります（変更が急激すぎて、車が道から滑り出すような状態）。

**期待される結果：**
- ε=0.05: 遅く、しかし安定した学習（慎重すぎる）。
- ε=0.2:  良いバランス（いわゆる**「ゴルディロックス」な値** — 童話の『3びきのくま』で、女の子が熱すぎず冷たすぎないちょうど良い温度のスープを選ぶように、大きすぎず小さすぎない最適な値を指します）。
- ε=0.4:  速く学習できる可能性があるが、**オーバーシュートや振動**が起きやすい（オーバーシュート = 最適な方策を通り過ぎてしまうこと。振動 = 定着せずにその周りを揺れ動くこと）。

### 実験2：学習率 (Learning Rate)

```
lr = 1e-4  (低速だが安定)
lr = 3e-4  (標準)
lr = 1e-3  (高速だがリスクあり)
```

**学習率は何を制御するのか？**

学習率は、丘を登る際の「歩幅」のようなものです（各ステップはニューラルネットワークの重みの1回の更新に対応し、報酬を改善する方向に少しずつ移動します）：
- 小さすぎる：頂上に着くまでに永遠に時間がかかる（収束が遅い）。
- 大きすぎる：頂上を通り過ぎて反対側に落ちてしまう（**発散（diverge）** — 学習が着実に進む代わりに、報酬が崩壊したり激しく変動したりする）。
- ちょうど良い：頂上に向かって着実に前進する。

**実生活の例：** ギターの弦のチューニング。
- lr=1e-4: ペグ（弦を締めたり緩めたりするためのつまみ）をほんの少しずつ回す。正確だが永遠に時間がかかる。
- lr=3e-4: 通常のチューニング。数回で正しい音程を見つける。
- lr=1e-3: ペグを**グイッ（突然強く引く）**と回す。弦が**切れてしまう（断線）**かもしれません（大きすぎる更新がトレーニングを修復不能なほど破壊してしまうのと同じです）！

**期待される結果：**
- lr=1e-4: 最終的には良くなるが、非常に遅い。
- lr=3e-4: 全体的に最高のパフォーマンス。
- lr=1e-3: 最初は速いが、その後に不安定になる。

### 実験3：更新エポック数 (K)

```
K = 3   (保守的 — 各バッチを数回だけ学習)
K = 10  (標準)
K = 20  (攻撃的 — 各バッチを何度も学習)
```

**更新エポック数は何を制御するのか？**

新しい経験を集めるためのゲームプレイ（**ロールアウト**）の後、PPOはその経験を**バッチ**（状態、アクション、報酬のセット）にまとめます。そして、同じデータに対して K 回の**パス**（バッチ全体を走査してネットワークを更新すること）を実行します。
エポック数が多いほど、各バッチからより多くの学習を引き出せますが、**古いデータへの過学習（overfitting）**のリスクがあります（古い方策の下では真実だったパターンを暗記してしまい、方策が更新された後には通用しなくなること。去年の試験問題を暗記して、新しい試験に落ちる学生のようなものです）。

**実生活の例：** 20問の数学の問題集を練習している学生。
- K=3:  各問題を3回解く → まだ学習中。練習セットを丸暗記はしない。
- K=10: 各問題を10回解く → それらの特定の問題をしっかりマスターする。
- K=20: 各問題を20回解く → **数学を理解せずに答えを丸暗記してしまう**（モデルが特定のバッチに完璧に適合するが、汎用性を失う）！

> ⚠️ **「K=20 でも結果は良さそうに見えるけど、なぜ気にする必要があるの？」**
> PPOのクリッピング・トリックにより、1回ごとの更新量は制限されるため、K=20 が突然の崩壊を引き起こすことはありません。
> しかし、エージェントは「現在の最新の方策が実際に遭遇するはずの状態」を反映しなくなった古いデータに対して、気づかないうちに過剰に適応してしまいます。
> これにより**長期的な学習が遅くなります**。後半のパスになるほど情報の「鮮度」が落ちるため、1回のロールアウトから得られる真の学習効果が低下してしまいます。
> このダメージは劇的ではなく、徐々に進行するため、短い実験では見落とされやすいのです。

クリッピングは壊滅的な過学習を防ぎますが、エポック数が多すぎると全体的な学習速度が低下する可能性があります。

**期待される結果：**
- K=3:  効率が低い（バッチあたりの学習の可能性を使い切れていない）。
- K=10: 良いバランス。
- K=20: **鮮度の落ちたデータに対して方策が自信を持ちすぎる**リスク（現在の方策が遭遇するものとは異なる過去の経験に基づいて更新が行われ、サンプル効率をじわじわと低下させる）。

## 結果の読み方

グラフには、各ハイパーパラメータを変化させた3つの図が表示されます：

```
左のグラフ：   クリップ・イプシロン — どの ε が最も早く学習するか？
真ん中のグラフ：学習率 — どの lr が最も安定しているか？
右のグラフ：   更新エポック数 — どの K が最高の方策を見つけるか？
```

各線は、ランダム性を抑えるための **3つのシードの平均報酬** を示しています。

**注目すべき点：**
1. **学習速度：** どの線が最も早く高い報酬に達しているか？
2. **最終的なパフォーマンス：** 最終的に最も高い報酬を達成しているのはどの線か？
3. **安定性：** 振動（上下の揺れ）が最も少ないのはどの線か？

優れたハイパーパラメータは、これら3つのバランスが取れています！

## 手法：科学的な実験

この実験では**アブレーション（除去）研究（ablation study）**の手法を採用しています（1つの要素だけを取り除いたり変更したりして、その個別の影響を測定する手法。生物学で特定の組織を選択的に除去して機能を調べることに由来します）：
1. デフォルト値を決める：ε=0.2, lr=3e-4, K=10
2. 一度に1つのパラメータだけを変更する
3. 他のすべては固定する
4. 結果を比較する

これにより、各パラメータが個別に与える影響を知ることができます。

**実生活の例：** 新しい肥料が植物に効くかテストする：
- 肥料だけを変え、他（土、水、日光）はすべて同じにする。
- 植物がより良く育てば、それは肥料のおかげです！

## 実践における一般的な知見

| ハイパーパラメータ | 小さすぎる場合 | スイートスポット | 大きすぎる場合 |
|----------------|-----------|------------|-----------|
| **ε (clip)** | 収束が遅い | ε ≈ 0.2 | 不安定 |
| **lr** | 遅すぎる | 2.5e-4 〜 3e-4 | 発散 |
| **K (epochs)** | **データの無駄** (信号を抽出しきる前に捨てる) | K = 4-10 | 古いデータへの過学習 |
| **n_steps** | ノイズが多い | 128-2048 | **OOM メモリ不足** (RAMを使いすぎる) |
| **batch_size** | ノイズが多い | 32-256 | **OOM メモリ不足** (RAMを使いすぎる) |

これらの「スイートスポット」は環境によって変わることがあります！

## 重要な洞察：PPOは比較的堅牢である

ターゲットネットワークのないDQNなどの以前のアルゴリズムと比較して、PPOはハイパーパラメータの選択に対して比較的堅牢です。クリッピング・メカニズムが自然なセーフティネットを提供しています。

**実生活の例：** **ABS**（アンチロック・ブレーキ・システム — 急ブレーキ時に車輪がロックするのを防ぎ、ハンドル操作を可能にする安全機能）付きの車とそうでない車の違い：
- ABSなし（DQN）：一度の操作ミス（間違ったハイパーパラメータ）でスピンアウト。
- ABSあり（PPO）：車が自分自身を修正してくれる。妥当な範囲のハイパーパラメータであれば、どれもそこそこ機能する。

この堅牢性こそが、実務においてPPOが最も人気のある強化学習アルゴリズムである大きな理由です！

## 重要なポイント

| コンセプト | 平易な説明 |
|---------|---------------|
| **アブレーション研究** | その影響を見るために、一度に1つのことだけを変えること |
| **クリップ・イプシロン ε** | 安全境界線。通常 0.2 が最適 |
| **学習率 (Learning rate)** | **歩幅** — 各バッチの後にネットワークの重みをどれだけ調整するか。**2.5e-4 〜 3e-4** は科学的表記で 0.00025 〜 0.0003 を意味します。これらは無次元の倍数であり、時間ではありません。 |
| **更新エポック数 K** | 各バッチを何回再利用するか。4-10回が標準的 |
| **ランダムシード** | 各実験は異なる**ランダムシード**（乱数生成器に与えられる初期値。学習中のすべてのランダムな選択を制御する）で繰り返されます。複数のシードを使用することで、結果が安定したものか、単に運が良かっただけなのかが分かります。 |

## まとめ：方策勾配法の比較

```
REINFORCE              A2C                    PPO
     │                  │                      │
エピソード単位         nステップ更新           nステップ + クリッピング
単純だがノイズ大       高速だが不安定         安定 + 効率的
簡単な環境に最適       中程度の難易度         困難な環境に最適
                       の環境向け             (業界標準)
```

**このフェーズでアルゴリズムを1つだけ学ぶなら、PPOを学んでください。** それは以下の基礎となっています：
- OpenAIのChatGPTトレーニング（RLHFはPPOを使用）
- DeepMindのAlphaGoの後継モデル
- ほとんどの最新のロボティクス研究
- ビデオゲームをプレイするAI
