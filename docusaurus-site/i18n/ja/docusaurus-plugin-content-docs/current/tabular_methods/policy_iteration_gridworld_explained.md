# GridWorld の方策反復 🗺️

## それは何？

あなたが **4×4 のグリッド**（小さなチェス盤のようなもの）の上でボードゲームをしているところを想像してみてください。一方の角からスタートし、もう一方の角に到達する必要があります。一歩進むごとに 1 ポイントのコストがかかります（無駄な歩数は避けたいですよね！）。ゴールに到達しても追加の報酬はありません。単に、できるだけ早くそこにたどり着きたいだけです。

**方策反復 (Policy Iteration)** は、ボード上の **すべてのマスに対して最適な動き** を一度にコンピュータが導き出す方法です！

---

## 大きなアイデア：2つのステップを何度も繰り返す

助手と一緒に自分の部屋を掃除するようなものだと考えてください：

1. **ステップ 1 — 各マスの良さを把握する (方策評価)**
   助手がすべてのマスを歩き回り、こう書き留めます：「今の計画に従った場合、ここから出口に到達するまでに何歩かかるだろうか？」彼らは数字が変化しなくなるまで、これを何度も繰り返します。

2. **ステップ 2 — 計画を改善する (方策改善)**
   今度はあなたが各マスを見て、こう問いかけます：「ここからもっと良い方向に行ける可能性はないだろうか？」もしあれば、計画を更新します！

計画が変化しなくなるまでステップ 1 と 2 を繰り返します。それが **最適方策 (optimal policy)** です！

**現実世界の例：** 学校への最短ルートを見つけることを想像してください。まずルートを推測し、時間を計ります（ステップ 1）。次に、各街角で「ここから近道はないか？」と問いかけます（ステップ 2）。ルートを更新し、それ以上近道が見つからなくなるまで繰り返します！

## コードから分かったこと

私たちの 4×4 GridWorld には2つの終端状態（角）があり、エージェントは1ステップごとに -1 を支払います。方策反復はわずか **4 ラウンド**（合計 139 回の評価スイープ）で収束しました：

```
状態価値 V(s):           最適方策:
 0.0  -1.0  -1.9  -2.7    T   ←   ←   ↓
-1.0  -1.9  -2.7  -1.9    ↑   ↑   ↑   ↓
-1.9  -2.7  -1.9  -1.0    ↑   ↑   ↓   ↓
-2.7  -1.9  -1.0   0.0    ↑   →   →   T
```

**価値は完全に理にかなっています！** 終端状態の隣のマスは価値が -1（1歩離れている）です。2歩離れたマスは価値が -1.9 (= -1 + 0.9 × -1) となり、以下同様です。

## 現実世界の例

- **GPS ナビゲーション**：マップ上の*すべての*交差点で最適な曲がり方を割り出す。
- **エレベーター制御**：複数のリクエストがある場合、エレベーターはどの階に行くべきか？
- **工場のロボット**：倉庫のグリッド内で最も効率的な経路を計画する。

---

## 覚えておくべきキーワード

- **方策 (Policy)**：計画。各状態でどのような行動をとるか
- **価値関数 V(s)**：状態 s にいることがどれくらい良いか（高いほどゴールに近い）
- **方策評価 (Policy Evaluation)**：現在の計画がどれくらい良いかを計算すること
- **方策改善 (Policy Improvement)**：価値関数を使用して計画をより良くすること
- **最適方策 (Optimal Policy)**：可能な限り最高の計画。これ以上改善できない

大きなアイデア： **考えられるすべての計画を試す必要はありません！現在の計画を改善し続けるだけで、非常に少ないラウンド数で最高の計画を見つけることができます。**
