# マトリクスゲーム：最もシンプルなマルチエージェントの世界 🎲

## マトリクスゲームとは何か？

あなたと友人が、**じゃんけん**（グー、チョキ、パー）のいずれかを*同時に*出す場面を想像してください。お互いの出す手は見えません。勝敗は、次のような小さなテーブルによって決まります。

| | グー | パー | チョキ |
| :--- | :---: | :---: | :---: |
| **グー** | 0, 0 | -1, +1 | +1, -1 |
| **パー** | +1, -1 | 0, 0 | -1, +1 |
| **チョキ** | -1, +1 | +1, -1 | 0, 0 |

このテーブルこそが、ゲームの*世界のすべて*です。移動も、時間の経過も、マップもありません。ただ一回限りの意思決定があるだけです。この報酬行列（ペイオフマトリクス）が環境のすべてであるため、これを**マトリクスゲーム**と呼びます。

マトリクスゲームは、**マルチエージェント強化学習（MARL）**を研究する上で最もクリーンな場所です。なぜなら、学習中に変化し得るのは各プレイヤーの*ポリシー*（各行動を選択する確率）だけだからです。

---

## なぜ「マルチエージェント」なのか

シングルエージェントの強化学習では、環境は固定されています。風は常に同じように吹き、床が勝手に動くことはありません。エージェントは上達し、最終的に勝利します。

マトリクスゲームにおけるあなたの「環境」は、*別の学習するエージェント*です。相手が賢くなるにつれて、あなたにとっての「良い手」も*変化*します。これは**非定常性（non-stationarity）**と呼ばれ、マルチエージェント強化学習における中心的な課題です。

> もしあなたがグーを出し続ければ、相手はいずれ常にパーを出すようになるでしょう。そこであなたはチョキに切り替えます。すると相手はグーに切り替え、あなたはパーに切り替え……といった具合に。「最善の手」が固定されることはありません。

古典的な解決策は**混合戦略（mixed strategies）**です。一つの行動を決定的に選ぶのではなく、相手に利用されないようにランダムに手を選択します。

## ここで扱う3つのゲーム

### 1) じゃんけん（零和ゲーム：ゼロサム）
- 一方の利益は、もう一方の損失となります。
- **ナッシュ均衡**は、各プレイヤーが各行動を1/3の確率で選択することです。この確率から外れると、相手に利用される隙が生まれます。
- 2つのQ学習エージェントは、1/3-1/3-1/3の周りをふらふらと動くことが予想されます。一方が偏ればもう一方が反応するため、完全に静止することはありません。

### 2) 囚人のジレンマ（一般和ゲーム）
2人の容疑者が別々に取り調べを受けています。

| | 協調（Cooperate） | 裏切り（Defect） |
| :--- | :---: | :---: |
| **協調** | 3, 3 | 0, 5 |
| **裏切り** | 5, 0 | 1, 1 |

- 相手がどう動こうと、「裏切り」は「協調」に勝ります。これは**支配戦略**と呼ばれます。
- 両プレイヤーが合理的であれば、共に「裏切り」を選択し、両者とも1の利益しか得られません。両者が「協調」すれば3ずつ得られたはずですが、利己的な最善の反応がグループ全体の利益を損なってしまいます。
- Q学習は、きれいに（裏切り、裏切り）へと収束することが予想されます。

### 3) 鹿狩り（協調ゲーム）
2人のハンターが協力して大きな鹿（高い報酬）を仕留めるか、それぞれが個別にウサギ（小さいが確実な報酬）を捕まえるかを選択します。

| | 鹿（Stag） | ウサギ（Hare） |
| :--- | :---: | :---: |
| **鹿** | 4, 4 | 0, 3 |
| **ウサギ** | 3, 0 | 2, 2 |

- （鹿、鹿）は**報酬支配的**であり、両者にとって最善です。
- （ウサギ、ウサギ）は**リスク支配的**であり、相手を信頼できない場合には安全な選択です。
- 結果は初期条件に依存します。独立したQ学習エージェントは、ウサギの方が安全に学習できるため、より*悪い*均衡である（ウサギ、ウサギ）に陥ることがよくあります。

## 現実世界の例

- **デュオポリー（二社独占）における価格設定**：同じ通りにある2つのコーヒーショップが、毎朝価格を決めます。報酬行列の形によって、高い「協調的」な価格（店には良いが客には悪い）に落ち着くか、安売り競争に陥るかが決まります。
- **ネットワークプロトコル**：ルーターと送信機がタイミング戦略を選択します。ネットワークの混雑状況は、通信の成功と待機というマトリクスゲームのような報酬によって決定されます。
- **オークションでの入札**：各入札者は他人の出方を知らずに入札額を決めます。報酬は入札額のベクトル全体に依存します。ナッシュ均衡は単一の数値ではなく、*入札戦略*となります。

## コードが行っていること

各ゲームにおいて、以下の手順を踏みます：
1. 2つの状態を持たないQ学習エージェントを作成します（Q値は行動ごとの一つの数値に過ぎません。一回限りのゲームには「状態」が存在しないためです）。
2. 20,000ステップのループを実行します。各ステップで、両エージェントが同時にε-greedy行動を選択し、報酬を受け取り、Q値を更新します。
3. 直近500ステップのウィンドウにおける、各エージェントの**経験的行動頻度**を追跡します。抽象的な確率を見るのではなく、実際に直近でどの行動をどれくらい選んだかをカウントします（例：「過去500ラウンドで、グーを40%の確率で出した」など）。これにより、変化し続ける戦略をリアルタイムで具体的に把握できます。
4. 頻度の推移をプロットして `outputs/<game>.png` に保存し、最終的なQ値を出力します。

### 確認できること

| ゲーム | プロットの予想される結果 |
| :--- | :--- |
| **じゃんけん** | 両プレイヤーとも1/3付近を漂いますが、目に見えて小刻みに動きます。曲線が互いを追いかけ合う、典型的なサイクリング（循環）行動が見られます。 |
| **囚人のジレンマ** | 両プレイヤーの「裏切り」の頻度がすぐに1.0付近まで上昇します。「協調」は消え去ります。 |
| **鹿狩り** | 多くの実行では（ウサギ、ウサギ）に落ち着きます。運が良いと（鹿、鹿）に到達することもあります。スクリプトのシード値を変えて、結果が入れ替わるのを確認してみてください。 |

## 独立学習の限界

ここでのエージェントは**独立的（independent）**です。自分の報酬しか見えず、相手の行動やQ値は知りません。これは最も単純なベースラインですが、限界があります。

- 一般和ゲームにおいて、**収束を保証できません**。
- **悪い均衡**に陥ることがあります（鹿狩り）。
- **相手をモデリングできません**。

現実のマルチエージェントアルゴリズムは、相手の学習者を明示的に考慮することでこれらを解決します。以下に代表的なものを紹介します：

| アルゴリズム | 核心となるアイデア | 現実世界の例え |
| :--- | :--- | :--- |
| **仮想対局 (Fictitious play)** | 相手が各行動を選んだ頻度の累計を記録します。明日も相手はこれまでと同じように動くと仮定し、その信念に対する自身の最善の反応を選択します。 | 多くのチェスの対局を通じて相手の癖を把握し、それに応じてオープニングを調整する。 |
| **CFR (Counterfactual Regret Minimization)** | 毎ラウンド後に「他の行動を選んでいたらどれだけ良かったか（後悔）」を計算します。選ばなかった行動への後悔の大きさに応じて、徐々に確率をシフトさせます。ポーカーのような**不完全情報**ゲームで使われます。 | ポーカーのハンドが終わった後、「もっと賭けるべきだった。次はそうしよう」と反省する。 |
| **LOLA (Learning with Opponent-Learning Awareness)** | 相手も勾配更新を行っているという事実を考慮して、自身の勾配ステップを計算します。相手の次の更新を予測しながら自身の更新を最適化します。一歩先ではなく二歩先を読みます。 | 交渉において、「自分がXを提示すれば、相手はYで返してくるだろうから、自分はZから始めるべきだ」と考える。 |
| **MADDPG** | 各エージェントの「評論家（critic）」は、全員の観測と行動が見える**グローバルな視点**でトレーニングされます。一方で「アクター（actor）」は、実行時には自分の情報しか使いません（CTDE：中央集中型学習・分散型実行）。 | バスケットボールのコーチはコート全体を見ているが（中央集中型評論家）、各選手には自分が見える範囲の情報だけで動くように教える。 |

しかし、独立Q学習は最初の一歩として最適です。非定常性の問題が目の前に立ちはだかるのを確認できれば、その後の解決策の意味がより深く理解できるでしょう。

## 覚えておくべきキーワード

| 単語 | 意味 |
| :--- | :--- |
| **報酬行列（ペイオフマトリクス）** | 一回限りのマルチエージェントゲームを定義するテーブル |
| **ナッシュ均衡** | どの一人のエージェントも、自分だけ戦略を変えても得をしない戦略の組み合わせ |
| **混合戦略** | 複数の行動をランダムに使い分けるポリシー |
| **非定常性** | 学習が進むにつれて、環境（＝他のエージェント）が変化し続けること |
| **独立学習者** | 他の学習者の存在を無視して学習するエージェント |
| **零和（ゼロサム）** | 一方の得が、もう一方の損と正確に一致すること |
| **一般和** | 両者が勝つ、両者が負ける、あるいはその中間があり得ること |

## 一行まとめ

> **マトリクスゲームでは「環境」そのものが別の学習者であるため、最善の一手は常に動き続ける。**

これは、自己対戦からMADDPG、通信を伴うマルチエージェント学習に至るまで、今後出会うあらゆるマルチエージェントアルゴリズムの基礎となる考え方です。
