# 自己対戦（Self-Play）：エージェント自身と対戦させて学習させる ♟️

## 自己対戦（Self-Play）とは何か？

チェスがとても上手くなりたいけれど、対戦相手がいない子供を想像してみてください。そこで彼女は自分自身と対戦することにしました。左手対右手です。毎回のゲームで、*両方の*側が勝とうと努力します。そして、*両方の*側が何がうまくいったかを学習します。

これが**自己対戦（self-play）**です。単一のエージェントが両方のプレイヤーとして振る舞い、あらゆる一手が、次に動く誰か（自分自身）への教訓となります。教師も、エキスパートの対戦相手もいません。学習者自身が、自分を引き上げるためのはしごとなります。

自己対戦は手品のように聞こえるかもしれません。本当の対戦相手が必要なはずでは？と。しかし、自己対戦こそが、過去10年間の強化学習における最も有名なマイルストーンの原動力なのです：**AlphaGo Zero**、**AlphaZero**、**MuZero**、**OpenAI Five**。これらはすべて自己対戦を利用しています。理由は単純です。自分が上達すれば、対戦相手も同じ分だけ上達するからです。挑戦のレベルは、常にあなたのスキルに自動的に一致します。

---

## なぜ機能するのか

自己対戦を特別なものにしている3つの要素があります：

1. **無限の対戦相手**：対局がなくなることはありません。対戦相手は常にそこにいて、コストもかかりません。
2. **自分と共に成長するカリキュラム**：初心者は初心者としか対戦できません。自分が上達すれば、影である対戦相手も自動的に上達します。
3. **対称性**：零和ゲーム（一方の勝利がもう一方の損失となるゲーム）では、一つのQ値のセットで両側を表現できます。相手の番になったら符号を反転させるだけでよいのです。そのため、**単一の**Qテーブルがそれ自身を教えることができます。

○×ゲーム（Tic-tac-toe）は、これを確認するのに最適なテストベッドです。辞書に収まるほど小さいながらも、ランダムに手を選ぶだけでは戦略的なプレイヤーにほぼ確実に負けてしまう程度の複雑さを備えています。

## 現実世界の例え

- **壁に向かってテニスの練習をする**：壁に負けることはありませんが、サーブの練習はできます。自己対戦は、これを両側で行うようなものです。あなたは壁であり、プレイヤーでもあり、それらを交互に切り替えます。
- **両方の立場から議論するディベート部**：自分が個人的に信じていることとは逆の立場を常に擁護することで、より優れたディベーターが生まれます。あらゆる議論が、攻撃と防御の両方を訓練します。
- **AlphaGo Zero**：人間の対局データから一切学習しませんでした。ランダムな動きからスタートして、自分自身と数百万回の対局を行いました。数日のうちに、イ・セドルを破ったプログラムを含む、それまでのあらゆる囲碁プログラムを凌駕しました。

## コードが行っていること

「現在の手番のプレイヤー」のためのQテーブルを一つ学習させます：

```
Q[(盤面, 手番のプレイヤー)][行動] = そのプレイヤーにとっての期待リターン
```

トレーニングループは以下の通りです：

1. 空の盤面から開始します。`player = X` とします。
2. ε-greedy法を用いて、両方のプレイヤーが**同じエージェント**として行動します。
3. 各対局の後、履歴にあるすべての（盤面、プレイヤー、行動）のトリプルを遡り、Q学習の更新を適用します。
4. 報酬はターンごとに符号が反転します：もしXが勝てば、Xが行ったすべての手に +1 が与えられ（または将来の勝利状態から価値をブートストラップし）、Oが行ったすべての手には -1 が与えられます。
5. 探索率（ε）を 0.2 から 0.02 へと徐々に減少（減衰）させます。これにより、トレーニングの後半ではランダムな手を試すのではなく、最善の手を打つようになります。

2,500エピソードごとに、エージェントを**ランダムな対戦相手**と対戦させて評価します（評価中はQテーブルを更新せず、両者が貪欲に行動します）。十分に自己対戦を繰り返した後、エージェントはこれらの対局のほぼ100%で勝利または引き分けになるはずです。

| 学習済みエージェント vs 自分自身 (200ゲーム) | **200ゲームすべて引き分け**。○×ゲームは、両者が完璧にプレイすれば必ず引き分けになるゲームです。自己対戦ですべて引き分けになることは、収束の証です。 |

## 注意すべき細かな点

- **符号の反転が重要**：よくあるバグとして、「相手が自分の価値を最大化する」ことは、ブートストラップのターゲットにおいては「自分の価値を最小化する」ことであるのを忘れてしまうことがあります。コード内での更新は `target = reward - gamma * max(Q[next, opponent])` となっています。
- **ここでは対称性を利用していない**：実際の高度な実装では、盤面を正規化（回転や反転させて標準的な形式に統一）することで、8つの対称的な状態でQ値を共有します。ここでは、状態空間が全探索できるほど小さいため、そのステップは省略しています。
- **Qテーブルの増大**：50,000回の自己対戦後、数千個の状態・プレイヤーのキーが生成されます。ここでは問題ありませんが、チェスや囲碁の場合は代わりにニューラルネットワークが必要になります。そのため、**AlphaZeroではテーブルをCNN + MCTSに置き換えています**。

## 自己対戦がうまくいかないケース

- **非零和ゲーム**：対称的なプレイと「両者がハッピーになること」は両立しません。単に符号を反転させるだけでは対応できません。
- **非対称な役割**：「攻撃側」と「守備側」で行動空間が異なる場合、2つの独立したネットワークが必要になります。
- **戦略の循環**：純粋な自己対戦では、じゃんけんのような循環的な戦略に陥ることがあります。AlphaStarでは、過去のバージョンのエージェントを大量に保存した「リーグ（プール）」を維持し、そこからランダムに対戦相手を選ぶことで、現在の自分だけでなく様々なプレイスタイルに勝てるように学習させてこの問題を解決しました。
- **報酬ハック**：自己対戦は両者を賢くしますが、それは*あなたが定義した*ゲームにおいてのみです。もし報酬システムに意図しない抜け穴（例：勝つことよりも、単に長く生き残ることに報酬を与えてしまうなど）があれば、両者がその抜け穴を互いに利用し合い、本来のゲームの習得とは異なる奇妙で役に立たない行動をとるようになります。

## 覚えておくべきキーワード

| 単語 | 意味 |
| :--- | :--- |
| **自己対戦 (Self-play)** | 同じエージェントがゲームの両側をプレイすること |
| **零和 (Zero-sum)** | 一方の利益 ＝ もう一方の損失となる関係 |
| **対称性** | 符号を反転させることで、一つのQテーブルを両側で共有できること |
| **ポピュレーション・プレイ** | 過去の自分自身の多くのバージョンを対戦相手とする自己対戦（AlphaStar） |
| **カリキュラム** | 難易度が自然に上がっていく過程。自己対戦ではこれが無料で手に入る。 |
| **MCTS** | モンテカルロ木探索。AlphaZeroが自己対戦と組み合わせているプランニングアルゴリズム。 |

## 一行まとめ

> **自己対戦は上達を自らのはしごに変えます。自分が上手くなるたびに、対戦相手も自動的に上手くなるからです。**
