# Montezuma's Revenge での訓練 🏛️🔑

## なぜこのゲームが（強化学習の世界で）有名なのか

2015年、DeepMind の DQN は、生のピクセルから人間を超えるレベルで数十の Atari ゲームをプレイすることを学習しました。これは大きなニュースになりました。しかし、結果の表の中に、DQN が **0** 点（何もしないのと同じスコア）しか取れなかったゲームが1つありました。それが **Montezuma's Revenge** です。

なぜでしょうか？最初の部屋で求められることを見てみましょう：

1. 梯子を **降りる**。
2. 狭い足場を渡る。
3. 転がってくる頭蓋骨をジャンプして避ける（タイミングを間違えると死ぬ）。
4. 別の梯子を **登る**。
5. 鍵を掴む。

これには約 **100回の正確なボタン入力** が必要ですが、鍵を手にするまでゲームは **1ポイントも与えてくれません**。オープニングシーケンスの間中、報酬シグナルは平坦で特徴のない **ゼロ** のままです。

通常の強化学習エージェントは、実際に受け取った報酬に向かって調整することで学習します。もし手が届く範囲の報酬がどこもかしこもゼロなら、*学習すべき材料が何もない* ことになります。これは、完全に平らな谷の底を、下り坂の方向を探ることで見つけようとするようなものです。そのため、DQN は開始地点のプラットフォーム付近で永遠にピクピク動いているだけでした。Montezuma は、ランダムではなく **賢く** 探索しなければクリアできないゲームとして、**困難な探索 (hard exploration)** のベンチマークとなりました。

突破口は2018年の **Random Network Distillation (RND)** によってもたらされました。そのトリックこそが、まさに本教材のワーク項目1の主題です。エージェントが新しい画面に到達したときに自分自身に報酬を与える **内発的好奇心ボーナス** を追加したのです。すると突然、エージェントをステージの奥深くへと引き寄せる濃密なシグナルが生まれました。RND は Montezuma で人間を超えるスコアを記録しました（その後、Go-Explore、Agent57 などが登場しました）。

## 「Montezuma スタイル」の疎な報酬の現実世界の例

- **ダイヤル錠 / 暗号めいた手がかりのある宝探し**：部分点はありません。景品を手にするまでずっとゼロです。
- **論文の受理や、スタートアップの収益化**：数ヶ月間、外部からの報酬はゼロで、その後に（運が良ければ）大きな報酬が1つ得られます。
- **ビデオゲームのスピードランのルート**：数十回のフレーム単位の正確な入力を連続で行い、そのトリックが成功するかしないかまでフィードバックはありません。
- **脱出ゲーム**：いくつかの発見を鎖のようにつなぎ合わせるまで、部屋はほとんど何も教えてくれません。

これらすべてにおいて、「ただランダムに試す」のは絶望的です。**計画的に** 探索する必要があります。そして、「お、これは新しいぞ、続けよう」という内発的なシグナルこそが、あなたを計画的な探索へとどまらせるのです。

## なぜここでは本物の Montezuma で訓練しないのか

*本物* を適切に行うには以下のものが必要になります：

- 210×160 の RGB 画面を見るための畳み込みネットワーク
- フレーム・スタッキング（頭蓋骨がどちらに動いているか判断できるようにするため）
- RND モジュール（固定されたランダムな「ターゲット」と訓練される「予測器」の2つの追加ネットワーク）
- そして **数千万回の環境フレーム**（多くの GPU 時間）

これは研究プロジェクトであり、教育用スクリプトの範疇を超えています。そのため、`montezuma_revenge.py` では代わりに誠実な2つのことを行います：

### 1. 本物のゲームに「触れる」（`ale-py` がインストールされている場合）

Gymnasium を通じて `ALE/MontezumaRevenge-v5` をロードし、**一様ランダムなエージェントを 2000 ステップ** 走らせ、ゲームの合計報酬を報告します。表示される数値はほぼ確実に **0.0** です。「疎な報酬」という抽象的なフレーズが、自分で実行できる具体的な事実へと変わります。もし Atari パッケージがインストールされていない場合は、`pip install` コマンドを表示して次へ進みます。

### 2. スケールモデル `MiniMontezumaEnv` でテーブル型エージェントを訓練する

これは、Montezuma の最初の部屋と同じ **骨組み** を持った小さなグリッドワールドです：

```
###############
#S....#.......#
#.....#.......#
#.....#...T...#     S = スタート
#.....D.......#     K = 鍵 (Key)      D = ドア (鍵を持っている時だけ通れる)
#..K..#.......#     T = 宝物 (報酬が得られる唯一のタイル)
###############
```

勝つためには、**鍵** まで歩き（約6手）、それを拾い、**ドア** まで歩き（約4手、これで開く）、そこを通って **宝物** に到達する（約5手）必要があります。**約15回の完璧な動き** が必要で、**宝物にたどり着くまでフィードバックはゼロ** です。`has_key` フラグはエージェントの状態の一部であるため、鍵を掴むと発見すべき *新しい* 状態の部屋が丸ごと現れます。これは、本物のゲームで新しい画面が開くのと同じです。

次に、プレーンな **テーブル型 Q学習** を2回訓練します：

| エージェント | MiniMontezuma での結果 |
|-------|--------------------------|
| **好奇心なし (ε-グリーディ)** | 1,500 エピソードの間、報酬はずっと **0** のままです。鍵にすら到達できません。（どこかで聞いた話ですね？本物のゲームでの DQN と同じです。） |
| **予測誤差好奇心ボーナスあり** | 20〜25 エピソード以内に宝物に到達し、その後 **最適な 15 ステップのルート** を学習します。（これが Qテーブルに収まるように縮小された RND のアイデアです。） |

図には、2つの学習曲線が並べて表示され、さらに好奇心旺盛なエージェントが学習した実際のルートがグリッド上に描かれます（スタート → 鍵 → ドア → 宝物）。スクリプトはこのルートを ASCII フレームとしても出力します。

## 教訓

> **「疎な報酬」は、1つの変わった Atari ゲームの癖ではありません。成功のために長く特定の行動シーケンスが必要なあらゆる世界におけるデフォルトの状態です。** 報酬のみに頼るエージェント（バニラ DQN）は、文字通り学習を開始することすらできません。従うべき勾配が存在しないからです。好奇心ボーナスは、自己生成された濃密な「これは新しい、続けよう」というシグナルを作り出します。そしてそのシグナルこそが、エージェントをゼロの砂漠を越えさせて、最初の本物の報酬へと運んでくれるのです。その後のすべて（RND, Go-Explore, Agent57）は、同じ手法をニューラルネットワークでスケールアップしたバージョンに過ぎません。

## 覚えておくべきキーワード

| 単語 | 意味 |
|------|---------|
| **困難な探索 (Hard exploration)** | 賢く探索することでのみ成功できる問題。ランダムな探索は失敗する |
| **疎な報酬 (Sparse reward)** | 報酬がほとんどの場所でゼロであり、長い正しい手順の後にしか得られないこと |
| **Montezuma's Revenge** | 古典的な深層強化学習エージェント (DQN, A3C) が 0 点しか取れなかった Atari ゲーム。困難な探索の代表的なベンチマーク |
| **RND (Random Network Distillation)** | 予測誤差好奇心ボーナスを使用して Montezuma を攻略した 2018 年の手法 |
| **Go-Explore** | 「有望な状態を記憶し、そこに戻ってから探索を再開する」という、Montezuma を攻略した別の手法 |
| **スケールモデル (Scale model)** | 困難な問題の *構造* を維持したまま、素早く研究できるようにした小さく安価な環境 |

## 一言まとめ

> **Montezuma's Revenge は、「一度も受け取ったことのない報酬は何も教えてくれない」ということを強化学習の世界に教えたゲームです。そしてその解決策は、当時も今も、本物の景品を見つけるまでエージェントが探索に対して自分自身に報酬を与える「好奇心ボーナス」なのです。**
