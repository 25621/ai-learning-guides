# 探索戦略の比較 🔦

## 一言で言えば何が問題か

強化学習（RL）エージェントは、相反する2つのことを行わなければなりません：

- **利用 (Exploit)**：これまでで最もうまくいったことを行う。
- **探索 (Explore)**：もっと良い方法があるかもしれないので、新しいことを試す。

「利用」に偏りすぎると、平凡なルーチンに一生甘んじることになります。「探索」に偏りすぎると、いつまでも利益を得られません。単に探索するかどうかだけでなく、*どのように* 探索するかが、Montezuma's Revenge を解決できるエージェントと、スコアが 0 のままのエージェントの分かれ目となります。

このスクリプトでは、**5つ** の探索戦略を同じ2つの困難なタスクで直接対決させ、それぞれの個性を明らかにします。

## 現実世界の例：ランチのお店選び

あなたは200軒のレストランがある新しい街に引っ越してきたばかりだとします。

- **ε-グリーディ** = 「今のお気に入りの店に行く。ただし10回に1回はサイコロを振って、*完全にランダム*にレストランを選ぶ。」幅広くサンプルを収集しますが、*目的がありません*。すでに嫌いになった店を何度も選び直すこともあります。
- **楽観的初期化 (Optimistic initialisation)** = 「試していないレストランはすべて、そうでないことが証明されるまでは街で一番の店だと仮定する。」200軒すべてを丹念に回り、現実を突きつけられるたびに候補から消していきます。これなら本当に素晴らしい店をすぐに見つけられます。
- **UCB (Upper Confidence Bound)** = 「お気に入りの店を優先するが、ほとんど試したことがない店には*ボーナス*を与える。その店について知っていることが少ないほど、ボーナスを大きくする。」どこの知らない店を試すべきかについては賢いですが、すべての決定は局所的です。ルート全体の計画を立てるのではなく、*今*一番良く見える選択肢を選びます。
- **カウントベースの報酬ボーナス** = UCB に似ていますが、*新しさそのものを楽しむ*ようになります。新しい場所での食事はそれ自体が満足感（内発的報酬）を与え、その満足感がどの近辺を探索するかという長期的な計画を形成します。
- **予測誤差報酬ボーナス** = 「自分の予測を*裏切る*ような食事に興奮を覚える。」予想通りの新しい店？まあまあです。自分のメンタルモデルと劇的に違う店？魅力的であり、そのような店をさらに探すように計画を更新します。

## 5つの戦略 (`compare_exploration.py` に収録)

### 1. ε-グリーディ — デフォルトだが、探索ではなく「ふらつき」

基本的には欲張りに（グリーディに）行動しますが、確率 ε で一様にランダムな行動をとります。DQN などでの標準的なベースラインです。困難なタスクにおける致命的な欠点は、**各ステップが独立したコイン投げである**ことです。`N` 回の連続した正しい動きを偶然行うには、コインが `N` 回連続で表を出さなければなりません。これは指数関数的にあり得ない確率になります。ε-グリーディは「探索」ではなく「小刻みな揺れ」に過ぎません。

### 2. 楽観的初期化 — 「退屈だと分かるまでは潔白」

すべての Q値を、理論上可能な最大リターン `R_max / (1 − γ)` から開始します。すると、エージェントが一度も試したことのない行動は世界で最高のものに見えるため、**グリーディな** 方策はそれを試さざるを得なくなります。実際に訪れて価値が真実に向かって下がった後、ようやく落ち着きます。未試行の領域に対する楽観主義は、（Q学習のブートストラップを介して）**価値関数を通じて自動的に伝播**します。そのためエージェントは、一歩ずつ、まだ見ぬ世界へと引き寄せられていきます。コストはほぼゼロで、追加の記録保持も不要です。小規模なテーブルの世界では、これが最も強力な深層探索者となります。

### 3. UCB スタイルの行動選択 — 報酬ではなく「選択」にボーナス

`argmax_a [ Q(s,a) + c·√(ln t / N(s,a)) ]` を選びます。価値の高い行動を好みつつ、あまり試していない行動を水増しします。多腕バンディット問題で有名です。落とし穴は、ボーナスが **行動選択ルール** の中にしか存在せず、報酬には含まれないことです。そのため、ボーナスは価値関数の中を流れません。UCB は「*この* 状態で全行動を試したか確認する」のには適していますが、「遠くの未探索領域へのルートを計画する」のには向いていません。

### 4. カウントベースの報酬ボーナス — 好奇心の古典的な形

**報酬** に `1/√(N(s,a))` を加えます（時間の経過とともに減衰する重み `beta` を使用）。報酬に含まれているため、Q学習によって伝播されます。つまり、新しい領域につながる状態が価値を持つようになります。これは MBIE-EB や古典的な「探索ボーナス」のアイデアです。

### 5. 予測誤差報酬ボーナス — ICM/RND バージョンの好奇心

小さな学習済み予測モデルからの `−log P(s'|s,a)` を報酬に加えます（同様に `beta` を使用）。5つの中で最も鋭い新しさのシグナルです。決定論的な世界では、一度見た遷移の驚きは瞬時に 0 に近くなります（`1/√N` のようにゆっくり減衰するのとは対照的です）。ICM や RND のテーブル版と言えます。

## 2つのテストタスク

- **タスク A — MiniMontezuma**：鍵→ドア→宝物のグリッドワールド。宝物に到達したときのみ報酬が得られます（完璧に動いて約15手）。「長い疎な報酬の連鎖を生き残れるか」をテストします。
- **タスク B — DeepSea(N)**：深層探索の教科書的な連鎖。長さ `N = 5, 8, 11, 14` で実行します。報酬は `N` 回の正しい動きの先に隠れており、各ステップにはわずかな即時コストがかかります。目先の利益しか見ないエージェントはコストを嫌って報酬にたどり着けません。「連鎖が *長く* なっても戦略が機能するか」をテストします。

## 実際の結果

**タスク A — MiniMontezuma:**

| 戦略 | 最初の宝物到達 | 最終解決率 |
|----------|---------------:|-----------------:|
| ε-グリーディ | 到達せず | 0.00 |
| 楽観的初期化 | 約エピソード 1 | 1.00 |
| UCB 行動選択 | 約エピソード 3 | 約 0.95 |
| カウント報酬ボーナス | 約エピソード 82 | 約 0.41 |
| 予測誤差報酬ボーナス | 約エピソード 23 | 1.00 |

**タスク B — DeepSea, 報酬を見つけたシードの割合:**

| 戦略 | N=5 | N=8 | N=11 | N=14 |
|----------|----:|----:|-----:|-----:|
| ε-グリーディ | 0 | 0 | 0 | 0 |
| 楽観的初期化 | 1.0 | 1.0 | 1.0 | 1.0 |
| UCB 行動選択 | 1.0 | 1.0 | 0.0 | 0.0 |
| カウント報酬ボーナス | 1.0 | 1.0 | 約 0.1 | 0.0 |
| 予測誤差報酬ボーナス | 約 0.9 | 約 0.8 | 約 0.9 | 約 0.2 |

*（乱数シードによって数値は多少変動しますが、傾向は揺るぎません。）*

## 教訓

1. **ε-グリーディは探索ではない。** どちらの困難なタスクも解決できません。ランダムなふらつきでは、長い正しいシーケンスを繋ぐことは不可能です。（それにも関わらず、多くのコードでデフォルトとして使われています。簡単なタスクならそれで十分であり、実装が非常に単純だからです。）

2. **本当の探索とは、未知に対して楽観的であること。** 楽観主義を「初期値」に入れるか（戦略 2）、「行動選択」に入れるか（戦略 3）、「自己生成報酬」に入れるか（戦略 4-5）に関わらず、共通しているのは「*未探索のものを魅力的に見せる*」ことです。そうすれば、学習があなたをそこへ連れて行ってくれます。

3. **疎な報酬のグリッドでは、4つの「本物の」戦略はすべて機能する。** 中でも予測誤差ボーナスは、最も鮮明な「これは新しい」というシグナルを出すため、最短で到達します。

4. **楽観主義が長い距離を伝わらなければならない「深い」連鎖では、楽観的初期化が圧倒的なチャンピオンになる。** 追加コストなしで価値関数を通じて楽観主義を伝播させるからです。UCB は最初に崩壊します（ボーナスが価値関数に入らないため、深く計画を立てられません）。報酬ボーナスはより良い結果を出しますが、ボーナスが減衰する前に長い連鎖の奥まで楽観主義を押し通すには、通常のテーブル型 Q学習では時間がかかりすぎます。

5. **まさにこの点が、深層探索をピクセル入力にスケールさせるために追加の火力が必要だった理由です。** Bootstrapped DQN、本物のニューラルネットを使った RND（楽観主義が一マスずつ伝わるのではなく、似た状態間で汎化される）、Go-Explore（有望な状態を文字通り記憶してそこに戻る）などです。ここでのテーブル型のトイ・モデルは、その *原理* を示しています。実用的なシステムは、これらの原理に汎化能力を持つネットワークを組み合わせたものです。

## 覚えておくべきキーワード

| 単語 | 意味 |
|------|---------|
| **探索と利用のトレードオフ** | 新しいことを試すか、知っていることを使うか。RL における中心的な緊張関係 |
| **ふらつき (Dithering)** | 行動にランダムなノイズを加えることによる「探索」（ε-グリーディ、ガウスノイズ）。困難なタスクには弱い |
| **不確実性に直面した際の楽観主義** | 未知のものを、確認するまでは素晴らしいものとして扱うという包括的な原則 |
| **楽観的初期化** | すべての値を可能な最大リターンから開始することで、上記の原則を実装する手法 |
| **UCB** | 上限信頼限界： `argmax (価値 + 訪問回数に応じて減るボーナス)` を選択する |
| **深層探索 (Deep exploration)** | 単一の行動ではなく、長く一貫した「珍しい」行動のシーケンスを必要とする探索 |
| **beta アニーリング** | エージェントがいずれ探索をやめて利用に専念するように、好奇心の重みを時間とともに減らすこと |

## 一言まとめ

> **ε-グリーディは単なるノイズに過ぎません。真の探索戦略はすべて、楽観的な価値、行動選択ボーナス、または自己生成された新しさ報酬を通じて未探索領域を魅力的に見せることで機能します。最適な選択は、報酬が単に「疎（sparse）」であるか、それとも鍵のかかった金庫のように長く正確な手順を必要とする「深い（deep）」ものであるかによって決まります。**
