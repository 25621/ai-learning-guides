# 保守的Q学習 (Conservative Q-Learning: CQL) 🛡️

## それは何ですか？

他人が行った過去の株式取引の巨大な台帳を読んで、投資を学んでいるところを想像してください。台帳には「買い」「売り」「維持」が記録されていますが、**誰も実際に行わなかった取引の記録はありません**。

ここで、自信過剰な学生が台帳を見てこう言ったとします：
「もし誰かが毎週月曜日に宝くじを買っていたらどうだろう？ それは素晴らしい取引になったはずだ！」

問題は、**台帳には「月曜日に宝くじを買う」というデータが一切ない**ため、この学生は単に妄想しているだけだということです。しかし、その妄想上の取引は紙の上では素晴らしく見えるため、学生の「方策」はそれを実行したがります。

この妄想問題こそが**分布シフト（distribution shift）**です。オフライン学習者は、データセットが一度もテストしていないアクションを好む傾向があります。なぜなら、その楽観的な予測を否定するデータが存在しないからです。CQLはこの問題の治療薬です。

---

## オフラインでQ学習が失敗する仕組み

通常のQ学習のターゲットは以下の通りです：

```
target(s, a) = r + γ · max_{a'} Q(s', a')
```

この `max_{a'}` が危険です。データセットに状態 `s'` でのアクション `a'` の記録がない場合、ネットワークはQ値を単に*推測*します。そして、ニューラルネットワークは未知の入力に対してQ値を**過大評価**する傾向があります。ターゲットはその過大評価を引き継ぎ、ネットワークはその大きな数値を予測するように学習し、次のステップではデータが裏付ける範囲をさらに超えて**外挿（extrapolate）**します。方策は幻影を追いかけ始めるのです。

もし新しいデータを収集し続けることができれば、これは自己修正されます（幻影のアクションが実際には悪いことが判明するため）。しかし、**オフライン強化学習では新しいデータを収集できません。** 幻影は永遠に消えないのです。

---

## CQLのトリック

CQL (Kumar et al., 2020) は、損失関数にペナルティ項を追加します：

```
cql_loss(s)  =  log Σ_a exp Q(s, a)   -   Q(s, a_dataset)
```

2つのパーツに分かれています：

1. **`log Σ_a exp Q(s, a)`** （読み：全アクションにわたる log-sum-exp）：これは全アクションにわたる **ソフト・マキシマム（soft maximum）** です。1つの勝者を厳密に選ぶのではなく、すべてのアクションを一度に考慮する滑らかで微分可能な `max` の近似です。これにペナルティを与えることで、Q値を**全体的に**押し下げます（すべての予測を一律に下げます）。特に、幻影が潜んでいる「最も高いQ値を持つアクション」が強く押し下げられます。
2. **`- Q(s, a_dataset)`** ：データセットに実際に記録されているアクションのQ値を高く保つように報酬を与えます。これにより、上記の押し下げから「データ分布内」のQ値を守ります。

純粋な効果：**見知らぬアクションのQ値は引き下げられ、既知のアクションのQ値は引き上げられます。** 学習されたQ値は、真のQ値の「下限（lower bound）」となります。**`argmax`** 方策（単にQ値が最大のアクションを選ぶルール）は、幻影を追いかけるのをやめます。

全体の損失関数：

```
L  =  Bellman_MSE   +   α · cql_loss
```

（ここで **`Bellman_MSE`** は通常のQ学習の標準誤差であり、ネットワークの現在の推測と、それ自身の将来の推測がどれだけ一致していないかを測定します。）

`α` は保守性の度合いを決めるノブです。小さすぎると分布シフトが忍び寄ります。大きすぎるとエージェントは保守的になりすぎて、データを超えて改善することができなくなります。

## 実生活の例

- **保守的なチェスコーチ。** 既にプレイされた対局からしか学べません。向こう見ずなコーチは「前例のないこの仮説的な一手は素晴らしいかもしれない！」と言います。CQLは「そのデータはない。実際のプレイヤーが試した手に固執しよう」と言うコーチです。
- **レストランのメニュー選択。** グルメサイトのレビューは、裏メニューまでは網羅していません。素朴な方策は、妄想上の星5評価に基づいて裏メニューを推薦してしまうかもしれません。CQLは、信頼できるほど十分に注文されたメニューだけを推薦します。
- **ログからのロボットの把握。** ロボットはコップ、ボトル、本を掴む動画を持っていますが、ナイフの動画はありません。CQLは「ナイフの刃の部分を掴む」ことを自信満々に勧めることを拒否します。

## コードの動作

スクリプト `cql.py` は以下のことを行います：

1. `d4rl_dataset.py` によって作成された**4つのデータセットをロード**します。
2. **`medium-replay`** をトレーニングセットとして選択します。これは最も現実的（混合した質）であり、素朴な手法が最もダメージを受けやすいものです。
3. **3つのエージェントを純粋にオフラインでトレーニング**します。`α` 以外の条件はすべて同一です：
   - `α = 0`   →  素朴なオフラインDQN（ペナルティなし。壊れたベースライン）
   - `α = 1.0` →  穏やかなCQL
   - `α = 5.0` →  強力なCQL
4. **2,500勾配ステップごとに各エージェントを評価**します。実際の環境で10エピソード動かします。これが唯一の環境との接触であり、トレーニング自体は環境を見ることはありません。
5. **学習曲線**を `outputs/cql.png` にプロットします。

## 期待される結果

典型的な実行結果の表示：

```
最終評価リターン (10エピソード平均, greedy):
  naive offline DQN (alpha=0)         ->  ~30-150  (不安定。しばしばクラッシュする)
  CQL (alpha=1.0)                     ->  ~300-450
  CQL (alpha=5.0)                     ->  ~450-500
```

学習曲線のグラフ：

- **赤色の曲線** (`α = 0`) は、初期は上昇しますが、分布シフトの幻影が **ベルマン・ターゲット（Bellman target）** に感染すると、しばしば**崖から転落**します。（ベルマン・ターゲットとは、Qネットワークのトレーニングにおける「正解」として使用される数値 `r + γ · max Q(s', ·)` のことです）。幻影のQ値がこのターゲットを汚染すると、勾配を更新するたびに状況が悪化します。**ベルマン損失（Bellman loss）**（Qネットワークの予測とベルマン・ターゲットの間の平均二乗誤差）は正常に見えてしまいます。これがこの問題の**狡猾さ（treachery）**です。ネットワークは自分自身の誤った信念と完璧に一致しているため、損失関数は警告を発しません。
- **オレンジ色の曲線** (`α = 1.0`) は、よりゆっくり上昇しますが、**高い位置を維持**します。
- **緑色の曲線** (`α = 5.0`) は最も安定しており、通常は最高の結果を出します。

ベルマン損失のパネルは、もう一つの明白な証拠を示しています。素朴なDQNの損失は、方策が最悪であるにもかかわらず小さく留まることがあります。これは、ネットワークが自分自身の妄想と内部的に一致しているためです。

## この分野におけるCQLの位置づけ

CQLは、分布シフトに対する理論的でシンプルな修正を提供したため、非常に大きなインパクトを与えました。その系譜は以下の通りです：

```
DQN (オンライン)
   │
   ▼
素朴なオフラインDQN  ── 分布シフトにより崩壊
   │
   ▼
CQL (Kumar 2020)     ── 保守的なペナルティを追加：Q値は下限となる
   │
   ▼
IQL (Kostrikov 2021) ── 最初から未知のアクションに対してQ値を問い合わせない
   │
   ▼
Decision Transformer (Chen 2021) ── Q学習を完全にスキップし、強化学習をシーケンスモデリングとして扱う
                                     （LLMが次の単語を予測するのと同様に、過去の状態と期待する合計報酬
                                      から「次のアクション」を予測する）
```

この系譜の各ステップは、同じ問いに対する異なる答えです：
**「一度も見たことがないものについて、どうすればQネットワークへの問い合わせを避けられるか？」**

## 覚えておくべきキーワード

| 単語 | 意味 |
|------|---------|
| **分布シフト (Distribution shift)** | トレーニングされた方策が、データにないアクションを求めてしまう現象 |
| **分布外 (Out-of-distribution: OOD)** | データセットに一度も記録されていない (s, a) ペア |
| **真のQ値 (True Q)** | 状態 s でアクション a を取った場合の、将来の期待リターンの真の値 |
| **保守的Q値 (Conservative Q)** | 過大評価を避け、真のQ値以下に留まろうとする学習されたQ関数 |
| **Logsumexp** | `max` 関数の滑らかで微分可能な近似 |
| **アルファ (α)** | CQLの保守性を調整するノブ。未知のアクションのQ値をどれだけ強く押し下げるか |

---

## 一行まとめ

> **CQLは、データセットが試さなかったアクションのQ値を罰する「悲観的ペナルティ」を追加することで、方策が妄想に陥るのを防ぎます。**
