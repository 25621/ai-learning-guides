# 行動クローニング (Behavioral Cloning: BC) 🐒

## それは何ですか？

テニスを習いたいと想像してみてください。あなたはウィンブルドンの試合の録画を何百時間も視聴し、単に**選手がしていることを真似します**。彼らのショットが「最高」のショットだったかどうかを考えるのではなく、ただ自分の体の位置を彼らに合わせ、同じようにスイングするだけです。

それが行動クローニングです。**報酬も、計画もありません。ただ模倣するだけです。**

強化学習（RL）の用語で言えば： `(状態, 行動)` のペアのデータセットを用意し、画像分類モデルが猫か犬かを予測するのと全く同じように、状態から行動を予測するようにニューラルネットワークをトレーニングします。「ラベル」は、データ収集者が取ったアクションそのものです。

---

## 「真の」オフライン強化学習との違い

| アプローチ | 報酬を使用するか？ | データを超えることができるか？ |
|----------|---------------|---------------------|
| **BC**   | ❌ いいえ | ❌ いいえ — せいぜいデータの平均的な質に並ぶ程度 |
| **CQL**（など） | ✅ はい | ✅ はい — 混合データから良い遷移を繋ぎ合わせることができる |

BCは強化学習を「教師あり学習」の視点で捉えたものです。非常にシンプルで、時には驚くほど強力であり、普遍的なベースラインとなります。**もしオフライン強化学習アルゴリズムが同じデータセットでBCに勝てないなら、そのアルゴリズムは何も成し遂げていないも同然です。**

## 実生活の例

- **ドライブレコーダーの映像から運転を学習する。** 道路を見て、人間が行ったハンドルの角度を予測します。2つの画期的な例があります：
  - **ALVINN (1989)** — 最初期のニューラルネットワークによる運転システム。カメラとレーザーの入力からバンを操作して高速道路を走るための、わずか3層の小さなネットワークでした。
  - **NVIDIA PilotNet (2016)** — ドライブレコーダーの映像でエンドツーエンドにトレーニングされた最新のディープCNN。手動のルールを一切介さず、人間の運転手を模倣するだけで車線維持と基本的なステアリングを学習しました。
- **見習いがマスターシェフを模倣する。** 「シェフがすることなら何でもする」。シェフが優秀ならうまくいきますが、シェフが下手なら下手なシェフが出来上がります。
- **GitHub Copilot。** オートコンプリートは「人間が次にどんなコードを入力するか」を予測するようにトレーニングされています。これはソースコードのログの純粋な模倣です。
- **年上の兄弟の真似をする。** 子供たちは、年上の兄弟が「なぜ」そうするのかを考えるようになるまで、何年も真似を続けます。

## 数式（たった一行）

データセット内の各 `(s, a)` について、以下を最小化します：

```
loss = -log π(a | s)        (交差エントロピー)
```

これだけです。方策 `π` は行動のロジット（logits）を出力する単なるMLPです。トレーニングはMNISTと全く同じです。専門用語を分解してみましょう：
- **`π` (Pi):** 「方策（Policy）」の標準的な記号。何をすべきかを決定するルールやニューラルネットワークのことです。
- **MLP (Multi-Layer Perceptron):** 基本的で標準的なニューラルネットワーク。
- **ロジット (Logits):** ネットワークが確率に変換する前に吐き出す、正規化されていない生のスコア。
- **交差エントロピー (Cross-entropy):** 正解に低い確率を割り当てたときにモデルに罰則を与えるための標準的な数式。
- **MNIST:** 手書き数字の有名な初心者向けデータセット。

BCを通じてエージェントにゲームを教えるのは、MNISTの手書き数字を認識するようにネットワークをトレーニングするのと文字通り同じです。MNISTでは、入力は画像で出力は数字（0-9）です。BCでは、入力はゲームの状態で出力は行動（例：「左に動く」）です。

## コードの動作

スクリプト `behavioral_cloning.py` は以下のことを行います：

1. `d4rl_dataset.py` によって作成された**4つのデータセットすべてをロード**します（`random`, `medium`, `expert`, `medium-replay`）。
2. 各データセットについて、交差エントロピーを用いて10,000ステップの勾配更新を行い、**個別のBC方策をトレーニング**します。報酬（reward）カラムは完全に無視されます。
3. 2,500ステップごとに、実際の CartPole-v1 環境で現在の学習済み方策を動かし（20エピソードの平均）、**評価**を行います。
4. 以下のグラフをプロットします：
   - 棒グラフ：データセットごとの最終的なBCの報酬。
   - 学習曲線グラフ：各BCバリアントがトレーニングを通じてどのように向上するか。

## 期待される結果

典型的な実行結果の表示：

```
最終評価リターン:
  BC on random          ->    ~20  ± 少々   (≈ ランダムプレイ)
  BC on medium          ->   ~150  ± 大     (≈ medium方策と同等)
  BC on expert          ->   ~480  ± 小     (≈ expert方策と同等)
  BC on medium-replay   ->    ~60  ± 大     (≈ 混合データの平均)
```

棒グラフは明確なストーリーを物語っています：**BCの報酬は、データセットの平均報酬を追従します。** BCは、混合データセットの中の「良い」部分を「悪い」部分より優先する方法を持っていないため（どちらも等しく模倣の対象となるため）、その上限を超えることはできません。

これが結論です：**BCはデータの限界をそのまま引き継ぎます。**

## BC vs CQL — 最も明快な比較

**medium-replay** データセット（最も現実的で、質の混合したケース）において：

| 手法 | およその最終報酬 | 理由 |
|--------|--------------------:|------|
| BC     | ~60   | 初期段階の失敗した試行と、後の成功した試行の「平均」を模倣してしまう |
| CQL    | ~400+ | 報酬を使用してQ値の高い遷移を優先し、混合データから良い方策を繋ぎ合わせる |

つまり、CQLは**データを超え**、BCは**データに並び**ます。これこそが、オフライン強化学習が単なる「模倣学習」ではなく、独立した研究分野である理由です。データの質が混ざっている場合（現実のログは常にそうです）、報酬を考慮する手法の方がより多くの成果を回収できます。

**expert** データの場合、比較は逆転します。BCはエキスパートに並びます（~480）。なぜCQLがエキスパートに勝つのではなく「引き分け」になるのか不思議に思うかもしれません。CQLは「保守的」であるように設計されており、データセットに見られないアクションを罰するため、結局エキスパートがしたことと全く同じことをすることになります。エキスパートを超えることはできません（最大スコアが既に達成されているため）が、エキスパートの戦略を能動的に壊すこともありません。ただBCのパフォーマンスと並ぶだけです。

これは有名な「データの質 vs アルゴリズム」のトレードオフです：

```
                            エキスパートデータ  →  BCが勝利、CQLは引き分け
   アルゴリズムの洗練度      ↑         
                            混合データ         →  CQLがBCを圧倒
                            
                            ランダムデータ     →  どちらも失敗。探索が必要
```

## 現代の強化学習におけるBCの役割

- **オンライン強化学習の事前学習。** 多くの最新システム（RT-1, Voyager, ゲームAIなど）は、デモンストレーションに対するBCから開始し、その後にPPOやSACを用いてオンラインで微調整を行います。
- **RLHF (人間からのフィードバックを用いた強化学習)。** InstructGPTのステップ1は教師あり微調整（SFT）であり、これは人間が書いた回答に対する純粋なBCです。PPOと報酬モデルはその後に登場します。
- **DAgger (Ross et al., 2011)。** **誤差の累積（compounding-error）**問題を修正するための賢い拡張。
  *BCが完璧にクローンしたとしても、なぜ誤差の累積が問題になるのでしょうか？* たとえBCモデルが99%正確であっても、1%のミスはいつか起こります。ミスが起きると、エージェントは完璧なデータセットには存在しない未知の状態に入ってしまいます。混乱したエージェントはさらに大きなミスを犯し、既知のデータからさらに遠ざかり、最終的には壊滅的な失敗（断崖絶壁から転落するなど）へと積み重なって（累積して）いきます。
  *修正方法：* エキスパートにずっと運転してもらうこともできますが、エキスパートの時間は高価です。代わりに、DAggerではBC方策に運転させます。方策がミスをして奇妙な状態に迷い込んだとき、そこで停止してエキスパートに「今、ここであなたならどうしますか？」と尋ね、その答えをデータセットに追加します。BC方策が訪れる状態についてのみエキスパートに再度問い合わせるのは、常に問い合わせるのではなく、自分自身の特定のミスから回復する方法をエキスパートに教えてもらう必要があるからです。
- **Decision Transformer (Chen et al., 2021)。** アクションの予測を、期待する「報酬（return-to-go）」で条件付ける「賢い」BC。本質的にオフライン強化学習を「次トークン予測」へと変換したものです。

## 覚えておくべきキーワード

| 単語 | 意味 |
|------|---------|
| **模倣学習 (Imitation learning)** | 「デモンストレーターをコピーする」手法の総称。BCはその最もシンプルな形 |
| **誤差の累積 (Compounding error)** | BCの小さなミスが、データセットに存在しない状態へと導き、そこでさらなるミスが重なる現象 |
| **デモンストレーションデータ** | エキスパートによって作成された軌跡データ。BCのトレーニングセットとなる |
| **データの限界 (Data ceiling)** | BCの報酬はデータセット内の平均報酬に制限されるという性質 |
| **DAgger** | 誤差の累積問題を対話的に修正する手法 |

## 一行まとめ

> **行動クローニングは、(状態, 行動)のペアに対する単なる教師あり学習であり、データが良いときは強力ですが、データが混合しているときは無力です。**
