# DPO：審判を介さず直接ソースへ

## 核心となるアイデア

従来のRLHFには、まず報酬モデルを学習させ、次にPPOを使用してそのスコアを追いかけるという2つの段階があります。DPO（Direct Preference Optimization：直接好みの最適化）は、次のような賢い問いを投げかけます。

*「報酬モデルが単なる中間ステップであるなら、それをスキップできないだろうか？」*

その答えは「イエス」です。DPOは、好みのペア（比較データ）から言語モデルを直接学習させます。独立した審判（報酬モデル）も、PPOのサンプリングループも、調整が必要なKL係数も不要です。一つの洗練された数式を使用し、教師あり学習のように振る舞います。

これにより、DPOは実行がよりシンプルで、安定しており、高速になります。そのため、多くのオープンソースの整列（アライメント）済みモデルにおいて、急速にデフォルトの選択肢となりました。

## 現実世界の例え

あなたが、学生にエッセイの書き方を指導しているとします。

**PPOのアプローチ**は、エッセイを採点する教師を雇い、学生に次々とエッセイを書かせ、教師の採点に基づいて調整させるというものです。

**DPOのアプローチ**は、学生に一度に2つのエッセイを見せ、「こっちの方が良い。これに近い書き方をし、もう一方のような書き方は避けるように」と伝えるものです。中間に教師は存在しません。学生は比較から直接調整を行います。

どちらも機能しますが、DPOは通常、個別の教師を訓練して維持する必要がないため、より早く完了します。

## 学習の仕組み（直感的な理解）

DPOは、報酬モデリングと同じ好みのペア（プロンプト、選択された回答（chosen）、拒否された回答（rejected））を使用します。それぞれのペアに対して、2つの質問を投げかけます。

1. モデルは、リファレンスモデル（元のモデル）よりも、選択された回答を生成する**確率が高く**なったか？
2. モデルは、リファレンスモデルよりも、拒否された回答を生成する**確率が低く**なったか？

学習では、これら両方の数値を同時に正しい方向へと押し進めます。重要なのは、リファレンスモデルが常に比較対象として存在することです。これはPPOにおけるKLペナルティと同じ役割を果たします。モデルの変化は許容されますが、その変化は常に開始点からの**相対的なもの**となります。

DPOの論文における微細で美しい結論は、この単一の損失関数が「報酬モデルを学習させてから、KLペナルティ付きのPPOを実行する」ことと数学的に等価であるということです。目的地は同じですが、道程がよりシンプルになっています。

## 実験結果が示すこと

2,000個の好みのペアを用いて、300エポックにわたりポリシーを直接学習させました。

![DPO training](outputs/dpo_implementation.png)

- **左図**：モデルが拒否された回答よりも選択された回答を好むように学習するにつれて、DPO損失が低下しています。
- **中央図**：好みの精度（ポリシーが選択された回答に対して、より高い潜在的報酬を割り当てる頻度）が約99%まで上昇しています。
- **右図**：潜在的報酬のマージン（差）が拡大しています。DPOは明示的な「報酬」を定義しませんが、選択された回答と拒否された回答の対数確率の差（betaでスケーリングされたもの）を報酬として読み取ることができます。この差が着実に広がっていることは、モデルが自身の好みに自信を深めていることを意味します。

PPOと比較して、これがいかにクリーンであるかに注目してください。サンプリングループも、探索ノイズも、稼働中の個別の報酬モデルもありません。各エポックは、好みのデータセットに対する純粋な教師あり学習スタイルの更新となります。

## RLHFパイプラインにおける位置付け

DPOは、従来のパイプラインのステップ2（報酬モデリングとPPO）に代わるものです。

- **従来型：** 好み → 報酬モデル → PPO → 整列済みモデル
- **DPO：** 好み → 整列済みモデル（完了）

注意点としては、DPOは固定された好みのデータセットで学習するということです。PPOは、各ラウンドで新しい回答をサンプリングするため、理論上はより広範囲を探索できます。実際には、「精査された好みのデータセットに基づいて整列させる」というほとんどのユースケースにおいて、DPOが優勢です。

## 研究室の外でこれが重要である理由

「中間的な測定を省く」というパターンは、至る所で見られます。

- 水泳のコーチが、各ラップをストップウォッチで計測するのではなく、横に並んでフォームを実演して修正する。
- 写真家が、「良い写真」の採点基準を作るのではなく、一度に2枚の写真を見比べて良い方を選んで編集する。
- 採用担当者が、30項目のチェックリストで各人を採点するのではなく、2通の履歴書を比較する。

「ランク付け」が必要なだけであれば、絶対的な尺度は必要ありません。DPOは、その洞察を言語モデルに適用したものです。

## 一行まとめ

**DPOは、中間に報酬モデルを介することなく、好みのペアから直接より優れたモデルを作成します。PPOよりもシンプルで、多くの場合、同等の成果が得られます。**
