# 報酬モデリング：コンピューターに「人の好み」を教える

## 核心となるアイデア

報酬モデルとは、「小さな審判員」のようなものです。同じ質問に対する2つの回答を提示し、人間がどちらをより好んだかを教えます。これを繰り返すことで、報酬モデルは「人間が好むであろう回答」に対して、より高いスコアを与える方法を学習していきます。

なぜこのような審判が必要なのでしょうか？それは、言語モデルに期待する多くの要素を数式で書き表すことが難しいためです。「役に立つ」「礼儀正しい」「文章が上手い」といったことを定義する単一の方程式は存在しません。しかし、人間であれば、2つの選択肢のうちどちらが良いかをほぼ常に判断できます。報酬モデルは、こうした単純な「こっちの方が良い」という投票結果を、学習アルゴリズムが利用可能なスコアへと変換してくれるのです。

## 現実世界の例え

友人にブラウニーの焼き方を教えている場面を想像してみてください。

あなたは「何が良いブラウニーを作るか」について50ページのルールブックを渡したりはしません。代わりに、2つのバッチ（焼き上がった単位）を試食して、こう言います。

「こっちの方が美味しいよ」

これを数回繰り返すと、友人はパターンに気づき始めます。「しっとりしている方がいつも選ばれるな」「焼きすぎた方はいつも不評だな」といった具合です。友人は、あなたの比較から自分の中に採点基準を作り上げていきます。

報酬モデルは、まさにこれと同じことを数値で行います。選択された回答が*なぜ*優れているのかを知る必要はありません。ただ「こちらがあちらに勝る」という例をたくさん提示されることで、好みに一致したスコアを徐々に学習していくのです。

## 学習の仕組み（直感的な理解）

各学習データは、「プロンプト」「**選択された**回答（chosen）」「**拒否された**回答（rejected）」の3点セットです。モデルには、拒否された回答よりも選択された回答に対して、どれだけの差であっても構わないので、より高いスコアを付けるように促します。

学習における調整の考え方は非常にシンプルです。

- 選択された方のスコアが低すぎる？ → スコアを上げる。
- 拒否された方のスコアが高すぎる？ → スコアを下げる。
- すでに正しい順序で明確な差がある？ → 何もしない。

この調整手法は「ブラッドリー・テリー・モデルの損失（Bradley-Terry loss）」と呼ばれ、現代のRLHFシステムにおける標準的な手法となっています。

## 実験結果が示すこと

2,000個の合成された好みのペアを使用して報酬モデルを学習させました。下のグラフは、同じ学習プロセスの3つの側面を示しています。

![Reward model training](outputs/reward_modeling.png)

- **左図**：損失（loss）が急速に低下しています。モデルが自身のランキング付けに自信を深めていることを示しています。
- **中央図**：好みの精度（どちらが良いかを当てる確率）がほぼ100%まで上昇しています。ほぼすべてのペアにおいて、選択された回答が拒否された回答よりも高いスコアを得ています。
- **右図**：選択された回答と拒否された回答のスコア分布が引き離されています。開始直後は重なり合っていましたが、学習後には選択された回答の分布が明確に右側に位置しています。

この「分離」こそが重要です。後続のステップ（PPOやDPO）では、このスコアを最適化の目標として利用できるようになります。

## RLHFパイプラインにおける位置付け

RLHFは「人間の好みにモデルを整列（アライメント）させる」プロセスです。報酬モデルは、その3つのステップの最初にあたります。

1. **報酬モデル（本ファイル）**：好みの投票結果をスコアに変換する。
2. **PPOファインチューニング**：元の振る舞いを維持しつつ、言語モデルをより高いスコアへと押し進める。
3. **DPO**：報酬モデルを完全にスキップする、より新しいショートカット。

つまり、報酬モデリングは「人間の判断」と「マシンの最適化」をつなぐ架け橋なのです。この架け橋を掛け間違えると、その後のすべてのステップが誤った方向へと導かれてしまいます。

## 研究室の外でこれが重要である理由

同じアイデアは多くの場所で見られます。

- **レコメンデーションシステム**は、クリック、スキップ、視聴時間から、あなたの好みを学習します。
- **検索エンジン**は、「どの結果がクリックされたか」からランキングを学習します。
- **レストラン**は、客が好きなものについてエッセイを書くのを待つのではなく、リピート注文から人気メニューを把握します。

「評価（採点）」するよりも「比較」する方が簡単な場合、報酬モデルは最適なツールとなります。
