# PPOファインチューニング：モデルを壊さずに磨き上げる

## 核心となるアイデア

回答を採点する報酬モデルが手に入ったら、次は言語モデルがより高いスコアを得られる回答を生成するようにしたいと考えます。PPO（Proximal Policy Optimization：近接方策最適化）は、まさにこれを行う手法ですが、モデルがスコアを追い求めるあまり、普通の文章の書き方を忘れてしまわないように「安全ベルト」を追加します。

これは「磨き上げ」の工程だと考えてください。モデルはすでに流暢に話すことができます。私たちは、報酬モデルが好む方向にモデルを少しだけ促しつつ、そのモデルらしさを維持させるのです。

## 現実世界の例え

すでに料理の腕は良いものの、特定の料理評論家を喜ばせようと努力しているシェフを想像してみてください。

各料理に対し、評論家はスコアを付けます。シェフには2つのプレッシャーがかかります。

1. **より高いスコアを得ること。** 評論家が好むように料理を作ること。
2. **自分を見失わないこと。** もしシェフが自分のスタイルを完全に捨て、スコアを稼ぐためだけに塩をカップ一杯分も投入したりすれば、料理は奇妙なものになってしまいます。客は来なくなるでしょう。

PPOはこの両方のプレッシャーを捉えています。

- **報酬（reward）**の部分は、審判が好む回答へとモデルを押し進めます。
- **KLペナルティ**の部分は、学習開始前の話し方へとモデルを引き戻します。KL（カルバック・ライブラー情報量）は、「新しい振る舞いが古い振る舞いとどれだけ異なっているか」を測定する指標です。

これらが合わさることで、「より良くあれ、だが自分自身であれ」というメッセージになります。

## 学習の仕組み（直感的な理解）

各学習ラウンドは以下のようになります。

1. いくつかのプロンプトを用意し、現在のモデルに回答を生成させます。
2. 報酬モデルを使用して、それらの回答を採点します。
3. **リファレンスモデル**（学習開始前のモデルのコピー）と比較します。新しい回答が大きく異なっている場合は、報酬からKLペナルティを差し引きます。
4. スコアが良かった回答に近づくよう、モデルを少し調整します。

PPOの「Proximal（近接）」とは、*「大きなジャンプ（急激な変化）をしない」*という意味です。各更新は、慎重で小さな一歩です。ポリシーの学習における大きなジャンプは学習の破綻（クラッシュ）を引き起こすことがあり、それが単純な方策勾配法などの以前の手法が非常に不安定だった理由です。

## 実験結果が示すこと

初期状態のポリシーと学習済みの報酬モデルから開始します。PPOを150イテレーション実行し、回答のバッチをサンプリングしてポリシーを更新します。

![PPO training](outputs/ppo_fine_tuning.png)

- **左図**：報酬モデルの平均スコアが着実に上昇しています。ポリシーは、審判が好む回答を生成することを学んでいます。
- **中央図**：リファレンスモデルからのKL発散が増加しています。ポリシーが開始点から離れていっていることを示しています。これは予想通りの動きですが、もしこれが抑制されずに増大し続けると、モデルは支離滅裂な内容を生成し始めます。
- **右図**：整形された報酬（生の報酬からKLペナルティを引いたもの）は、最初は生の報酬に近い値で推移しますが、KLが増加するにつれて引き離されていきます。ペナルティが、離れすぎることに対して「代償を払わせる」という役割を果たしています。

実際のRLHFシステムでは、スコアは上昇しつつもモデルの一貫性が保たれるようにKL係数を調整します。ペナルティが小さすぎると、モデルは奇妙なフレーズを繰り返すなどの方法で報酬を「ハック」してしまいます。逆に大きすぎると、モデルは全く改善されません。

## RLHFパイプラインにおける位置付け

これは、従来のRLHFのレシピにおけるステップ2です。

1. 好みデータから報酬モデルを学習させる。
2. **その報酬モデルを使用し、PPOで言語モデルをファインチューニングする。**
3. （オプション）よりシンプルな経路を望む場合は、DPOでステップ2をスキップする。

PPOは、OpenAIやAnthropicといった企業が、InstructGPTや初期のChatGPTを含む第一世代の整列済みモデルで使用した主要な手法です。

## 研究室の外でこれが重要である理由

「改善しつつも、逸脱しない」というパターンは、どこにでも見られます。

- 難しい曲を練習しているピアニストは、一箇所を完璧に弾くために自身のテクニック全体を変えたりはしません。そんなことをすれば、リサイタルの他の部分が崩れてしまうからです。
- 登録数を増やすためにウェブサイトを微調整する企業も、既存のユーザーにとってブランドが認識可能な状態を維持しなければなりません。
- 工場の工程で一つのつまみを調整する場合も、他の設定は既知の良好な値の近くに保ちます。

PPOは、数学で記述された、この普遍的なアイデアの慎重なバージョンなのです。

## 一行まとめ

**PPOファインチューニングは、報酬を高める方向へモデルを押し進める一方で、KLペナルティによって元の振る舞いから離れすぎないように保ちます。「改善しつつも、自分自身であれ」ということです。**
