# 学習済みモデルを用いたプランニング（MPC）🔮

## 核心となるアイデア

あなたは**世界モデル（World Model）**、すなわち未来を予測するニューラルネットワークを手に入れました。さて、これをどう活用すべきでしょうか？

最も直接的な活用法は**プランニング（計画）**です。一瞬一瞬、「このプランを試したらどうなるか？あのプランなら？別のプランなら？」とモデルに問いかけます。そして、最も良さそうなプランを選びますが、実行するのは**その最初の1ステップだけ**です。なぜならモデルは完璧ではないため、1つのアクションだけを実行し、実際の環境から得られた新しい状態を観察し、再びゼロからプランを立て直すのです。

この手法は、**モデル予測制御（Model Predictive Control: MPC）**と呼ばれます。

---

## 現実世界の例え

あなたはレストランでメニューを見ています。その場で5コースすべての注文を確定させる必要はありません。最初のコースを注文し、お腹の膨れ具合を確認してから、デザートを注文するかどうかを決め直します。

あるいは、曲がりくねった道を運転している場面を想像してください。今後30秒間のハンドル操作をあらかじめ固定することはありません。常に前を見て数秒先を予測し、次のハンドル操作を行い、そしてまた予測（プランニング）を繰り返します。

この**「遠くを計画し、近くを実行し、計画し直す」**というループがMPCです。

---

## 「ランダムシューティング（Random Shooting）」の仕組み

より洗練されたプランナーも存在します。例えば：
- **CEM（Cross-Entropy Method）**：各ラウンドで上位k個のプランだけを残すことで、プランの分布を反復的に洗練させる。
- **MCTS（Monte Carlo Tree Search：モンテカルロ木探索）**：シミュレーション統計に基づいた探索木を構築する。AlphaGoやMuZeroで使用されている。
- **勾配ベースのプランナー**：アクションに対するモデルの予測を微分し、勾配を直接追いかける。

ここでは、最もシンプルで効果的な**ランダムシューティング**を使用します。

```
現在の状態 s が与えられたとき：
    1. 長さ H=15 のランダムな行動シーケンスを N=200 個サンプリングする。
    2. 各シーケンスについて、状態 s から世界モデルを通じてシミュレーションを行い、
       各ステップでの整形された報酬（shaped reward）を合計する。
       （200個の「夢」を並列でシミュレーション。非常に高速！）
    3. 予測された合計報酬が最も高いシーケンスを見つける。
    4. そのシーケンスの「最初の」アクションだけを現実の環境で実行する。
    5. 現実の次の状態を観察し、残りのプランは破棄する。
    6. ステップ1に戻り、ゼロからプランを立て直す。
```

200プラン × 15ステップ = 1現実ステップあたり3,000回の想像上の遷移。世界モデルは、ニューラルネットワークの1回のバッチ順伝播（フォワードパス）でこれらを一括実行します。通常、数ミリ秒で完了します。

---

## なぜ毎ステップ計画し直すのか？

それはモデルが不完全だからです。誤差はロールアウト（シミュレーションの実行）が進むにつれて累積していきます（`world_model.py` で生成され、`outputs/world_model.png` に保存されたグラフで確認できます）。ステップ0でのプランが信頼できるのは最初の数手だけであり、ステップ15に達する頃にはモデルは「幻覚」を見ているような状態になります。そのため、**最初の一手**だけを信頼し、最新の現実の状態に基づいてプランをリフレッシュするのです。

これは、人間がチェスで100手先までのプランを立ててそれに固執しないのと同じ理由です。状況は変化しますし、先を予測すればするほど、現実に即さなくなっていくからです。

---

## 注意点：報酬はプランナーに「何か」を伝える必要がある

CartPole（カートポール）では、現実の報酬はポールが倒れるまで毎ステップ `+1` です。モデルはほとんどのプランに対して忠実に `+1, +1, +1, ...` と予測します。なぜなら、モデル内ではランダムなプランでもすぐには終了しないことが多いため、すべてのプランが同じスコアになってしまうのです。これではプランナーは何を選べばよいか分かりません。

解決策：プランニング中、現実の報酬を**滑らかな代理報酬（proxy reward）**に置き換えます。

```python
reward_proxy(state) = 1
                    - |ポールの角度| / 0.21         # ポールは直立しているか？ (1=はい)
                    - 0.1 * |カートの位置| / 2.4   # カートは中央にあるか？ (1=はい)
```

これにより、最終的にポールが倒れてしまうプランは、直立を維持するプランよりも明らかに低いスコアになります。プランナーはこれらをランク付けできるようになります。

> **現実的な教訓：** 「もう1秒生き延びた」といった平坦な報酬シグナルは、短期的なプランニングには役に立ちません。高密度で整形された（shaped）シグナルが助けになります。

## コードが行っていること

`model_based_planning.py`：

1. `world_model.py` で保存された世界モデルの重みを**ロード**します。（見つからない場合は、その場で再学習させます。）
2. 現実の CartPole-v1 環境でMPCを**20エピソード実行**します。
3. 比較対象として、一様ランダムなポリシーでも**20エピソード実行**します。
4. 両方の結果を並べてプロットし、平均値を出力します。

### 実行時に確認できること

| ポリシー | 平均報酬（生存ステップ数） |
| :--- | :---: |
| ランダム | 〜22（CartPoleの典型的な値。ポールはすぐに倒れる） |
| MPC（本手法） | 〜150–500（シードにより変動。多くのエピソードで500近くに到達） |
| 最大可能値 | 500 |

この**5〜25倍の改善**は、ポリシーネットワークも価値関数も、追加の学習も一切なしで達成されています。ただ、世界モデルと、1ステップあたり200回の「夢」があるだけです。

`outputs/model_based_planning.png` のグラフでは、各エピソードごとに2本の色付きのバーが表示されます。MPCはほぼ常にランダムよりも高く、多くのエピソードで500ステップの天井に達しています。

## モデルベース・プランニングの長所

- **サンプル効率が良い**：学習はすべてランダムな遷移の1バッチから行われました。有用なポリシーを導き出すために、それ以上の環境との相互作用は必要ありませんでした。
- **目標変更が容易**：エージェントを別の方法で制御したいですか？その場合は代理報酬を変更するだけです。再学習は不要です。（お遊びで、カートの速度を最大化するようにしてみてください。）
- **解釈可能**：エージェントが検討したプラン、予測された軌跡、およびスコアを確認することができます。

## 短所（およびその対策）

- **ランダムシューティングは洗練されていない**：プランを盲目的にサンプリングします。より高次元の場合は、**CEM**（上述のクロスエントロピー法）や、モデルを局所的に線形近似して解析的に解く**iLQR**（反復線形二次レギュレータ）、あるいは微分可能なモデルを通じて勾配を追うことで行動を改善する**勾配ベース**のプランナーに切り替えます。
- **モデル誤差の累積**：長いホライズン（予測期間）では予測が逸脱します。対策として、同じデータで学習させた複数のモデルを用いる**確率的アンサンブル**（PETS, Chua et al. 2018など）が使われます。これにより、プランナーはモデル間の不一致に気づき、不確実なプランにペナルティを与えることができます。
- **最終的に欲しいのは現実の報酬である**：報酬整形（Reward shaping）は役立ちますが、より複雑なタスクでは、世界モデルの*内部*で学習された**価値関数**を使用します。これは、手作りの代理報酬を必要とせず、任意の状態からの長期的なリターンを推定する学習済みの「評論家（critic）」です。潜在的な想像空間の中でアクター・クリティックを完全に学習させる**Dreamer**や、モンテカルロ木探索と学習済み価値ネットワークを組み合わせた**MuZero**などは、このアイデアを利用しています。

## 現代のシステムとのつながり

今実行した「**学習されたダイナミクス + プランニング**」というレシピは、現代のAI研究における最も強力な強化学習システムのバックボーンとなっています。

- **MuZero** (DeepMind)：学習された世界モデルとモンテカルロ木探索を組み合わせたもの。ルールを事前に知ることなく、囲碁、チェス、将棋、Atariをマスターしました。
- **Dreamer / DreamerV3** (Hafner et al.)：学習された**潜在空間（latent-space）**世界モデルの*内部*でポリシーを学習させます。これは、モデルが未来を予測する前に、生の画像や状態をコンパクトで抽象的な表現に圧縮することを意味します。100以上のベンチマークで最高水準の性能を達成しています。
- **PETS / PlaNet / TD-MPC**：これらは、まさにこのアイデアをロボット工学のような複雑な連続制御タスクに拡張したアルゴリズム群です。

あなたは今、わずか数百行のコードで、それらの強力なシステムと同じファミリーの、最小のメンバーを構築したのです。

## キーワード

| 用語 | わかりやすい説明 |
| :--- | :--- |
| **MPC** | モデル予測制御。先を計画し、一度だけ行動し、計画し直す。 |
| **ランダムシューティング** | たくさんのランダムなプランを採点し、最高のものを1つ選ぶ。 |
| **ホライズン (H)** | プランがどれくらい先まで予測するか（ステップ数）。 |
| **N サンプル** | 1ステップあたりに検討する候補プランの数。 |
| **後退ホライズン** | 1つのプランに固執せず、毎ステップ計画し直すこと。 |
| **代理報酬 / 報酬整形** | プランナーが最適化に利用できる、滑らかで有用な代わりの報酬シグナル。 |

## 一行まとめ

> **世界モデルを手に入れれば、プランニングとは「100通りの未来を夢見て、最高の最初の一手を選び、それを繰り返す」ことである。**

これがモデルベース強化学習のすべての秘密です。
