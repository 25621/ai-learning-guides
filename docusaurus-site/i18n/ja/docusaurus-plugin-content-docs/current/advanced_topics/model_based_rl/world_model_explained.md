# 世界モデルの学習：エージェントに「夢」を見せる方法 🌍

## 「世界モデル（World Model）」とは何か？

**世界モデル**とは、エージェントが持つ*「宇宙（環境）の内部コピー」*のことです。状態と行動をモデルに与えると、次に何が起こるかを予測します。

```
(状態, 行動)  ──►  ニューラルネットワーク  ──►  (次の状態, 報酬)
```

これは現実の世界そのものではありません。現実を観察し、それを模倣することを学ぶことでエージェントが**自ら構築したシミュレーター**です。

学習が完了すると、モデルを使用して、エージェントは実際に動くことなく「もしも（what-if）」の質問を投げかけることができます。

> *「今左に押して、それから右に2回押したら、どこにたどり着くかな？ポールは倒れてしまうだろうか？」*

エージェントは、現実の一歩を踏み出す間に、モデルの中で100通りのプランをじっくり考えることができます。これが世界モデルの存在意義です。

---

## 現実世界の例え

あなたがパズルを解くときのことを考えてみてください。すべてのピースをすべてのスロットに実際に動かして試したりはしません。ピースAをここに置いたらどうなるかを**想像（イメージ）**します。その脳内シミュレーションが間違っているように見えれば、指一本動かさずにその案を却下します。

あなたの脳には、長年の経験から学んだ「物体がどのように振る舞うか」という世界モデルが備わっています。それがあるからこそ、行動を起こす前に結果をシミュレートできるのです。

その他の例：

- **チェスプレイヤー**：数手先の動きを想像する。
- **ドライバー**：「今ブレーキをかければ、後ろの車との距離は十分保てる」と考える。
- **ブロックを積み上げる子供**：「大きなものを上に置くと、タワーがグラグラするぞ」。（以前にタワーを倒してしまった経験から、このモデルを学習したのです。）

いずれの場合も、**「メンタルモデル + 想像 = リスクの少ないより良い意思決定」**となります。

---

## エージェントはどうやってモデルを構築するのか？

エージェントはただ**観察**するだけです。具体的には以下の通りです：

1. **データを収集する**：任意のポリシー（ランダムなものでも可）で現実の環境としばらく対話させます。すべての遷移を保存します：
   ```
   (状態, 行動, 報酬, 次の状態)
   ```
2. **ニューラルネットワークを学習させる**：`(状態, 行動)` から `次の状態` と `報酬` を予測するように訓練します。これは教師あり学習です。保存された各遷移がラベル付きの例となります。入力は「エージェントが見たこと、したこと」であり、ラベルは「その次に実際に起きたこと」です。
3. **検証する**：データの10%を検証用に残しておき、モデルの予測と実際の結果を照らし合わせます。誤差が少なければ、モデルは環境の**ダイナミクス**（行動によって状態がどのように変化するか）を捉えたことになります。

ここでの工夫：`次の状態` を直接予測するのではなく、**差分（delta）**である `次の状態 − 現在の状態` を予測します。物理現象の多くは漸進的であり（「カートがほんの少し動いた」など）、小さな目標値の方がニューラルネットワークにとって学習しやすいためです。

## 設定内容

| 項目 | 設定値 | 理由 |
| :--- | :--- | :--- |
| 環境 | `CartPole-v1` | 4次元の状態、2つの行動。モデル化が容易。 |
| データ | ランダムなポリシーによる20,000遷移 | 状態空間を広くカバーするため。 |
| ネットワーク | MLP, 2層 × 128ユニット, ReLU | MLPは標準的なニューラルネットワーク。128ニューロンの隠れ層2層。十分な容量があり、高速に学習可能。 |
| 損失関数 | `(差分状態, 報酬)` に対するMSE | MSE（平均二乗誤差）は標準的な回帰損失。 |
| 最適化手法 | Adam, 学習率 1e-3, 30エポック | Adamは適応的な最適化手法で、特別な調整なしで使用可能。 |

学習全体は、CPUでも数秒で完了します。

---

## 「良いモデル」とはどのようなものか？

2つの診断指標が重要です：

### 1. 1ステップの予測精度（検証MSE）

これは「モデルが未来の**1ステップ**をどれだけ正確に予測できるか」です。30エポック後には、検証MSEは **1e-4 から 1e-3** の範囲になるはずです。これは非常に小さく、ポールの角度やカートの位置が小数点以下数桁まで正確であることを意味します。

### 2. kステップのロールアウトにおける**累積誤差**

これが*本当の*テストです。ある状態をモデルに入力し、その予測値を再びモデルに戻す、という操作を `k` ステップ連続で行います。誤差は増大していきます。なぜなら、各ステップで前の予測の上に少しずつノイズが乗るからです。

```
ステップ 1:  L2誤差 ≈ 0.01  （ほぼ完璧）
ステップ 5:  L2誤差 ≈ 0.05
ステップ 10: L2誤差 ≈ 0.15
ステップ 20: L2誤差 ≈ 0.40  （目に見えて逸脱している）
```

*(L2誤差 = 予測された次の状態と実際の状態の間のユークリッド距離。「4次元の状態空間において、モデルの推測がどれだけ外れているか」を表します。)*

**これが重要な理由：** モデルを使って15ステップ先を計画する場合、15ステップ目の*正確な*状態は間違っているかもしれません。しかし、「良いプラン vs 悪いプラン」の相対的な順位が保たれていれば、プランニングは依然として機能します。（これは `model_based_planning.py` が利用している点です。）

`outputs/world_model.png` のグラフには、これら2つの診断結果が並べて表示されています。トレーニング損失の曲線は対数スケールできれいに下降し、ロールアウト誤差の曲線は緩やかに上昇しています。

## なぜ「差分（Delta）」を予測するのか？

ネットワークに同じ問題を提示する2つの方法を比較してみましょう：

| ターゲット | 典型的な大きさ | 難易度 |
| :--- | :---: | :--- |
| `次の状態` | 0〜2.4（カート位置） | ネットワークは位置**および**微小な変化の両方を再現しなければならない |
| `次の状態 - 現在の状態` | 〜0.02 | ネットワークは微小な変化だけを学習すればよい |

差分を予測することには別の利点もあります。もしネットワークの出力がゼロ（未学習の初期ネットワークによくあること）だった場合、予測結果は単に「何も動かなかった」となり、1タイムステップの予測としては理にかなった安全なデフォルト値になります。対照的に、絶対的な `次の状態` を直接予測しようとすると、最初は完全にデタラメな値が出力されるため、トレーニングの初期段階が非常に不安定になります。

## これによって得られるもの

学習済みの世界モデルは、以下の基礎となります：

- **プランニング**：想像上の行動シーケンスを探索する（`model_based_planning.py` を参照）。
- **Dynaスタイルの拡張**：想像上のデータでQネットワークをトレーニングし、サンプル効率を数倍に高める。
- **好奇心 / 探索**：モデルがうまく予測できない状態をあえて訪れる。
- **Dreamer / World-Models論文の手法**：初期のデータ収集以外、現実世界との対話なしに、モデルの内部だけで**ポリシー**を完全にトレーニングする。

## 限界と注意点

- **分布外（Out-of-distribution）への逸脱**：モデルは自分が経験した範囲の世界しか知りません。あまりに強気に計画を立てすぎると、モデルが一度も訪れたことのない領域に足を踏み入れ、そこでの予測は純粋な空想（デタラメ）になってしまいます。
- **誤差の累積**：グラフが示すように、累積誤差のため、長い**ホライズン**（遠い未来の予測）にわたるプランニングは信頼性が低くなります。現代のシステムでは、**確率的アンサンブル**（複数のモデルをトレーニングし、それらが一致するかを確認する手法。PETSやDreamerなどで採用）を用いることで、モデルの不確実性を把握し、リスクの高い未知の経路を避けるようにしています。
- **確率的な環境**：標準的な決定論的な回帰モデルは「平均的な」結果しか予測できず、起こり得る結果の広がりを完全に見落としてしまいます。複雑な現実世界の環境では、不確実性やランダム性を正確に表現するために、確率モデル（ガウス分布を出力するものや、世界の状態を圧縮空間内の確率分布としてエンコードする**潜在確率モデル**など）が必要になります。

## キーワード

| 用語 | わかりやすい説明 |
| :--- | :--- |
| **世界モデル** | 環境を模倣するニューラルネットワーク |
| **ダイナミクス** | `(s, a) → s'` という変化の関数 |
| **報酬モデル** | `(s, a) → r` という報酬の関数（通常、モデルに含まれる） |
| **1ステップ予測** | 現実の状態からモデルが出力するもの |
| **ロールアウト** | 1ステップ予測を繰り返し、出力を再び入力に戻すこと |
| **誤差の累積** | ロールアウトが進むにつれて増大する小さな誤差 |
