# Dyna-Q：「想像」することでより速く学習する 🧠

## Dyna-Qとは何か？

ミアという女の子が、新しい学校の道順を覚えている場面を想像してみてください。彼女は毎日廊下を歩き、新しい発見をします。「食堂を過ぎると図書室がある」「スミス先生の教室は階段の近くの2階だ」。

**単純なQ学習（Q-learning）**の生徒は、*今日*自分が行った行動からのみ学習します。今日、教室から食堂まで歩いただけなら、その一つの経路についての記憶しか更新しません。

**Dyna-Q**の生徒は違います。実際に歩いた後、彼女は少し腰を下ろし、覚えている過去の歩行を**頭の中で再現（リプレイ）**します。再現するたびに、彼女の頭の中の地図は強化されます。数週間後には、彼女は学校を隅々まで把握しています。それは、彼女がより多く歩いたからではなく、**すでに見たものについてより多く考えたから**です。

これこそが、強化学習エージェントにとってのDyna-Qの役割です。Dyna-Qは、実際の経験**および**、学習の過程で構築したモデルから引き出した「想像上の経験」から学習します。

---

## 3つの構成要素

Dyna-Qは「Q学習 + モデル + プランニング」です。現実世界での一歩は、**3つ**の役割を果たします。

1. **直接的強化学習（Direct RL）**：`(s, a, r, s')` からの通常のQ学習の更新。
2. **モデル学習**：「状態 *s* で行動 *a* をしたとき、報酬 *r* を得て状態 *s'* で終わった」と記録する。
3. **プランニング（計画）**：モデルの記憶からランダムに *n* 個の `(s, a)` ペアを選び、それらのステップが今起きたかのように**見せかけて**、さらに *n* 回のQ学習の更新を行う。

この3番目のステップが魔法の種です。`n = 50` の場合、現実世界での1ステップがQテーブルに対する**51回の更新**を引き起こします。エージェントは、純粋なQ学習者よりも（現実のステップ数換算で）約50倍速く学習します。

## ループの図解

```
                   ┌────────────────────────────────────┐
                   │                                    │
   現実世界  ──► 行動 a をとる ──► (r, s') を観測する    │
                            │                           │
              ┌─────────────┼──────────────┐            │
              ▼             ▼              ▼            │
        Q学習の更新      Model[s,a] ← (r,s')   プランニング ─┘
                                           (n 回の想像上の更新)
```

モデルは単なるルックアップテーブルです：
`(状態, 行動) → (報酬, 次の状態)`。構築もクエリも低コストで行えます。

## 現実世界の例

- **チェスの研究**：グランドマスターは、自分の対局や名局を頭の中で再現することに何時間も費やします。すべての再現は「プランニング」であり、すでに起きた経験からの追加の学習です。
- **ミュージシャンの音階練習**：難しい小節を一回弾いた後、次に進む前に頭の中でさらに10回リハーサルを行います。指は動いていませんが、脳は更新されています。
- **自動運転車**：赤信号で停車している間、タイヤを摩耗させることなくポリシーを微調整するために、シミュレーション内で過去100回の車線変更を再現します。

## コードが行っていること

サットン（Sutton）とバート（Barto）の書籍（[図 8.2](http://incompleteideas.net/book/the-book.html)）に登場する古典的な**Dyna Maze**を使用します。これは、いくつかの壁がある6×9のグリッドで、中ほど左側にスタート `S`、右上隅にゴール `G` があります。

3つのバリエーションを実行し、それぞれ30個のランダムシードで平均化します。

| 設定 | 現実の1ステップあたりのプランニングステップ数 | 意味 |
| :--- | :--- | :--- |
| `n = 0` | 0 | 通常のQ学習 |
| `n = 5` | 5 | 少しの想像上の練習 |
| `n = 50` | 50 | たくさんの想像上の練習 |

スクリプトは、トレーニングが進むにつれて**1エピソードあたりの平均現実ステップ数**を報告します。ステップ数が少ないほど、エージェントはゴールへのより直接的な経路を学習したことを意味します。

### 実行時に確認できること

この迷路の最短経路は約9ステップです。ε-greedy探索を行う、十分に訓練されたエージェントは、1エピソードあたり平均約10ステップになります。50エピソード実行すると、3つの設定すべてがそこに収束しますが、違いは**「いかに速く」**収束するかです。

| 設定 | 1エピソードあたりのステップ数（最後の10エピソード） | 意味 |
| :--- | :---: | :--- |
| `n = 0` | 〜10 | 収束した。しかし、ここまで来るのに30〜50エピソードの彷徨いが必要だった。 |
| `n = 5` | 〜10 | 約10エピソード以内に収束した。 |
| `n = 50` | 〜10 | 約3〜5エピソード以内に収束した。 |

注目すべきは最終的な数値ではなく、**学習曲線**です。`outputs/dyna_q.png` に保存されるグラフでは、3つの曲線が全く異なる速度で床（最小ステップ数）に向かって潜り込んでいく様子が見て取れます。`n = 50` はわずか数エピソードでそこに到達しますが、`n = 0` は実行のかなり後半までステップ数が減り続けます。（このような小さな決定論的な迷路では、通常のQ学習も最終的には到達しますが、Dyna-Qははるかに少ない現実のエピソード数で済みます。現実のステップが高コストである環境では、これが非常に重要になります。）

---

## なぜこの迷路でこれほど効果的なのか

理由は2つあります：

1. **環境が決定論的であること。** 各 `(s, a)` は常に同じ `(r, s')` を提供するため、1回訪問するだけでモデルは正確になります。想像上の経験は、現実の経験と同じくらい価値があります。
2. **現実のステップは高コストだが、想像上のステップは無料であること。** 想像上の更新はわずか数回のテーブル参照に過ぎませんが、現実のステップはエージェントが実際に歩く必要があります。現実の相互作用にコストがかかる場合（現実のロボットや現実のゲームなど）、Dyna-Qは非常に優れた**サンプル効率**を発揮します。

---

## Dyna-Qが苦戦するケース

- **確率的な環境。** もし `(s, a)` が多くの異なる `s'` 値につながる可能性がある場合、「最後の結果を覚える」モデルは嘘をつくことになります。解決策：訪問回数を保存するか、確率モデルを学習させます。
- **非定常な環境。** 世界が変化する場合（例えば、開いていたドアが突然閉まったり、近道が現れたりする場合）、モデルは古くなり、間違った予測をするようになります。**Dyna-Q+** は、*探索ボーナス*を追加することでこれを解決します。長い間再訪問されていない状態に小さな追加報酬を与えることで、エージェントが世界が変化していないか確認しに行くように促します。
- **広大な状態空間。** `(s, a)` をキーとする辞書は、数百万の状態や連続的な状態には対応できません。それこそが、**学習された（ニューラルネットワークによる）世界モデル（World Model）**が埋めるべきギャップです。詳細は次の `world_model.py` を参照してください。

## 覚えておくべきキーワード

| 単語 | 意味 |
| :--- | :--- |
| **モデル（Model）** | `(状態, 行動) → (報酬, 次の状態)` の記憶 |
| **プランニングステップ** | モデルからの想像上のデータを使用してQ更新を行うこと |
| **直接的RL（Direct RL）** | 現実のデータを使用してQ更新を行うこと |
| **サンプル効率（Sample efficiency）** | AIモデルやアルゴリズムが、特定の性能レベルに達するためにどれだけ効果的にトレーニングデータを使用しているかの指標 |
| **Dyna** | 学習とプランニングを交互に行うサットンのアーキテクチャ |

---

## 一行まとめ

> **Dyna-Qは実行すること、そして想像することの両方から学習し、想像することにコストはかからない。**

このアイデアは、現代のニューラルネットワークの形をとって、これまでに構築された中で最も強力な強化学習エージェントのいくつか（MuZero、Dreamer、World Models）の動力源となっています。
