# Implement InfoNCE

## Key Insight

[InfoNCE](/shared/glossary/#infonce) is the [loss](/shared/glossary/#loss-function) at the heart of [CLIP](/shared/glossary/#clip) and almost every [dual encoder](/shared/glossary/#dual-encoder), and writing it yourself demystifies it: you [L2-normalize](/shared/glossary/#cosine-similarity) the image and text [embeddings](/shared/glossary/#embedding), build the N×N grid of [cosine-similarity](/shared/glossary/#cosine-similarity) scores with a single [matmul](/shared/glossary/#matmul), and apply [cross-entropy](/shared/glossary/#cross-entropy) so that in each row the diagonal entry — the one true pair — wins the [softmax](/shared/glossary/#softmax) over all the mismatched ones. *Symmetric* means you do this twice, once down the rows (each image picks its caption) and once down the columns (each caption picks its image), then average the two; skipping one half quietly biases the model toward one [modality](/shared/glossary/#modality). Verifying the [gradients](/shared/glossary/#gradients) against a finite-difference estimate — nudging one input by a tiny ε and checking the loss moves by the amount the gradient predicts — is the cheapest way to catch a sign flip or a wrong axis before you waste a full training run.
